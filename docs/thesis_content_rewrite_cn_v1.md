## 中文摘要

随着大语言模型上下文窗口从数千扩展到数万乃至十万级词元，键值缓存（KV Cache）的显存与带宽开销已成为长上下文推理部署的主要瓶颈。对 KV Cache 进行低比特量化是降低显存占用、提升并发能力的直接路径，但现有方法对量化误差的分析与校准仍主要停留在数值重建层面，难以充分解释长上下文任务中出现的检索失败、生成退化与跨模型不稳定现象。本文认为，仅以张量数值误差来组织 KV Cache 量化研究是不充分的；相较之下，量化前后注意力行为的保持程度更值得被显式纳入分析与设计。据此，本文将“行为”提升为统一的分析与设计原则，用以组织校准、混合精度分配与预算建议三个层面的决策。

围绕这一原则，本文首先以 **INT8 规范验证路径** 作为最干净的端到端实例，说明行为对齐校准能够形成可离线固化、可运行时加载、可系统集成的校准产物。与旧叙事不同，本文不再将该路径解释为“行为目标普适优于 MSE”的证据，而是将其界定为一条更便于审计和归因的框架验证链。随后，本文将研究重点扩展到混合精度 KV 分配问题，考察行为导出的层敏感度画像是否能够支持逐层预算决策，并在多模型、多规模与多任务设置下给出更稳健的经验结论：KV 分配结果更适合被解释为随模型家族、模型规模和任务类型变化的分区结构，而不是某一个单一配置或单一方法的普适最优规律。

实验部分覆盖 Qwen2.5-1.5B、Qwen2.5-7B、LLaMA-3.1-8B、Qwen2.5-14B、Mistral-7B 等模型与若干长上下文评测任务。结果表明：第一，INT8 路径为行为导向框架提供了当前最便于归因、最便于审计的验证链，其作用应理解为“框架路径验证”，而非“校准目标的普适胜利”；第二，混合精度分配器的表现随模型家族、模型规模和任务类型变化显著，7B 呈现出典型的 aggregation-split 结构，8B 表现为 fixed-k 叙事开始失效的转折点，14B 则显示 heuristic 与 uniform 仍是强基线，3B 进一步暴露出明显的首层瓶颈异常；第三，基于行为导出敏感度画像的 **auto-k** 可以将脆弱的固定预算 sweep 转化为更有依据的预算建议机制，在 Mistral-7B 上给出当前最明确的正面结果，并在 8B、14B 上表现出实质竞争力，但仍不足以支持“跨模型统一最优”的结论。

综合而言，本文的主要贡献不在于提出一个普适更优的单点方法，而在于建立一条更稳健的研究主线：以行为为统一分析对象，以 INT8 规范路径验证框架可执行性，以跨模型分区结构重写 allocator 的经验事实，并将 auto-k 作为该框架下的有力扩展而非理论中心。这样的表述更符合当前证据边界，也更适合作为后续 clean-provenance 覆盖验证和进一步方法扩展的基础。

**关键词**：大语言模型；键值缓存；量化；行为对齐；混合精度分配；长上下文推理

# 第一章 绪论

## 1.1 研究背景

近年来，大语言模型在知识问答、长文档理解、代码生成和复杂推理等任务上取得了显著进展。伴随模型参数规模和上下文窗口的持续增长，推理阶段的系统开销也迅速攀升。对于采用 Decoder-only Transformer 的主流大语言模型而言，长上下文推理的主要瓶颈已不再仅仅是模型权重本身，而是随序列长度线性增长的键值缓存（KV Cache）。在上下文长度增大、并发批次提升的场景下，KV Cache 往往成为限制部署规模、吞吐能力和响应延迟的关键因素。

在自回归生成过程中，模型每生成一个新 token，都需要与历史 token 的 Key/Value 进行注意力计算。为避免重复计算，推理系统通常将历史 Key 和 Value 缓存在显存中。KV Cache 的引入显著降低了重复计算开销，但也带来了显著的显存负担。对一个具有 $L$ 层、每层 $H_{kv}$ 个 KV 头、头维度为 $d_k$、序列长度为 $S$ 的模型而言，KV Cache 的近似存储量可表示为：

$$
M_{\mathrm{KV}} = 2 \times L \times H_{kv} \times d_k \times S \times b
$$

其中 $b$ 表示每个元素所占字节数，因子 2 对应 Key 与 Value 两部分。由此可见，KV Cache 的存储量与序列长度呈线性关系，并受到层数、KV 头数和表示精度的共同影响。

量化是缓解这一问题的自然方向。将 FP16 的 KV Cache 压缩为 INT8、INT4 甚至更低比特表示，可以直接降低显存占用，从而在相同硬件预算下支持更长上下文或更高并发。然而，低比特压缩的代价并非只体现在数值误差上，更会通过注意力 logits、softmax 归一化和生成决策放大为行为偏移。尤其是在长上下文任务中，一些看似数值误差可控的量化配置，可能导致检索能力突然崩溃或生成分布明显偏移。因此，仅依赖数值重建误差来组织 KV Cache 量化研究，已经不足以支撑新的长上下文推理需求。

## 1.2 问题定义与研究动机

现有 KV Cache 量化研究通常关注两个问题：一是如何在尽量不损失质量的前提下降低缓存位宽；二是如何通过 kernel 优化或缓存管理进一步提升推理效率。尽管已有工作在工程上取得了显著进展，但其分析与校准方式往往建立在均方误差、百分位裁剪或逐张量数值重建等目标之上。这类目标在许多温和量化设置下是有效的，但它们并不直接对应模型真正受到影响的对象：注意力行为。

在自注意力计算中，Key 和 Value 并不是被单独消费的静态张量，而是通过 Query-Key 点积、softmax 归一化以及对 Value 的加权求和共同影响最终输出。量化误差一旦进入这一链条，其影响大小就不再由数值误差本身单调决定，而是取决于误差如何改变注意力权重分布、是否击中关键 token、以及这种偏移是否会累积放大到生成阶段。也就是说，KV Cache 量化的核心问题并不只是“量化后张量是否接近”，而是“量化后注意力行为是否仍然保持”。

基于这一观察，本文将“行为”提升为统一的分析对象。本文所谓行为，并不等于一个抽象的、不可操作的概念，而是具体落实为注意力分布、层敏感度画像、预算分配偏好等可被计算、记录和消费的中间对象。在这一视角下，离线校准不再只是搜索一个数值上最小误差的 scale，而是产生一个行为导出的校准产物；逐层混合精度分配也不再是机械地扫若干固定的 $k$，而是把预算问题重写为一个依据敏感度画像作出的决策问题。

进一步地，本文的动机并不止于“再提一个方法”。通过对现有实验和新数据的系统吸收，我们发现更值得写入论文主线的，不是某个行为导向方法对所有基线的普适性胜利，而是三个更稳的事实：第一，INT8 提供了一条适合验证框架可执行性的规范路径；第二，一旦进入 allocator 层，经验结果更适合被读成随模型家族、模型规模和任务类型变化的分区结构，而不是单一法则；第三，fixed-k 的脆弱性使得 auto-k 预算建议成为自然且必要的扩展。因此，本文真正试图回答的问题，是：在 KV Cache 量化中，什么才是合适的统一分析对象？这一对象如何同时组织校准与预算分配？而当预算分配进入多模型与多任务环境后，什么样的经验读法才是最稳健的？

## 1.3 国内外研究现状

围绕 KV Cache 压缩与高效推理，现有研究大体可分为四条相互关联的方向。第一条是 **量化格式设计**。代表性工作如 KIVI 提出了 Key 逐通道、Value 逐 token 的非对称量化格式，强调 K 与 V 在分布形态和量化敏感性上的差异；AsymKV 等工作进一步从层和角色的角度探索极低比特下的非对称配置。第二条是 **校准信号与校准目标设计**。在这一方向上，多数方法仍依赖 MSE、percentile clipping 等数值代理，而较少直接面向注意力行为或生成行为本身。第三条是 **混合精度与逐层预算分配**。KVTuner、KVmix 等工作已经表明，不同层在 KV 量化中的重要性并不一致，逐层 budget allocation 本身就是一个值得单独研究的问题。第四条是 **与量化相邻的 attention-driven control / eviction 路线**，如 H2O、SnapKV、AhaKV、DuoAttention 等，它们虽然不直接解决量化问题，但共同说明了 attention-derived signal 可以成为压缩决策的有效依据。

从这些工作可以看出，现有研究已经分别在“如何表示”“如何校准”“如何分配预算”“如何按注意力重要性压缩”几个方向给出了有效解法，但这些方向之间仍缺少一条统一的解释链。现有量化工作很容易陷入两种叙事偏差：一是把某种数值校准目标直接上升为质量最优的充分条件；二是把某个混合精度策略写成可跨模型迁移的普适法则。随着数据覆盖的扩大，这两种叙事都变得越来越不稳。本文的工作正是在这一背景下展开：我们不试图否定已有工作在其各自设计轴上的价值，而是希望把校准与预算分配纳入同一个以行为为中心的框架中重新解释，并用更保守、更可辩护的分区结构读法替代“单点方法优于一切”的旧叙事。

## 1.4 本文研究问题

围绕上述动机，本文聚焦以下三个相互递进的研究问题。

**研究问题一：在 KV Cache 量化中，什么才是合适的统一分析对象？**  
如果量化误差的真正影响是通过注意力行为传播的，那么校准目标是否应从单纯的数值误差，转向更直接反映行为偏移的对象？在这一问题上，INT8 是否可以提供一条最干净、最可验证的框架验证路径？

**研究问题二：当问题从校准扩展到预算分配时，最优读法是否仍然是“寻找一个统一最优配置”？**  
随着模型家族、模型规模和任务类型的扩展，fixed-k 保护层数、aggregation 方式、heuristic 基线之间的相对关系会如何变化？allocator 结果是否更适合被读成多种分区结构，而非单一规律？

**研究问题三：在 fixed-k 明显不稳的情况下，行为导出的敏感度画像是否能够支持更自动化的预算建议？**  
如果逐层预算选择确实依赖模型家族和画像形态，那么是否可以基于行为导出的层敏感度画像提出一种更合理的预算建议机制，从而减少对手工 fixed-k sweep 的依赖？

## 1.5 本文研究内容与主要贡献

围绕上述问题，本文将研究内容组织为一个由浅入深的三层链条。

第一层是 **以行为为中心的解释框架**。本文提出，不应将 KV Cache 量化理解为纯粹的数值压缩问题，而应以注意力行为为统一分析对象。在这一框架下，校准、层敏感度分析和预算分配不再是相互割裂的工程步骤，而是围绕同一分析对象展开的不同决策层。

第二层是 **INT8 规范验证路径**。本文将 INT8 路径界定为一个规范验证实例：它不是论文的最终压缩目标，也不承担“行为目标优于所有数值目标”的证明任务，而是作为最干净的端到端验证链，说明行为导向的校准与运行时集成能够形成可离线固化、可运行时消费、可审计的系统路径。

第三层是 **allocator 的分区结构读法与 auto-k 扩展**。在多模型、多任务设置下，本文不再把 allocator 问题写成“哪个 fixed-k 或哪个 aggregation 是最终答案”，而是将其重写为随模型家族、规模与任务而变化的分区结构。基于这种现实，本文进一步将 auto-k 定位为一种由行为导出画像支撑的预算建议机制，即框架下游的强扩展，而不是全文理论中心。

基于当前证据，本文的主要贡献可概括为四点。

**贡献一：提出以行为为统一分析对象的 KV Cache 量化框架。**  
本文将注意力行为提升为贯穿校准与预算分配的统一设计对象，明确区分行为对齐校准与行为引导分配两个层次，为后续方法与实验组织提供一条统一的解释链。

**贡献二：建立 INT8 规范验证路径，并明确其正确边界。**  
本文不再将 INT8 写成“低比特最强方案”，而是将其定位为最可审计、最可归因的框架验证链。实验表明，INT8 的主要价值在于验证框架可落地，而不是证明 `KL > MSE` 的普适结论。

**贡献三：将 mixed-precision allocator 的经验结果重写为分区结构读法。**  
在 7B、8B、14B、Mistral、3B 等不同模型上，allocator 的表现并不支持单一统一法则。7B 提供了最清晰的 aggregation-split 现象，8B 成为 fixed-k 旧叙事开始失效的转折点，14B 进一步说明 heuristic 与 uniform 仍是强基线，3B 则暴露出明显的首层瓶颈异常。本文据此将 allocator 结果组织为多种分区结构，而非单一胜负表。

**贡献四：提出并重定位 auto-k 为强扩展能力。**  
为解决 fixed-k 不稳和手工 sweep 脆弱的问题，本文引入基于行为导出画像的 auto-k 预算建议机制。当前证据表明，auto-k 已不再只是概念性设想，而可以被谨慎地写成具有初步跨模型支撑的预算建议扩展，尤其在 Mistral 上给出最明确的正面信号，并在 8B、14B 上进入第一竞争梯队；但本文明确不将其表述为跨模型统一最优解。

# 第二章 相关工作与理论基础

## 2.1 Transformer、KV Cache 与长上下文推理基础

本文研究对象是基于 Decoder-only Transformer 的大语言模型推理系统。在该架构下，模型通过自回归方式逐 token 生成输出，每一步都需要与历史 token 的 Key 和 Value 进行注意力计算。为了避免重复计算，推理系统通常将历史层的 Key/Value 张量缓存在显存中，这部分缓存即 KV Cache。

对本文而言，需要保留的理论基础并不在于完整复述 Transformer 的所有细节，而在于说明三点。第一，KV Cache 是长上下文推理中最主要的可变成本之一，其大小与层数、KV 头数、头维度、序列长度和表示精度共同决定。第二，GQA 架构将多个 Query 头映射到较少的 KV 头上，从而显著影响缓存开销与量化敏感性。第三，量化误差进入的是注意力计算链，而不是一条与模型行为无关的存储链。

基于这些事实，本文后续所有方法设计都围绕以下基本判断展开：当上下文长度持续增长时，如何压缩 KV Cache 不再是单纯的存储问题，而是一个直接作用于注意力行为的系统问题。

## 2.2 KV Cache 量化的三条设计轴

现有 KV Cache 量化研究可以沿三条设计轴来理解。

第一条是 **量化格式轴**，即 Key 和 Value 采用何种位宽、量化粒度和对称性设计。例如，量化可以是对称或非对称的，可以是逐张量、逐组、逐通道，也可以在 Key 与 Value 两侧使用不同的粒度与范围定义。

第二条是 **校准信号轴**，即量化参数是根据什么目标来选取的。现有方法大多使用 MSE、percentile clipping 或与之相近的数值代理，但也有越来越多的动机促使研究者考虑更接近行为层的目标。

第三条是 **预算分配轴**，即当统一位宽不再足够时，如何把较高精度的预算分配到不同层、不同角色甚至不同 token 上。这个问题本质上已经超出了“量化器怎么设”的范畴，而进入了“预算该怎么花”的 allocator 问题。

这三条设计轴是理解已有工作的最清晰方式，也决定了本文在文献中的准确位置。本文不主张自己在每一条轴上都提出了全新方案，而是强调：已有工作往往只在单条轴上优化，而本文尝试用一个以行为为中心的视角把这三条轴重新组织起来。

**表 2-1 内容设计：KV Cache 压缩研究的三条设计轴**

| 设计轴 | 关键问题 | 典型决策 | 与本文的关系 |
|---|---|---|---|
| 量化格式 | Key/Value 如何表示 | 对称/非对称，per-group / per-channel / per-token | 本文吸收其格式洞见，但不把创新点写成“新格式” |
| 校准信号 | 参数依据什么目标选择 | MSE、percentile、行为导向信号 | 本文重点重写此轴的解释对象 |
| 预算分配 | 高精度预算如何分配 | fixed-k、heuristic、uniform、auto-k | 本文重点重写此轴的经验读法 |

## 2.3 量化格式方向：K/V 非对称设计

在量化格式方向上，最重要的代表工作是 KIVI。KIVI 从 KV Cache 的统计分布出发，提出了 Key 逐通道、Value 逐 token 的非对称量化方式。这一格式的重要意义在于，它不再把 K 和 V 视为必须同构处理的对象，而是明确承认两者在分布结构和量化敏感性上的差异。后续 AsymKV 等工作又进一步将这种非对称思想推进到层级和极低比特设置中，说明 K/V 角色差异并非偶然现象，而是一个值得系统利用的结构特征。

本文与这一方向的关系应当被准确表述。本文并不宣称自己提出了一个完全脱离 KIVI 家族的新低比特格式，也不将论文贡献写成“数值上打败 KIVI-style baseline”。更准确的定位是：本文通过以行为为中心的诊断与解释框架，为 K/V 非对称格式提供一种更具机制性的解释，并进一步把这种解释延伸到 allocator 与预算建议层。换言之，本文对这一方向的主要贡献是“解释 + 组织”，而不是“另起炉灶造一个全新格式”。

## 2.4 校准信号方向：从数值代理到行为对象

在校准方向上，大多数工作仍然将量化看作一个数值重建问题。使用 MSE 或 percentile 的理由并不难理解：它们计算简单、实现稳定，并且在许多量化任务中足够有效。然而，对于直接作用于注意力计算的 KV Cache 而言，仅仅保持数值接近并不总能保证行为保持。softmax 的非线性会放大某些误差、吞掉另一些误差，使得数值误差大小与最终行为偏移之间并不存在稳定的一一对应关系。

本文在这一点上的核心主张不是简单地说“KL 一定比 MSE 更强”，而是说：如果真正关心的是长上下文推理中的行为保持，那么至少应该允许一种以行为为对象的校准视角存在。本文将这种视角落实为行为对齐校准，并进一步用 INT8 的规范验证路径说明：行为导向的校准不仅可以作为诊断口径存在，也可以形成可运行时消费的离线校准产物。

需要强调的是，本文在这一方向上的新口径比旧论文更收敛。当前证据足以支持“行为是一个有用的统一分析对象”，但并不足以支持“行为导向目标在所有设置下都普适优于数值代理”这一强主张。因此，本文在写作上应主动保留这一边界。

## 2.5 预算分配方向：混合精度与逐层预算分配

一旦统一位宽不再足以兼顾质量与成本，问题就转化为 allocator 问题。KVTuner、KVmix 等工作已经表明，不同层在 KV Cache 量化中的重要性并不一致，逐层预算分配不是简单的工程调参，而是一个独立的研究方向。这些工作推动了混合精度 KV 分配的出现，也说明 allocator 的研究价值并不依附于某一个具体量化器。

本文与这一路线的区别在于：本文不把 allocator 直接写成一个更强的优化器，而是强调 allocator 应当被理解为行为导出敏感度画像的下游消费层。也就是说，allocator 的输入不再只是经验性的层编号或盲扫出来的固定预算，而可以是上游画像提供的层敏感度信息。这样一来，allocation 与 calibration 就不再是两条分离的故事线，而是被同一个统一分析对象连接起来。

## 2.6 相邻方向：attention-driven memory control 与 eviction

除量化以外，还有一条与本文高度相关但并不完全同题的路线，即 attention-driven control / eviction。H2O 通过 heavy-hitter token 识别来选择保留哪些缓存位置；SnapKV 通过观察窗口估计头部的关注模式，从而压缩长上下文下的缓存；AhaKV 和 DuoAttention 等工作进一步说明，attention-derived signals 可以用来做更细粒度的缓存控制与头级调度。

这些方法与本文并不是直接竞争关系。它们主要解决“保留什么”或者“按什么注意力线索控制缓存”的问题，而本文解决的是“保留的 KV 用什么精度表示”以及“有限高精度预算如何分配”的问题。尽管问题不同，这些工作仍然具有重要参考价值，因为它们共同支持了一个更宽泛的判断：attention-derived signal 是一种可操作的压缩决策依据，而本文正是将这一判断引入了 calibration 与 allocation 的联合框架中。

## 2.7 本文定位

综合以上三条设计轴与相邻路线，本文的定位可以概括为以下四点。

第一，本文不是一个“全新低比特格式论文”。在低比特量化格式上，本文吸收了 KIVI / AsymKV 所代表的 K/V 非对称设计思想，但不把自己写成其直接替代者。

第二，本文也不是一个“单纯证明行为目标优于 MSE”的校准论文。INT8 路径的价值在于验证框架路径，而不是给出一个普适的校准目标胜负判决。

第三，本文不是一个“单一 allocator 打赢所有基线”的胜负论文。随着模型家族和规模的扩展，allocator 结果更适合被写成分区结构化的经验事实。

第四，本文真正的独特位置在于：它把校准与预算分配放到同一个以行为为中心的框架下，并把 auto-k 定位成这一框架下的自然扩展。换句话说，本文的中心不是某个点状方法，而是一条更稳健的问题组织方式。

基于这一定位，下一章不再沿着“先讲一个校准方法，再并列讲一个 allocator 方法”的写法展开，而是按照统一分析对象、两类离线产物和一条运行时路径来组织方法设计。这样处理的目的，是把前文讨论的三条设计轴真正收束为一个可执行、可解释的系统框架。

# 第三章 行为引导的 KV Cache 量化与分配框架设计

## 3.1 设计原则：以行为为统一分析对象

本文框架的出发点是：在 KV Cache 量化中，需要被分析和尽量保持的对象，不仅包括张量数值本身，更包括由这些张量共同决定的注意力行为。所谓行为，并不意味着把问题抽象成无法落地的概念层，而是将其落实为量化前后注意力分布的差异、层级敏感度画像以及基于这些画像作出的预算决策。只有把这些对象视为同一条设计链上的不同层，校准与预算分配才能被纳入同一个统一框架中讨论。

这一设计原则直接带来两个方法层次的区分。第一层是 **行为对齐校准（behavior-aligned calibration）**，它直接面向行为相关信号生成离线校准产物，用于回答“某一层应该如何被量化”。第二层是 **行为引导分配（behavior-guided allocation）**，它消费上游校准导出的层敏感度画像，用于回答“有限高精度预算应该花在哪些层上”。这一分层是本文的重要方法论边界：校准与分配相关，但并不等价；前者直接面向行为目标构造离线产物，后者则利用行为导出的画像完成预算决策。

## 3.2 框架总体结构

从实现角度看，本文框架可以被拆解为两个离线产物和一个运行时消费路径。

第一类产物是 **校准产物（calibration artifact）**。它来自离线校准流程，包含逐层或逐组的 scale、裁剪参数、必要的辅助项以及 provenance 信息。其核心用途是保证量化参数可复现、可审计、可在运行时直接加载。

第二类产物是 **策略产物（policy artifact）**。它来自 allocation 阶段，包含每层的目标位宽、被保护层列表、平均位宽预算以及在 auto-k 情况下的候选预算与推荐预算。其核心用途是把预算决策从代码逻辑中剥离出来，使混合精度路由成为一条显式、可替换的系统路径。

运行时阶段同时消费这两类产物：一方面使用校准产物确定量化器如何工作，另一方面使用策略产物决定哪些层走高精度路径、哪些层走低精度路径。这样的结构有两个直接好处。其一，它将“参数如何定”和“预算如何分”从概念上彻底分离，避免方法写作时把二者混成一个模糊的优化故事。其二，它为未来的 K/V-asymmetric allocation 和 prompt-adaptive policy selection 留出了清晰接口，而不需要重写框架本身。

## 3.3 行为对齐校准（behavior-aligned calibration）

behavior-aligned calibration 的目标是生成一个可离线固化的校准产物。在实现上，离线流程通过校准样本收集若干层的 Query、Key、Value 表征，并在候选量化参数空间中搜索更合适的配置。与传统只面向张量数值的校准不同，这一流程强调其目标对象应与注意力行为尽量对齐。

在当前工作中，behavior-aligned calibration 的主要价值不应被写成“证明 KL 一定比 MSE 更强”。更准确的表述是：它提供了一种更合理的统一分析视角，并在 INT8 的规范路径中生成了可部署、可审计的校准产物。也就是说，这一阶段的核心贡献是建立框架路径，而不是给出一个普适的 loss superiority 结论。

从产物内容上看，一个典型的校准产物应包括：

- 模型标识、revision、随机种子与校准数据来源
- 针对各层或各组的 scale / clipping 参数
- 若框架保留某些辅助项，如温度或保护性参数，也应显式写入
- 产物生成时间、搜索空间版本与 provenance 元数据

这些内容的存在，使得校准不再只是“在线统计后立即推理”的黑箱过程，而成为可以审计、复查和复现实验路径的一部分。

**表 3-1 内容设计：校准产物结构**

| 字段类别 | 典型内容 | 作用 |
|---|---|---|
| 模型元数据 | model name、revision、seed、calib set | 记录 provenance |
| 量化参数 | per-layer scale、clip percentile、group size | 定义量化器行为 |
| 辅助项 | 可选温度或保护性参数 | 保证路径可复现 |
| 运行约束 | bit-width、quantization mode、artifact version | 便于运行时校验 |

## 3.4 INT8 规范验证路径

INT8 在本文中承担的不是“最终低比特目标”的角色，而是一个规范验证实例。其意义在于：在这个相对温和、可归因的 setting 中，离线校准、产物固化、运行时加载和 fused decode 路径之间可以被清楚地串成一条端到端链。由于 INT8 的量化程度不像 INT4 那样激进，许多变量之间的相互干扰较小，因此它更适合作为“框架验证链”，而不是“极限压缩战场”。

这一本章中的定位必须在论文里讲清楚。INT8 路径验证的是“行为导向框架是否可落地、可复现、可集成”，而不是“行为目标是否在所有条件下优于 MSE”。后者是旧主线想证明但未被当前证据充分支持的说法；前者则是当前证据真正能稳住的说法。因此，第三章写 INT8 时，应强调它的规范性与可审计性，而不是过度强调其数值胜负。

## 3.5 行为引导的预算分配（behavior-guided allocation）

当问题从校准阶段转向混合精度预算分配阶段时，核心问题发生了变化。此时需要决定的已不再是某个层内量化参数，而是“哪些层值得保留更高精度”。在当前框架中，这一决策来自行为导出的层敏感度画像。

当前 allocator 的核心思想是：利用上游校准阶段暴露出的层级差异，将其压缩成一个每层一个分数的敏感度画像，再将这一画像转化为具体预算策略。可选策略包括：

- top-k：直接保护最敏感的前 $k$ 层
- threshold：保护超过阈值的层
- heuristic：按位置给出固定模式的保护层
- uniform：不给行为引导，直接统一预算
- auto-k：根据 sensitivity coverage 自动提出候选预算

需要强调的是，这一阶段被称为 **行为引导**，而不是“行为对齐”。原因在于 allocator 并没有在每个候选 budget policy 上重新直接优化原始行为目标，而是将上游行为导出的敏感度画像作为决策依据。这样的表述更加准确，也能避免把 allocator 写成一个比现有证据更强的理论主张。

## 3.6 auto-k：基于敏感度画像的预算建议扩展

fixed-k 的主要问题在于它对模型家族和规模过于敏感。某个在 7B 上表现不错的 $k$，在 8B 或 14B 上可能完全失效；而 hand-sweep 出来的最佳预算不仅难迁移，也难以被写成一个真正的方法贡献。auto-k 的出现，正是为了回应 fixed-k 的这一本质脆弱性。

本文对 auto-k 的定位应十分明确：它不是理论中心，也不是普适赢家，而是行为引导分配框架下的一种 **基于敏感度画像的预算建议机制**。它的作用是把层敏感度画像转化为一组更有可能有效的预算候选，从而减少盲目的 fixed-k sweep。

在当前实现中，auto-k 可以被理解为一种 coverage-based proposer：按层敏感度排序后，计算前若干层对总 sensitivity mass 的覆盖比例，并据此生成若干候选预算。这样一来，auto-k 的价值并不在于“总是最优”，而在于“把手工穷举缩减为少量更有信息量的候选”。这也是本文将其写成强扩展而非主理论的原因。

## 3.7 运行时集成与扩展接口

在运行时阶段，系统同时消费校准产物与策略产物。生成循环根据策略产物确定各层位宽路由，再调用对应的量化与反量化路径执行推理。这一实现方式的关键意义在于：框架已经把“量化器参数”和“预算决策”拆成两类可显式管理的对象，因此未来的扩展可以直接挂接在现有接口之上。

当前最值得保留为未来工作接口的有三条。第一是 **K/V 非对称分配**，即把当前对 K 和 V 角色差异的诊断真正转化为角色级预算决策。第二是 **质量—成本 Pareto 分析**，即把 allocator 从质量比较推进到质量、延迟与显存三维联合优化。第三是 **prompt 自适应分配**，即根据输入任务类型或输入画像动态选择策略产物。需要强调的是，这三条都应作为“框架已预留接口、当前尚未进入主结果”的未来方向，而不应提前写成已完成的主贡献。

综上，第三章真正建立起来的是一条从统一分析对象到两类离线产物，再到运行时消费路径的完整链条。第四章的实验安排将严格沿着这条链条展开：先讨论 INT8 规范验证路径能够支持的结论边界，再讨论预算分配在不同模型上的分区结构，最后讨论 auto-k 作为扩展能力所处的位置。

# 第四章 实验与分析

## 4.1 实验设置与证据分层

### 4.1.1 实验目的与写作边界

本章的目标不是罗列所有实验，而是围绕本文主线组织最关键的证据。与旧版叙事不同，实验结果不再被写成“某一方法全面优于所有 baseline”，而是分别服务于以下四个问题：

1. INT8 规范路径到底验证了什么；
2. allocator 的经验事实到底应如何被解释；
3. auto-k 到底是何种强度的扩展；
4. 哪些结果可以进入主文，哪些只能作为支持性证据。

因此，本章在写法上需要显式区分证据层级。当前阶段的结果足以支撑“主文候选级”叙事，但仍不能被写成最终发布级的主表结论。对正文而言，这一分层不是附带说明，而是保障主线可信度的重要组成部分。

## 4.2 INT8 规范验证路径：能证明什么，不能证明什么

INT8 在本章中首先承担框架验证的职责。当前最稳的结论并不是“INT8 上 KL 显著优于 MSE”，而是：在 INT8 这一较温和但可部署的设置下，行为对齐校准可以形成完整的校准—产物—运行时路径，并在系统实现上闭环。

这一节建议用一张边界表把结论写死，而不是在文字中反复申明。核心数据应包括：

- 在 1.5B INT8 上，KL 与 MSE 在当前鲁棒选择协议下收敛到同一参数点
- 产出的 calibration scales 逐位一致
- 下游指标也一致，例如：
  - PPL = 9.3367
  - Needle = 100%
  - RULER pass rate = 60.12%

这组结果的最重要意义在于：它证明了“框架路径成立”，而不是“行为校准胜过 MSE”。如果继续把 INT8 写成 `KL > MSE` 的胜利证据，整条新主线就会再次失去可信度。

**表 4-3 内容设计：INT8 结论边界表**

|---|---|---|---|
| KL 与 MSE 在 INT8 是否分化？ | 1.5B 上同点收敛 | INT8 足以验证框架路径 | KL 在 INT8 普适优于 MSE |
| INT8 的价值是什么？ | 校准产物可固化、可运行时消费 | 规范验证实例 | 低比特最终方案 |

## 4.3 allocator 的跨模型分区结构读法

当研究问题从校准转向预算分配时，数据所支持的最强结论已经改变。进入这一节之前，需要先明确两点。其一，heuristic 在多个模型上都表现出强基线地位，因此不能再被当作陪衬方法处理。其二，auto-k 已经具备写入正文的实证支撑，但其角色更接近扩展性的预算建议机制，而不是统摄全文的理论中心。在这一前提下，当前最稳的经验读法不再是“找到一个统一最优的 k 或 aggregation”，而是 allocator 的表现随着模型家族、模型规模和任务类型发生结构性变化。也就是说，当前结果更适合被组织成一组分区结构，而不是单一胜负叙事。

这一步是整篇论文主线升级的关键。旧叙事试图寻找一个能跨 1.5B、7B、8B、14B 迁移的 fixed-k 或 aggregation 规律，但在当前对比集合下，这条路已明显失去稳定性。与其继续为某个单一规律辩护，不如将 allocator 结果组织为分区结构化的经验事实，并以此为基础重组全文实验章。

| 8B | fixed-k 旧叙事开始失效 | auto-k 有竞争力，但非最优 | 单调 scale-shift |
| 14B | heuristic / uniform 仍强 | 宽预算高质量区间 | 某一 allocator 全面取胜 |
| Mistral | auto-k 取得最明确正面结果 | auto-k 最清晰的正面案例 | auto-k 普适最优 |
| 3B | 首层瓶颈（first-layer bottleneck） | 异常区间 / 特殊模型状态 | 一般小模型规律 |

## 4.4 7B：最具辨识度的分区结构发现

7B 是当前最适合被写进正文核心发现的模型。其最重要的价值，不在于某一个具体数字最大，而在于它暴露出 allocator 行为的结构性分裂：不同 aggregation 下的行为导引策略表现出显著不同的质量模式，形成清晰的 aggregation-split 结构。

这意味着 7B 并不是在告诉我们“某个 aggregation 一定更好”，而是在告诉我们 allocator 的经验表现可能在结构上分裂成不同状态。对论文而言，这类 finding 远比“某个方法赢了 0.1 分”更重要，因为它会直接改变我们应该如何组织 allocator 理论与实验解释。

因此，7B 在全文中的角色应当被明确写成“定义 allocator 分区结构的关键发现”，而不是新的普适规律。

## 4.5 8B：fixed-k 旧叙事失效的转折点

8B 的价值在于它打断了旧的 fixed-k 叙事。随着 budget 扩大，fixed-k 并没有沿着一条简单的“模型越大，保护层越多越好”的曲线发展。相反，8B 的结果告诉我们：fixed-k 在这一阶段已经变得不稳，而基于画像的预算建议开始变得必要。

在当前数据下，8B 上的 auto-k 已经进入强二梯队，显著强于 heuristic 的较弱区间和若干 mean-k 配置，但仍略低于 best fixed-k。最稳的写法应当是：

- auto-k 在 8B 上已经是有竞争力的扩展；
- 但 8B 不支持把 auto-k 写成最优统一解；
- 8B 也不再支持“best-k 随规模单调上移”的旧故事。

建议在这一节中直接给出 8B 的代表性数值：

- `bakv_k11 = 9.5214`
- `bakv_auto_cov80_max = 9.3543`
- `bakv_auto_cov90_max = 9.3491`
- `bakv_auto_cov70_max = 9.2949`

这些数字的意义不在于排名本身，而在于它们共同说明：auto-k 已经进入可写入正文的竞争区，但 fixed-k 仍未完全失效。

## 4.6 14B：强基线与宽预算带

14B 提供了与 8B 不同但同样重要的信号。它进一步证明，heuristic 与 uniform 不应被当成弱基线来处理。在 14B 上，多种策略共同落入一个较宽的高质量区间，这说明 allocator 问题在更大模型上呈现出“宽带稳定”而非“单点最优”的特征。

当前 14B 的代表性结果包括：

- `uniform_int4_k4v4 = 7.2345`
- `bakv_auto_cov90_max = 7.1501`
- `bakv_auto_cov80_max = 7.1213`
- `heuristic_k3 = 7.1171`

这一结果的正确读法不是“uniform 打败了所有行为导向方法”，也不是“auto-k 已经接近最优所以可以写成赢家”。更稳妥的读法是：

1. 14B 进一步证明强基线仍然活跃；
2. auto-k 已进入第一竞争梯队；
3. allocator 的合理解释仍然是 regime，而不是某个统一法则。

## 4.7 Mistral：auto-k 最清晰的正面证据

Mistral-7B 是当前 auto-k 最值得写进主文的正面结果来源。在 Mistral full sweep 中，`bakv_auto_cov80_max` 获得了当前最好的平均分，并在多个任务上给出明确的正向结果。这意味着 auto-k 已经不再只是一个“合理想法”，而是具备了完整实验链支持的扩展能力。

当前可直接写入的代表性结果为：

- `bakv_auto_cov80_max = 14.7640`
- `heuristic_k3 = 14.6036`
- `bakv_auto_cov70_max = 14.4000`

分任务上，`gov_report` 和 `hotpotqa` 由 auto-k 取得最好结果，而 `narrativeqa` 仍由 `heuristic_k3` 保持更强表现。这一点非常重要，它说明即使在 auto-k 最清晰的正面案例中，heuristic 也没有彻底失效。因此，本节的正确结论应是：

- Mistral 提供了 auto-k 最明确的正面证据；
- 但它仍不能被写成 auto-k 的普适性证明。

## 4.8 heuristic 应被重新定位为强基线

旧叙事中最危险的一点，是容易把 heuristic 写成一个弱基线，好让行为引导策略显得更强。当前数据已经明确否定了这种写法。在 14B 和 Mistral 等模型上，heuristic 不仅不是陪衬，反而常常定义了质量上界附近的竞争区间。

从论文结构上看，heuristic 的重新定位实际上强化了本文的主线。因为一旦 heuristic 是强基线，那么任何行为引导分配器的价值都不能再靠“赢过弱对手”来证明，而必须通过更稳的结构性现象来说明。这恰恰支持了本文将 allocator 写成分区结构问题，而不是优胜性问题。

## 4.9 auto-k 的定位：强扩展，而非理论中心

综合 8B、14B、Mistral 和 3B 的结果，当前对 auto-k 的最稳表述应当是：它是一种具备实证支撑的预算建议机制，已经从局部设想升级为框架下游的强扩展能力，但其显式胜出结果主要仍集中在 Mistral，因此不应被写成全文的理论中心，更不应被写成跨模型普适最优方法。

## 4.10 3B：首层瓶颈与首层恢复

3B 是当前实验中最有理论新意的异常现象之一。它的主角不是 auto-k，而是首层恢复（first-layer rescue）。本模型显示出极强的早层瓶颈特征，使得保护第一层能够显著提升结果，而中层 heuristic 则可能灾难性失败。

当前 audit 给出的关键数据非常适合写成一段高度凝练的结果：

- `bakv_k1` 保护层为 `[0]`
- `heuristic_k1` 保护层为中层 `[18]`
- `bakv_k1 = 6.9023`
- `heuristic_k1 = 3.48`

这个结果的重要性在于，它说明 allocator 的分区结构不仅会温和移动，还可能出现性质明显不同的异常区间。因此，3B 更适合被写成一个异常现象，而不是新的普适规律；首层恢复则应被写成一个有明确数据支撑的结构性发现。

# 第五章 结论与展望

## 5.1 研究结论

本文围绕 KV Cache 量化中的分析对象、校准路径与 allocator 读法三个层面，对行为导向量化问题进行了系统研究。本文最终形成的结论更强调结构与边界，而不是单点胜负。

首先，本文的证据支持将注意力行为而非单纯数值误差作为统一分析对象来重组 calibration 与 allocation 的叙述。行为导向视角的价值，不在于立即给出一个普适胜利的 loss 结论，而在于把 calibration 与 allocation 两个原本相对分离的层次放到同一解释框架下。

其次，INT8 在本文中被明确重定位为规范验证实例。它验证的是框架路径的可执行性、可审计性与系统集成能力，而不是“KL 普适优于 MSE”的强结论。这一重定位使得 INT8 获得了清晰且稳固的论文角色。

再次，在 allocator 层，本文的主要经验发现不是单一方法胜出，而是随模型家族、规模与任务而变化的分区结构。7B 呈现清晰的 aggregation-split，8B 显示 fixed-k 旧叙事开始失效，14B 强化了 heuristic / uniform 的基线地位，Mistral 提供了 auto-k 的最明确正面证据，3B 则暴露出首层瓶颈异常。这些结果共同说明 allocator 问题不应再被写成统一胜负叙事。

最后，本文将 auto-k 重新定位为框架下游的强扩展能力。它已经有跨模型的实证支持，尤其在 Mistral 上最为明确，并在 8B、14B 上进入第一竞争梯队，但尚不足以支撑普适赢家的写法。将其表述为基于敏感度画像的预算建议机制，更符合当前证据边界，也更利于后续继续推进。

## 5.2 研究局限性

当前工作仍存在若干明确局限。

第一，本文当前主线虽然已具备主文候选级支撑，但结果生产层仍未完成 clean-provenance 覆盖，因此最终胜负表、紧致数值结论和可直接发布的最终结论仍需等待后续覆盖验证。

第二，auto-k 虽已显示出较强潜力，但其显式胜利目前仍主要集中在 Mistral，其他模型上更多体现为“进入第一竞争梯队”，而不是“稳定取得最优”。因此，本文不能将 auto-k 上升为统一规律。

第三，虽然已有诊断与实验共同表明 Key 与 Value 在脆弱性上存在结构差异，但本文尚未把这一观察系统地推进到 K/V 非对称 allocator 层，因此角色级预算分配仍停留在未来工作阶段。

第四，当前 extend-task 结果的信息量并不均匀。`dureader` 与 `lcc` 可提供有效支撑，但 `trec`、`vcsum` 等任务当前更适合作为能力边界或支持性材料，而不适合作为 allocator 主结论依据。

## 5.3 未来工作展望

基于本文当前结果，后续最自然的推进方向包括以下三条。

第一，**K/V 非对称分配**。既然现有诊断已经表明 K 与 V 在敏感性上的差异并不对称，那么下一步最自然的问题就是：能否在同一框架下为 K 和 V 分配不同的精度预算，从而把角色差异真正转化为 allocator 层的策略。

第二，**质量—成本 Pareto 分析**。如果 allocator 真正要从“质量现象”升级为“预算分配方法”，就必须联合考虑质量、延迟与显存，而不是只比较任务指标。将 latency、memory 与 quality 放在同一 Pareto 面上，才可能把 allocator 写成一个更完整的方法问题。

第三，**prompt 自适应策略选择**。当前策略产物仍是静态的，但不同任务、不同输入画像很可能需要不同预算策略。若能在现有框架上进一步引入条件化选择器，那么以行为为中心的框架价值将进一步从“分析与解释”扩展到“更细粒度的动态决策”。

## 5.4 结语

总的来看，本文并未追求用更强的说法覆盖当前证据边界，而是选择在现有数据能够支撑的范围内，建立一条更稳定的研究叙述：以行为为统一分析对象，以 INT8 规范验证路径说明框架可执行性，以分区结构解释预算分配现象，并将 auto-k 写成受控且有支撑的扩展方向。这样的处理虽然放弃了某些更激进的结论，却使全文主线更加稳固，也更接近一篇能够经受答辩与评审检验的完整论文。
