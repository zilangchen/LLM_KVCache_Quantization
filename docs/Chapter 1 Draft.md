### 1.1

随着大语言模型（Large Language Model, LLM）的参数规模从数十亿增长至数千亿、上下文窗口从 4K 扩展至 128K tokens，GPT-4、LLaMA 3[1]、Qwen2.5[2] 等模型在文本生成、阅读理解和逻辑推理等任务上展现出卓越能力。长上下文处理能力的持续提升，使大语言模型逐步从短文本问答走向长文档理解、复杂推理与多轮生成等更具实际价值的应用场景。然而，随着上下文长度和服务并发需求同步上升，推理阶段的显存占用与延迟开销也快速放大，并逐渐成为制约大模型规模化部署的核心瓶颈。

现代大语言模型普遍采用 Transformer[3] 架构。在自回归解码（autoregressive decoding）过程中，模型每生成一个新的 token，都需要与全部历史 token 的键值对（Key-Value pair）进行注意力计算。为避免对历史上下文重复执行 K/V 投影与注意力累积，推理系统通常将已计算得到的键值对缓存起来，形成键值缓存（KV Cache）。这一机制是自回归解码得以高效执行的关键基础，能够将单步解码中的重复计算复杂度从 $O(n^2)$ 降低到 $O(n)$；但与此同时，缓存本身的存储开销也会随序列长度持续增长。

从存储结构上看，KV Cache 的显存占用与模型层数、每层 KV 头数、头维度、序列长度以及存储精度近似成正比，可写为

$$
M_{KV} = 2 \times L \times H_{kv} \times d_k \times S \times b,
$$

其中 $L$ 表示层数，$H_{kv}$ 表示每层 KV 头数，$d_k$ 表示头维度，$S$ 表示序列长度，$b$ 表示每个元素占用的字节数，前面的系数 2 对应 Key 和 Value 两部分存储。在长上下文和批量推理场景下，这一开销会随上下文长度与并发批次同步放大，甚至可能超过模型参数本身。vLLM[4] 等推理框架通过 PagedAttention 改善了显存碎片和缓存管理效率，但这类方法并不直接减少 KV Cache 的绝对存储量，因此无法从根本上消除长上下文部署中的容量压力。

在这一背景下，对 KV Cache 进行低成本压缩就具有了明确的工程价值。若将 FP16 精度的缓存进一步压缩为 INT8 或 INT4 表示，其显存占用理论上可分别降至原始的 $1/2$ 与 $1/4$，从而在相同硬件预算下支持更长的上下文窗口或更高的并发批次。也正因如此，KV Cache 量化已成为长上下文推理优化中的关键方向之一。然而，压缩率的提升并不自动等于模型功能稳定性的保持，尤其是在更具吸引力的低比特区间内，如何在显存收益与模型可用性之间取得平衡，构成了本文进一步需要回答的问题。

### 1.2

在上述背景下，低比特量化，尤其是 INT4 量化，具有最强的压缩吸引力。与更高精度表示相比，INT4 能够在相同硬件预算下显著降低 KV Cache 的存储开销，从而为更长上下文和更高并发提供直接空间。然而，现有面向 KV Cache 的低比特路径并不总能稳定保持模型可用性；在长上下文场景下，这类方法往往会同时表现出精确检索能力下降与语言建模质量退化等问题。这表明，对于 KV Cache 而言，“压得更低”并不自动等于“仍然可用”，低比特压缩本身已经构成一个需要被单独回答的研究问题。

现有 KV Cache 量化方法大多仍以数值重建误差作为校准目标，例如均方误差（MSE）或基于分位数的裁剪策略。这类目标隐含地假设：张量误差越小，模型功能损伤也越小。然而，在注意力计算中，这一假设并不稳固。由于 softmax 的非线性作用，微小的 Key 扰动也可能引起注意力分布的明显重排；相反，某些较大的数值误差则可能在归一化过程中被部分吸收。因此，数值误差的大小与下游功能偏移之间并不存在稳定的一一对应关系。与此同时，静态阈值也难以充分适应不同层、不同头以及不同上下文位置上的分布差异。

这意味着，KV Cache 量化中的困难并不只是一个继续调节超参数或进一步缩小重建误差的工程问题，而是与注意力结构本身的误差传播方式有关。模型真正感知到的，并不是孤立的张量数值距离，而是这些误差经过注意力计算后所诱导出的功能偏移。特别是在长上下文和低比特场景下，同一量级的数值扰动在不同层、不同头乃至不同角色上，可能对应完全不同的实际后果。也正因如此，单纯追求全局数值误差最小，并不能保证模型在推理时仍然保持稳定和可用。

由此，本文进一步提出一个更基础的问题：在 KV Cache 量化中，真正需要被保持的，是否并不是单纯的张量重建精度，而是某种更贴近模型实际功能稳定性的组织对象。换言之，问题的关键不再只是如何继续减小 MSE 或调整裁剪阈值，而是是否应当围绕一个更能反映下游推理变化的对象来重新组织校准过程。只有先把这一层问题立住，后续关于量化路径、位宽选择以及预算决策的讨论，才具有一致的分析基础。

围绕这一问题意识，下一节将对模型量化、KV Cache 量化与高效推理三条相关研究脉络作高层概述，并在此基础上于后文进一步收束为本文的核心假设与研究问题。更系统的技术基础与相关工作比较将放在第二章展开，而正式的误差传播分析与方法定义则留待第三章完成。

围绕长上下文推理与 KV Cache 压缩，现有研究大体可以从模型量化、KV Cache 专属量化和高效推理三条脉络来理解。前者为低比特表示、校准与误差控制提供了方法学基础，后两者则分别从缓存压缩与系统优化角度缓解长上下文部署压力。需要说明的是，本节只对这一研究背景作绪论层面的高层概述，不展开系统的文献比较；更详细的理论基础、代表性工作对比及本文定位将在第二章进一步展开。

在模型量化方向，已有研究围绕权重与激活的低比特表示、训练后量化以及校准策略形成了较为成熟的方法体系。以 SmoothQuant、GPTQ 和 AWQ 为代表的工作，分别从量化难度转移、近似误差补偿和激活感知保护等角度推进了大模型的压缩与部署。这些方法主要面向模型参数与激活，但其中关于量化表示、误差控制和校准流程的基本思想，也为后续 KV Cache 压缩提供了重要的方法学参考。

在 KV Cache 专属量化方向，随着长上下文推理对缓存显存的压力持续上升，研究开始直接面向缓存本身设计压缩路径。以 KIVI、KVQuant 和 ZipCache 为代表的工作，分别从 K/V 异构量化轴、极低比特量化和重要性驱动压缩等角度探索了不同的 KV Cache 压缩方案。这些研究表明，缓存压缩并不是对一般模型量化方法的简单复用，而需要结合注意力计算特征、缓存访问模式与部署约束进行专门设计。

除量化压缩外，高效推理方向也在同步推进。以 FlashAttention 和 PagedAttention 为代表的工作，分别从注意力计算访存优化和缓存管理改进入手，提高了长上下文推理的系统效率。前者主要降低注意力计算中的中间访存开销，后者则改善缓存分页与显存碎片问题。这类工作与 KV Cache 量化并非替代关系，而是从不同层面共同服务于长上下文推理的可部署性。

### 1.4

在前述研究背景、问题动机与研究现状概述的基础上，可以看到，KV Cache 量化的核心难点并不只是继续提高压缩率，而是如何用一个更贴近模型真实功能变化的视角来重新组织这一问题。真正需要回答的，不是一般意义上的“还能不能继续压缩”，而是：在长上下文和低比特约束下，是否存在一个能够同时服务校准与预算决策、并与模型实际可用性更直接相关的分析起点。

核心假设：在 KV Cache 量化中，相比直接以张量数值重建误差组织校准与预算决策，以注意力行为（attention behavior）作为统一组织对象，更贴近模型真实功能损伤，也更有可能为不同位宽下的校准路径与预算分配提供一致的判断基础。这里所说的注意力行为，在引言层面主要指注意力分布及其输出行为的整体稳定性，而不是某个单一张量距离指标。

研究问题一（RQ1）：行为引导校准是否能够形成稳定、可复现的基础保真路径？这一问题关注的是，在最保守、最易审计的位宽设置下，若以注意力行为为导向组织校准过程，能否先建立一条可靠的规范路径，作为后续更激进的低比特探索的出发点。

研究问题二（RQ2）：在低比特场景下，行为引导路径能否把由对称低比特压缩触发的显著失稳，恢复为一条可用、可审计的低比特路径？这里关心的不是某个具体实现是否在个别设置下占优，而是当压缩进入更有吸引力也更有风险的低比特区间后，这种组织方式能否把失效路径重新变成可分析、可部署的低比特实例。

研究问题三（RQ3）：行为敏感度画像能否支撑跨模型预算分配，并揭示稳定的策略适用区间结构？这一问题将视角从校准进一步推到预算决策层，关注同一组织原则能否在不同模型家族、不同规模和不同任务条件下形成具有解释性的策略分化，而不是只给出一种单点式配置。
