\chapter{绪论}

\section{研究背景}

随着大语言模型（Large Language Model, LLM）的参数规模从数十亿增长至数千亿、
上下文窗口从 4K 扩展至 128K tokens，
GPT-4、LLaMA~3~\cite{grattafiori2024llama3}、Qwen2.5~\cite{qwen2025qwen25} 等模型
在文本生成、阅读理解和逻辑推理等任务上展现出卓越能力，
但推理阶段的显存与延迟开销也随之急剧增长，成为制约大规模部署的核心瓶颈。

现代大语言模型普遍采用 Transformer~\cite{vaswani2017attention} 架构，
其自回归解码（autoregressive decoding）过程中，每生成一个新的 token 都需要与所有历史 token 的键值对（Key-Value pair）进行注意力计算。
为避免重复计算，推理系统会将已计算的键值对缓存起来，形成键值缓存（KV Cache）。
KV Cache 机制是自回归解码的核心加速手段，能够将单步解码的时间复杂度从 $O(n^2)$ 降低至 $O(n)$，
但其代价是显存占用随序列长度线性增长。

具体而言，KV Cache 的显存占用与模型层数、每层 KV 头数、头维度以及序列长度均成正比，
可表示为：
\begin{equation}
  M_{\mathrm{KV}} = 2 \times L \times H_{kv} \times d_k \times S \times b
  \label{eq:kv_memory}
\end{equation}
式中 $L$ 为层数、$H_{kv}$ 为每层 KV 头数、$d_k$ 为头维度、$S$ 为序列长度，
$b$ 为每个元素占用的字节数（FP16 下为 2），因子 2 对应 Key 和 Value 两个矩阵。

在长上下文、批量推理场景下，
KV Cache 的总显存消耗可轻易超过模型参数本身（具体推导与数值示例见第~\ref{sec:ch2-kv-memory}~节）。
vLLM~\cite{kwon2023vllm} 等推理框架通过 PagedAttention 技术改善了显存碎片问题，
但并未从根本上减少 KV Cache 的绝对存储量。

量化（quantization）技术是降低 KV Cache 显存占用的重要手段。
通过将 FP16 精度的键值对压缩为 INT8 甚至 INT4 表示，可以将 KV Cache 的显存分别降至原始的 $1/2$ 和 $1/4$，
从而在有限的 GPU 显存中支持更长的上下文或更大的并发批次。

\section{问题定义与研究动机}

在上述背景下，低比特（INT4）量化是最具吸引力的压缩目标，
理论上可将 KV Cache 压缩至 FP16 的 $1/4$，
在相同硬件预算下支持 4 倍的上下文长度或并发批次。
然而，现有以数值重建误差为校准目标的 INT4 量化方法在长上下文场景下表现急剧恶化：
Needle 检索通过率可从 100\% 降至 0\%，困惑度退化超过 15\%。
对称 INT4 量化的模型在长上下文下
不仅无法完成精确检索任务，其语言建模能力也发生灾难性崩溃。
传统校准目标（MSE、percentile）未能充分揭示这种失效，
数值重建误差看似合理，注意力行为却可能完全崩溃。
这一矛盾促使本文从注意力行为保真的角度重新审视校准目标：
在不同 bit-width 下，哪些行为应当被优先保持？
数值代理何时足够？何时必须直接对齐行为？

现有 KV Cache 量化方法主要采用均方误差（MSE）或百分位裁剪（percentile clipping）等数值重建目标。
两者的共同盲点在于：softmax 的非线性使得数值误差大小与注意力分布偏移不存在单调关系，
微小的 Key 扰动可能剧烈改变注意力分布，较大的误差反而可能被归一化效应吸收；
静态阈值亦无法自适应不同层、不同头的分布差异
（详细机制分析见第~\ref{sec:ch2-quant-softmax}~节）。

本文采用注意力行为保真视角：KV Cache 量化的合适衡量对象
不是存储表示的数值误差，而是量化前后注意力行为
（attention behavior）的分布差异。本文将这一视角命名为
\textbf{行为对齐}（Behavior Alignment, BA）。
若将 FP16 精度下的注意力权重分布记为 $\bm{p}$，
量化后的注意力权重分布记为 $\bm{q}$，
则行为偏移可以用 Kullback--Leibler 散度衡量：
\begin{equation}
  D_{\KL}(\bm{p} \| \bm{q}) = \sum_{i} p_i \log \frac{p_i}{q_i}
  \label{eq:kl_divergence}
\end{equation}
其中，$D_{\KL}$ 为 Kullback--Leibler 散度，衡量量化引入的注意力行为偏移程度。

本文围绕校准目标、低比特失效模式与部署效率三个问题展开
（详见第~\ref{sec:ch1-contribution}~节）。

\section{国内外研究现状}

KV Cache 量化建立在模型量化、KV 专属压缩与高效注意力计算三大方向之上。

\subsection{模型量化}

模型量化通过降低数值精度压缩模型体积、加速推理，
代表工作包括权重-激活联合量化 SmoothQuant~\cite{xiao2023smoothquant}、
训练后权重量化 GPTQ~\cite{frantar2023gptq} 与激活感知权重量化 AWQ~\cite{lin2024awq}。
这些方法主要面向权重，但其量化理论和误差分析为 KV Cache 量化提供了方法学参考。

\subsection{KV Cache 量化}

KIVI~\cite{liu2024kivi} 揭示了 Key 与 Value 在量化敏感性上的不对称特征，
提出逐通道 Key + 逐 token Value 的非对称量化策略；
KVQuant~\cite{hooper2024kvquant} 探索了极低比特非均匀量化方案；
ZipCache~\cite{he2024zipcache} 基于 token 重要性实现混合精度 KV Cache 压缩。
然而上述工作的校准目标仍停留在数值层面，
K/V 各自的量化敏感性边界及其与 GQA 架构的关系亦缺乏系统性消融。

\subsection{高效注意力计算}

FlashAttention~\cite{dao2022flashattention,dao2024flashattention2} 通过分块计算与内核融合
降低了注意力计算的访存开销，但主要面向 FP16 精度；
vLLM~\cite{kwon2023vllm} 的 PagedAttention 改善了 KV Cache 显存碎片。
本文基于 Triton~\cite{tillet2019triton} 实现 INT8/INT4 融合量化解码注意力核函数，
将反量化与注意力计算整合在单个 GPU 核函数中。

更详细的工作对比与技术分析见第~\ref{chap:related-work}~章。

\section{本文研究内容与主要贡献}
\label{sec:ch1-contribution}

基于上述背景，本文提出三个递进的研究问题：

\textbf{研究问题一：KV Cache 量化应优化什么目标？其有效性是否随模型规模变化？}
现有方法以数值重建误差（如 MSE、absmax percentile）为校准目标，
但数值层面的最优未必对应推理行为的保真。
是否存在一个更贴近下游推理行为的优化目标？
当模型规模增大、GQA 头数 $H_{kv}$ 改变时，
数值代理与行为对齐之间的差异是否保持一致？

\textbf{研究问题二：低比特量化为何失效？失效模式是否与 $H_{kv}$ 结构性关联？}
对称 INT4 在 Needle 检索上降至 0\%，PPL 退化超过 15\%。
该失效是全局性的还是存在可定位的结构性根因？
量化噪声在 Key 和 Value 之间如何分配？
不同 GQA 配置（$H_{kv}=2,4,8$）下的退化模式是否呈现结构性关联？

\textbf{研究问题三：诊断结论能否导出有效的低比特设计，并伴随可部署的效率收益？}
如果低比特失效有可定位的根因，
是否可以基于诊断结论设计差异化的量化策略，
在保持压缩率的同时恢复检索能力？
在部署层面，融合量化解码核函数的效率增益
是否也依赖 $(H_{kv}, \text{seq\_len})$ 的组合，
从而为量化方案的部署选型提供依据？

围绕以上三个问题，本文给出三个主要贡献。

\textbf{贡献一：attention-KL 诊断透镜与 Key 主导退化机制。}
本文提出以 attention-KL 作为量化前后注意力行为的统一诊断度量，
在四个主模型（Qwen2.5-1.5B/7B/14B、LLaMA-3.1-8B）及 Mistral-7B 对照上
首次用受控 K/V 消融定量刻画 INT4 KV Cache 的 Key 主导退化机制。
在 Qwen2.5-14B 上，保持 Key 为 FP16 可恢复 93\% 的 PPL 退化，
保持 Value 为 FP16 仅恢复 64\%；
该比例跨 K-only/V-only 消融在四个主模型上方向一致。
该诊断自然推导出 per-channel Key + per-token Value 非对称量化格式，
为 KIVI~\cite{liu2024kivi} 从工程经验独立提出的同一格式
提供了首个 attention-KL 视角的机制性解释——
KIVI 的格式从"经验碰巧有用"升级为"诊断推论下的必然选择"。
校准目标层面，本文首次系统报告 attention-KL 与 MSE 的选择
在 bit-width 与模型规模上呈现"趋同-分化-再收敛"经验规律：
INT8 下两目标在鲁棒选择协议（clip-rate $\leq 1\%$）的参数空间约束下
收敛到同一 $(\text{clip\_percentile}, \text{group\_size})$ 并产生逐位一致的校准产物；
INT4 下在 1.5B（$H_{kv}=2$）分化、在 7B（$H_{kv}=4$）再次收敛
（详见第~\ref{sec:exp-main}~节）。

\textbf{贡献二：INT4-RoleAlign---可复现可审计的离线校准接口。}
在上述非对称格式上，本文以离线 attention-KL 搜索
取代 KIVI 的运行时 absmax/min 统计量，实例化 INT4-RoleAlign，
把量化参数固化为可审计的校准产物 JSON——
这为合规场景（金融、医疗）提供了量化过程可追溯的部署接口。
该实例化在四个主模型上 Needle 检索通过率一致保持 100\%，
KV Cache 压缩约 73\%，
PPL 退化随模型规模减弱（1.5B: 13.7\%，7B: 6.1\%，8B: 2.4\%），
并在 LongBench 合成评分上相对同格式 KIVI-style 基线
提供小幅边际提升（1.5B: $+0.09$，7B: $+0.06$）。
Qwen2.5-14B 作为外部效度锚点进一步验证 Needle 在 4K--32K 全通过、
RULER 在 4K--16K 达到 96.6\%--98.5\%
（14B PPL 在 32K prefix 上评测，与其他模型的完整 test split
采用不同协议，作为外部效度观察而非跨模型单调性证据）。
INT4-RoleAlign 配置下揭示的 Needle 与 PPL 解耦现象
（同一配置可保持 Needle 100\% 的同时伴随可观 PPL 退化）
说明低比特量化的评价不能仅依赖 PPL，
这一发现对后续 KV 量化方法的评测设计具有方法论意义
（详见第~\ref{sec:exp-int4}~节）。

\textbf{贡献三：$(H_{kv}, \text{seq\_len})$ 空间中的融合核效率相位图与分层部署策略。}
本文实现了 Triton 融合量化解码注意力核，
并在 $(H_{kv}, \text{seq\_len})$ 二维空间给出清晰的效率相位图。
$H_{kv}=8$ 在 32K 序列下相对同配置 INT4 参考实现降低解码延迟约 40\%；
8B（$H_{kv}=8$, LLaMA）与 14B（$H_{kv}=8$, Qwen）跨越 1.75$\times$ 参数量差距
与不同模型族，在短序列效率特征上高度一致，
支持 $H_{kv}$（而非模型规模）为融合核效率的结构性驱动变量。
基于该相位图，本文给出 NVIDIA H20 平台的分层部署策略：
$H_{kv}=8$ 在序列长度 $\geq$ 8K 时优先使用融合核；
$H_{kv}=4$ 在序列长度 $\geq$ 32K 出现边际收益；
$H_{kv}=2$ 推荐使用 SDPA。
该"相位图"评估范式为未来的 efficient inference kernel 工作
提供了一个可推广的汇报模板——
kernel 的收益应按场景相位图呈现，而非单一平均加速。

综上，attention-KL 提供了 GQA 架构下 KV Cache 量化的统一校准与诊断原则，
$H_{kv}$ 则是与低比特失效模式、K/V 角色分化和融合解码效率
一致性变化的结构性相关变量（非严格因果识别，详见第~\ref{subsec:exp-threats}~节效度威胁声明）。

\section{论文组织结构}

本文共分五章。
第一章阐述研究背景与动机，综述国内外研究现状，
介绍研究内容与主要贡献。
第二章介绍 Transformer 注意力机制、量化理论基础及 KV Cache 量化相关工作。
第三章详述行为对齐视角下的两级设计：
INT8 对称路径（KL 校准 + 自适应保护）
与在 KIVI-style 非对称格式上的 behavior-guided 实例化 RoleAlign
（per-channel Key + per-token Value 格式 + BA percentile 离线搜索），
以及 Triton 融合量化解码核函数的系统实现。
第四章按主要贡献对应的证据组织实验：
校准目标有效性的 bit-width 与规模依赖、
Key 主导退化的结构性诊断与低比特实例化验证、
$(H_{kv}, \text{seq\_len})$ 空间中的融合核效率分析与部署建议，
最后进行综合讨论。
第五章总结全文核心观察，回扣引言问题，
分析局限性并展望未来方向。
