\chapter{绪论}

\section{研究背景}

随着大语言模型（Large Language Model, LLM）的参数规模从数十亿增长至数千亿、
上下文窗口从 4K 扩展至 128K tokens，
GPT-4、LLaMA~3~\cite{grattafiori2024llama3}、Qwen2.5~\cite{qwen2025qwen25} 等模型
在文本生成、阅读理解和逻辑推理等任务上展现出卓越能力，
但推理阶段的显存与延迟开销也随之急剧增长，成为制约大规模部署的核心瓶颈。

现代大语言模型普遍采用 Transformer~\cite{vaswani2017attention} 架构，
其自回归解码（autoregressive decoding）过程中，每生成一个新的 token 都需要与所有历史 token 的键值对（Key-Value pair）进行注意力计算。
为避免重复计算，推理系统会将已计算的键值对缓存起来，形成键值缓存（KV Cache）。
KV Cache 机制是自回归解码的核心加速手段，能够将单步解码的时间复杂度从 $O(n^2)$ 降低至 $O(n)$，
但其代价是显存占用随序列长度线性增长。

具体而言，对于一个具有 $L$ 层 Transformer、每层 $h_{KV}$ 个键值头（KV head）、头维度为 $d_h$ 的模型，
在序列长度为 $s$ 时，KV Cache 的显存占用可表示为：
\begin{equation}
  M_{\text{KV}} = 2 \times L \times h_{KV} \times d_h \times s \times b
  \label{eq:kv_memory}
\end{equation}
其中，因子 $2$ 对应键（Key）和值（Value）两个矩阵，$b$ 为每个元素占用的字节数（FP16 下 $b=2$）。

以 Qwen2.5-7B-Instruct 模型（28 层，GQA~\cite{ainslie2023gqa}，4 个 KV 头，头维度 128）为例，
在 FP16 精度、32K 上下文下，单个请求的 KV Cache 即占用约 1.75~GB 显存。
在批量推理场景下，KV Cache 的总显存消耗将随批大小成倍增长，可轻易超过模型参数本身的显存占用。
vLLM~\cite{kwon2023vllm} 等推理框架通过 PagedAttention 技术改善了显存碎片问题，但并未从根本上减少 KV Cache 的绝对存储量。

量化（quantization）技术是降低 KV Cache 显存占用的重要手段。
通过将 FP16 精度的键值对压缩为 INT8 甚至 INT4 表示，可以将 KV Cache 的显存分别降至原始的 $1/2$ 和 $1/4$，
从而在有限的 GPU 显存中支持更长的上下文或更大的并发批次。

\section{问题定义与研究动机}

在上述背景下，低比特（INT4）量化是最具吸引力的压缩目标，
理论上可将 KV Cache 压缩至 FP16 的 $1/4$，
在相同硬件预算下支持 4 倍的上下文长度或并发批次。
然而，现有的 INT4 量化方法在长上下文场景下表现急剧恶化：
Needle 检索通过率可从 100\% 降至 0\%，困惑度退化超过 15\%。
对称 INT4 量化的模型在长上下文下
不仅无法完成精确检索任务，其语言建模能力也发生灾难性崩溃。
传统校准目标（MSE、percentile）未能充分揭示这种失效，
数值重建误差看似合理，注意力行为却可能完全崩溃。
这一矛盾促使我们从注意力行为保真的角度重新审视校准目标：
在不同 bit-width 下，哪些行为应当被优先保持？
数值代理何时足够？何时必须直接对齐行为？

现有 KV Cache 量化方法主要采用两类校准策略：
均方误差（Mean Squared Error, MSE）和百分位裁剪（percentile clipping）。
MSE 校准通过最小化量化前后张量的逐元素数值误差来确定量化参数；
百分位裁剪则通过截断分布尾部的异常值来缩小量化范围，以降低大多数数值的量化误差。

但这两种策略有一个共同的盲点：它们优化的是数值层面的重建误差，
而数值误差小并不意味着模型行为保真。
softmax 的非线性使得误差大小与注意力偏移之间不存在单调关系：
微小的 Key 扰动可能剧烈改变注意力分布，
较大的误差反而可能被归一化效应吸收。
百分位裁剪同样缺乏与下游注意力行为的关联，
对不同层、不同头的分布差异无法自适应。

本文采用注意力行为保真视角：KV Cache 量化的合适衡量对象
不是存储表示的数值误差，而是量化前后注意力行为
（attention behavior）的分布差异。
若将 FP16 精度下的注意力权重分布记为 $\bm{p}$，
量化后的注意力权重分布记为 $\bm{q}$，
则行为偏移可以用 Kullback-Leibler 散度衡量：
\begin{equation}
  D_{\KL}(\bm{p} \| \bm{q}) = \sum_{i} p_i \log \frac{p_i}{q_i}
  \label{eq:kl_divergence}
\end{equation}
其中，$D_{\KL}$ 为 Kullback-Leibler 散度，衡量量化引入的注意力行为偏移程度。

基于这一视角，本文在 Qwen2.5-1.5B/7B/8B/14B 上、
在贪心解码与 robust selection（$\max~\text{clip rate} \leq 1\%$）
约束下系统研究两个核心问题：
(1) 校准目标的有效性是否依赖 bit-width？
即数值代理（MSE/percentile）与行为对齐（attention-KL）
在 INT8 与 INT4 下是否给出等价的量化参数选择；
(2) 当 bit-width 下降时哪些行为会优先退化，
诊断结果如何指导实用的低比特配置。
我们将这一视角命名为"行为对齐"（Behavior Alignment）。

\section{国内外研究现状}

KV Cache 量化的研究建立在模型量化和高效注意力计算两大技术方向之上，
以下从三个方面简述相关研究现状，更详细的分析将在第二章展开。

\subsection{模型量化}

模型量化通过降低数值精度来压缩模型体积、加速推理。
代表性工作包括面向权重-激活联合量化的 SmoothQuant~\cite{xiao2023smoothquant}、
基于近似二阶信息的训练后量化方法 GPTQ~\cite{frantar2023gptq}、
以及激活感知权重量化策略 AWQ~\cite{lin2024awq}。
上述方法主要面向模型权重量化，与本文关注的 KV Cache 量化在目标和技术路径上有所不同，
但其量化理论和误差分析方法为本文提供了重要参考。
更详细的技术分析见第二章。

\subsection{KV Cache 量化}

KV Cache 量化是近年来的研究热点。
KIVI~\cite{liu2024kivi} 揭示了 Key 和 Value 在量化敏感性上的不对称特征，
提出逐通道 Key + 逐 token Value 的非对称量化策略；
KVQuant~\cite{hooper2024kvquant} 探索了面向极低比特的非均匀量化方案；
ZipCache~\cite{he2024zipcache} 则基于 token 重要性实现了混合精度 KV Cache 压缩。
上述方法在量化策略和压缩技术上取得了重要进展，
但其校准仍依赖于数值误差指标，未从注意力行为保真的角度建立校准目标；
K/V 各自的量化敏感性边界及其与模型架构的关系也缺乏系统性消融验证。
更详细的工作对比见第二章。

\subsection{高效注意力计算}

FlashAttention~\cite{dao2022flashattention,dao2024flashattention2} 通过分块计算与内核融合
显著降低了注意力计算的访存开销，但其主要面向 FP16 精度的标准注意力，
尚未充分支持量化 KV Cache 的在线反量化融合。
本文基于 Triton~\cite{tillet2019triton} 实现了 INT8/INT4 融合量化解码注意力核函数，
将反量化与注意力计算整合在单个 GPU 核函数中。
更详细的技术讨论见第二章和第三章。

\section{本文研究内容与主要贡献}

基于上述背景，本文提出三个递进的研究问题：

\begin{description}
  \item[\textbf{RQ1}] \emph{KV Cache 量化应优化什么目标？}
    现有方法以数值重建误差（如 MSE、absmax percentile）为校准目标，
    但数值层面的最优未必对应推理行为的保真。
    是否存在一个更贴近下游推理行为的优化目标？
  \item[\textbf{RQ2}] \emph{低比特量化为何系统性失效？}
    对称 INT4 在 Needle 检索上降至 0\%，PPL 退化超过 15\%。
    该失效是全局性的还是存在可定位的结构性根因？
    量化噪声在 Key 和 Value 之间如何分配？
    不同模型架构（特别是不同 GQA 头数 $H_{kv}$）对量化容忍度有何影响？
    进一步，用于补偿量化偏移的逐头温度校正机制 $\tau^{-1}$（inverse temperature）
    是否也会随 $H_{kv}$ 呈现架构依赖，
    从而为"量化$\times$架构"的交互提供新的诊断视角？
  \item[\textbf{RQ3}] \emph{诊断结论能否导出有效的低比特设计？}
    如果低比特失效有可定位的根因，
    是否可以基于诊断结论设计差异化的量化策略，
    在保持压缩率的同时恢复检索能力？
    其代价边界（PPL 退化、延迟变化）如何？
\end{description}

围绕以上三个问题，本文给出三个主要贡献和一个经验观察。

\textbf{贡献一：bit-width 依赖的校准目标有效性。}
我们系统比较 attention-KL 与 MSE 两种校准目标在 INT8 与 INT4 下的作用。
在 INT8 结合 robust selection（$\max~\text{clip rate} \leq 1\%$）约束下，
两种目标对 $(\text{clip\_percentile}, \text{group\_size})$ 的选择
完全趋同，产生 448 个 per-group scale 在 float32 下逐位一致的校准产物；
而在 INT4 下，两种目标分别选择 $\text{clip\_percentile}=99.5$ 与 $99.0$，
所有 scale 元素不同，中位相对差异 12.35\%，
逐头温度校正 $\tau^{-1}$ 的最大差异达 1.5。
这一对比表明：在低比特场景下数值代理不等价于行为对齐，
校准目标的有效性呈 bit-width 依赖。

\textbf{贡献二：受控诊断——Key 主导退化与 retrieval/PPL 解耦。}
通过 K-only 与 V-only 因子分解，我们在三个模型族上一致观察到
INT4 KV Cache 退化由 Key 侧主导：
K@INT4 / V@INT16 配置下 1.5B 模型 PPL 崩溃至 $\sim 1291$，
而 K@INT16 / V@INT4 配置 PPL 与 FP16 相当。
通过 Needle retrieval 与 PPL 的并行评测，
我们观察到二者在若干配置下显著解耦——
同一 INT4-RoleAlign 配置可以在 Needle 上保持 100\% pass rate
的同时 PPL 退化 13.7\%（1.5B 最坏情况）。
这一组诊断说明：评价低比特 KV Cache 量化不能仅用 PPL，
行为层面的指标是必要补充。

\textbf{贡献三：KIVI-style 格式上的行为引导实例化 RoleAlign。}
基于上述诊断，我们在 KIVI~\cite{liu2024kivi} 提出的
per-channel Key + per-token Value 非对称格式上，
通过离线 attention-KL 搜索确定每层的 BA percentile 以替代
运行时 absmax/min，得到行为引导的实例化 INT4-RoleAlign。
在 Qwen2.5-1.5B、Qwen2.5-7B、LLaMA-3.1-8B 与 Qwen2.5-14B
四个模型上，INT4-RoleAlign 在 4K--32K 上下文范围内保持
Needle 检索 100\%，KV Cache 压缩约 73\%，
PPL 退化随模型规模增大而减弱（1.5B: 13.7\%，7B: 6.1\%，
8B: 2.4\%，14B: 6.1\%）；在 LLaMA-3.1-8B 的 64K 上下文
极限长度下，INT4-RoleAlign 与 FP16 在同一位置出现相同的
long-context 退化模式，说明量化本身没有引入新的失败边界。
相关 Triton 融合量化解码核函数为上述实例化提供系统支撑，
统计检验采用 Bootstrap CI + BH-FDR 框架（$\alpha=0.05$）。

\textbf{经验观察：$\tau^{-1}$ 的 GQA 尺度依赖。}
在上述实验之外，我们还观察到逐头温度校正
$\tau^{-1}$ 的效果与 GQA 头数 $H_{kv}$ 呈反向依赖：
$H_{kv}=2$（Qwen2.5-1.5B）时 $\tau^{-1}$ 略改善 PPL 约 1.6\%，
$H_{kv} \geq 4$（Qwen2.5-7B、LLaMA-3.1-8B、Qwen2.5-14B）时反而
恶化 1.8\%--6.0\%。这一模式与 GQA 下头间共享导致的噪声稀释效应
$\sigma_{\text{eff}} \propto \sigma / \sqrt{N_{\text{rep}}}$
在 query 交互近似独立的简化假设下一致
（详见第~\ref{sec:ch3-invtau}~节的假设边界讨论）。
基于这一观察，本文默认关闭 $\tau^{-1}$，
仅在 $H_{kv}=2$ 的小模型部署场景下作为可选启发式提供。

综上，本文的核心主张是：在本文协议下，
KV Cache 量化校准目标的有效性呈 bit-width 依赖，
数值代理在 INT8 下足以近似行为对齐但在 INT4 下不再等价；
基于这一视角，我们在 INT8 上建立稳定校准参考，
在 INT4 上通过受控诊断识别 Key 主导退化与 retrieval/PPL 解耦，
并在 KIVI-style 非对称格式上给出一个行为引导的实例化 RoleAlign，
以约 73\% KV Cache 压缩率保持四个模型的 Needle 检索能力，
同时清晰界定 PPL 退化与延迟代价的边界。

\section{论文组织结构}

本文共分五章。
第一章阐述研究背景与动机，综述国内外研究现状，
介绍研究内容与主要贡献。
第二章介绍 Transformer 注意力机制、量化理论基础及 KV Cache 量化相关工作。
第三章详述行为对齐视角下的两级设计：
INT8 对称路径（KL 校准 + 自适应保护）
与在 KIVI-style 非对称格式上的 behavior-guided 实例化 RoleAlign
（per-channel Key + per-token Value 格式 + BA percentile 离线搜索），
以及 Triton 融合量化解码核函数的系统实现。
第四章按主要贡献对应的证据组织实验：
INT8 与 INT4 下 KL/MSE 校准目标的 bit-width 依赖对比、
Key 主导退化与 retrieval/PPL 解耦的受控诊断、
KIVI-style 格式上的 RoleAlign 实例化验证与能力边界分析，
最后进行综合讨论。
第五章总结全文核心观察，回扣引言问题，
分析局限性并展望未来方向。
