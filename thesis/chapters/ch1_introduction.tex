\chapter{绪论}

\section{研究背景}

随着大语言模型（Large Language Model, LLM）的参数规模从数十亿增长至数千亿、
上下文窗口从 4K 扩展至 128K tokens，
GPT-4、LLaMA~3~\cite{grattafiori2024llama3}、Qwen2.5~\cite{qwen2025qwen25} 等模型
在文本生成、阅读理解和逻辑推理等任务上展现出卓越能力，
但推理阶段的显存与延迟开销也随之急剧增长，成为制约大规模部署的核心瓶颈。

现代大语言模型普遍采用 Transformer~\cite{vaswani2017attention} 架构，
其自回归解码（autoregressive decoding）过程中，每生成一个新的 token 都需要与所有历史 token 的键值对（Key-Value pair）进行注意力计算。
为避免重复计算，推理系统会将已计算的键值对缓存起来，形成键值缓存（KV Cache）。
KV Cache 机制是自回归解码的核心加速手段，能够将单步解码的时间复杂度从 $O(n^2)$ 降低至 $O(n)$，
但其代价是显存占用随序列长度线性增长。

具体而言，对于一个具有 $L$ 层 Transformer、每层 $h_{KV}$ 个键值头（KV head）、头维度为 $d_h$ 的模型，
在序列长度为 $s$ 时，KV Cache 的显存占用可表示为：
\begin{equation}
  M_{\text{KV}} = 2 \times L \times h_{KV} \times d_h \times s \times b
  \label{eq:kv_memory}
\end{equation}
其中，因子 $2$ 对应键（Key）和值（Value）两个矩阵，$b$ 为每个元素占用的字节数（FP16 下 $b=2$）。

以 Qwen2.5-7B-Instruct 模型（28 层，GQA~\cite{ainslie2023gqa}，4 个 KV 头，头维度 128）为例，
在 FP16 精度、32K 上下文下，单个请求的 KV Cache 即占用约 1.75~GB 显存。
在批量推理场景下，KV Cache 的总显存消耗将随批大小成倍增长，可轻易超过模型参数本身的显存占用。
vLLM~\cite{kwon2023vllm} 等推理框架通过 PagedAttention 技术改善了显存碎片问题，但并未从根本上减少 KV Cache 的绝对存储量。

量化（quantization）技术是降低 KV Cache 显存占用的重要手段。
通过将 FP16 精度的键值对压缩为 INT8 甚至 INT4 表示，可以将 KV Cache 的显存分别降至原始的 $1/2$ 和 $1/4$，
从而在有限的 GPU 显存中支持更长的上下文或更大的并发批次。

\section{问题定义与研究动机}

现有的 KV Cache 量化方法在校准策略（calibration strategy）的选择上，
通常采用基于均方误差（Mean Squared Error, MSE）或百分位裁剪（percentile clipping）的方案。
MSE 校准通过最小化量化前后张量的逐元素数值误差来确定量化参数；
百分位裁剪则通过截断分布尾部的异常值来缩小量化范围，以降低大多数数值的量化误差。

然而，上述方法存在本质局限性。MSE 校准优化的是数值层面的重建误差，
但数值误差的最小化并不等同于模型行为的保真。
在注意力机制中，键和值经过缩放点积（scaled dot-product）运算后通过 softmax 函数映射为概率分布。
softmax 函数的非线性特性意味着，即使键矩阵中微小的数值偏差，
也可能在某些注意力模式下被放大，导致注意力权重分布发生显著改变；
反之，某些看似较大的数值误差可能因 softmax 的归一化效应而对最终分布影响甚微。
百分位裁剪策略同样面临问题：其裁剪阈值的选择缺乏与下游注意力行为的关联，
对不同层、不同头的特征分布差异缺乏自适应能力。

本文的核心洞察在于：KV Cache 量化的最终目标不是最小化存储表示的数值误差，
而是保持量化前后注意力行为（attention behavior）的一致性。
具体而言，若将 FP16 精度下的注意力权重分布记为 $\bm{p}$，
量化后的注意力权重分布记为 $\bm{q}$，
则量化质量应当以两者之间的分布差异来衡量：
\begin{equation}
  D_{\KL}(\bm{p} \| \bm{q}) = \sum_{i} p_i \log \frac{p_i}{q_i}
  \label{eq:kl_divergence}
\end{equation}
其中，$D_{\KL}$ 为 Kullback-Leibler 散度，衡量量化引入的注意力行为偏移程度。

基于上述洞察，本文提出以注意力权重分布的 KL 散度作为校准目标函数，
直接优化量化参数使得注意力行为尽可能保持不变。
这一"行为对齐"（Behavior Alignment）的思路将校准过程从数值优化层面提升到语义保真层面，
为 KV Cache 量化提供了新的理论视角和技术路径。

\section{国内外研究现状}

KV Cache 量化的研究建立在模型量化和高效注意力计算两大技术方向之上，
以下从三个方面简述相关研究现状，更详细的分析将在第二章展开。

\subsection{模型量化}

模型量化通过降低数值精度来压缩模型体积、加速推理。
代表性工作包括面向权重-激活联合量化的 SmoothQuant~\cite{xiao2023smoothquant}、
基于近似二阶信息的训练后量化方法 GPTQ~\cite{frantar2023gptq}、
以及激活感知权重量化策略 AWQ~\cite{lin2024awq}。
上述方法主要面向模型权重量化，与本文关注的 KV Cache 量化在目标和技术路径上有所不同，
但其量化理论和误差分析方法为本文提供了重要参考。
更详细的技术分析见第二章。

\subsection{KV Cache 量化}

KV Cache 量化是近年来的研究热点。
KIVI~\cite{liu2024kivi} 揭示了 Key 和 Value 在量化敏感性上的不对称特征，
提出逐通道 Key + 逐 token Value 的非对称量化策略；
KVQuant~\cite{hooper2024kvquant} 探索了面向极低比特的非均匀量化方案；
ZipCache~\cite{he2024zipcache} 则基于 token 重要性实现了混合精度 KV Cache 压缩。
上述方法在量化策略和压缩技术上取得了重要进展，
但其校准仍依赖于数值误差指标，未从注意力行为保真的角度建立校准目标；
K/V 各自的量化敏感性边界及其与模型架构的关系也缺乏系统性消融验证。
更详细的工作对比见第二章。

\subsection{高效注意力计算}

FlashAttention~\cite{dao2022flashattention,dao2024flashattention2} 通过分块计算与内核融合
显著降低了注意力计算的访存开销，但其主要面向 FP16 精度的标准注意力，
尚未充分支持量化 KV Cache 的在线反量化融合。
本文基于 Triton~\cite{tillet2019triton} 实现了 INT8/INT4 融合量化解码注意力核函数，
将反量化与注意力计算整合在单个 GPU 核函数中。
更详细的技术讨论见第二章和第三章。

\section{本文研究内容与主要贡献}

本文面向大语言模型长上下文推理的效率瓶颈，
围绕 KV Cache 量化的校准策略、融合计算、K/V 精度诊断和实证评估展开研究，主要贡献如下：

\textbf{贡献一：行为对齐校准框架。}
提出以注意力权重分布的 KL 散度作为校准目标函数，
结合自适应 Scale 保护机制保障长上下文鲁棒性，
将校准从数值优化层面提升到语义保真层面。
KL 散度同时用作诊断透镜，揭示不同量化配置下的注意力行为失效模式，
引导精度分配设计决策。
逐头温度校正（$\tau^{-1}$）作为框架的可选增强机制保留。

\textbf{贡献二：Triton 融合量化解码注意力核函数。}
设计并实现支持 INT8/INT4 两种精度的 Triton 融合核函数，
在单个 GPU kernel 调用中完成在线反量化与注意力计算，
消除两阶段方案中中间张量的显存读写开销，原生支持 GQA 机制。

\textbf{贡献三：诊断驱动的低比特扩展路径。}
KL 诊断透镜揭示对称 per-group INT4 的系统性失效模式，
K/V 消融实验在四个模型（三个模型族）上一致确认 Key 量化为退化主导因素，
且该敏感性呈明显的 GQA 架构依赖特征。
基于此诊断，提出两条实用低比特路径：
MixedKV（K@INT8+V@INT4）作为诊断驱动的实用扩展，
以及 INT4-RoleAlign——吸收 per-channel K + per-token V 非对称轴设计，
通过行为对齐校准搜索最优 K/V percentile 参数。
INT4-RoleAlign 在 Qwen2.5-1.5B 上将对称 INT4 的 Needle 通过率从 0\% 恢复至 100\%，
PPL 退化从 KIVI-style 的 12\% 降至 1.2\%；
在三个模型（两个模型族）上呈现竞争性。
chunk\_size 敏感性分析（Qwen2.5-1.5B）表明 BA 校准在 fully quantized 协议下的鲁棒性（Qwen2.5-1.5B 单模型验证）。

\textbf{贡献四：多维度实证验证与系统发现。}
INT8 主线在四个模型（三个模型族）、五个评测基准上进行全矩阵验证，
INT4-RoleAlign 在三个模型（两个模型族）上进行跨架构验证。
采用 Bootstrap 置信区间与 BH-FDR 校正的统计框架，
发现 K/V 敏感性的 GQA 架构依赖特征，
并诚实报告三项未达标指标及其详细归因分析。

本文的技术路线遵循递进式扩展：INT8 行为对齐量化作为经过完整验证的规范实例（canonical instance），MixedKV（K@INT8+V@INT4）作为实用性扩展，INT4-RoleAlign 作为低比特修复扩展。三者共享同一诊断框架，但证据成熟度依次递减。

\section{论文组织结构}

本文共分五章。
第一章阐述研究背景与动机，综述国内外研究现状，介绍研究内容与主要贡献。
第二章介绍 Transformer 注意力机制、量化理论基础及 KV Cache 量化方法的相关工作。
第三章详述行为对齐校准框架的设计思路与数学推导，
包括 KL 散度校准方法、自适应保护机制、角色感知非对称量化的方法设计，
以及 Triton 融合核函数的实现和系统集成方案。
第四章呈现多模型、多维度的实验结果与统计分析，
涵盖 INT8 主线验证、诊断驱动的 INT4 边界分析、K/V 精度消融、
角色感知非对称量化评估、与 KIVI-style 的正面对比、
量化评估粒度鲁棒性分析以及综合讨论。
第五章总结全文，分析局限性，并展望未来研究方向。
