\chapter{绪论}

\section{研究背景}

随着大语言模型（Large Language Model, LLM）的参数规模从数十亿增长至数千亿、
上下文窗口从 4K 扩展至 128K tokens，
GPT-4、LLaMA~3~\cite{grattafiori2024llama3}、Qwen2.5~\cite{qwen2025qwen25} 等模型
在文本生成、阅读理解和逻辑推理等任务上展现出卓越能力，
但推理阶段的显存与延迟开销也随之急剧增长，成为制约大规模部署的核心瓶颈。

现代大语言模型普遍采用 Transformer~\cite{vaswani2017attention} 架构，
其自回归解码（autoregressive decoding）过程中，每生成一个新的 token 都需要与所有历史 token 的键值对（Key-Value pair）进行注意力计算。
为避免重复计算，推理系统会将已计算的键值对缓存起来，形成键值缓存（KV Cache）。
KV Cache 机制是自回归解码的核心加速手段，能够将单步解码的时间复杂度从 $O(n^2)$ 降低至 $O(n)$，
但其代价是显存占用随序列长度线性增长。

具体而言，对于一个具有 $L$ 层 Transformer、每层 $h_{KV}$ 个键值头（KV head）、头维度为 $d_h$ 的模型，
在序列长度为 $s$ 时，KV Cache 的显存占用可表示为：
\begin{equation}
  M_{\text{KV}} = 2 \times L \times h_{KV} \times d_h \times s \times b
  \label{eq:kv_memory}
\end{equation}
其中，因子 $2$ 对应键（Key）和值（Value）两个矩阵，$b$ 为每个元素占用的字节数（FP16 下 $b=2$）。

以 Qwen2.5-7B-Instruct 模型（28 层，GQA~\cite{ainslie2023gqa}，4 个 KV 头，头维度 128）为例，
在 FP16 精度、32K 上下文下，单个请求的 KV Cache 即占用约 1.75~GB 显存。
在批量推理场景下，KV Cache 的总显存消耗将随批大小成倍增长，可轻易超过模型参数本身的显存占用。
vLLM~\cite{kwon2023vllm} 等推理框架通过 PagedAttention 技术改善了显存碎片问题，但并未从根本上减少 KV Cache 的绝对存储量。

量化（quantization）技术是降低 KV Cache 显存占用的重要手段。
通过将 FP16 精度的键值对压缩为 INT8 甚至 INT4 表示，可以将 KV Cache 的显存分别降至原始的 $1/2$ 和 $1/4$，
从而在有限的 GPU 显存中支持更长的上下文或更大的并发批次。

\section{问题定义与研究动机}

在上述背景下，低比特（INT4）量化是最具吸引力的压缩目标，
理论上可将 KV Cache 压缩至 FP16 的 $1/4$，
在相同硬件预算下支持 4 倍的上下文长度或并发批次。
然而，现有的 INT4 量化方法在长上下文场景下表现急剧恶化：
Needle 检索通过率可从 100\% 降至 0\%，困惑度退化超过 15\%。
对称 INT4 量化的模型在长上下文下
不仅无法完成精确检索任务，其语言建模能力也发生灾难性崩溃。
传统校准目标（MSE、percentile）\textbf{无法解释}这种失效，
数值重建误差看似合理，注意力行为却可能完全崩溃。
这个矛盾暴露了当前校准范式的盲点：
仅优化数值误差不足以保障量化后的模型行为。
我们需要一种新的校准原则，既能指导参数搜索，又能诊断失效模式。

现有 KV Cache 量化方法主要采用两类校准策略：
均方误差（Mean Squared Error, MSE）和百分位裁剪（percentile clipping）。
MSE 校准通过最小化量化前后张量的逐元素数值误差来确定量化参数；
百分位裁剪则通过截断分布尾部的异常值来缩小量化范围，以降低大多数数值的量化误差。

但这两种策略有一个共同的盲点：它们优化的是数值层面的重建误差，
而数值误差小并不意味着模型行为保真。
softmax 的非线性使得误差大小与注意力偏移之间不存在单调关系：
微小的 Key 扰动可能剧烈改变注意力分布，
较大的误差反而可能被归一化效应吸收。
百分位裁剪同样缺乏与下游注意力行为的关联，
对不同层、不同头的分布差异无法自适应。

本文认为，KV Cache 量化的最终目标不是最小化存储表示的数值误差，
而是保持量化前后注意力行为（attention behavior）的一致性。
若将 FP16 精度下的注意力权重分布记为 $\bm{p}$，
量化后的注意力权重分布记为 $\bm{q}$，
则量化质量应当以两者之间的分布差异来衡量：
\begin{equation}
  D_{\KL}(\bm{p} \| \bm{q}) = \sum_{i} p_i \log \frac{p_i}{q_i}
  \label{eq:kl_divergence}
\end{equation}
其中，$D_{\KL}$ 为 Kullback-Leibler 散度，衡量量化引入的注意力行为偏移程度。

由此，我们提出以 KL 散度作为校准目标，
直接优化量化参数以最小化注意力行为偏移。
这一思路，即"行为对齐"（Behavior Alignment），
将校准从数值层面推进到语义保真层面。

\section{国内外研究现状}

KV Cache 量化的研究建立在模型量化和高效注意力计算两大技术方向之上，
以下从三个方面简述相关研究现状，更详细的分析将在第二章展开。

\subsection{模型量化}

模型量化通过降低数值精度来压缩模型体积、加速推理。
代表性工作包括面向权重-激活联合量化的 SmoothQuant~\cite{xiao2023smoothquant}、
基于近似二阶信息的训练后量化方法 GPTQ~\cite{frantar2023gptq}、
以及激活感知权重量化策略 AWQ~\cite{lin2024awq}。
上述方法主要面向模型权重量化，与本文关注的 KV Cache 量化在目标和技术路径上有所不同，
但其量化理论和误差分析方法为本文提供了重要参考。
更详细的技术分析见第二章。

\subsection{KV Cache 量化}

KV Cache 量化是近年来的研究热点。
KIVI~\cite{liu2024kivi} 揭示了 Key 和 Value 在量化敏感性上的不对称特征，
提出逐通道 Key + 逐 token Value 的非对称量化策略，
其校准依赖运行时的 absmax 估计（无离线校准步骤）；
KVQuant~\cite{hooper2024kvquant} 探索了面向极低比特的非均匀量化方案；
ZipCache~\cite{he2024zipcache} 则基于 token 重要性实现了混合精度 KV Cache 压缩。
上述方法在量化策略和压缩技术上取得了重要进展，
但其校准目标仍落在数值重建误差这一层级，
未从注意力行为保真的角度重新定义校准对象；
K/V 各自的量化敏感性边界及其与模型架构的关系也缺乏系统性消融验证。
本文的 INT4-RoleAlign 将沿用 KIVI 的 per-channel Key + per-token Value 格式，
但以离线 attention-KL percentile 搜索替代其运行时 absmax 估计，
把校准信号从数值重建提升到注意力行为保真。
更详细的工作对比见第二章。

\subsection{高效注意力计算}

FlashAttention~\cite{dao2022flashattention,dao2024flashattention2} 通过分块计算与内核融合
显著降低了注意力计算的访存开销，但其主要面向 FP16 精度的标准注意力，
尚未充分支持量化 KV Cache 的在线反量化融合。
本文基于 Triton~\cite{tillet2019triton} 实现了 INT8/INT4 融合量化解码注意力核函数，
将反量化与注意力计算整合在单个 GPU 核函数中。
更详细的技术讨论见第二章和第三章。

\section{本文研究内容与主要贡献}

基于上述背景，本文提出三个递进的研究问题：

\begin{description}
  \item[\textbf{RQ1}] \emph{KV Cache 量化应优化什么目标？}
    现有方法以数值重建误差（如 MSE、absmax percentile）为校准目标，
    但数值层面的最优未必对应推理行为的保真。
    是否存在一个更贴近下游推理行为的优化目标？
  \item[\textbf{RQ2}] \emph{低比特量化为何系统性失效？}
    对称 INT4 在 Needle 检索上降至 0\%，PPL 在最敏感的模型上退化至约 14\%。
    该失效是全局性的还是存在可定位的结构性根因？
    量化噪声在 Key 和 Value 之间如何分配？
    不同模型架构（特别是不同 GQA 头数）对量化容忍度有何影响？
  \item[\textbf{RQ3}] \emph{诊断结论能否导出有效的低比特设计？}
    如果低比特失效有可定位的根因，
    是否可以基于诊断结论设计差异化的量化策略，
    在保持压缩率的同时恢复检索能力？
    其代价边界（PPL 退化、延迟变化）如何？
\end{description}

围绕以上三个问题，本文提出一个行为对齐原则并将其推进至低比特设计：
KV Cache 量化应当保持注意力行为而非最小化数值重建误差。
该原则具体表现为四项核心贡献（C1--C4），分别回应三个研究问题
（C1 与 C2 回应 RQ1，C3 回应 RQ2，C4 回应 RQ3）；
此外在 C1--C4 的诊断过程中意外浮现一条二阶规律，
作为 \emph{补充发现}（非预先设定的贡献目标）一并报告。

\textbf{贡献一：统一原则。}
以注意力分布的 KL 散度（attention-KL）作为一个
兼具校准与诊断双重用途的目标。
它在离线阶段指挥量化参数的搜索；
当量化失效时，同一散度又让我们得以反过来看清失效发生在哪里、为什么发生。
校准和诊断由此落在同一把尺度上。

\textbf{贡献二：INT8 规范验证实例。}
在该原则下给出 INT8 的 canonical validated instance，
在三个模型族上实现 44\% KV 显存降低与 8--38\% 延迟降低，
证明行为对齐校准的有效性，并为低比特诊断建立参照基线。

\textbf{贡献三：低比特失效诊断。}
同一诊断透镜发现对称 INT4 的系统性失效并非偶然：
受控 K/V 消融在三个模型族上一致指向 Key 量化为退化
主导因素，且敏感性与 GQA 架构密切相关。
该发现为精度分配提供了定量依据。

\textbf{贡献四：诊断导出的低比特设计与能力边界。}
上述诊断直接导出角色感知的低比特设计。
INT4-RoleAlign 采用 per-channel Key + per-token Value
非对称量化轴与行为对齐 percentile 校准。
在 Qwen2.5-1.5B、Qwen2.5-7B、LLaMA-3.1-8B 三个模型上
Needle 从 0\%$\to$100\%，
在 73\% KV Cache 压缩率下恢复检索能力。
PPL 退化因模型规模而异（8B: 2.4\%，7B: 6.1\%，1.5B: 13.7\%），
反映出 INT4 量化的一个结构性特征：
局部检索对量化噪声的容忍度远高于全局语言建模。
INT4-RoleAlign 沿``原则$\to$诊断$\to$设计$\to$边界''的路径展开，
构成本文的核心方法贡献与能力边界分析。

\paragraph{补充发现（诊断副产物）：温度校正的 GQA 尺度依赖性。}
C1--C4 已构成本文的四项核心贡献。
在执行上述诊断流程的同一次 KL 参数扫描中，
我们还顺带发现了一条规律：逆温度校正（inv\_tau）的有效性
与 GQA 头数呈反向依赖——
在 $H_{\text{kv}}{=}2$（Qwen2.5-1.5B）上 inv\_tau 改善 PPL 1.6\%，
而在 $H_{\text{kv}}{\geq}4$（Qwen2.5-7B 与 LLaMA-3.1-8B）上反而恶化 3.4--6.0\%。
这并非我们预先设定的研究目标，
而是诊断流程的一个副产物（byproduct），因此不计入正式贡献清单。
我们给出的是一种启发式论证：
共享 KV 头的量化噪声在下游聚合时似乎被衰减，
使得温度锐化在头数较多时反而过度。
严格意义上这需要 i.i.d.\ 假设，
在 GQA 结构下并不严格成立，因此我们将完整形式化推导留作未来工作，
并在第四章以三个模型规模的实证结果作为该启发式的经验依据。

\textbf{系统支撑。}
为使上述框架在实际推理中可用，
本文实现了 Triton 融合量化解码核函数，
并以 5 个种子的主实验（PPL 确定性另以 10 个种子验证）、
Bootstrap 置信区间与 BH-FDR 多重比较校正作为统计框架，
保障所有量化结论的可复现性与统计可靠性。
LLaMA-3.1-8B 上 INT8 校准对数据规模敏感这一已知局限，
将在第四章作为限制条款明确披露。

总的来说，KV Cache 量化的关键不是最小化数值误差，
而是保持注意力行为。attention-KL 在 INT8 上
经过了实证验证，在 INT4 诊断中定位了 Key 主导退化的根因，
并最终导出 INT4-RoleAlign——
后者以约 73\% 的 KV Cache 压缩率
将三个模型的 Needle 检索从对称 INT4 基线的 0\% 恢复至 100\%，
同时清楚地标出 PPL 退化随 GQA 头数减少而加剧的边界。
诊断流程中浮现的 inv\_tau × GQA 规律，
则是这一框架在优化目标之外留下的一个额外观察。
