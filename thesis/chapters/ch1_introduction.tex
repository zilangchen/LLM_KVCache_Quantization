\chapter{绪论}

\section{研究背景}

随着大语言模型（Large Language Model, LLM）的参数规模从数十亿增长至数千亿、
上下文窗口从 4K 扩展至 128K tokens，
GPT-4、LLaMA~3~\cite{grattafiori2024llama3}、Qwen2.5~\cite{qwen2025qwen25} 等模型
在文本生成、阅读理解和逻辑推理等任务上展现出卓越能力，
但推理阶段的显存与延迟开销也随之急剧增长，成为制约大规模部署的核心瓶颈。

现代大语言模型普遍采用 Transformer~\cite{vaswani2017attention} 架构，
其自回归解码（autoregressive decoding）过程中，每生成一个新的 token 都需要与所有历史 token 的键值对（Key-Value pair）进行注意力计算。
为避免重复计算，推理系统会将已计算的键值对缓存起来，形成键值缓存（KV Cache）。
KV Cache 机制是自回归解码的核心加速手段，能够将单步解码的时间复杂度从 $O(n^2)$ 降低至 $O(n)$，
但其代价是显存占用随序列长度线性增长。

具体而言，KV Cache 的显存占用与模型层数、每层 KV 头数、头维度以及序列长度均成正比，
可表示为：
\begin{equation}
  M_{\mathrm{KV}} = 2 \times L \times H_{kv} \times d_k \times S \times b
  \label{eq:kv_memory}
\end{equation}
式中 $L$ 为层数、$H_{kv}$ 为每层 KV 头数、$d_k$ 为头维度、$S$ 为序列长度，
$b$ 为每个元素占用的字节数（FP16 下为 2），因子 2 对应 Key 和 Value 两个矩阵。

在长上下文、批量推理场景下，
KV Cache 的总显存消耗可轻易超过模型参数本身（具体推导与数值示例见第~\ref{sec:ch2-kv-memory}~节）。
vLLM~\cite{kwon2023vllm} 等推理框架通过 PagedAttention 技术改善了显存碎片问题，
但并未从根本上减少 KV Cache 的绝对存储量。

量化（quantization）技术是降低 KV Cache 显存占用的重要手段。
通过将 FP16 精度的键值对压缩为 INT8 甚至 INT4 表示，可以将 KV Cache 的显存分别降至原始的 $1/2$ 和 $1/4$，
从而在有限的 GPU 显存中支持更长的上下文或更大的并发批次。

\section{问题定义与研究动机}

在上述背景下，低比特（INT4）量化是最具吸引力的压缩目标，
理论上可将 KV Cache 压缩至 FP16 的 $1/4$，
在相同硬件预算下支持 4 倍的上下文长度或并发批次。
然而，现有以数值重建误差为校准目标的 INT4 量化方法在长上下文场景下表现急剧恶化：
Needle 检索通过率可从 100\% 降至 0\%，困惑度退化超过 15\%。
对称 INT4 量化的模型在长上下文下
不仅无法完成精确检索任务，其语言建模能力也发生灾难性崩溃。
传统校准目标（MSE、percentile）未能充分揭示这种失效，
数值重建误差看似合理，注意力行为却可能完全崩溃。

现有 KV Cache 量化方法主要采用均方误差（MSE）或百分位裁剪（percentile clipping）等数值重建目标。
两者的共同盲点在于：softmax 的非线性使得数值误差大小与注意力分布偏移不存在单调关系，
微小的 Key 扰动可能剧烈改变注意力分布，较大的误差反而可能被归一化效应吸收；
静态阈值亦无法自适应不同层、不同头的分布差异
（详细机制分析见第~\ref{sec:ch2-quant-softmax}~节）。

这一矛盾并非工程调参问题，
而是指向注意力结构本身的误差传播特性。
设标准注意力为
\begin{equation*}
  z_i = \frac{q^\top k_i}{\sqrt{d}}, \qquad
  a_i = \mathrm{softmax}(z)_i, \qquad
  o = \sum_i a_i v_i;
\end{equation*}
量化后 $k_i, v_i$ 变为 $\hat k_i, \hat v_i$，
输出误差可严格分解为以下 telescoping 形式（详见第三章方法部分）：
\begin{equation}
  \hat{o} - o
  = \underbrace{\sum_i (\hat a_i - a_i)\, v_i}_{\text{分布侧贡献项}}
  + \underbrace{\sum_i \hat a_i\, (\hat v_i - v_i)}_{\text{聚合侧贡献项}}
  \label{eq:error_decomposition}
\end{equation}
该分解是精确的代数 telescoping 展开（非 K/V 因果分离）：
第一项源自 K 扰动经 softmax 非线性传播引起的 attention 分布偏移 $(\hat a_i - a_i)$，
在 FP16 的 V 下加权求和；
第二项源自 V 扰动 $(\hat v_i - v_i)$，
在量化后 attention $\hat a_i$（已受 K 扰动影响）下的直接聚合贡献。
两项共同刻画量化对注意力行为（分布 $\bm{a}$ 与输出 $\bm{o}$ 两个面）的完整损伤；
真正被下游感知的不是 $\|\hat k - k\|$ 或 $\|\hat v - v\|$ 这样的数值距离，
而是由其诱导的行为扭曲。

基于上述结构性观察，本文提出以下核心假设：
\begin{quote}
  \textbf{核心假设（注意力行为的联合保持度）：}
  在 KV Cache 量化中，把\textbf{注意力行为}定义为 \emph{分布侧} $\bm{a}$ 与
  \emph{聚合侧} $\bm{o}$ 的\emph{联合保持度}，比以张量数值范数作为优化对象更贴近模型真实的功能损伤。
  误差分解式~\eqref{eq:error_decomposition}
  表明两侧误差通过 $\hat a_i$ 与 $\hat v_i - v_i$ 相乘的聚合侧项\emph{耦合}，
  因此保持分布 $\bm{a}$ 的同时也约束了聚合侧项的第一因子——
  这是工程上 behavior 保持度可以用\textbf{分布侧 KL 散度作为直接可优化代理}的结构性理由
  （第三章 §\ref{sec:ch3-problem} 给出形式化推导）。
  聚合侧 $\bm{o}$ 的保真度在本文实证中由端到端任务指标
  （PPL、Needle、LongBench 等）作\emph{补充验证}，而非游离于 框架 之外。
\end{quote}
在具体实现上，本文以 Kullback--Leibler 散度刻画分布侧偏移：
若 $\bm{p}$ 记 FP16 注意力权重分布、$\bm{q}$ 记量化后分布，则
\begin{equation}
  D_{\KL}(\bm{p} \| \bm{q}) = \sum_{i} p_i \log \frac{p_i}{q_i}
  \label{eq:kl_divergence}
\end{equation}
$D_{\KL}$ 是 attention 分布侧偏移的 operational proxy。
之所以选 KL 作为代理，是因为分布 $\bm{a}$ 直接出现在方程~\eqref{eq:error_decomposition}~两项的系数中，
小 KL 对应小的分布偏移 $(\hat a_i - a_i)$，
进而约束分布侧贡献项的量级；
聚合侧贡献项的保真性则由校准后的模型
在端到端任务指标上的经验表现被间接验证。
本文第三章将 $D_{\KL}$ 具体实例化为校准层目标函数
（第~\ref{sec:ch3-calibration}~节）；
该校准产物是否能进一步作为跨层敏感度度量、
指导分配层的预算决策，
作为延伸方向由第四章实验部分呈现。

本文后续将在方法章（第三章）落地基于该假设的校准实例，
在实验章（第四章）以跨模型长上下文任务验证其与输出侧保真的相关性。


\section{国内外研究现状}

KV Cache 量化建立在模型量化、KV 专属压缩与高效注意力计算三大方向之上。

\subsection{模型量化}

模型量化通过降低数值精度压缩模型体积、加速推理，
代表工作包括权重-激活联合量化 SmoothQuant~\cite{xiao2023smoothquant}、
训练后权重量化 GPTQ~\cite{frantar2023gptq} 与激活感知权重量化 AWQ~\cite{lin2024awq}。
这些方法主要面向权重，但其量化理论和误差分析为 KV Cache 量化提供了方法学参考。

\subsection{KV Cache 量化}

KIVI~\cite{liu2024kivi} 观察到 Key 与 Value 在量化敏感性上的不对称特征，
提出逐通道 Key + 逐 token Value 的非对称量化策略；
KVQuant~\cite{hooper2024kvquant} 探索了极低比特非均匀量化方案；
ZipCache~\cite{he2024zipcache} 基于 token 重要性实现混合精度 KV Cache 压缩。
然而上述工作的校准目标仍以数值重建为导向，
未系统地将量化产物与下游推理行为（attention 分布与聚合输出）的保持度直接对齐，
亦未将行为敏感度度量进一步作为跨层预算分配的输入。

\subsection{高效注意力计算}

FlashAttention~\cite{dao2022flashattention,dao2024flashattention2} 通过分块计算与内核融合
降低了注意力计算的访存开销，但主要面向 FP16 精度；
vLLM~\cite{kwon2023vllm} 的 PagedAttention 改善了 KV Cache 显存碎片。
本文在 INT8 canonical path 上基于 Triton~\cite{tillet2019triton}
实现融合量化解码核函数，作为 behavior-guided 校准的系统落地实例之一（详见第三章）。

更详细的工作对比与技术分析见第~\ref{chap:related-work}~章。

\section{本文研究内容与主要贡献}
\label{sec:ch1-contribution}

基于上述背景与核心假设，本文提出三个递进的研究问题：

\textbf{研究问题一（RQ1）：在 KV Cache 量化中，什么样的分析对象比单纯数值误差更贴近模型真实的功能损伤？}
现有方法以数值重建误差（MSE、absmax percentile 等）为校准目标，
但数值层面的最优未必对应推理行为的保真
（详见第~\ref{sec:ch2-quant-softmax}~节的 softmax 非线性机制分析）。
从注意力计算的误差传播结构出发（方程~\eqref{eq:error_decomposition}），
是否存在一个更贴近下游推理行为的统一分析对象，
可以在不同 bit-width 与模型规模下为校准与诊断提供共同基础？

\textbf{研究问题二（RQ2）：以注意力行为为中心的校准原则能否落成一套完整可用的量化系统？}
若上述假设成立，
能否将其在不同量化位宽上实例化为完整可落地的量化路径？
具体地：INT8 上 behavior-guided 校准能否在 FP16 基础上几乎无损地保持推理质量？
在更困难的 INT4 低比特场景下，
该原则与前沿动态校准方法（如 KIVI~\cite{liu2024kivi}）相比是否仍具竞争力？

\textbf{研究问题三（RQ3）：同一行为原则能否延伸到更高层的预算分配决策？如果能，它揭示了什么跨模型结构？}
校准过程的副产品——每层对注意力行为的敏感度画像——
是否可以作为跨层预算分配（layer-wise bit allocation）的输入？
若将同一原则延伸至 behavior-guided allocator 与
进一步的 profile-guided 预算建议机制（AutoK），
跨模型家族（Qwen / LLaMA / Mistral）、跨规模（1.5B 至 14B）、
跨任务（LongBench 多任务）的实验会揭示什么样的 operating 适用区间 结构？

围绕以上三个问题，本文给出三个主要贡献（C1 / C2 / C3），
分别回应 RQ1 / RQ2 / RQ3。

\textbf{贡献一（C1）：Behavior-guided 统一框架。}
我们提出以 attention 行为（分布 $a$ 的保持度）为核心的统一原则，
把 KV Cache 量化中\emph{两个}此前常被独立处理的层——
\emph{静态校准}与\emph{预算分配}——
统一到同一 behavior sensitivity profile 作为共享输入的框架下。
第三章 §\ref{sec:ch3-problem} 给出误差分解
\begin{equation*}
  \hat{o} - o
  = \underbrace{\textstyle\sum_i (\hat a_i - a_i)\, v_i}_{\text{分布侧项}}
  + \underbrace{\textstyle\sum_i \hat a_i\, (\hat v_i - v_i)}_{\text{聚合侧项}},
\end{equation*}
作为 框架 的\textbf{理论起点}；§\ref{sec:ch3-allocator} / §\ref{sec:ch3-autok}
把同一 profile 延伸到 layer-wise allocator 与 AutoK。
此 框架 的独立贡献不在"击败弱 基线"，
而在\textbf{组织了 calibration 与 allocation 的共享基础}——
这是相对 KIVI 等 calibration-only 方法与 KVTuner 等 allocator-only 方法的\emph{结构性}区别。

\textbf{贡献二（C2）：两层实例化——INT8 canonical path 与 INT4 RoleAlign/AutoK。}
C1 的 框架 在两层、两个 bit-width 上实现闭环验证：
\begin{itemize}[leftmargin=2em, itemsep=1pt, topsep=2pt]
  \item \textbf{INT8 canonical path}（第~\ref{subsec:exp-int8-canonical}~节 + 表~\ref{tab:t1-int8-canonical}）：
    Qwen2.5-1.5B 上 INT8-ours 相对 FP16 mean $\Delta{=}{+}0.02$，
    在 3 个 LongBench task 上\textbf{在实证上不可区分}，
    证明 框架 的 calibration 层在 canonical 规模下可闭环；
  \item \textbf{INT4-RoleAlign}（第~\ref{sec:exp-int4-kivi}~节 + 表~\ref{tab:t2-int4-kivi}/\ref{tab:ch3-rolealign-vs-kivi}）：
    在 KIVI-style 的 per-channel K + per-token V 非对称格式上以\textbf{离线 KL 搜索}
    取代运行时 absmax/min，生成可审计的校准产物 JSON；
    跨 4 模型 vs KIVI 同 format 下 PPL $\Delta$ 控制在 $0.15$ PPL 内、Needle 均 $100\%$；
  \item \textbf{AutoK}（第~\ref{sec:ch3-autok}~节 + 表~\ref{tab:t4-mistral-autok}）：
    框架 的自然扩展，基于 profile 的 cov$80\%$ 覆盖度 阈值
    自动提出 预算；Mistral-7B 上作为 strongest single-family 正面案例
    在 5-task 中赢 3 个。
\end{itemize}
Triton 融合量化解码核被定位为 INT8 canonical path 的\textbf{系统落地手段}
（C2 的工程子项，第~\ref{sec:ch3-triton}~节），
不作为独立贡献——性能收益详见第~\ref{sec:exp-deployment}~节部署附录。

\textbf{贡献三（C3）：跨模型 适用区间 地图作为 empirical 洞察。}
通过在 Qwen2.5-\{3B, 7B, 14B\}、LLaMA-3.1-8B、Mistral-7B 上
在\emph{同量级 INT4 预算 band} 内对比 4 policy（uniform / BA-$k$ / heuristic-$k$ / AutoK），
我们揭示一张由 \textbf{(family, scale, task)} 共同决定的 operating 适用区间 地图
（第~\ref{sec:exp-cross-model}~节，表~\ref{tab:t3-cross-model-main}，图~\ref{fig:sensitivity-heatmap} / \ref{fig:t7-pareto} / \ref{fig:regime-map}）：
\begin{itemize}[leftmargin=2em, itemsep=1pt, topsep=2pt]
  \item \textbf{Small-scale early-layer rescue}（Qwen2.5-3B，§\ref{subsec:exp-per-model-3b}）——
    BA-$k_1$ 保护 layer 0 把 NarrativeQA 从 heuristic 的 $3.08$ 分回救到 $7.17$，
    是 behavior-guided 相对位置启发式\textbf{最强的结构性差异化证据}；
  \item \textbf{Large-scale top-tier without stable winner}（Qwen2.5-14B，§\ref{subsec:exp-per-model-14b}）——
    4 policy 挤在 top-1 的 $3.5\%$ relative 距离内，没有单一跨任务赢家；
  \item \textbf{Mistral-specific AutoK win}（§\ref{subsec:exp-per-model-mistral}）——
    cov$80$ 预算下单模型最清晰的 AutoK 正面案例；
  \item \textbf{Heuristic as strong 基线}——
    在多个 适用区间 区间里，等距位置保护与 behavior-guided 选中集合重合，
    不是方法失败而是 适用区间 的\emph{结构性观察}。
\end{itemize}
这一 适用区间 map 把本文的贡献从"方法跑分更高"\textbf{抬升为"揭示结构分区"}——
即 behavior-guided allocator 的真实价值不在数字胜利，
而在提供了一个\emph{读数工具}让不同 适用区间 可被识别与命名。

综合三条贡献，本文给出 KV Cache 量化的第一个
\textbf{以 attention 行为为统一组织原则}的 框架 + 实例化 + 跨模型实证工作。
图~\ref{fig:framework-overview} 给出完整的论证脉络（§\ref{sec:ch1-structure} Roadmap）。
