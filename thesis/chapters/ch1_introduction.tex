\chapter{绪论}

\section{研究背景}

随着大语言模型（Large Language Model, LLM）的参数规模从数十亿增长至数千亿、上下文窗口从 4K 扩展至 128K tokens，GPT-4、LLaMA~3~\cite{grattafiori2024llama3}、Qwen2.5~\cite{qwen2025qwen25} 等模型开始覆盖长文档理解、复杂推理与多轮生成等更长输入场景。上下文窗口扩展带来了更强的任务覆盖能力，也把推理阶段的显存占用与延迟开销推到前台：当上下文长度和服务并发同步上升时，缓存存储很快成为制约大模型规模化部署的核心瓶颈。

现代大语言模型普遍采用 Transformer~\cite{vaswani2017attention} 架构。在自回归解码（autoregressive decoding）过程中，模型每生成一个新的 token，都需要与全部历史 token 的键值对（Key-Value pair）进行注意力计算。为避免对历史上下文重复执行 K/V 投影与注意力累积，推理系统通常将已计算得到的键值对缓存起来，形成键值缓存（KV Cache）。这一机制是自回归解码得以高效执行的关键基础，能够将单步解码中的重复计算复杂度从 $O(n^2)$ 降低到 $O(n)$；但与此同时，缓存本身的存储开销也会随序列长度持续增长。

从存储结构上看，KV Cache 的显存占用与模型层数、每层 KV 头数、头维度、序列长度以及存储精度近似成正比，可写为
\begin{equation}
  M_{\mathrm{KV}} = 2 \times L \times H_{kv} \times d_k \times S \times b
  \label{eq:kv_memory}
\end{equation}
其中 $L$ 表示层数，$H_{kv}$ 表示每层 KV 头数，$d_k$ 表示头维度，$S$ 表示序列长度，$b$ 表示每个元素占用的字节数，前面的系数 2 对应 Key 和 Value 两部分存储。在长上下文和批量推理场景下，这一开销会随上下文长度与并发批次同步放大，甚至可能超过模型参数本身。vLLM~\cite{kwon2023vllm} 等推理框架通过分页注意力（PagedAttention）改善显存碎片和缓存管理效率，但这类方法并不直接减少 KV Cache 的绝对存储量，因此无法从根本上消除长上下文部署中的容量压力。

在这一背景下，对 KV Cache 进行低成本压缩就具有了明确的工程价值。若将 FP16 精度的缓存进一步压缩为 INT8 或 INT4 表示，其显存占用理论上可分别降至原始的 $1/2$ 与 $1/4$，从而在相同硬件预算下支持更长的上下文窗口或更高的并发批次。也正因如此，KV Cache 量化已成为长上下文推理优化中的关键方向之一。然而，压缩率的提升并不自动等于模型功能稳定性的保持，尤其是在更具吸引力的低比特区间内，如何在显存收益与模型可用性之间取得平衡，构成了本文进一步需要回答的问题。

\section{问题定义与研究动机}

在上述背景下，KV Cache 量化的价值首先来自直接的存储压缩。INT8 可以作为较保守的保真基线，更低位宽则能进一步释放显存空间，支持更长上下文或更高并发。但缓存压缩不能只按 bit 数下降来理解：位宽降低以后，Key 与 Value 的扰动会进入注意力计算，并可能改变检索、摘要和语言建模等任务行为。对 KV Cache 而言，“压得更低”并不自动等于“仍然可用”，低比特压缩需要单独回答稳定性问题。

现有 KV Cache 量化方法大多仍以数值重建误差作为校准目标，例如均方误差（MSE）或基于分位数的裁剪策略。这类目标隐含地假设：张量误差越小，模型功能损伤也越小。然而，在注意力计算中，这一假设并不稳固。由于 softmax 的非线性作用，微小的 Key 扰动也可能引起注意力分布的明显重排；相反，某些较大的数值误差则可能在归一化过程中被部分吸收。因此，数值误差的大小与下游功能偏移之间并不存在稳定的一一对应关系。与此同时，静态阈值也难以充分适应不同层、不同头以及不同上下文位置上的分布差异。

KV Cache 量化的困难因此不只是继续调节超参数或压低重建误差。模型在推理时感知到的是量化误差经过注意力计算后的功能偏移：注意力分布是否被重排，聚合输出是否偏离，任务行为是否随之改变。特别是在长上下文和低比特场景下，同一量级的数值扰动落在不同层、不同头或不同角色上，可能产生完全不同的后果。单纯追求全局数值误差最小，并不能保证模型在推理时保持稳定和可用。

本文据此提出一个更基础的问题：KV Cache 量化需要保持的不只是张量重建精度，还包括推理行为的稳定性。本文关心量化后注意力分布是否被重排，以及这种重排是否改变输出和任务表现。只有先明确这一对象，后续关于校准目标、量化路径和预算分配的讨论才有共同的分析基础。

围绕这一问题意识，下一节从模型量化、KV Cache 量化与高效推理三条脉络概述相关研究，并进一步收束为本文的核心假设与研究问题。第二章给出更系统的技术基础与相关工作比较，第三章展开正式的误差传播分析与方法定义。

\section{国内外研究现状概述}

围绕长上下文推理与 KV Cache 压缩，现有研究主要形成三条脉络。模型量化研究提供低比特表示、校准与误差控制的基础方法；KV Cache 专属量化直接处理缓存存储压力；高效推理系统则从注意力计算和缓存管理角度降低部署开销。三条路线共同说明，长上下文部署不是单一算法问题，而是压缩精度、缓存组织和系统执行共同作用的结果。

在模型量化方向，已有研究围绕权重与激活的低比特表示、训练后量化以及校准策略形成了较为成熟的方法体系。以 SmoothQuant~\cite{xiao2023smoothquant}、GPTQ~\cite{frantar2023gptq} 和 AWQ~\cite{lin2024awq} 为代表的工作，分别从量化难度转移、近似误差补偿和激活感知保护等角度推进了大模型的压缩与部署。这些方法主要面向模型参数与激活，但其中关于量化表示、误差控制和校准流程的基本思想，也为后续 KV Cache 压缩提供了重要的方法学参考。

在 KV Cache 专属量化方向，随着长上下文推理对缓存显存的压力持续上升，研究开始直接面向缓存本身设计压缩路径。以 KIVI~\cite{liu2024kivi}、KVQuant~\cite{hooper2024kvquant} 和 ZipCache~\cite{he2024zipcache} 为代表的工作，分别从 K/V 异构量化轴、极低比特量化和重要性驱动压缩等角度探索了不同的 KV Cache 压缩方案。这些研究表明，缓存压缩并不是对一般模型量化方法的简单复用，而需要结合注意力计算特征、缓存访问模式与部署约束进行专门设计。

除量化压缩外，高效推理方向也在同步推进。FlashAttention~\cite{dao2022flashattention,dao2024flashattention2} 通过更高效的分块注意力计算降低中间访存开销；分页注意力~\cite{kwon2023vllm} 则通过缓存分页改善显存碎片和缓存利用率。这类工作与 KV Cache 量化并非替代关系，而是从不同层面共同服务于长上下文推理的可部署性。

总体来看，现有研究已经从模型量化、KV Cache 专属压缩与高效推理等角度推进了长上下文部署，但这些路线解决的层面并不相同。本文的切入点是 KV Cache 量化中的行为保持问题：量化后的缓存不仅要占用更少显存，还要在注意力计算中保持可用的推理行为。基于这一判断，下一节给出本文的核心假设与研究问题。

\section{核心假设与研究问题}

KV Cache 量化的核心难点不只是继续提高压缩率，而是如何在压缩后保持模型的实际推理行为。本文把注意力行为作为分析起点，关注量化后注意力分布、聚合输出和任务表现是否仍与全精度路径保持一致。

\noindent\textbf{核心假设：} 在 KV Cache 量化中，注意力行为比张量重建误差更接近模型实际功能损伤。本文关心量化后注意力分布是否保持稳定、加权聚合输出是否偏离，以及这些变化是否影响任务行为。基于这一假设，校准目标和预算分配可以共享同一个观察对象，而不必被拆成彼此孤立的局部问题。

\noindent\textbf{研究问题一：行为引导校准能否形成稳定基线？} 这一问题关注最保守、最易审计的位宽设置。若以注意力行为为导向组织校准过程，本文首先需要建立一条可靠的 INT8 基准路径，作为后续低比特探索的出发点。

\noindent\textbf{研究问题二：低比特路径能否从对称失效恢复到可用？} 当位宽继续降低时，对称压缩可能触发检索和语言建模层面的失稳。本文要回答的是，承认 K/V 角色差异之后，行为引导方法能否把失效的低比特路径恢复为可分析、可审计、可部署的实例。

\noindent\textbf{研究问题三：行为敏感度能否解释跨模型预算分配差异？} 这一问题把视角从校准推到预算决策层。本文关注同一行为读数能否在不同模型家族、不同规模和不同任务条件下读出策略适用区间，而不是只给出一种单点配置。

围绕上述核心假设与三个研究问题，下一节将进一步概述本文的研究内容、技术路线与主要贡献。

\section{研究内容、技术路线与主要贡献}
\label{sec:ch1-contribution}

围绕前述核心假设与三个研究问题，本文将 KV Cache 量化组织为一条以注意力行为为主线的研究路径。校准层回答如何选择量化参数，低比特路径回答如何在更激进的压缩下保持可用，分配层回答有限预算应优先保护哪些层或角色。

从研究内容上看，本文首先尝试从注意力行为视角重新组织 KV Cache 量化问题，用一个更贴近模型实际功能稳定性的切入点连接不同层级的量化决策；在此基础上，再围绕不同位宽形成可执行的方法实例，并通过跨模型比较提炼策略适用区间图谱。因此，本文关注的并不只是某个局部设置中的压缩收益，而是一条从问题界定、方法落地到结构读数逐步收束的论证链条。

从技术路线看，本文先在相关工作和理论基础上明确问题边界，再在方法章节中给出校准目标、路径实例和分配机制，最后在实验章节中检验其保真度、低比特恢复能力与跨模型结构读数。全文按“问题 - 方法 - 证据”展开，不把若干技巧并列陈列。

\noindent\textbf{贡献一：提出以注意力行为保持为核心的 KV Cache 量化视角。} 本文不再只用张量重建误差组织校准，而是把注意力分布、聚合输出和任务行为的稳定性作为共同观察对象。这个视角把校准和预算分配放到同一条分析主线上，为后续方法设计提供统一的判断基础。

\noindent\textbf{贡献二：给出从 INT8 基准路径到低比特恢复的可审计方法实例。}
本文首先在 INT8 基准路径上验证行为引导校准的基础保真能力，
再在低比特场景中通过 INT4-RoleAlign 区分 Key 与 Value 的功能角色，
并进一步用 AutoK 将行为敏感度画像转写为预算建议。
需要说明的是，Triton~\cite{tillet2019triton}、
离线校准产物和在线推理管线属于系统支撑层，
其作用是保证方法可部署、可复现与可审计，
而不与三条顶层贡献并列。

\noindent\textbf{贡献三：识别跨模型预算策略的适用区间。} 本文不把跨模型实验写成某一种策略在所有模型家族、规模和任务条件下都最优，而是识别不同条件共同作用下的策略适用区间。这样的读法能够解释 heuristic 何时已经足够、何时会选错关键层，也能解释 AutoK、早层保护和高性能簇共存等现象为何出现在不同模型上。

上述研究内容与技术路线共同构成了全文的论证主线，各章节的具体功能分工将在下一节进一步说明。

\section{论文结构安排}
\label{sec:ch1-structure}

本文共分五章。第一章为绪论，主要阐述研究背景，提出问题定义、核心假设与研究问题，并概述本文的研究内容、技术路线与主要贡献，完成全文立题。第二章为相关工作与技术基础，介绍 Transformer、KV Cache 与模型量化的必要技术前置，综述相关研究并归纳本文的研究空白与切入位置。第三章为方法设计，围绕注意力行为保持展开本文的方法框架与实现路径。第四章为实验评估与结果分析，在统一实验协议下验证基准保真、低比特恢复和跨模型预算分配三类问题，并进一步讨论部署效率与结论边界。第五章为结论，总结全文的研究结论与学术价值，讨论研究局限与未来工作，并以结语完成全文收束。
