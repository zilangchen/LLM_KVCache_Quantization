\section{校准产物格式}
\label{sec:app-calib-schema}
% ============================================================

离线校准阶段产出的 JSON 文件遵循以下 schema：

\begin{lstlisting}[
  basicstyle=\small\ttfamily,
  frame=single,
  breaklines=true,
  numbers=left,
  numberstyle=\tiny,
  caption={校准产物 JSON schema},
  label={lst:calib-schema}
]
{
  "model_name": "Qwen/Qwen2.5-1.5B-Instruct",
  "calib_method": "kl_selected",
  "num_layers": 28,
  "layers": {
    "0": {
      "k_scale": <float32>,
      "v_scale": <float32>,
      "inv_tau": [<float32>, ...],  // per-head, length = num_heads
      "k_zero_point": <float32>,    // optional, asymmetric only
      "v_zero_point": <float32>     // optional, asymmetric only
    },
    "1": { ... },
    ...
  },
  "metadata": {
    "quant_bits": 8,
    "calibration_samples": 128,
    "calibration_seq_len": 2048,
    "timestamp": "2026-xx-xx"
  }
}
\end{lstlisting}

其中：
\begin{itemize}[nosep]
  \item \texttt{k\_scale} / \texttt{v\_scale}：逐层对称量化的缩放因子，始终以 float32 存储。
  \item \texttt{inv\_tau}：逐头温度校正因子 $\tau^{-1}$，用于在注意力 softmax 前调节 logits 的分布锐度。
  \item \texttt{k\_zero\_point} / \texttt{v\_zero\_point}：仅在 KIVI 风格非对称量化模式下出现。
\end{itemize}

\end{table}

从表~\ref{tab:app-ruler-subtask} 可以看出，RULER 的整体通过率
主要由 MK-NIAH 贡献（各模式均在 96\%--99\% 区间），
而 S-NIAH 的绝对通过率普遍较低（$<$10\%），
这是因为 S-NIAH 采用严格的精确匹配标准，
长上下文下的单点检索对模型注意力精度要求极高。
INT4 量化模式在所有子任务上均完全失败（通过率 0\%），
与主文中 Needle 的观察一致，进一步确认了 4-bit 量化在 32K 上下文下的根本性精度不足。

  \item \textbf{校准方法}：KL 散度行为对齐校准（kl\_selected）
  \item \textbf{校准模型}：Qwen2.5-1.5B-Instruct
  \item \textbf{校准样本}：128 条 WikiText-103 文本，序列长度 2048 tokens
  \item \textbf{生成日期}：2026-02-12
  \item \textbf{量化精度}：INT8 对称量化，per-group（group\_size=16）
  \item \textbf{产物内容}：28 层 $\times$（k\_scale, v\_scale, per-head inv\_tau）
\end{itemize}

\textbf{已知局限。}
该校准产物生成时，校准脚本中 Q 向量的预处理流程尚未包含
input\_layernorm 和 RoPE 位置编码（CAL-019/020 问题），
该问题已在后续版本中修复。
1.5B 模型的 INT8 量化在主线实验中展现了与 FP16
几乎一致的质量（PPL 退化 $-$0.18\%，Needle 100\%），
当前结果提示该校准产物对 1.5B 主线的影响可能有限，
但尚未完成修复后的重校准对比验证，
完整的验证列为未来工作（第~\ref{sec:conclusion-future} 节）。

7B 和 8B 模型的校准文件在远端 GPU 服务器上独立生成，
各模型的 per-layer scale 和 per-head inv\_tau 参数均为模型特异性的。
这些校准产物的 provenance 依赖远端实验记录，
当前本地仓库未完整收录其校准日志。

\textbf{Tier 3 终止决策。}
基于上述结果，扩展实验（Tier 3: 跨模型全矩阵验证）被终止。
理由：float32 修复虽有效，但无法稳定扩大 ours vs baseline 的相对优势，
继续扩展的 ROI 不足以支持实验资源投入。
