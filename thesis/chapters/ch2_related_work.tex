\chapter{相关工作与技术基础}
\label{chap:related-work}

行为对齐量化框架的设计依赖于 Transformer 注意力机制、量化理论
和 KV Cache 管理三方面的基础知识。
本章从这三个维度展开综述，
并在回顾 KV Cache 量化代表性工作后定位本文的研究空白。

% ============================================================
\section{Transformer 架构与自注意力机制}
% ============================================================

\subsection{Decoder-only 架构定位}

当前主流大语言模型（LLM）——包括本文实验所用的 LLaMA~\cite{grattafiori2024llama3}
与 Qwen~\cite{qwen2025qwen25} 系列——均采用 Transformer~\cite{vaswani2017attention}
的仅解码器（Decoder-only）变体，
在自注意力层引入因果掩码以适配自回归语言建模。
本文全部方法与实验针对该架构展开，后续章节默认此背景。

\subsection{自注意力计算}

自注意力机制是 Transformer 的核心组件。给定输入序列的隐藏状态矩阵 $\bm{H} \in \mathbb{R}^{n \times d}$，
其中 $n$ 为序列长度，$d$ 为隐藏维度，通过三个可学习的线性投影矩阵 $\bm{W}_Q$、$\bm{W}_K$、
$\bm{W}_V$ 分别计算查询（Query）、键（Key）和值（Value）矩阵：
\begin{equation}
  \bQ = \bm{H} \bm{W}_Q, \quad \bK = \bm{H} \bm{W}_K, \quad \bV = \bm{H} \bm{W}_V
  \label{eq:qkv_proj}
\end{equation}
其中 $\bQ, \bK \in \mathbb{R}^{n \times d_k}$，$\bV \in \mathbb{R}^{n \times d_v}$。注意力
输出通过缩放点积注意力（Scaled Dot-Product Attention）计算：
\begin{equation}
  \mathrm{Attention}(\bQ, \bK, \bV)
    = \softmax\!\left(\frac{\bQ \bK^\top}{\sqrt{d_k}}\right) \bV
  \label{eq:attention}
\end{equation}
其中 $\sqrt{d_k}$ 为缩放因子，用于防止当 $d_k$ 较大时点积结果过大导致 softmax 函数
进入梯度极小区域。在因果解码场景下，还需对注意力权重矩阵施加下三角掩码，将未来位置
的注意力分数置为 $-\infty$。

\subsection{多头注意力与分组查询注意力}

多头注意力（Multi-Head Attention, MHA）将 $\bQ$、$\bK$、$\bV$ 分别拆分为 $h$ 个
注意力头，各头独立进行注意力计算后拼接输出：
\begin{equation}
  \mathrm{MHA}(\bQ, \bK, \bV)
    = \mathrm{Concat}(\mathrm{head}_1, \ldots, \mathrm{head}_h) \bm{W}_O
  \label{eq:mha}
\end{equation}
其中每个注意力头的计算为 $\mathrm{head}_i = \mathrm{Attention}(\bQ_i, \bK_i, \bV_i)$，
$\bm{W}_O \in \mathbb{R}^{d \times d}$ 为输出投影矩阵。MHA 的优势在于使模型能够在不同的
表示子空间中同时捕捉不同类型的依赖关系。

标准 MHA 中，Q、K、V 的头数相同，即 $H_q = H_k = H_v = h$。
多查询注意力（Multi-Query Attention, MQA）\cite{shazeer2019fast}
将所有 Q 头共享单一 K、V 头，极大地减少了 KV Cache 开销，
但可能导致质量退化。
分组查询注意力（Grouped Query Attention, GQA）\cite{ainslie2023gqa}
在 MHA 与 MQA 之间取得了平衡：将查询头
分为若干组，每组共享同一组 K 和 V 头。设 Q 头数为 $H_q$、KV 头数为 $H_{kv}$，则每
$H_q / H_{kv}$ 个 Q 头共享一对 K、V 头（本文后续章节统一使用 $H_q$ 和 $H_{kv}$）。
GQA 在保持模型质量的前提下，显著减少了 KV 投影的参数量与 KV Cache 的存储开销。

以本文实验所用的 Qwen2.5 系列模型 \cite{qwen2025qwen25} 为例，Qwen2.5-1.5B-Instruct
具有 12 个 Query 头与 2 个 KV 头（即每 6 个 Query 头共享一对 KV 头），
而 Qwen2.5-7B-Instruct 具有 28 个 Query 头与 4 个 KV 头（即每 7 个 Query 头共享一对 KV 头）。这一设计使得 KV Cache 的存储量仅为标准 MHA 的 $1/6$ 至 $1/7$，
为长序列推理带来了可观的显存节省。

\section{KV Cache 机制与显存分析}
\label{sec:ch2-kv-memory}
% ============================================================

\subsection{KV Cache 的作用与工作原理}

在自回归解码过程中，模型逐词元生成输出序列。在生成第 $t$ 个词元时，需要计算当前词元的
查询向量 $\bq_t$ 与所有历史词元（位置 $1$ 至 $t-1$）的键向量 $\bk_1, \ldots, \bk_{t-1}$
的注意力分数，并对相应的值向量 $\bv_1, \ldots, \bv_{t-1}$ 进行加权求和。如果每步都
重新计算所有历史词元的 K、V 投影，计算复杂度将随已生成序列长度的增加而线性增长，导致
严重的计算冗余。

KV Cache 机制通过将每层每步计算得到的 K、V 张量缓存在 GPU 显存中来避免重复计算。具体而言，
推理过程分为两个阶段：

\textbf{预填充（Prefill）阶段}：一次性处理全部提示词（prompt）词元，计算所有层的 K、V 矩阵
并存入缓存。此阶段为计算密集型（compute-bound），可充分利用 GPU 的并行计算能力。

\textbf{解码（Decode）阶段}：每步仅处理新生成的一个词元，计算该词元对应的 $\bk_t$、$\bv_t$，
将其追加到 KV Cache 中，然后从缓存读取完整的历史 K、V 序列进行注意力计算。此阶段为
访存密集型（memory-bound），因为每步仅有一个词元的计算量，但需要读取整个 KV Cache。

\subsection{KV Cache 显存占用分析}

对于一个具有 $L$ 层 Transformer 解码器的模型，每层有 $H_{kv}$ 个 KV 注意力头，
每个头的维度为 $d_k$，当序列长度为 $S$ 时，KV Cache 的总显存占用可表示为：
\begin{equation}
  M_{\mathrm{KV}} = 2 \times L \times H_{kv} \times d_k \times S \times b
  \label{eq:kv_mem}
\end{equation}
其中因子 $2$ 表示 K 和 V 各占一份存储，$b$ 为每个元素的字节数（FP16 时 $b = 2$，
INT8 时 $b = 1$，INT4 时 $b = 0.5$）。

以 Qwen2.5-7B-Instruct 为例，该模型具有 $L = 28$ 层、$H_{kv} = 4$ 个 KV 头、
$d_k = 128$ 的头维度。在序列长度 $S = 32768$（32K）、FP16 存储条件下，单个请求的
KV Cache 显存占用为：
\begin{equation}
  M_{\mathrm{KV}} = 2 \times 28 \times 4 \times 128 \times 32768 \times 2
    \approx 1.75 \; \mathrm{GB}
  \label{eq:kv_mem_example}
\end{equation}

这意味着即便不考虑模型权重本身的存储（约 14~GB），仅单个请求在 32K 上下文下就需要
近 1.75~GB 的 KV Cache 显存。在批处理场景下，该开销随批大小线性增长。当处理 128K 甚至
更长的上下文时，KV Cache 的显存需求将成为推理系统的主要瓶颈，严重限制可服务的并发
请求数与最大序列长度。这一现实需求推动了 KV Cache 量化压缩技术的研究。

\section{模型量化技术基础}
% ============================================================

量化（Quantization）是一种模型压缩技术，通过将浮点数值映射为低比特整数表示来减少存储
开销和计算代价。本节介绍模型量化的基本原理，涵盖对称量化、非对称量化以及校准方法，
为后续 KV Cache 量化方案的讨论奠定理论基础。

\subsection{对称量化}

对称量化（Symmetric Quantization）将浮点数值映射到以零为中心的均匀整数网格上。
给定一个浮点张量 $\bx$ 和目标比特宽度 $n$，对称量化的映射过程如下 \cite{jacob2018quantization}：

首先，计算量化缩放因子（scale）：
\begin{equation}
  s = \frac{\max(|\bx|)}{2^{n-1} - 1}
  \label{eq:sym_scale}
\end{equation}

然后，将浮点值映射为整数：
\begin{equation}
  x_q = \mathrm{clamp}\!\left(\left\lfloor \frac{x}{s} \right\rceil, \; -2^{n-1}, \; 2^{n-1} - 1\right)
  \label{eq:sym_quant}
\end{equation}
其中 $\lfloor \cdot \rceil$ 表示四舍五入取整，$\mathrm{clamp}(\cdot)$ 将结果截断到
合法整数范围内。反量化（Dequantization）过程为 $\hat{x} = x_q \times s$，由此引入的
量化误差为 $\epsilon = x - \hat{x}$。

对于 INT8 对称量化，整数表示范围为 $[-127, 127]$，可表示 255 个离散值，通常足以在
不显著损失精度的前提下有效压缩张量。对于 INT4 对称量化，整数范围缩小为 $[-7, 7]$，
仅有 15 个离散值，量化误差显著增大，需要更精细的校准策略以控制精度损失。INT4 量化
还支持比特打包（bit-packing）优化：将两个 4 位整数值存储在一个 8 位整数中，进一步
将存储空间压缩至原始 FP16 的 $1/4$。

在实际应用中，逐张量（per-tensor）量化为整个张量使用单一 scale，实现简单但对异常值
敏感。分组量化（group-wise quantization）将张量沿特定维度按固定分组大小（group size）
划分为多个子组，每组独立计算 scale，能够更精细地适应局部数值分布，在量化精度与存储
开销之间取得更好的平衡。

\subsection{非对称量化}

非对称量化（Asymmetric Quantization）通过引入零点（zero point）偏移来处理数值分布
不对称的场景。给定浮点张量 $\bx$ 和目标比特宽度 $n$，非对称量化的映射定义为：
\begin{equation}
  s = \frac{\max(\bx) - \min(\bx)}{2^n - 1}, \qquad
  z = \mathrm{clamp}\!\Bigl(\bigl\lfloor -\tfrac{\min(\bx)}{s} \bigr\rceil, \; 0, \; 2^n{-}1\Bigr)
  \label{eq:asym_params}
\end{equation}
其中 $s$ 为缩放因子，$z$ 为零点。量化映射为：
\begin{equation}
  x_q = \mathrm{clamp}\!\left(\left\lfloor \frac{x}{s} \right\rceil + z, \; 0, \; 2^n - 1\right)
  \label{eq:asym_quant}
\end{equation}
反量化过程为 $\hat{x} = (x_q - z) \times s$。

相比对称量化，非对称量化能够更好地利用有限的整数范围来覆盖非零中心的数值分布。
在 KV Cache 量化的语境下，研究表明 Key 张量的不同通道往往具有不同的均值偏移，此时
逐通道（per-channel）非对称量化能够更准确地捕捉各通道的数值范围，减小量化误差。
例如，KIVI \cite{liu2024kivi} 方法即采用 per-channel 非对称量化处理 Key 矩阵，取得了
优于对称量化的效果。

\subsection{校准方法综述}

量化校准（Calibration）在量化前利用少量校准数据确定 scale、裁剪阈值等参数。
常见策略可按优化目标分为两类：
\emph{数值空间校准}以张量重建误差为目标，
代表方法包括百分位裁剪（抑制异常值影响）、
均方误差最小化~\cite{jacob2018quantization}；
\emph{分布空间校准}以输出分布相似性为目标，
代表方法是以 KL 散度为目标的 NVIDIA TensorRT INT8
激活校准方案~\cite{migacz2017tensorrt}：
\begin{equation}
  s^* = \argmin_{s} \; \KL\!\left(P_{\mathrm{orig}} \,\|\, P_{\mathrm{quant}}(s)\right)
  \label{eq:kl_calib}
\end{equation}
后者的核心思想是保持模型输出行为的一致性而非仅追求数值精度。
对于需要穿透 softmax 非线性变换的 KV Cache 量化场景，
通用校准方法的行为保真度不足以直接迁移，
这一不足正是下一节所综述的 KV Cache 专属方法试图解决的问题。

% ============================================================
\section{KV Cache 量化相关工作}
% ============================================================

随着大语言模型的上下文窗口不断扩大，KV Cache 的显存开销已成为制约长序列推理效率的关键
瓶颈。近年来的相关工作可沿三条线索梳理：
量化压缩与校准方法、K/V 非对称性与低比特修复策略、
以及 GQA 感知的高效推理优化。
本节对代表性工作进行系统综述，并定位本文以 $H_{kv}$ 为结构性关联变量的研究空白。
量化理论的全面综述可参见 Nagel 等人~\cite{nagel2021whitepaper} 的白皮书，
Hubara 等人~\cite{hubara2018quantized} 则系统讨论了低精度训练与推理的基础框架。

\paragraph{K/V 非对称格式与校准方向。}
KIVI~\cite{liu2024kivi} 提出无需微调的非对称 KV Cache 量化：
Key 采用逐通道（per-channel）、Value 采用逐词元（per-token）的非对称量化，
其核心观察是 Key 通道分布差异大而 Value 时序分布差异大。
\emph{范围披露：}本文的 KIVI-style baseline 实现了上述核心量化轴，
但不包含原论文可选的异步量化流水线与 token 分级存储等工程优化。
KVQuant~\cite{hooper2024kvquant} 通过非均匀码本和 Pre-RoPE Key 量化将精度推至
sub-4-bit 并规避 RoPE 旋转对量化网格的扭曲。
AsymKV~\cite{tao2024asymkv} 从层深度维度观察 K$>$V 敏感性不对称，
AQUA-KV~\cite{shutova2025aquakv} 利用 K/V 时序相关性构造自适应量化，
这些工作共同印证了 K/V 差异化处理的必要性。

\paragraph{异常值消除与混合精度。}
QuaRot~\cite{ashkboos2024quarot} 通过 Hadamard 旋转变换均匀化 Key 通道异常分布，
使标准对称量化即可有效工作；与本文从诊断角度揭示 Key 通道异质性形成互补——
两者可在未来工作中结合。
在混合精度与显著性方向，
ZipCache~\cite{he2024zipcache} 基于显著 token 差异化精度分配，
GEAR~\cite{kang2024gear} 以残差低秩近似补偿量化误差，
QServe~\cite{lin2024qserve} 提出 W4A8KV4 端到端联合量化，
IntactKV~\cite{liu2024intactkv} 与 Outlier Tokens Tracing~\cite{su2025outliertoken}
则以高精度保留枢纽 token。
QJL~\cite{zandieh2024qjl} 基于 Johnson-Lindenstrauss 随机投影
将压缩推至 1-bit 极限，SKVQ~\cite{duanmu2024skvq} 则引入滑动窗口差异化精度。

\paragraph{驱逐与量化的正交性。}
KV 驱逐是与量化正交的另一类压缩策略：
H$_2$O~\cite{zhang2024h2o}、StreamingLLM~\cite{xiao2024efficient}、
SnapKV~\cite{li2024snapkv}、PyramidKV~\cite{cai2024pyramidkv}、
DuoAttention~\cite{xiao2024duoattention} 与 HeadKV~\cite{fu2025headkv}、
ChunkKV~\cite{liu2025chunkkv} 等工作分别从注意力重要性、层级衰减、
逐头贡献度、语义 chunk 等角度决定 KV 对的保留/丢弃，
可与量化技术组合使用以进一步降低显存开销。

\paragraph{权重量化的方法学启示。}
虽非直接作用于 KV Cache，权重量化的核心思想对本文具有方法学参考价值：
SmoothQuant~\cite{xiao2023smoothquant} 通过等价变换平滑转移量化难度、
GPTQ~\cite{frantar2023gptq} 以 Hessian 信息实现误差补偿、
AWQ~\cite{lin2024awq} 以激活感知保护重要通道、
LLM.int8()~\cite{dettmers2022gpt3int8} 以混合精度处理异常值维度。
这些思想（非均匀码本、误差补偿、异常值处理）对 KV Cache 量化的极低比特方向有参考价值。

与上述工作相比，本文的核心特征在于以 GQA 头数 $H_{kv}$ 为结构性关联变量，
在同一框架中联合分析校准目标灵敏度、K/V 退化不对称性、温度校正方向
与融合解码核函数的效率特征。
在目前公开文献中，现有工作尚未在同一框架中将上述四个维度与 $H_{kv}$ 显式关联。

在方法层面，KV Cache 量化方案可从两个\textbf{正交}维度刻画：
（1）\emph{量化格式}，即对称/非对称、per-group/per-channel/per-token 等量化轴选择，
决定了整数网格的形状和覆盖模式；
（2）\emph{校准目标}，即 MSE、percentile、KL 散度等准则，决定了量化参数的优化方向。
现有工作中，KIVI 在量化格式维度引入了 K/V 非对称轴的创新，
但校准仍依赖在线动态统计量（absmax/min）；
本文在校准目标维度采用 attention-KL 作为行为对齐度量，
并在跨模型实验中观察到该度量的有效性呈 bit-width 与规模的双重依赖。
两个维度的改进在理论上不互斥，可组合使用。

此外，FP8 KV Cache（vLLM/TensorRT-LLM 已默认支持）是另一类工程路径，
其优势在于保留指数-尾数结构对异常值更鲁棒；
本文选择 INT8/INT4 路径主要出于压缩率考虑——
INT4 相对 FP8 提供 2$\times$ 压缩率优势，
且行为对齐校准原则与量化格式解耦，可迁移至 FP 路径。
FP8 与 INT 路径的系统性对比列入后续工作（第~\ref{sec:conclusion-future}~节）。

% ============================================================
\subsection{量化对注意力分布的影响}
\label{sec:ch2-quant-softmax}
% ============================================================

近期文献从不同角度观察到量化对 softmax 分布的扰动效应。
QeRL~\cite{agarwal2025qerl}（ICLR 2026）指出
量化固有地增加采样熵（``quantization inherently increases the sampling entropy''），
导致输出概率分布趋于平坦化，尽管其分析针对的是 output logits 而非 attention 内部。
KVTuner~\cite{xu2025kvtuner}（ICML 2025）通过 Lemma 1 数学证明了
Key 量化误差在 attention score 中被放大 13.9 倍（INT8$\to$INT4），
并观察到``高 KV cache 量化误差的注意力头通常展现非稀疏的注意力模式''。
Softmax Bias Correction~\cite{bondarenko2023softmax}（ICCV 2023 Workshop）
分析了 softmax 对量化的 SQNR 劣化 8 倍，指出量化后高达 99\% 的 softmax 值被舍入为零。

与温度校正最相近的先行工作是 Velickovic 等人的 \emph{Softmax-Not-Enough}
\cite{velickovic2024softmax}（NeurIPS 2024 Workshop），该工作证明 softmax 分布
锐度随序列长度衰减并在语言模型头（LM head）应用自适应温度；
以及 AhaKV~\cite{gu2025ahakv}（arXiv 2025），该工作基于注意力熵
构造\emph{全局} softmax scale 用于 KV Cache 驱逐的\emph{重要性打分}。
两者均确立了"softmax 温度作为可调参数"的方法学先例，但都与本文在三个维度上存在关键差异：
(i) 本文的 $\tau^{-1}_{l,h}$ 是\emph{逐头}而非全局；
(ii) 用于\emph{量化补偿}而非 LM-head 温度调整或驱逐打分；
(iii) 通过\emph{行为对齐 KL 校准}在离线阶段确定参数值，而非运行时启发式。

上述工作均未针对 KV Cache 量化场景系统报道温度校正的规模依赖性。
QeRL 聚焦于 output logits，KVTuner 的 Lemma 1 止步于误差放大的量化分析，
Softmax-Not-Enough 着眼于 LM head，AhaKV 用于驱逐打分；
四者均未将因果链延伸至 attention 内部 softmax 的平坦化补偿与 GQA 架构的交互。
在目前公开文献中，
尚无工作系统报告 KV Cache 量化中逐头温度校正 $\tau^{-1}$ 与 GQA 头数 $H_{kv}$
之间的尺度依赖反转（$H_{kv}=2$ 改善、$H_{kv}\geq 4$ 恶化）。
本文的行为对齐框架（第~\ref{sec:ch3-invtau}~节）在此基础上
对逐头温度校正因子 $\tau^{-1}$ 进行了系统的跨模型消融，
意外发现其有效性与 GQA 头数呈反向依赖
（第~\ref{subsec:exp-rolealign-invtau}~节）。

然而，校准策略的改进仅解决了"量化什么"的问题；
要将量化带来的显存节省转化为实际的推理加速，还需要在计算层面进行融合优化。

\section{高效注意力计算}
% ============================================================

注意力计算的效率是大语言模型推理系统的关键性能瓶颈之一。本节介绍高效注意力计算
领域的重要进展，包括 FlashAttention、PagedAttention 以及 Triton 编程模型，
这些技术与 KV Cache 量化在推理优化中具有互补关系。

\subsection{FlashAttention}

FlashAttention \cite{dao2022flashattention} 是一种 IO 感知的精确注意力计算算法。
标准注意力实现需要将完整的 $n \times n$ 注意力权重矩阵写入 GPU 高带宽显存（HBM），
当序列长度 $n$ 较大时，这一中间结果的读写开销成为性能瓶颈。FlashAttention 通过分块
计算（tiling）和在线 softmax（online softmax）技术，将注意力计算分解为多个可在
GPU 片上共享内存（SRAM）中完成的小块，避免了将完整注意力矩阵写回 HBM，从而将
注意力计算的 IO 复杂度从 $O(n^2)$ 降低至 $O(n^2 d / M)$（其中 $M$ 为 SRAM 大小），
在不引入任何近似的前提下实现了显著的速度提升和显存节省。

FlashAttention-2 \cite{dao2024flashattention2} 在此基础上进一步优化了 GPU 线程块间的
并行策略和工作分配方式。具体而言，FlashAttention-2 将外层循环调整为遍历 Q 块（而非 K、V 块），
减少了对共享内存的非矩阵乘法运算的访问，并在 Q 序列维度上实现了线程块级并行，
使得在长序列场景下的计算吞吐量进一步提升。

\subsection{PagedAttention 与 KV Cache 管理}

vLLM 系统 \cite{kwon2023vllm} 提出了 PagedAttention 技术，将操作系统中虚拟内存的
分页管理思想引入 KV Cache 管理。传统的 KV Cache 实现需要为每个请求预分配连续的显存
空间，按最大可能序列长度分配，导致严重的显存碎片和浪费。PagedAttention 将 KV Cache
划分为固定大小的物理块（block），通过块表（block table）实现逻辑序列到物理块的映射，
支持非连续存储和跨请求的 KV Cache 共享。这一设计使得 GPU 显存利用率从约 20\%--40\%
提升至接近 100\%，显著提高了推理系统的吞吐量。

PagedAttention 与 KV Cache 量化技术具有天然的互补性：量化减少每个词元的 KV 存储字节数，
PagedAttention 优化存储的分配与管理方式，两者结合能够在更有限的 GPU 显存预算下服务
更多并发请求和更长的上下文。

\subsection{Triton 编程模型}

Triton \cite{tillet2019triton} 是一种面向 GPU 计算的领域特定语言（Domain-Specific Language,
DSL），旨在降低高性能 GPU kernel 编写的门槛。与直接使用 CUDA 编程相比，Triton 提供了
更高层次的抽象：开发者以分块（tile）为粒度组织计算逻辑，编译器自动处理线程映射、
共享内存管理、访存合并等底层优化。

在 KV Cache 量化场景下，Triton 支持将反量化与注意力计算融合为单一 kernel：
在 SRAM 中就地反量化并直接参与注意力，避免中间浮点张量的 HBM 读写开销。
本文第三章基于 Triton 实现了 INT8 与 INT4 融合 decode attention kernel。

\section{本章小结}
% ============================================================

基于上述综述，现有 KV Cache 量化方法存在以下\textbf{研究空白}：

\begin{enumerate}[leftmargin=2em, itemsep=2pt]
  \item \textbf{校准目标与注意力行为脱节。}
    KIVI、KVQuant 等方法的校准目标侧重于张量级别的数值精度（MSE 或百分位裁剪），
    缺乏对量化前后注意力权重分布一致性的显式优化。
    在 softmax 非线性变换下，数值误差的最小化并不等同于注意力行为的保真。
  \item \textbf{INT4 长上下文质量退化缺乏深入分析。}
    多数工作在短上下文评测中报告了可接受的精度，
    但对 32K 及以上长上下文下的系统性质量退化（尤其是精确检索任务）缺乏深入讨论。
  \item \textbf{量化与计算的融合优化不足。}
    主流推理框架对量化 KV Cache 的支持仍以"先反量化、再计算"的两阶段方案为主，
    未充分利用融合核函数消除中间张量的访存开销。
  \item \textbf{K/V 不对称敏感性缺乏系统性消融。}
    KIVI~\cite{liu2024kivi} 等工作观察到 Key 和 Value 在统计特性上的差异，
    并据此设计了非对称量化轴（Key 逐通道、Value 逐 token），
    但尚未通过系统性的比特宽度消融实验来定量分析
    K 和 V 各自对量化精度的独立敏感性，
    也未探究该敏感性在不同模型架构（特别是不同 GQA 头数）间的差异模式。
  \item \textbf{缺乏以 $H_{kv}$ 为统一关联变量的跨维度分析。}
    上述四个空白并非孤立：校准灵敏度、K/V 退化、温度校正与融合核效率
    均可能受 GQA 头数 $H_{kv}$ 的结构性影响，
    但现有工作尚未将 $H_{kv}$ 作为统一的分析轴联合考察。
\end{enumerate}

针对这些问题，下一章将详细介绍以 attention-KL 为统一校准与诊断原则的
行为对齐量化框架，包括两阶段校准策略、非对称格式上的 RoleAlign 实例化、
以及 Triton 融合量化解码核函数的设计。
第四章将以 $H_{kv}$ 为结构性关联变量，
通过跨模型受控实验系统填补上述研究空白。
