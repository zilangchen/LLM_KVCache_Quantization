\begin{abstractzh}

大语言模型的上下文窗口从数千扩展至数十万 token，
键值缓存（KV Cache）的显存占用随序列长度线性增长，
已成为制约长上下文推理部署的核心瓶颈。
INT4 量化理论上可将 KV Cache 压缩至原始的 $1/4$，
但现有方法以数值重建误差进行校准，
无法解释对称 INT4 在长上下文下的灾难性失效：
Needle 检索通过率从 100\% 降至 0\%，困惑度退化超过 15\%。

本文提出以注意力权重分布的 KL 散度（attention-KL）
作为统一的校准目标与诊断透镜。
作为校准目标，它搜索使注意力行为偏移最小化的量化参数；
作为诊断透镜，它揭示不同量化配置下的失效模式与结构性根因。
该原则首先在 INT8 层面得到验证：
行为对齐校准在三个模型族上保持质量的同时
实现 KV Cache 显存降低约 44\%、解码延迟降低 8--38\%，
为低比特诊断建立了参照基线。
进而，同一诊断透镜揭示对称 INT4 失效的结构性根因。
受控 K/V 消融一致确认 Key 量化主导质量损失，
且退化呈显著的分组查询注意力（GQA）架构依赖特征。

除此之外，诊断框架还意外揭示了一个二阶规律：
逆温度校正（inv\_tau）与 GQA 头数呈反向依赖，
在 $H_{kv}=2$ 时改善 PPL 1.6\%，
而在 $H_{kv} \geq 4$ 时反而恶化 3.4--6.0\%。
该现象与 GQA 噪声稀释的直觉论证一致
（$\sigma_{\text{eff}} \propto \sigma / \sqrt{N_{\text{rep}}}$），
体现了行为对齐诊断框架的二阶价值：
在优化量化参数的主要目标之外，
它能够意外暴露架构与噪声机制之间的耦合约束。

上述诊断直接导出本文的核心贡献 INT4-RoleAlign，
一种诊断驱动的角色感知低比特设计：
对 Key 和 Value 采用差异化的非对称量化轴
（per-channel Key + per-token Value），
通过行为对齐 percentile 搜索确定校准参数。
INT4-RoleAlign 在 Qwen2.5（1.5B/7B）、LLaMA-3.1-8B 三个模型上
将 Needle 检索从 0\% 恢复至 100\%，
实现 73\% 的 KV Cache 压缩。
PPL 退化因模型规模而异（8B: 2.4\%，7B: 6.1\%，1.5B: 13.7\%），
表明局部信息检索对量化噪声的鲁棒性远高于全局语言建模质量。
从原则到诊断、从设计到边界，
行为对齐校准贯通了递进的逻辑链路。

\keywordszh{大语言模型；键值缓存；量化；行为对齐校准；非对称量化；GQA 架构；温度校正}

\end{abstractzh}
