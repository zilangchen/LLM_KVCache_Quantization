\begin{abstractzh}

随着大语言模型上下文窗口从数千扩展至数十万 token，
键值缓存（KV Cache）的显存占用随序列长度线性增长，
已成为制约长上下文推理部署的核心瓶颈。
低比特量化是最具吸引力的压缩手段——
INT4 理论上可将 KV Cache 压缩至原始的 $1/4$，
但现有方法以数值重建误差进行校准，
无法解释对称 INT4 在长上下文下的灾难性失效：
Needle 检索通过率从 100\% 降至 0\%，困惑度退化超过 15\%。

本文的核心洞察在于：
KV Cache 量化的关键不是最小化数值重建误差，
而是保持注意力行为的一致性。
据此，本文提出以注意力权重分布的 KL 散度（attention-KL）
作为统一的校准目标与诊断透镜：
作为校准目标，它搜索使注意力行为偏移最小化的量化参数；
作为诊断透镜，它揭示不同量化配置下的失效模式与结构性根因。

该原则最终导出本文的核心贡献——INT4-RoleAlign，
一种诊断驱动的角色感知低比特设计：
对 Key 和 Value 采用差异化的非对称量化轴
（per-channel Key + per-token Value），
通过行为对齐 percentile 搜索确定校准参数。
INT4-RoleAlign 在 Qwen2.5（1.5B/7B）、LLaMA-3.1-8B 三个模型上
将 Needle 从 0\% 恢复至 100\%，
PPL 退化从 KIVI-style 的 2.4\%--12\% 降至 0.3\%--1.2\%，
在 75\% 的 KV Cache 压缩率下实现上述质量保持。

INT4-RoleAlign 的设计源于行为对齐原则的完整论证链。
该原则首先在 INT8 上得到验证——
行为对齐校准在三个模型族上保持质量的同时
实现 KV Cache 显存降低约 44\%、解码延迟降低 8--38\%，
为低比特诊断建立了参照基线。
同一诊断透镜揭示对称 INT4 失效的结构性根因：
受控 K/V 消融一致确认 Key 量化主导质量损失
且呈显著的分组查询注意力（GQA）架构依赖特征，
直接导出 INT4-RoleAlign 的角色感知设计。

本文的结果表明：行为对齐校准不仅是更优的量化目标，
更能揭示失效根因并导出角色感知的低比特设计，
完成从原则到诊断到设计的完整闭环。

\keywordszh{大语言模型；键值缓存；量化；行为对齐校准；非对称量化}

\end{abstractzh}
