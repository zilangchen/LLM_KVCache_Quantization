\begin{abstractzh}

大语言模型的上下文窗口从数千扩展至数十万 token，
键值缓存（KV Cache）的显存占用随序列长度线性增长，
已成为制约长上下文推理部署的核心瓶颈。
INT4 量化理论上可将 KV Cache 压缩至原始的约 $1/4$，
但以数值重建误差为校准目标的现有方法
无法解释对称 INT4 在长上下文下的灾难性失效：
Needle 检索通过率从 100\% 跌至 0\%，
困惑度在最敏感的模型上退化至约 14\%。

本文的出发点是：KV Cache 量化应当保持注意力行为，
而不是最小化数值重建误差。
据此我们提出以注意力分布的 KL 散度（attention-KL）
作为一个兼具校准与诊断双重用途的目标——
它在离线阶段指挥量化参数的搜索，
当量化失效时，又让我们得以反过来看清失效发生在哪里、为什么发生。
这样就把校准和事后诊断置于同一个行为对齐的尺度之下。

本文的核心贡献是在该原则下给出 INT4-RoleAlign，
一种诊断驱动的角色感知低比特设计：
对 Key 和 Value 采用角色差异化的量化粒度
（per-channel Key 与 per-token Value），
并以行为对齐 percentile 搜索确定校准参数。
INT4-RoleAlign 在 Qwen2.5-1.5B/7B、LLaMA-3.1-8B 三个模型上
将 Needle 检索从对称 INT4 基线的 0\% 恢复至 100\%（对齐 FP16），
实现相对 FP16 约 73\% 的 KV Cache 压缩
（剩余约 2\% 来自 per-channel scale 与 zero-point 的元数据开销）。
wikitext-2 上的 PPL 退化与 GQA 头数呈反向依赖：
$H_{\text{kv}}{=}8$ 时 2.4\%、$H_{\text{kv}}{=}4$ 时 6.1\%、
$H_{\text{kv}}{=}2$ 时 13.7\%。
这种对比表明，局部检索对 INT4 量化噪声的容忍度
远高于全局语言建模质量。

该设计是由一条完整的诊断证据链导出的。
我们首先在 INT8 上验证原则：
行为对齐校准在 Qwen2.5-1.5B/7B 上 PPL 退化不足 0.3\%、Needle 保持 100\%，
同时实现 KV Cache 显存降低约 44\%、解码延迟降低 8--38\%；
LLaMA-3.1-8B 上对校准数据规模的敏感性作为已知局限在第四章披露。
同一诊断透镜随后揭示对称 INT4 失效的结构性根因：
受控 K/V 消融在三个模型族上一致确认 Key 量化主导质量损失，
并呈现与 GQA 架构密切相关的特征，
从而直接导出角色感知的量化设计。
INT4-RoleAlign 沿用 KIVI 的 per-channel Key / per-token Value 格式，
但以离线 attention-KL percentile 搜索替代其运行时 absmax 估计，
把校准信号从数值重建提升到注意力行为保真。

在这个诊断过程中，我们还意外发现了一条规律。
逆温度校正（inv\_tau）与 GQA 头数呈反向依赖：
在 $H_{\text{kv}}{=}2$ 时改善 PPL 1.6\%，
而在 $H_{\text{kv}}{\geq}4$ 时反而恶化 3.4--6.0\%。
这一规律与 GQA 噪声稀释的启发式论证定性一致
（共享 KV 头的噪声在下游聚合时看似被衰减），
但严格推导需要 i.i.d.\ 假设，
这在 GQA 结构下并不严格成立，完整形式化证明留作未来工作。
这条规律并非我们设计之初所追求的，
而是在同一次 KL 参数扫描中顺带浮现的。

综合来看，行为对齐不止于一个更好的校准目标。
它让我们看到 INT4 在哪里开始失效，
塑造了随后的非对称设计，
也清楚地界定了这一设计在哪里会开始失效。
以上结果基于 5 个种子的主实验
（另以 10 个种子验证 PPL 的确定性），
配以 Bootstrap 置信区间与 BH-FDR 多重比较校正后的显著性检验。

\keywordszh{键值缓存；量化；行为对齐校准；非对称量化；长上下文推理}

\end{abstractzh}
