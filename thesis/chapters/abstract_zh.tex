\begin{abstractzh}

大语言模型的上下文窗口从数千扩展至数十万 token，
键值缓存（KV Cache）的显存占用随序列长度线性增长，
已成为制约长上下文推理部署的核心瓶颈。
INT4 量化理论上可将 KV Cache 压缩至原始的 $1/4$，
但现有以数值误差为校准目标的方法在长上下文下
表现急剧恶化：Needle 检索通过率可从 100\% 降至 0\%，
困惑度退化超过 15\%。
本文从注意力行为保真的角度重新审视校准目标，
以量化前后注意力分布的 Kullback--Leibler 散度
（attention-KL）作为统一的校准与诊断原则，
并以 GQA 头数 $H_{kv}$ 为结构性关联变量
联合分析校准灵敏度、退化模式与融合解码效率。

在 Qwen2.5-1.5B/7B/14B 与 LLaMA-3.1-8B 四个模型上
（其中 14B 作为外部效度锚点，仅 1 seed 覆盖 Needle 4K--32K 与 RULER 4K--16K，
不计入 PPL 单调趋势声明），
研究发现校准目标的有效性呈 bit-width 与规模依赖：
INT8 下两种目标完全趋同，
INT4 下在小模型（$H_{kv}=2$）上分化而在大模型（$H_{kv}=4$）上再次趋同。
受控诊断表明 INT4 退化由 Key 侧主导，
14B 上保持 Key 为 FP16 可恢复 93\% 的 PPL 退化；
在 INT4-RoleAlign 配置下 Needle retrieval 与 PPL 在四个模型上一致解耦
（其他 INT4 配置如 symmetric-only 下 Needle 随 PPL 同降）。
该诊断指向 per-channel Key + per-token Value 的差异化格式——
与 KIVI~\cite{liu2024kivi} 从工程经验出发独立提出的格式一致。
在该格式上以离线 attention-KL 搜索实例化的 INT4-RoleAlign
在四个模型上保持 Needle 100\%，KV Cache 压缩约 73\%，
PPL 退化随模型规模减弱（1.5B: 13.7\%，7B: 6.1\%，8B: 2.4\%）。
14B 提供外部效度：Needle 在 4K--32K 全通过，
RULER 在 4K--16K 达到 96.6\%--98.5\%；
需说明 14B 的 PPL 退化为 7.6\%，
与 1.5B$\to$8B 观察到的规模单调递减趋势不一致，
应视为在更大规模外部效度锚点下的补充观察而非主趋势的一部分。
同时需明确，INT4-RoleAlign 相对 KIVI-style baseline 在 PPL 数值上未体现一致优势，
其贡献在于以 attention-KL 诊断透镜为 per-channel K $+$ per-token V
非对称格式提供可解释的离线校准接口，而非单纯量化质量提升。
融合量化解码核函数的效率增益
在 $(H_{kv}, \text{seq\_len})$ 空间中呈规模依赖：
$H_{kv}=8$ 在 32K 下延迟减少约 40\%，
$H_{kv}=2$ 下融合核在所有测试长度上均慢于 SDPA。

\keywordszh{大语言模型；键值缓存量化；行为对齐校准；非对称量化；GQA 架构}

\end{abstractzh}
