\begin{abstractzh}

随着大语言模型长上下文推理窗口不断扩大，其自回归推理过程中键值缓存的显存占用随序列长度线性增长，因此在真实部署场景中，解码瓶颈不仅仅在计算负载上，同时也转向持续增长的 KV Cache 存储与访存开销。降低缓存位宽能够直接减少显存占用，但现有量化路径常用的张量级重建误差校准并不一定能充分刻画量化缓存进入注意力计算后的功能偏移。Key 侧扰动可能经 logits 与 softmax 改写注意力分布，Value 侧扰动则主要通过加权聚合影响输出表示。基于这一传播差异，本文提出一种面向高效推理的 KV Cache 行为对齐量化框架，将注意力分布、聚合输出与任务表现作为共同的校准和审计对象，连接量化参数选择、低比特恢复和层间预算分配三个决策环节。

本文首先在 INT8 位宽下建立以 attention-distribution KL 作为量化校准目标的基准路径，用于检验行为引导校准在保守位宽下的保真效果。随后提出 \texttt{INT4-RoleAlign}，在 per-channel Key 与 per-token Value 的非对称格式下，分别刻画 Key 侧注意力排序偏移和 Value 侧输出扰动，避免将两类传播机制压缩为单一重建目标。离线校准产物进一步被聚合为逐层行为敏感度画像，并用于 behavior-guided fixed-$k$ 分配、位置启发式基线比较和 \texttt{AutoK} 覆盖率预算建议，形成从校准到预算分配的一致证据链。

KV Cache 量化与预算分配在覆盖 Qwen2.5、LLaMA-3.1 与 Mistral 三个模型族、六个开源指令模型的实验中呈现明确的模型族、规模和任务依赖性。具体来看，INT8 基准路径在保守位宽下保持了接近 FP16 的任务读数，说明以注意力分布 KL 组织校准可以形成稳定的行为保真锚点。进入 INT4 后，对称量化的失稳主要表现为 Key 侧低比特噪声触发注意力排序偏移，问题不只是 bit 数下降，而是扰动落入了更敏感的计算路径；采用 per-channel Key 与 per-token Value 的角色感知格式后，低比特检索行为可以恢复到可用区间。跨模型预算实验进一步表明，层级敏感度并不存在统一形状，\texttt{AutoK}、早层保护、位置启发式和 uniform 策略分别落在不同适用区间，因此预算分配应被理解为模型族、规模和任务条件共同约束下的候选选择问题。系统读数则显示，INT4 带来的缓存容量收益稳定可预期，而融合解码的时间收益需要结合 $H_{kv}$、序列长度和后端实现判断。总体而言，本文构建了一条以注意力行为保持为核心、结合 K/V 角色差异恢复与条件化预算分配的 KV Cache 行为对齐量化框架。

\keywordszh{大语言模型；KV Cache 量化；注意力行为保持；低比特推理；行为引导校准；预算分配}

\end{abstractzh}
