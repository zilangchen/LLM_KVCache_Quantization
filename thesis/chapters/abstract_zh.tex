\begin{abstractzh}

长上下文推理窗口扩大后，自回归解码需要持续保留越来越长的 K/V 历史，KV Cache 的显存占用和读取开销随序列长度同步增长。降低缓存存储位宽可以直接减少容量压力，但量化后的缓存并不是停在张量空间中被使用。Key 侧扰动会先进入 logits 与 softmax，并可能改变注意力质量的分配，Value 侧扰动则在注意力权重下进入输出表示。仅用张量级重建误差选择量化参数，难以完整描述这两条传播路径带来的功能偏移。围绕这一问题，本文构建面向高效推理的 KV Cache 行为对齐量化框架，以注意力分布、聚合输出和任务表现作为校准与审计读数，贯通量化参数选择、低比特恢复和层间预算分配。

方法设计从一组校准样本开始。全精度参考配置与候选量化配置并行前向，系统记录 attention-distribution KL 和注意力聚合输出扰动，并把下游任务读数保留为任务层审计信号。这些读数先用于筛选校准参数，随后按层汇总为行为敏感度画像，用来观察不同模型的关键层分布。围绕这组画像，本文比较 behavior-guided fixed-$k$ 分配、位置启发式基线和 uniform 策略，并用 \texttt{AutoK} 按敏感度排序与覆盖率阈值生成模型级 $k$ 候选。由此，离线行为读数不再只停留在校准阶段，而是继续进入低比特恢复、跨模型结构判读和预算候选生成。

在 Qwen2.5、LLaMA-3.1 与 Mistral 三个模型族的六个开源指令模型中，KV Cache 量化和预算分配呈现出清楚的模型族、规模和任务依赖性。Qwen2.5-1.5B 的 task-core 协议中，INT8 基准路径相对 \texttt{FP16} 的三任务均值差约为 $+0.02$，表明行为引导校准在保守位宽下可以形成稳定参照。位宽进入 \texttt{INT4} 后，对称量化在 Qwen 系列中触发检索阶跃崩塌，K/V 角色诊断显示 Key 侧低比特噪声是更直接的失稳来源。角色感知低比特路径 \texttt{INT4-RoleAlign} 将检索行为恢复到可用区间。跨模型预算实验进一步读出四类结构，Mistral-7B 给出最清晰的 \texttt{AutoK} 单模型支持，Qwen2.5-3B 对应早层保护有效区间，LLaMA-3.1-8B 呈现中等规模共识区间，Qwen2.5-14B 则表现为高性能簇接近而无稳定最优。\texttt{INT4} 路径带来约 $73.4\%$ 的 KV Cache 容量压缩，融合解码的时间收益还需要结合 $H_{kv}$、序列长度和后端实现判断。上述结果说明，以注意力行为保持为核心、以 K/V 角色差异恢复为低比特路径、以 \texttt{AutoK} 和行为敏感度画像为预算接口的框架，可以把 KV Cache 量化从单纯降位宽推进到可审计的行为对齐问题。

\keywordszh{大语言模型；KV Cache 量化；注意力行为保持；低比特推理；行为引导校准；预算分配}

\end{abstractzh}
