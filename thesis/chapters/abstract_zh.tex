\begin{abstractzh}

长上下文推理使大语言模型的解码瓶颈从计算负载进一步转向持续增长的 KV Cache 存储与访存开销。降低缓存位宽能够直接减少显存占用，但张量级重建误差并不能充分刻画量化缓存进入注意力计算后的功能偏移：Key 侧扰动可能经 logits 与 softmax 改写注意力分布，Value 侧扰动则主要通过加权聚合影响输出表示。针对这一问题，本文提出一种面向高效推理的 KV Cache 行为对齐量化框架，将注意力分布、聚合输出与任务表现作为共同的校准和审计对象，用以连接量化参数选择、低比特恢复和层间预算分配三个决策环节。

方法上，本文首先构建以 attention-distribution KL 为代理目标的 INT8 基准路径，用于在保守位宽下建立可复核的行为保真锚点；随后提出 \texttt{INT4-RoleAlign}，在 per-channel Key 与 per-token Value 的非对称格式下，分别刻画 Key 侧注意力排序偏移和 Value 侧输出扰动，避免将两类传播机制压缩为单一重建目标。离线校准产物进一步被聚合为逐层行为敏感度画像，并用于 behavior-guided fixed-$k$ 分配、位置启发式基线比较和 \texttt{AutoK} 覆盖率预算建议，从而形成从校准到预算分配的一致证据链。

在覆盖 Qwen2.5、LLaMA-3.1 与 Mistral 三个模型族、六个开源指令模型的实验中，KV Cache 量化与预算分配呈现明确的模型族、规模和任务依赖性。在 Qwen2.5-1.5B 上，INT8 基准路径相对 FP16 的三任务均值差约为 $+0.02$，支持其作为保守位宽下的行为保真实例；对称 INT4 在 Qwen 系列上出现检索阶跃崩塌，K/V 角色诊断进一步显示 Key 侧低比特噪声是更直接的失稳触发源。跨模型分配实验表明，Mistral-7B 上 \texttt{AutoK} 给出最清晰的单模型支持，core mean 为 14.76，extend mean 为 15.69；Qwen2.5-3B 呈现早层保护有效区间，其中 behavior-guided $k=1$ 的 core mean 为 6.90，高于中层 heuristic 的 3.48；Qwen2.5-14B 中 Uniform INT4 与 AutoK-cov90 的 core mean 分别为 7.23 与 7.15，说明高性能策略接近而无稳定最优。系统评估显示，INT4 路径在四个代表性模型上可实现约 73.4\% 的 KV Cache 容量压缩，但融合解码的时间收益受 $H_{kv}$、序列长度与后端实现共同调制。本文因此表明，KV Cache 量化不应被简化为统一位宽压缩问题，而应被建模为注意力行为保持、K/V 角色差异恢复与条件化预算分配共同决定的结构化问题。

\keywordszh{大语言模型；KV Cache 量化；注意力行为保持；低比特推理；行为引导校准；预算分配}

\end{abstractzh}
