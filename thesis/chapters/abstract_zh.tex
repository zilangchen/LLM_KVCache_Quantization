\begin{abstractzh}

大语言模型的上下文窗口从数千扩展至数十万 token，
键值缓存（KV Cache）的显存占用随序列长度线性增长，
已成为制约长上下文推理部署的核心瓶颈。
INT4 量化理论上可将 KV Cache 压缩至原始的 $1/4$，
但现有以数值误差为校准目标的方法在长上下文下
表现急剧恶化：Needle 检索通过率可从 100\% 降至 0\%，
困惑度退化超过 15\%。
这一矛盾提示我们从注意力行为保真的角度重新审视校准目标，
以量化前后注意力分布之间的 Kullback--Leibler 散度
（attention-KL）刻画行为偏移。

在 Qwen2.5-1.5B 上、在贪心解码与
robust selection（$\max~\text{clip rate}\leq 1\%$）约束下，
我们直接比较 attention-KL 与 MSE 两种校准目标的作用，
发现校准目标的有效性呈明显的 bit-width 依赖。
在 INT8 下，两种目标对
$(\text{clip\_percentile},\text{group\_size})$
的选择完全趋同，产出的每组 per-group scale 在 float32 下
逐位一致；而在 INT4 下，两种目标分别选择
$\text{clip\_percentile}=99.5$ 与 $99.0$，
所有 scale 元素不同，中位相对差异 12.35\%。
在本文 INT4 设定下，数值代理不足以稳定替代行为对齐目标。

进一步的受控诊断揭示了两个结构性特征：
通过 K-only 与 V-only 因子分解，
INT4 退化在三个模型族上一致呈 Key 主导——
仅对 Key 量化为 INT4 即复现全 INT4 的崩溃，
而仅对 Value 量化为 INT4 几乎无影响；
同时 Needle retrieval 与 PPL 之间存在明显解耦——
同一 INT4 配置可以在 Needle 上保持 100\% pass rate
的同时 PPL 退化 13.7\%。
基于这些诊断，我们在 KIVI~\cite{liu2024kivi} 提出的
per-channel Key + per-token Value 非对称格式上，
以离线 attention-KL 搜索为每层确定 BA percentile，
得到行为引导的实例化 RoleAlign。
在上述四个模型上，RoleAlign 以约 73\% 的 KV Cache 压缩率
保持 Needle 检索 100\%，
PPL 退化随模型规模增大而减弱
（1.5B: 13.7\%；7B: 6.1\%；8B: 2.4\%；14B: 6.1\%）；
在 LLaMA-3.1-8B 的 65{,}536-token Needle 设定下，
RoleAlign 复现了 FP16 自身的 long-context failure mode。

此外，我们观察到逐头温度校正 $\tau^{-1}$ 呈现
GQA 头数相关的架构依赖（$H_{kv}=2$ 时略有改善，
$H_{kv}\geq 4$ 时反而恶化），
因此将其定位为可选启发式而非常规组件。

\keywordszh{大语言模型；键值缓存；量化；行为对齐校准；非对称量化；GQA 架构}

\end{abstractzh}
