\subsection{评测任务、数据与指标}
\label{subsec:ch4-benchmarks}
\label{subsec:exp-benchmarks}

本章的评测从两类问题展开。一类关心量化后模型行为是否仍然稳定，另一类问题关心缓存变小之后，真实推理中的显存占用、访存压力和解码时间是否同步下降。PPL 检查固定窗口内语言建模概率质量的整体漂移。Needle-in-a-Haystack~\cite{kamradt2023needle} 用精确匹配通过率观察长距离目标片段是否还能被取回。LongBench 风格合成任务把比较推进到任务输出层面，并在同一数据生成与评分管线内读出方法差异。RULER~\cite{hsieh2024ruler} 则提供更高压力的组合式长上下文检查。官方 LongBench 真实数据对照放在第~\ref{subsec:ch4-longbench-official}~节，用于检查评测协议与外部真实数据方向是否一致。正文主评测矩阵以 LongBench 风格合成任务为核心。

PPL 部分统一采用 32K 上下文评测协议，\texttt{chunk\_size} 固定为 128。交叉熵累计时，每个位置可访问的历史窗口由这一设置限定。显存充足的模型按完整测试集报告 PPL。Qwen2.5-14B 受 H20 显存约束，使用测试集固定前缀评测。因此 14B 的 PPL 绝对值用于该模型内部对照，跨模型读数仍按同协议结果解释。

Needle-in-a-Haystack 捕捉检索型失效，实验覆盖 4K、8K、16K 与 32K 四个上下文长度，并在每个长度内使用统一的 needle 深度设置扫描。正文报告精确匹配通过率，它作为长距离目标片段取回能力的主读数。

任务级主矩阵采用 LongBench 风格合成任务。task-core 子集覆盖单文档问答、多文档问答和摘要三类功能，问答任务使用 F1，摘要任务使用 Rouge-L，少量扩展任务使用 Edit Similarity 等文本相似度指标。该协议服务于本文内部的方法比较，分数按同一生成和评分口径解释，并与社区官方榜单保持区分。

RULER 检查组合式长上下文任务中的稳定性，正文使用宏平均通过率汇总。Needle 是单点检索探针，暴露目标片段取回的临界点。RULER 覆盖多个子任务，观察长上下文能力是否在更复杂任务组中同步退化。两者共同构成长上下文行为侧的压力测试。

系统效率部分单独报告 TPOT 与 KV Cache 占用。TPOT 对应 Decode 阶段生成单个 token 的时间，KV Cache 占用对应缓存容量收益。峰值显存和 batch 维度容量边界作为附录扩展读数，用于补充审计整体运行压力。第~\ref{sec:ch4-deployment}~节据此讨论当前 H20 环境中的部署收益与时间边界。

\subsection{统计检验、显著性与数据溯源}
\label{subsec:ch4-statistics}
\label{subsec:exp-statistics}

第四章的证据按来源区分。正文主结果来自同一实验矩阵、同一解码设置和同一统计规则下重新执行并核验的读数，承担主要结论。官方 LongBench 小范围对照、完整 LongBench-style 聚合和上下文长度趋势作为协议一致性与补充审计材料。较早批次或回溯整理的补充读数进入正文时，在对应表注中标明来源和用途。

随机性控制采用统一的固定种子与统一生成策略。生成式质量评测统一使用 5 个固定 seeds，吞吐量测试统一使用 8 个固定 seeds。生成式任务采用贪婪解码，从而剥离采样噪声。
PPL 评测采用固定输入序列上的确定性 likelihood 累计。seed 用于实现路径一致性核验，不构成独立统计重复。
在此基础上，质量指标的区间估计采用基于 10{,}000 次重采样的 95\% Bootstrap 置信区间，配对比较采用符号翻转置换检验，多重比较则使用 Benjamini--Hochberg 控制假发现率。在小样本下，Bootstrap 区间稳定性与 sign-flip 检验 power 都有限，因此正文中的“未显著”表示当前样本下不可分辨。

PPL 使用固定输入序列、固定 tokenization、固定窗口与固定 loss mask，属于确定性 likelihood 指标。其可靠性主要来自协议一致性与效应量，正文读取数值幅度，不读取统计显著性。
任务级分数、吞吐量等会随种子或系统噪声波动的指标，才配合 Bootstrap、sign-flip 与 BH-FDR 一并解释。

后文主表、主图与典型剖面均沿用上述层级。各协议的绝对分数按各自口径读取。官方 LongBench 真实数据对照服务于第~\ref{sec:ch4-rq1}~节的一致性检验。完整 LongBench-style 聚合、统计检验汇总与上下文长度质量趋势见附录第~\ref{sec:app-longbench-full}~节。部署效率读数对应当前 H20 硬件、固定软件栈与当前后端组合。

\section{低比特路径从对称失效到 RoleAlign 恢复}
\label{sec:ch4-rq2}
\label{sec:exp-int4-kivi}
\label{sec:exp-main}

第~\ref{sec:ch4-rq1} 节验证了 \texttt{INT8} 基准位宽。本节把控制变量切换到低比特路径，沿对称格式可用性、K/V 触发侧定位和同一非对称格式内的校准来源比较展开。

本节的证据锚点依次是表~\ref{tab:ch4-int4-cliff}、表~\ref{tab:ch4-kv-ppl}、表~\ref{tab:ch4-kv-multitask} 与表~\ref{tab:ch4-rolealign-kivi}。它们分别支撑对称 \texttt{INT4} 在部分架构上跨过功能临界点、Key 侧低比特噪声是更直接的触发源、K/V 角色差异能在任务指标上复现，以及共享格式下 \texttt{INT4-RoleAlign} 与 \texttt{KIVI-style} 的主要差别来自校准来源这四个判断。

\subsection{对称 INT4 的架构依附性失效与阶跃崩塌}
\label{subsec:ch4-int4-cliff}

\texttt{INT8} 基准路径说明行为引导校准可以在较保守位宽上保持任务行为。位宽继续压到 \texttt{INT4} 后，对称量化面对的是另一类压力。有效量化等级从 \texttt{INT8} 的 255 个降到对称 \texttt{INT4} 的 15 个，部分模型直接跨过检索功能临界点，从“可用”跳到“不可用”。不同架构越过这一临界点的难易程度并不相同，因此本节将其表述为\textbf{架构依附性失效}。

表~\ref{tab:ch4-int4-cliff} 中最突出的现象，是对称 \texttt{INT4} 并没有带来连续的小幅退化。Qwen2.5-1.5B 和 Qwen2.5-7B 的 Needle 都从 FP16 下的 100\% 掉到 0\%，PPL 也分别从 $8.93$、$6.71$ 上升到 $19.54$、$85.49$。同一设置下，LLaMA-3.1-8B 仍保留 98\% Needle，PPL 从 $6.92$ 变为 $6.97$。这组对照显示，对称 \texttt{INT4} 在部分模型上越过了检索功能的触发边界。

这种阶跃形态与 Key 侧误差进入 attention logits 的方式有关。目标 token 的 $QK^\top$ 分数一旦被低比特噪声推出高排名区域，softmax 会把概率质量转移到其他位置，检索读数就可能从可用直接跌到不可用，而不一定表现为 50\%、30\%、10\% 这类线性滑落。LLaMA-3.1-8B 的例外也说明，这个边界受架构条件调制。当前读数与 $H_{kv}=8$ 以及较小的 $H_q/H_{kv}$ 重复因子相符，但模型族、规模和训练数据差异也可能共同作用。接下来需要拆开 K/V 两侧误差，定位更直接的触发源。

\subsection{K/V 角色敏感性与 MixedKV 完整诊断}

\section{部署效率评估}
\label{sec:ch4-deployment}
\label{sec:exp-deployment}

本节考察这些路径在真实推理系统中的时间与显存代价，重点观察低比特路径的缓存容量收益如何转化为解码时间收益，以及这种转化随 $H_{kv}$、序列长度和后端实现发生怎样的变化。

本节按“入口读数 $\rightarrow$ 长序列放大 $\rightarrow$ 经验边界 $\rightarrow$ 部署建议”的顺序组织。第~\ref{subsec:ch4-tpot-4k}~节给出 4K 序列下的后端路径入口读数。第~\ref{subsec:ch4-tpot-scaling}~节展示融合路径收益如何随上下文增长而放大。第~\ref{subsec:ch4-crossover}~节将这些读数整理为性能交叉边界。第~\ref{subsec:ch4-kvcache}~节结合 \texttt{KV Cache} 显存压缩率与当前硬件约束给出部署读数。

\subsection{性能交叉边界与 GQA 影响}
\label{subsec:ch4-crossover}

第~\ref{subsec:ch4-tpot-4k}~节给出入口层读数，第~\ref{subsec:ch4-tpot-scaling}~节给出单模型扩展证据。本节将这些读数整理为性能交叉边界。这里的边界指经验量 $\Delta T = T_{\mathrm{Triton}} - T_{\mathrm{ref}}$ 的符号变化。所有长度点上 $|\Delta T|<2$\,ms 的情况统一标记为不显著。本节关注融合路径何时进入可读出的加速区间，以及这条边界如何随 $H_{kv}$ 与序列长度共同移动。

表~\ref{tab:ch4-deployment-boundary} 的 Panel B 展示了 $H_{kv}$ 不同取值下的边界移动。Qwen2.5-1.5B（$H_{kv}=2$）的 $\Delta T$ 在 4K/8K/16K/32K 上分别为 +1.67 / +4.23 / +9.78 / +15.92\,ms，融合路径在全长度下都不划算。Qwen2.5-7B（$H_{kv}=4$）的 $\Delta T$ 在 4K/8K/16K 仍为正或近似零，直到 32K 才降为 -4.90\,ms，交叉点被推迟到更长序列。Qwen2.5-14B（$H_{kv}=8$）从 8K 起越过不显著区间，并在更长上下文下继续扩大。由这组路径可读出的经验边界是：在当前硬件与后端组合下，$H_{kv}$ 越高，交叉点越早。序列越长，融合路径越容易进入收益区间。

同 $H_{kv}=8$ 的 8B 与 14B 对照补充了这一边界。LLaMA-3.1-8B 与 Qwen2.5-14B 的参数规模相差约 1.75 倍，却共享相同的 $H_{kv}=8$。以表~\ref{tab:ch4-tpot-4k}~的入口层读数（生成长度 128）为统一协议，两者在 4K 处的交叉幅度高度一致，分别为 $-0.39$\,ms 与 $-0.40$\,ms。在当前模型集合中，$H_{kv}$ 比参数量更接近性能交叉边界的主导相关变量。

第~\ref{subsec:ch4-crossover}~节得到的系统读数是一条经验边界。$H_{kv}=2$ 基本落在融合路径不划算区，$H_{kv}=4$ 到长序列才越过边界，$H_{kv}=8$ 在较早长度点进入收益区间。部署路径选择由 $H_{kv}$ 与序列长度共同决定。

表~\ref{tab:ch4-kvcache-memory} 表明，INT4 路径在四个代表性模型上都实现了近乎相同的缓存压缩率。Qwen2.5-1.5B 从 115.5\,MB 压缩到 30.7\,MB，Qwen2.5-7B 从 230.9\,MB 压缩到 61.5\,MB，LLaMA-3.1-8B 从 527.9\,MB 压缩到 140.5\,MB，Qwen2.5-14B 从 791.8\,MB 压缩到 210.8\,MB，压缩率均约为 73.4\%。在当前 INT4 路径下，\texttt{KV Cache} 的显存收益稳定且可预估。

结合前文时间边界，可以得到当前平台上的部署读数。INT4 路径带来的显存节省是稳定容量收益，融合 decode 路径的速度收益则随 $H_{kv}$ 与序列长度变化。在 NVIDIA H20 (sm\_90, 130 SM, 98 GB HBM) 平台上，$H_{kv}=8$ 且序列长度达到 \texttt{8K} 及以上时，Triton 融合路径进入优势区间。$H_{kv}=4$ 的模型需要接近 \texttt{32K} 的长序列才越过交叉边界。$H_{kv}=2$ 或短于 \texttt{8K} 的序列仍更适合参考路径。跨硬件、跨 batch 或更换 backend 后，交叉边界需要重新测量。

附录第~\ref{sec:app-efficiency-plots}~节进一步给出 Qwen2.5-1.5B 设置下的 batch 维度容量边界、吞吐与显存扩展读数，用于补充审计本节部署建议的适用范围。

本节得到的系统层读数可以概括为容量收益与时间收益的分离。INT4 的 \texttt{KV Cache} 显存收益稳定可预期，Triton 融合路径的时间收益由 $H_{kv}$ 与序列长度共同调制。下一节转入对这些结果为何呈现出当前形状的机制辨析。

\section{实验结果讨论与效度威胁}
\label{sec:ch4-discussion}
\label{sec:exp-discussion}

第~\ref{sec:ch4-rq1}--\ref{sec:ch4-deployment} 节分别给出了基准路径保真、低比特恢复、跨模型策略适用区间与部署效率读数。本节把这些结果合并为四个机制判断，依次讨论 KL 与 MSE 校准目标、\texttt{INT4} 结果的分层机制、heuristic 强基线以及外推边界。

\subsection{KL 与 MSE 校准目标的规模依赖性机制辨析}
\label{subsec:ch4-disc-kl-mse}

第~\ref{sec:ch4-rq1}~节已经说明，\texttt{INT8} 基准路径在主协议与官方 LongBench 对照中都能保持接近 FP16 的任务读数。attention-distribution KL 的作用需要放到低比特扰动中观察。KV Cache 误差进入注意力计算后，关键风险往往表现为 attention ranking 的改变。第三章第~\ref{sec:ch3-calibration}~节给出了 KL 作为行为代理的形式化理由，本节进一步说明它与 MSE 这类数值重建代理在实验中的分化条件。

本章结果显示，KL 的作用强弱取决于扰动是否进入注意力分布层。对称 \texttt{INT4} 在 Qwen 系列中触发检索崩塌时，Key 侧低比特噪声已经足以改写关键 token 的相对位置，分布侧代理能够直接读到这类风险。若扰动被位宽冗余、模型鲁棒性或架构条件吸收，attention ranking 没有明显移动，KL 与 MSE 可能收敛到相同选择。

这种趋同已经出现在 Qwen2.5-7B-Instruct 的 \texttt{INT4-RoleAlign} 校准中。KL 与 MSE 最终都选择 $k_{\mathrm{pct}}=100.0$、$v_{\mathrm{pct}}=99.9$，对应 PPL 均为 7.1121，差值为 0.00。附录第~\ref{subsec:app-7b-kl-mse}~节保留了这组结果的溯源表。该读数说明，在这一设置中，模型条件已经压缩了两类代理的差异。

由此可见，KL 在本文中的价值来自行为代理层。attention ranking 容易被量化噪声改写时，KL 更贴近行为损伤，并能为后续敏感度画像和预算分配提供同源读数。模型规模、位宽或架构条件使扰动保持在稳定区间内时，KL 与 MSE 会逐步趋同。这个结果把 behavior-guided 主线落在可观察的分布偏移上，校准目标之间的差异也随模型条件而变化。

\subsection{INT4 结果的分层机制辨析}
\label{subsec:ch4-disc-int4}
\label{subsec:exp-int4-honest}

第 4.3 节的低比特结果呈现出清楚的分层。\texttt{per-channel K + per-token V} 的角色感知非对称格式把检索行为从对称 \texttt{INT4} 的崩塌状态恢复到可用区间。表~\ref{tab:ch4-rolealign-kivi} 中三组完整配对模型的 PPL 差距都被压在 $|{\Delta}\mathrm{PPL}|\le 0.15$ 范围内，Needle 检索共同恢复到 100/100\%。因此，low-bit 恢复的一阶条件来自 role-aware 非对称格式本身。

在这一格式之上，\texttt{RoleAlign} 的价值主要体现为组织方式。K-path 以 attention-distribution KL 为主代理，并在离线搜索纪律下确定参数。V-path 由独立输出扰动代理主导。两条路径共享校准流程、结果接口与稳健选择纪律。与 \texttt{KIVI-style} 的同格式对比没有形成一边倒的 PPL 差距，却说明 \texttt{RoleAlign} 可以把低比特路径整理为可复核、可延伸，并能继续连接分配器的实例化路径。

前文 4.3.1–4.3.2 的结果还表明，low-bit cliff 的强度由模型族、规模与 $H_{kv}$ 共同调制。LLaMA-3.1-8B 在对称 \texttt{INT4} 下表现为架构例外，Qwen 系列在 K4V8 下更容易完全坍缩。这类现象体现的是架构依附的误差传播结果。更小的 \texttt{chunk\_size}、更接近 streaming 的 decode 粒度、更大的官方任务覆盖，以及更强的 role-aware allocator 反馈，仍可能继续放大 behavior proxy 与 numerical proxy 的差异。

\subsection{Heuristic 强基线的结构性含义}
\label{subsec:ch4-disc-heuristic}

第 4.4 节的 allocator 结果显示，position-based heuristic 在多个模型与预算带内接近 behavior-guided 策略，甚至在选中层集上出现局部重合。这是跨模型适用区间的一部分。heuristic 是\textbf{强基线}，原因来自结构本身。在某些适用区间下，关键层集中在很窄的位置区间，“保护哪几层”更接近窄选择空间，位置启发式在这些区间中能够逼近行为引导策略，说明可行层集已经被模型结构压窄。

Qwen2.5-3B 的早层保护剖面给出另一端结构。当敏感层高度集中于最早层，而位置启发式的假设偏向中层时，差距会放大到质量提升级别。启发式强基线与 3B 上的明显失配共同说明，跨模型预算分配的关键在于识别可行层集何时很窄、何时会被位置先验错过。行为引导分配器的价值就在于把这两类区间区分出来。

\subsection{威胁效度与外推边界}
\label{subsec:ch4-disc-validity}

第四章的外推边界与评测协议直接相关。正文主矩阵使用 LongBench 风格合成基准任务，官方 LongBench 真实数据对照覆盖 Qwen2.5-1.5B 的三个任务和单一种子。主协议方向与这组真实数据对照保持一致，后续仍需要更大的官方任务覆盖来扩展外部效度。

比较口径也限定了分配实验的读法。cross-model allocator 结果建立在同量级 INT4 预算带上。第 4.4 节给出了适用区间结构和 per-model 结构落点，严格 matched-budget 条件下的形式化胜负判定留给后续比较协议。

系统读数则绑定当前 NVIDIA H20、固定软件栈、\texttt{batch=1} 与贪婪解码设定。性能交叉边界在其它 GPU、batch 或 runtime 上可能移动，因此第 4.5 节给出的是当前平台上的经验决策规则。

数据来源层面，正文主结论以统一协议下重新核验的主结果为准。个别补充行或外部效度参考来自较早批次或回溯整理时，表注会标明来源和用途。沿着这些边界，第四章已经给出基准保真、低比特恢复、预算适用区间和部署收益的主要读数。更完整的官方数据集验证、更严格的同预算比较、更广的硬件矩阵，以及更成熟的 role-aware allocator 与 prompt 级自适应路由，将继续扩展这些结论的适用域。

\section{本章小结}
\label{sec:ch4-summary}

本章沿实验顺序检验第三章提出的行为对齐量化框架。结果先确认 INT8 基准路径的保真能力，再显示对称 \texttt{INT4} 的 K/V 角色失稳，随后把 \texttt{INT4-RoleAlign} 恢复、跨模型预算区间和系统后端边界串联起来。这样，本章报告的不只是单点分数，而是量化参数、低比特格式、层级预算和 Decode 后端之间的相互约束。

在基准保真方面，表~\ref{tab:ch4-int8-canonical} 与表~\ref{tab:ch4-longbench-official} 给出了保守位宽下的验证结果。Qwen2.5-1.5B 的 task-core 协议中，\texttt{INT8-Canonical} 相对 \texttt{FP16} 的三任务平均差异为 mean $\Delta=+0.02$，三个官方 LongBench 子任务的小范围对照也延续了同一方向。这说明 attention-distribution KL 组织的行为引导校准可以在 \texttt{INT8} 位宽下形成稳定保真锚点，为后续更激进的低比特路径提供参照。

低比特恢复实验进一步说明，位宽降低后的主要风险并不均匀分布在 K/V 两侧。表~\ref{tab:ch4-int4-cliff}、表~\ref{tab:ch4-kv-ppl}、表~\ref{tab:ch4-kv-multitask} 与图~\ref{fig:ch4-kv-ruler32} 显示，对称 \texttt{INT4} 在 Qwen 系列上触发阶跃崩塌，Key 侧低比特噪声是更直接的失稳来源。切换到 \texttt{per-channel K + per-token V} 后，表~\ref{tab:ch4-rolealign-kivi} 中三组完整配对模型的 Needle 检索都恢复到 100/100\%。这一结果把第三章的 K/V 传播路径分析落到实验读数中，也说明 \texttt{RoleAlign} 的价值不仅在质量恢复，还在于把离线行为读数组织成可继续进入分配器的校准产物。

跨模型预算实验把这一校准产物继续推进到层级决策。表~\ref{tab:ch4-regime-main}、图~\ref{fig:ch4-autok-protection}、图~\ref{fig:ch4-regime-heatmap} 以及两个典型剖面表显示，行为敏感度画像在不同模型中呈现出不同形状。Mistral-7B 提供 \texttt{AutoK} 的清晰单模型支持，Qwen2.5-3B 对应早层保护有效区间，LLaMA-3.1-8B 形成中等规模共识区间，Qwen2.5-14B 则表现为高性能簇共存而无稳定最优。heuristic 在部分区间中仍是强基线，这说明一些模型的可行层集本来就很窄。行为引导分配器的作用在于识别这些区间，并把固定 $k$ sweep 中分散的现象整理为可解释的结构读数。

系统实验给出了低比特路径进入真实后端后的边界。表~\ref{tab:ch4-tpot-4k}、表~\ref{tab:ch4-deployment-boundary} 与表~\ref{tab:ch4-kvcache-memory} 表明，当前 \texttt{INT4} 路径在代表性模型上带来约 73.4\% 的 KV Cache 容量压缩，但融合 decode 的时间收益随 $H_{kv}$、序列长度和后端组合变化。在 H20、\texttt{batch=1} 和当前 backend 条件下，$H_{kv}=8$ 且序列长度达到 \texttt{8K} 及以上时，Triton 融合路径开始进入优势区间。容量收益和时间收益因此需要分开读取，前者来自存储位宽下降，后者还取决于访存流量、调度开销和 GQA 映射方式。

综合来看，第四章给出了本文方法的实验落点。\texttt{INT8} 基准路径验证行为校准的保真基础，\texttt{INT4} 诊断暴露 K/V 角色差异，\texttt{INT4-RoleAlign} 把低比特路径恢复到可用区间，行为敏感度画像与 \texttt{AutoK} 则把同一组离线行为读数转化为预算候选。第五章将在这一基础上，进一步归纳全文贡献、适用边界和后续扩展方向。
