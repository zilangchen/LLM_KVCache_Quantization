\chapter{实验与结果分析}
\label{chap:experiments}

第三章提出了以 attention-KL 为核心的行为对齐量化框架。
本章通过四个核心 claims 验证该框架的有效性与设计范式：
行为对齐校准在 INT8 下产出可部署实例（Claim~1），
同一诊断透镜揭示低比特失效的结构性根因（Claim~2），
诊断直接导出 INT4-RoleAlign 恢复检索能力（Claim~3），
以及 INT4-RoleAlign 在 PPL 与延迟上的明确能力边界（Claim~4）。

\subsection{模型与硬件}
\label{subsec:exp-model-hardware}
% ------------------------------------------------------------

本文选取三个具有代表性的开源大语言模型作为实验对象，涵盖不同参数规模和架构设计：

\begin{itemize}[leftmargin=2em, itemsep=2pt]
  \item \textbf{Qwen2.5-1.5B-Instruct}~\cite{qwen2025qwen25}：
    主模型，28 层 Transformer，分组查询注意力（GQA，2 个 KV 头），头维度 128。
    在该模型上执行全量实验矩阵，包括所有量化模式、全部评测基准和敏感性分析。
    模型 revision 固定为 \texttt{989aa7}，确保严格可复现。
  \item \textbf{Qwen2.5-7B-Instruct}~\cite{qwen2025qwen25}：
    同架构的放大验证，28 层 Transformer，GQA（4 个 KV 头），头维度 128。
    用于验证行为对齐校准在同一架构不同参数规模下的有效性。
  \item \textbf{LLaMA-3.1-8B-Instruct}~\cite{grattafiori2024llama3}：
    跨架构验证，32 层 Transformer，GQA（8 个 KV 头），头维度 128，
    原生支持 128K 上下文窗口。
    用于验证行为对齐校准的跨架构适用性。
  \item \textbf{Mistral-7B-Instruct-v0.3}~\cite{jiang2023mistral}：
    第四验证模型，32 层 Transformer，GQA（8 个 KV 头），头维度 128，
    与 LLaMA-3.1-8B 相同的 KV 头数但来自不同的模型族。
    在第~\ref{sec:exp-kv-sensitivity}~节的 K/V 消融和 MixedKV 实验中
    用于进一步验证框架的跨模型族适用性。
\end{itemize}

所有实验在单张 NVIDIA H20 GPU（98GB 显存）上完成。
软件环境为 Python 3.12、PyTorch 2.8.0（CUDA 12.8）、
Transformers 库和 Triton 编译器。
完整依赖版本记录在 \texttt{env/versions.txt}
和 \texttt{env/\allowbreak requirements\_\allowbreak freeze.txt} 中，支持严格复现。

\subsection{评测基准}
\label{subsec:exp-benchmarks}
% ------------------------------------------------------------

本文从语言建模质量、长上下文检索能力、长文档理解能力、合成推理压测和系统效率
五个维度构建评测体系，具体如下：

\textbf{（1）困惑度（PPL）。}
基于 WikiText-2\cite{merity2016wikitext} 数据集评测语言建模质量。
采用分块评估策略：\texttt{chunk\_size\allowbreak =128}，
\texttt{stride\allowbreak =512}，
\texttt{max\_length\allowbreak =1024}，累计评测至少 100 万 token。
需要说明的是，chunk size 为 128 意味着每个 chunk 内新生成的 KV 采用 FP16 精度（未量化），
量化仅作用于跨 chunk 继承的历史 KV Cache。

\textbf{（2）Needle-in-a-Haystack~\cite{kamradt2023needle}。}
测试量化后的精确检索保真度。在 4K、8K、16K、32K 四个上下文长度点，
每个长度均匀设置 20 个深度级别（depth level），在干扰文本中插入唯一的"金针"信息，
要求模型在生成时准确检索并复述该信息。
通过率（pass rate）定义为通过精确匹配（exact match）的测试用例占比。

\textbf{（3）LongBench-style 合成评测。}
借鉴 LongBench~\cite{bai2024longbench} 的任务分类体系，
本文自行实现了确定性合成 QA 生成器，
覆盖 7 个代表性任务类别：
单文档问答（narrativeqa、dureader）、
多文档问答（hotpotqa）、
摘要生成（gov\_report、vcsum）、
少样本分类（trec）和
代码补全（lcc）。
合成样本通过在固定噪声上下文中植入锚点答案来构造，
确保完全离线可复现且与自定义推理管线集成。
各任务沿用 LongBench 对应指标类型：QA 类为 token-F1，
摘要类和中文 QA 为 Rouge-L，分类类为 Accuracy，代码类为 Edit Similarity。
主表汇总分数（\texttt{longbench\_score}）定义为 7 个任务指标的宏平均。
\emph{注意}：由于使用合成数据而非 LongBench 原始数据集，
绝对分数不可与 LongBench 官方排行榜直接对比，
但量化方法之间的相对差异仍可反映长上下文处理能力的保持程度。

\textbf{（4）RULER~\cite{hsieh2024ruler}。}
合成检索与推理能力的压力测试，包含 4 个子任务：
S-NIAH（单 passkey 检索）、MK-NIAH（多 key-value 并发检索，4 keys）、
VT（Variable Tracking，链式变量赋值，4 hops）和
CWE（Common Words Extraction，频率聚合，top-10）。
每个子任务在 4K、8K、16K、32K 四个长度点上评测，
总分为 4 个子任务准确率的宏平均。
本文自行实现了确定性合成任务生成器（task generator），
集成到自定义推理管线中，独立于 RULER 官方运行框架。

\textbf{（5）推理效率。}
包括三项系统指标：
Time-Per-Output-Token（TPOT，解码阶段每 token 延迟，单位 ms），
KV Cache 显存占用（单位 MB）以及
GPU 峰值显存（单位 MB）。
所有时间测量通过 CUDA 事件同步保证精确性。

    int4\_ours\_asym（INT4-RoleAlign）与 kivi\_style 共享非对称量化格式但使用 BA-guided percentile 校准，

各模式的关键差异在于四个维度：
（a）\emph{校准策略}——percentile 裁剪仅基于数值分布的百分位截断，
KL 行为对齐则以注意力权重分布的 KL 散度为优化目标；
（b）\emph{温度校正}——表中标记``有''表示校准阶段搜索了 $\tau^{-1}$ 参数
（int8\_ours 的 mainline 配置最终不启用温度校正，
以自适应保护作为替代；int4\_ours\_asym 同样不使用）；
（c）\emph{Decode Kernel}——triton\_fused 将反量化与注意力计算融合在单个
GPU 核函数中执行，消除了中间 FP16 张量的显存读写开销；
（d）\emph{Scale 来源与时间语义}——各模式的 Scale 计算时机和复用策略存在本质差异：
\begin{itemize}[leftmargin=2em, itemsep=1pt, topsep=2pt]
  \item \textbf{int8\_ours / int4\_ours}：Scale 由离线校准确定（per-layer per-group 静态 Scale），
    推理时固定不变，每个 token 步的 append 操作使用同一组预计算 Scale；
  \item \textbf{int8\_baseline / int4\_baseline}：Scale 在每次 append 时逐 token 动态计算
    （取当前输入的 absmax percentile），不同 token 步的 Scale 相互独立；
  \item \textbf{kivi\_style}：K 采用 prefill 阶段一次性计算的 per-channel Scale（后续 decode 步复用），
    V 采用每次 append 时逐 token 动态计算的 per-token Scale。
\end{itemize}

\subsection{统计框架}
\label{subsec:exp-statistics}
% ------------------------------------------------------------

为确保实验结论的统计可靠性，本文采用以下严格的统计框架：

\textbf{多种子运行。}
质量评测任务使用 5 个固定随机种子（1234, 1235, 1236, 1237, 1238），
吞吐量测试使用 8 个固定种子（1234--1241）。
所有评测统一采用贪婪解码（greedy decoding：$\text{temperature}=0.0$，
$\text{top\_p}=1.0$，$\text{top\_k}=0$），消除采样随机性，确保结果仅反映种子初始化差异。

\textbf{Bootstrap 置信区间。}
对每项指标计算基于 10\,000 次重采样的 95\% Bootstrap 置信区间~\cite{efron1994introduction}，
量化不确定性范围。

\textbf{符号翻转置换检验（sign-flip permutation test）。}
针对配对比较（如 int8\_ours vs int8\_baseline），采用基于种子配对的符号翻转置换检验，
在小样本（$n=5$）条件下提供非参数假设检验。

\textbf{BH-FDR 多重比较校正。}
当同一张表内存在多个配对比较时，采用 Benjamini-Hochberg 方法~\cite{benjamini1995controlling}
控制假发现率（False Discovery Rate），设定 $\alpha=0.05$。
BH-FDR 控制的是假发现率（而非族错误率 FWER），
在 NLP/ML 实验中为标准做法，具有更优的检验效力。
每张主表定义一个比较族（family），避免跨表展开导致比较数爆炸。


\medskip
\noindent\textit{本章的实验证据围绕四个核心 claims 组织，
而非按量化位宽罗列结果。
Claim~1（第~\ref{sec:exp-main}--\ref{sec:exp-generalization}~节）
验证行为对齐校准在 INT8 下产出可部署的操作点；
Claim~2（第~\ref{sec:exp-int4}--\ref{sec:exp-kv-sensitivity}~节）
通过诊断透镜揭示对称 INT4 失效的结构性根因；
Claim~3（第~\ref{sec:exp-rolealign}~节）
展示诊断直接导出的 INT4-RoleAlign 在三个模型上恢复检索能力；
Claim~4（第~\ref{subsec:exp-int4-boundary}~节）
界定 INT4-RoleAlign 在 PPL 与延迟上的能力边界。
第~\ref{sec:exp-kivi-unified}--\ref{sec:exp-discussion}~节
提供基线对比与综合讨论。}

\section{行为对齐原则验证：attention-KL 作为正确校准对象}
\label{sec:exp-main}
% ============================================================

\noindent\textbf{Claim 1：attention-KL 是正确的校准对象，INT8 给出 canonical validated instance。}
本节的双重目标是：（a）验证 attention-KL 比传统数值 proxy（MSE/percentile）更贴近
真实质量退化，（b）在此原则下给出经过完整验证的 INT8 规范实例。
INT8 的 256 个量化级别为检验 KL 校准效果提供了理想的受控设置——
精度损失足够小以观察校准策略的边际贡献，又足够大以暴露不良校准的缺陷。
以 Qwen2.5-1.5B-Instruct 为主模型，本节呈现在 32K 上下文长度下
各量化模式在全部评测基准上的综合表现。

表~\ref{tab:main-results} 的结果揭示了各量化模式在效率、显存和质量三个维度上的权衡关系，以下逐维度展开分析。

\textbf{解码延迟。}
在 32K 上下文下，INT8-ours 的 TPOT 为 47.14~ms，
较 INT8-baseline 的 51.43~ms 降低 8.3\%（$q=0.016$，BH-FDR 校正后显著）。
INT8-ours 的 TPOT 仍高于 fp16（24.39~ms），
因为 INT8 decode 阶段需要在核函数内执行反量化计算。
然而，在跨模型验证中（见第~\ref{sec:exp-generalization}~节），
更大模型上 Triton 融合核函数的加速效果更为显著：
7B 模型上 TPOT 从 60.52~ms 降至 50.07~ms（17.3\%），
8B 模型上从 91.77~ms 降至 57.23~ms（37.6\%），
均经 BH-FDR 校正后达到统计显著（$q=0.016$）。
这一加速归因于 Triton 融合核函数消除了中间 FP16 张量的显存分配与读写开销（核函数设计详见第~\ref{sec:ch3-triton}~节）。

\textbf{显存效率。}
INT8 量化将 KV Cache 的存储从 FP16 的 2 字节/元素压缩至 1 字节/元素。
在 32K 上下文下，int8\_ours 的 KV Cache 占用为 504~MB，
约为 fp16（896~MB）的 56.3\%，显存节省 43.8\%。
INT8 KV 存储的理论值为 448~MB（FP16 的 50\%），额外的 56~MB 为 per-group scale 参数的存储开销（$d_k/g$ 个 FP32 scale 每层每 KV），使实际占用 504~MB（56.3\%）。

\textbf{质量保持。}
INT8-ours 在全部 4 个上下文长度点（4K--32K）上
均保持 100\% Needle 通过率，
与 INT8-baseline 和 FP16 无统计显著差异
（符号翻转置换检验 $p=1.0$），
确认行为对齐校准有效保持了量化后注意力机制的检索精度。
困惑度方面，INT8-ours 的 PPL 为 8.95，较 INT8-baseline 的 8.93 仅退化 $-$0.20\%，
Bootstrap CI 和符号翻转检验均确认无统计显著差异
（$p=0.091$，$q=0.097$，BH-FDR 校正后不显著）。
三个模型的 PPL 退化幅度均在 $-$0.09\% 至 $-$0.26\% 范围内，
表明行为对齐量化在语言建模质量上不引入实质性退化。
在 LongBench-style 合成长上下文评测上，
表~\ref{tab:main-results} 中 INT8-ours mainline 配置的综合评分为 5.00（$\times$100），
与 INT8-baseline（4.78）和 FP16（4.82）可比。
附录表~\ref{tab:app-sig-quality} 基于所有配置变体的混合均值
（mainline + ablation）进行统计检验，
差异未达 BH-FDR 校正后的统计显著性（$q=0.144$），
进一步验证了行为对齐校准在多样化长上下文理解任务上的质量保持能力。

\textbf{RULER 评测的退化分析。}
\label{para:c6-fail}
RULER 是 INT8 唯一未达非劣性阈值的指标：
所有配置变体均值退化 $-$2.64\%（mainline 本身为 24.38\%，与 FP16 一致，
退化主要受非 mainline 消融变体拉低）。
退化有两层原因：（1）CWE/VT 子任务在所有模式（含 FP16）下均为 0\%，
为评测固有天花板（详见附录~\ref{sec:app-ruler-subtask}）；
（2）INT8-ours 特异性退化集中于 S-NIAH/MK-NIAH 两个检索子任务，
部分未启用 adaptive 保护的配置显著拉低均值。

\subsection{困惑度随上下文长度变化}
\label{subsec:exp-main-ppl}
% ------------------------------------------------------------

图~\ref{fig:main-quality-dashboard} 将 Claim~1 的质量证据压缩为三个主面板：
面板（a）给出 32K 上下文下的 PPL 主结果对比，

\subsubsection{校准策略的观察性比较}
\label{subsec:exp-ablation-calib}
% ------------------------------------------------------------

本节分析校准策略选择对量化质量的影响（RQ1）。
实验矩阵包含两种校准策略的变体：
KL 行为对齐校准和百分位裁剪（percentile）校准。
由于实验设计中各变体同时在校准策略、
核函数类型和温度校正等多个维度存在差异，
以下分析侧重于观察性比较而非严格因果归因。

表~\ref{tab:main-results} 中 INT8-baseline 与 INT8-ours
的质量指标对比提供了校准策略影响的重要观察：
INT8-baseline 采用百分位裁剪校准配合
PyTorch 参考核函数，
INT8-ours 采用 KL 行为对齐校准配合 Triton 融合核函数。
两者在 32K 长上下文下的 Needle 通过率
均保持 100\%，PPL 差异仅为 0.20\%，
LongBench 综合评分基本一致。
这表明 KL 校准在精度保持上至少不劣于百分位裁剪策略，
同时 KL 校准天然适配融合核函数的 scale 注入接口，
使得 INT8-ours 在保持质量的同时实现了 8\%--38\% 的 TPOT 降低。

需要指出的是，INT8-ours 与 INT8-baseline 同时在校准策略、
核函数实现和 adaptive 保护三个维度上存在差异，
上述对比反映联合设计的整体效果。
严格隔离校准策略的消融受限于计算资源未能完成。

注意力分布形态的核心特征——尖锐度和峰值位置——

额外样本提供的增量统计信息对校准结果的影响有限。

该结论能否推广至更大规模的模型或不同架构族仍需进一步实验确认。

\noindent\textbf{本节小结：}Adaptive static scale 保护是长上下文
Needle 通过率的决定性因素；
校准样本数 16--256 对量化质量影响极小。

\subsection{INT8 层面：INT8-ours vs KIVI-style}
\label{sec:exp-kivi}
% ------------------------------------------------------------

本小节将 INT8-ours 与 KIVI-style 基线进行对比，
评估行为对齐校准相对于非对称量化策略的竞争力。
KIVI~\cite{liu2024kivi} 提出了非对称量化策略——
Key 按通道（per-channel）量化、Value 按 token（per-token）量化，
这一设计利用了 Key 和 Value 矩阵在统计特性上的不对称性。

在展开对比之前，必须声明本文 KIVI-style 实现与 KIVI 原文的差异：

\begin{itemize}[leftmargin=2em, itemsep=2pt]
  \item 本实现仅采用 KIVI 的核心量化轴策略（per-channel K + per-token V 非对称量化），
    不包含原文中的残差补偿等完整工程优化细节；
  \item KIVI-style 的 decode 路径固定使用 \texttt{torch\_ref} 核函数
    （非 Triton 融合），因此 TPOT 对比需考虑 kernel 实现差异；
  \item KIVI-style 不使用温度校正（$\tau^{-1}$），
    在与 INT8-ours 对比时应理解为"无温度校正基线"；
  \item KIVI-style 的 INT4 路径使用 int8 容器存储（未 bit-pack），
    与 int4\_ours 的 0.5 byte/value bit-packing 存储方案不同，
    显存占用上不具直接可比性。
\end{itemize}

这表明以 KL 散度为目标的校准优化能够有效补偿对称量化在建模

对称量化方案提供了实证支持。

INT8-ours 的 Needle 通过率（100\%）与 KIVI-style（99\%）可比，
差异为 0\%（取最保守评估模型 LLaMA-8B）。
KIVI-style 在 1.5B 和 7B 模型上出现 99\% 的轻微退化（1 个 depth 级别失败），
可能与非对称量化在特定 Key 分布下的精度损失有关。

\textbf{效率对比。}
表~\ref{tab:kivi-comparison} 同时展示了两者在 32K 上下文下的推理效率。

尽管前者使用 PyTorch 参考实现而后者使用 Triton 融合核函数。
这是因为 KIVI-style 的 per-token Value 量化轴使得每次 decode 步仅需反量化
当前 token 的一组 scale 参数（而非 per-group 方案下的多组 scale），
部分抵消了非融合实现的额外显存读写开销。
KV Cache 显存方面，KIVI-style（462~MB）略低于 INT8-ours（504~MB），
差异源于两者不同的量化轴设计带来的 scale 张量存储开销差异。
综合质量和效率两个维度，INT8-ours 在 LongBench 上的优势（$+$1.37\%）
和 Needle 的完美通过率（100\% vs 99\%）使其在整体权衡中仍占优。

\noindent\textbf{小结：}KL 散度校准通过优化注意力行为一致性，有效补偿了对称量化在建模 K/V 不对称特性上的理论劣势，INT8-ours 在 LongBench 和 Needle 上均不劣于 KIVI-style。




% ============================================================
\section{低比特失效的结构性诊断：Key 主导与架构依赖}
\label{sec:exp-int4}
% ============================================================

\noindent\textbf{本节揭示：对称 INT4 导致系统性失效——Key 量化主导质量损失，且呈 GQA 架构依赖特征。}
当量化级别从 INT8 的 256 降至 INT4 的 15 时，
Needle 降至 0\%、PPL 退化$>$15\%——
15 个量化级别无法捕捉 Key 张量的通道间微结构。
这一结构性失效并非随机：它提示可通过精确诊断定位失效根因。

% ------------------------------------------------------------
\subsection{INT4 量化结果}
\label{subsec:exp-int4-results}
% ------------------------------------------------------------

INT4 量化将 KV Cache 的每个元素从 FP16 的 2 字节压缩至 0.5 字节
（通过 bit-packing），理论上可实现 4 倍的显存节省。
然而，4-bit 的表示范围仅有 16 个离散级别（对称量化下为 $[-7, 7]$），
这对量化精度构成了严峻挑战。

\textbf{Needle 通过率。}
INT4 量化在 Needle 检索任务上表现不佳。
在 1.5B 和 7B 模型上，INT4-baseline 和 INT4-ours 在 32K 的
Needle 通过率均为 0\%（完全失败）。
LLaMA-3.1-8B 是唯一的例外：INT4-baseline 100\%，INT4-ours 98\%，
退化 $-$2.0\%，超出非劣性阈值（$\geq$$-$1\%）。
值得注意的是，LLaMA-3.1-8B 的 INT4 量化表现异常稳健
（可能与其 8 个 KV 头提供更大的量化冗余有关），
但 KL 校准在 INT4 下仍引入了可测量的退化。

\textbf{困惑度。}
INT4-ours 的 PPL 同样未达到非劣性要求。
INT4-ours 的 PPL 在所有三个模型上均显著高于 INT4-baseline：
1.5B 模型 22.67 vs 19.54（退化 $-$16.0\%），
7B 模型 99.09 vs 85.49（退化 $-$15.9\%），
8B 模型 7.65 vs 6.97（退化 $-$9.8\%）。
退化幅度远超非劣性阈值（$\geq$$-$0.5\%），
表明 KL 校准在 INT4 精度下不仅未能改善 PPL，反而引入了额外退化。

% ------------------------------------------------------------
\subsection{INT4 局限性讨论}
\label{subsec:exp-int4-limitations}
% ------------------------------------------------------------

INT4 量化在长上下文场景下质量退化的根本原因在于表示精度不足。
具体而言：

\textbf{Needle 失败分析。}
Needle-in-a-Haystack 任务要求模型在大量干扰文本中精确定位一条目标信息，
这依赖于注意力机制能够在目标 token 位置产生足够尖锐的注意力权重。
INT4 量化的 16 级离散化（对称量化下仅 15 个有效级别）严重削弱了
Key 矩阵中微小数值差异的表达能力。当上下文长度增加到 32K 时，
attention score 的分母（$\sqrt{d}$ 归一化后的 $K \cdot Q$ 点积）需要在
数万个 token 中区分出目标位置，而 INT4 的量化噪声使得目标位置的
注意力得分被淹没在噪声中，导致 softmax 输出趋于均匀化，
模型无法有效检索目标信息。

\textbf{KL 校准的边际效益递减。}
虽然 KL 行为对齐校准在 INT8 下展现了显著优势，
但在 INT4 的极低精度条件下，其可优化空间受到量化网格本身的硬性限制。
当表示精度从 256 级（INT8）骤降到 16 级（INT4）时，
无论校准目标如何优化，量化后的注意力分布都不可避免地发生大幅偏移，
KL 校准只能在有限空间内寻找"最不坏"的解，而非"好"的解。

\textbf{量化噪声的定量分析。}
均匀量化的理论信噪比（Signal-to-Quantization-Noise Ratio, SQNR）
遵循经典公式 $\text{SQNR} = 6.02b + 1.76$~dB（其中 $b$ 为比特数）。
据此，INT8 的 $\text{SQNR} \approx 49.9$~dB，INT4 仅 $\approx 25.8$~dB，
两者相差 24.1~dB，对应量化噪声功率比 $10^{24.1/10} \approx 257$ 倍。
具体而言，对称量化的量化步长为
$\Delta = 2 x_{\max} / (2^b - 1)$，
量化噪声方差为 $\sigma_q^2 = \Delta^2 / 12$。
INT4 下 $\Delta_4 = 2 x_{\max} / 15$，
INT8 下 $\Delta_8 = 2 x_{\max} / 255$，
逐元素噪声方差之比 $\sigma_{q,4}^2 / \sigma_{q,8}^2 = (255/15)^2 = 289$。
这意味着 INT4 的逐元素量化噪声功率约为 INT8 的 289 倍，
远超 softmax 归一化所能自然消解的范围。

\textbf{量化步长与序列长度的交互效应。}
随序列长度 $n$ 增大，量化噪声扰动的极值以 $O(\sqrt{2 \ln n})$ 的速率增长
（极值统计理论），而目标检索信号不随 $n$ 增大，
因此存在临界序列长度 $n^*$，当 $n > n^*$ 时量化噪声以高概率淹没目标信号。
由于 INT4 的噪声方差约为 INT8 的 289 倍，其 $n^*$ 远小于 INT8，
这从理论上解释了 INT4 Needle 通过率随上下文增长急剧下降的趋势。
完整的形式化推导见附录~\ref{sec:app-sqnr-derivation}。

\textbf{GQA 结构下的量化误差稀释效应。}
分组查询注意力（GQA）中 $H_q$ 个 Query 头共享 $H_{kv}$ 个 KV 头。
设第 $g$ 个 KV 头的量化误差为 $\bm{\epsilon}_g$，
则该组内所有 $N_{\text{rep}} = H_q / H_{kv}$ 个 Query 头的注意力输出均受 $\bm{\epsilon}_g$ 影响，
但 $H_{kv}$ 个独立 KV 组各自引入统计独立的噪声。
在 GQA 的 concat+project 输出中，$H_{kv}$ 个独立 KV 组各自贡献等权独立噪声。
由独立随机变量求和的方差性质，合并输出的总误差方差按 $\sum_{g=1}^{H_{kv}} (1/H_{kv})^2 \cdot \sigma_g^2$ 缩放（假设各组噪声 i.i.d.），
即有效噪声功率 $\approx \sigma^2 / H_{kv}$。
三个实验模型的 GQA 配置差异如下：
Qwen2.5-1.5B（$H_{kv}=2$，有效噪声缩放 $1/2$）、
Qwen2.5-7B（$H_{kv}=4$，缩放 $1/4$）、
LLaMA-3.1-8B（$H_{kv}=8$，缩放 $1/8$）。
LLaMA-3.1-8B 的有效噪声功率仅为 Qwen2.5-1.5B 的 $(1/8)/(1/2) = 1/4$，
这为其在 INT4 下的异常稳健表现（Needle 100\%、PPL 退化仅 $-$9.8\%）
提供了定量解释：更多的 KV 头通过独立量化噪声的统计平均效应有效稀释了单个 KV 头的误差。

\textbf{结论与建议。}
综合以上定量分析，INT4 量化在长上下文场景下的质量退化可归纳为三层根因的叠加：
（1）\emph{表示层}——SQNR 从 49.9~dB 骤降至 25.8~dB，量化噪声功率增大约 289 倍；
（2）\emph{任务层}——序列长度增大使极值统计效应放大噪声对检索信号的淹没概率；
（3）\emph{架构层}——GQA 中 $H_{kv}$ 越小，误差稀释效应越弱，退化越严重。
INT4 量化在当前的对称量化、均匀网格设定下
不满足主线生产部署的质量要求。
要在 INT4 乃至更低比特下保持可接受的长上下文质量，
可能需要探索非均匀量化网格、基于 GQA 头数的自适应比特分配、
或量化感知微调（QAT）等更高级的技术手段，
这些方向将作为未来工作进一步研究。
第~\ref{sec:exp-kv-sensitivity}~节的 K/V 消融实验将进一步从实证层面验证上述理论分析——
K4V8（K@INT4, V@INT8）在 Qwen2.5 系列上 RULER 完全崩溃至 0\%，
而 LLaMA-3.1-8B 保持 31.12\%，
精确印证了 GQA 头数（$H_{kv}$）对 INT4 量化噪声稀释效应的决定性作用。
基于该诊断，MixedKV（K@INT8+V@INT4）作为实用替代方案，
在保持 K 高精度的同时压缩 V 至 INT4，
为 INT4 均匀量化的质量困境提供了一条可行的解决路径。

\paragraph{运行时 Scale 精度的影响。}
进一步分析发现，INT4 量化的 scale 参数在运行时链路中
可能经历从 float32 到 float16 的隐式精度降级，
该精度损失在分组量化的 scale 乘法链中逐层累积。
确保 scale 全链路以 float32 精度维护后的对照实验显示：
Qwen2.5-7B 的 INT4 PPL 从 86.56 改善至 52.19（$-$39.7\%），
LLaMA-3.1-8B 的 Needle 通过率恢复至 100\%（baseline 同步恢复），
但 Qwen2.5-1.5B 和 LLaMA-3.1-8B 的 PPL 仍未优于 baseline。
这表明运行时 scale 精度是影响 INT4 质量的重要因素之一，
但单独保障 scale 精度不足以建立 INT4 方案跨模型的一致优势。
该补充实验的完整结果见附录第~\ref{sec:app-eng066}~节。


% ============================================================
\subsection{K/V 精度敏感性分析}
\label{sec:exp-kv-sensitivity}
% ============================================================

\noindent\textbf{本节呈现全文核心发现：低比特退化由 Key 量化主导，呈 GQA 架构依赖。}
前节揭示对称 INT4 的系统性失效，但未区分该退化主要源于 Key 还是 Value。
本节通过受控 K/V 消融实验系统回答这一问题：
K/V 独立消融在三个模型族上一致确认 Key 量化噪声
是注意力行为偏移的主导来源，Value 对 INT4 具有显著鲁棒性。
这一诊断直接导出 MixedKV 和 RoleAlign 两条低比特设计路径。

% ------------------------------------------------------------
\subsubsection{消融实验设计}
\label{subsec:exp-kv-ablation-design}
% ------------------------------------------------------------

为分离 Key 和 Value 量化对质量的独立影响，设计以下三种消融配置：

\begin{itemize}[leftmargin=2em, itemsep=2pt]
  \item \textbf{K-only}（K@INT8, V@FP16）：仅对 Key 进行 INT8 量化，
    Value 保持 FP16 全精度，隔离 Key 量化的影响。
  \item \textbf{V-only}（K@FP16, V@INT4）：仅对 Value 进行 INT4 量化，
    Key 保持 FP16 全精度，隔离 Value 量化的影响。
  \item \textbf{K4V8}（K@INT4, V@INT8）：Key 采用 INT4 量化，
    Value 采用 INT8 量化，测试 Key 在极低精度下的退化程度。
\end{itemize}

三种消融配置的比特组合遵循以下诊断逻辑：
K-only 和 V-only 分别隔离单一维度的量化效应；
K4V8 则构造了一个``Key 激进、Value 保守''的极端对比，
用于验证 Key 量化是否为质量退化的主导因素。
每种配置在 Qwen2.5-1.5B、Qwen2.5-7B、LLaMA-3.1-8B 和 Mistral-7B
四个模型上使用 3 个随机种子（1234--1236）运行，
评测 RULER 和 LongBench 两个长上下文基准。

\subsubsection{核心发现：K 量化是长上下文质量退化的主导因素}
\label{subsec:exp-kv-key-bottleneck}
% ------------------------------------------------------------

表~\ref{tab:kv-ablation-ruler} 和表~\ref{tab:kv-ablation-longbench} 的结果揭示了
一个清晰而一致的模式：

\textbf{K@INT8 退化极小。}
K-only（K@INT8, V@FP16）在三个模型上 RULER 均接近 FP16 基线，
表明 INT8 精度对 Key 矩阵的表达能力充足。

\textbf{V@INT4 影响有限。}
V-only（K@FP16, V@INT4）同样保持接近 FP16 质量。
Value 仅作为 softmax 加权求和目标，
其量化噪声不影响 attention logits 分布，
且被多头平均进一步稀释。

\textbf{K@INT4 在 Qwen 系列上完全崩溃。}
K4V8 在 Qwen2.5-1.5B/7B 上 RULER 崩溃至 0\%，
产生典型重复退化（如 ``er er er er\ldots''），
印证第~\ref{subsec:exp-int4-limitations}~节分析：
Key 的 INT4 量化破坏 attention logits 排序结构。
LLaMA-3.1-8B（$H_{kv}=8$）上 K4V8 RULER 仍为 31.12\%，
体现 GQA 噪声稀释效应。

综上，\textbf{Key 决定``看哪里''、Value 决定``看到什么''}——
前者对量化噪声的敏感性远高于后者。

\subsubsection{退化模式的模型族依赖性}
\label{subsec:exp-kv-arch-dependency}
% ------------------------------------------------------------

K4V8 退化模式呈现出\textbf{显著的模型族/架构依赖性}：
Qwen2.5-7B（RULER 0\%）与 LLaMA-3.1-8B（RULER 31.12\%）
参数量相当（7B vs 8B），但 K4V8 表现迥异。
这一对比表明 INT4 鲁棒性不能仅由参数规模解释，
GQA 头数差异（$H_{kv}=4$ vs $H_{kv}=8$）是重要的架构因素。

此观察与第~\ref{subsec:exp-int4-limitations}~节的 GQA 噪声稀释理论一致：
LLaMA-3.1-8B 的 $H_{kv}=8$ 提供了 $1/8$ 的有效噪声缩放，
而 Qwen2.5-7B 的 $H_{kv}=4$ 仅有 $1/4$，
Qwen2.5-1.5B 的 $H_{kv}=2$ 更仅有 $1/2$。
更多的 KV 头通过独立量化噪声的统计平均效应，
使 LLaMA-3.1-8B 在 K@INT4 的极端条件下仍能维持部分检索能力。

与之相对，MixedKV（K@INT8, V@INT4）恢复了稳定且可用的检索质量。

\noindent\textbf{本节呈现论文的核心实验结论：诊断直接导出有效的低比特检索恢复设计。}
前两节的诊断链条——

Key 量化主导退化 + GQA 架构依赖（第~\ref{sec:exp-kv-sensitivity}~节）——

INT4-RoleAlign 在三个模型上将 Needle 从 0\% 恢复至 100\%，
实现 73\% KV Cache 压缩。
RULER 评测提供进一步支持性证据
（s\_niah $\geq$99\%、mk\_niah $\geq$93\%，4K--32K 上下文范围）。
然而，PPL 退化因模型规模而异（8B: 2.4\%，7B: 6.1\%，1.5B: 13.7\%），
解码延迟因缺少 Triton 融合核函数增加约 2--2.5$\times$，
揭示了检索恢复与语言建模保真之间的明确能力边界（Claim~4，
第~\ref{subsec:exp-int4-boundary}~节）。
这一``原则$\to$诊断$\to$设计$\to$边界''的完整链条构成本文的核心方法贡献。

\subsection{设计动机与差异}
\label{subsec:exp-rolealign-design}

本节提出的 \textbf{INT4-RoleAlign}（代码名 \texttt{int4\_ours\_asym}）
与 KIVI-style INT4 共享相同的量化格式（per-channel K + per-token V 非对称），
但在参数确定范式上存在本质差异：

\begin{itemize}[leftmargin=2em, itemsep=2pt]
  \item \textbf{校准范式}：KIVI-style 的量化参数完全由运行时数据的统计量
    （absmax/min）即时决定，属于无校准（calibration-free）方案；
    INT4-RoleAlign 的 K percentile 和 V percentile 通过离线 KL/attention-weighted
    搜索确定，属于校准引导（calibration-guided）方案。
  \item \textbf{设计动机}：KIVI-style 的非对称设计源于 K/V 统计特性的经验观察；
    INT4-RoleAlign 源于行为对齐框架的诊断发现——
    先通过 KL 分析定位对称 INT4 的行为失效模式，
    再通过受控消融确认 Key 是主导因素，
    最后吸收非对称轴策略作为解决方案。
  \item \textbf{参数求解}：KIVI-style 无搜索过程；
    INT4-RoleAlign 通过统一的 behavior-aligned 离线搜索确定
    role-aware 超参数（K percentile、V percentile），
    纳入 ours 框架的校准 pipeline。
\end{itemize}

\textbf{核心发现。}
（1）\emph{量化格式升级是 Needle 恢复的决定性因素}：
在全部三个模型上，从 per-group 对称切换到 per-channel/per-token 非对称后，
Needle 通过率均为 100\%（3 seeds 一致），
而对称 INT4 在 Qwen2.5-1.5B/7B 上为 0\%。
这表明对称 INT4 的失效并非校准策略不足，
而是量化格式无法捕捉 Key 通道间的数值异质性。

（2）\emph{PPL 退化呈现模型规模依赖性}：
INT4-RoleAlign 的 PPL 退化随模型规模递减——
LLaMA-3.1-8B 仅 2.4\%（与 KIVI-style 持平），
Qwen2.5-7B 为 6.1\%（略高于 KIVI-style 的 5.5\%），
Qwen2.5-1.5B 为 13.7\%（高于 KIVI-style 的 12.0\%）。
BA percentile 校准在 8B 上维持了与 KIVI-style 相当的 PPL 水平，
但在小模型上未能有效缩减退化。
值得注意的是，尽管 PPL 退化显著，三模型的 Needle 通过率均为 100\%，
\textbf{表明 INT4 量化主要影响全局语言建模质量，
而对局部信息检索能力的损伤极为有限}。
这一``PPL 退化 vs 检索保持''的解耦现象是本节的核心发现之一。

（3）\emph{GQA 架构依赖的收益边界}：
LLaMA-3.1-8B（$H_{kv}=8$）的 LongBench 分数与 KIVI-style 持平
（ours 6.31 $\approx$ kivi 6.31），PPL 退化也最低（2.4\%，与 KIVI-style 持平）。
这一现象与第~\ref{subsec:exp-kv-arch-dependency}~节的 GQA 架构依赖发现一致：
$H_{kv}=8$ 的架构中量化误差稀释效应较强，
KIVI 的运行时统计量已能提供较好的 scale 估计，
BA percentile 校准的边际改善收窄。
该结果进一步佐证了 $H_{kv}$ 是低比特量化容忍度的关键架构因素。

\subsection{$\tau^{-1}$ 在非对称量化上的负面结果}
\label{subsec:exp-rolealign-invtau}

基于 INT8 对称校准搜索得到的温度校正因子 $\tau^{-1}$
在 INT4 非对称量化上\emph{无稳定收益}：
Needle 从 100\%（20/20）退化至 95\%（19/20），
LongBench 综合评分下降约 15\%。
根因在于 per-channel K 量化的通道级 Scale 已足够精确，
原先为补偿 per-group 对称量化噪声设计的温度校正在非对称格式下产生过度补偿。
因此，INT4-RoleAlign 的最终配置\textbf{不包含 $\tau^{-1}$}。
这一结果揭示了行为对齐框架的一个重要特性：
有效组件的选择依赖于量化格式，不能无差别地跨格式迁移。

\subsection{证据边界声明}
\label{subsec:exp-rolealign-boundary}

INT4-RoleAlign 已完成 Needle、LongBench、PPL、RULER
四类质量评测以及 TPOT 和 KV Cache 显存的系统效率评测
（3 模型 $\times$ 3--5 seeds）。
RULER 评测覆盖 4K--32K 上下文，s\_niah contains $\geq$99\%、mk\_niah pass $\geq$93\%，
确认长上下文检索能力保持完整。
KV Cache 显存节省约 73\%（与理论 4$\times$ INT4 压缩一致），
但解码延迟（TPOT）因 \texttt{torch\_ref} 路径（无 Triton 融合核函数）
比 FP16 增加约 2--2.5$\times$。
因此，INT4-RoleAlign 当前定位为``质量保持 + 显存优化方案''，
系统延迟优势需在 Triton 融合核函数适配后方可实现。

\textbf{kivi\_style → INT4-RoleAlign：BA 校准在 Needle 保持一致的前提下对 LongBench 有微幅增强。}

但在任务导向的检索评测上与 KIVI-style 持平或略优。

\begin{sloppypar}
\textbf{int4\_kivi\_aligned → INT4-RoleAlign：设计差异。}\enspace
INT4-RoleAlign 不是 int4\_kivi\_aligned 的换皮版本，
而是 role-aware asymmetric 格式与 BA-guided percentile 校准的组合设计。
两者的关键差异在于参数确定范式：int4\_kivi\_aligned
在无校准基础上叠加预设~$\tau^{-1}$，
而 INT4-RoleAlign 通过离线 KL 搜索直接优化 percentile 参数。
\end{sloppypar}

\noindent\textbf{本节小结：}三方对比表明，BA-guided percentile 校准是 INT4-RoleAlign 优势的核心来源，$\tau^{-1}$ 在非对称格式下不可迁移。

% ============================================================

\noindent\textbf{Claim 4：INT4-RoleAlign 在 PPL 与延迟上存在明确能力边界。}
Claim~3 表明 INT4-RoleAlign 成功将 Needle 从 0\% 恢复至 100\%，
但 INT4 量化的代价不应被掩盖。
本节从语言建模质量与系统效率两个维度界定能力边界，
为部署决策提供定量依据。

\paragraph{PPL 退化：可控但不可忽略。}
第~\ref{subsec:exp-rolealign-results}~节的分析
已揭示 INT4-RoleAlign 的 PPL 退化呈模型规模依赖性
（8B: 2.4\%，7B: 6.1\%，1.5B: 13.7\%）。
此处进一步指出两个结构性特征。
\textbf{其一}，INT4-RoleAlign 在 Qwen2.5-1.5B 和 Qwen2.5-7B 上的 PPL 退化
略高于 KIVI-style（13.7\% vs 12.0\%、6.1\% vs 5.5\%），
仅在 LLaMA-3.1-8B（$H_{kv}=8$）上持平（2.4\% vs 2.4\%）。
这表明 BA percentile 校准的核心优势在于\emph{检索能力恢复}
而非 PPL 缩减——在小 $H_{kv}$ 模型上，
离线搜索的 percentile 参数
倾向于保护对检索至关重要的注意力聚焦模式，
而非全局概率分布的均匀保真。
\textbf{其二}，尽管 PPL 退化显著，三模型的 Needle 通过率均为 100\%。
这一``PPL 退化 vs 检索保持''的解耦现象构成本文的关键发现之一：
\emph{局部信息检索仅依赖少数关键位置的注意力聚焦，
对量化噪声的容忍度远高于需要全局概率保真的语言建模}。
该发现在实际部署中意味着，
对检索密集型应用（如 RAG、事实问答），
INT4 量化的实际影响远小于 PPL 数字所暗示的退化。

\paragraph{系统效率：显存收益已兑现，延迟收益待融合核函数。}
KV Cache 显存节省约 73\%（与理论 4$\times$ 压缩一致），
这一收益在当前实现中已完全兑现，
使得相同硬件预算下可支持约 3.7$\times$ 的上下文长度或并发批次。
然而，解码延迟（TPOT）因当前使用 \texttt{torch\_ref} 路径
（PyTorch 原生反量化 + 标准注意力计算）
比 FP16 增加约 2--2.5$\times$。
作为参照，INT8 路径通过 Triton 融合核函数
（第~\ref{sec:ch3-triton}~节）
实现了 8--38\% 的 TPOT 降低，
表明融合核函数是延迟收益的关键使能技术。
INT4 Triton 融合核函数的适配将是
INT4-RoleAlign 从``显存优化方案''升级为``全栈高效方案''的关键路径。

\paragraph{边界小结。}
INT4-RoleAlign 的能力呈现清晰的分层结构：
\emph{检索能力完整恢复}（Needle 100\%，RULER s\_niah $\geq$99\%），
\emph{显存压缩显著}（73\% KV Cache 节省），
但\emph{语言建模退化不可忽略}（PPL +2.4--13.7\%）
且\emph{延迟优势尚未兑现}（TPOT +2--2.5$\times$）。
上述边界的显式界定是行为对齐框架诊断能力的直接体现——
相较于仅报告最佳结果的传统做法，
诊断驱动的评估范式使我们能够精确识别能力的边界条件，
为后续优化（Triton INT4 融合核函数、
基于 $H_{kv}$ 的自适应比特分配）
指明方向而非留下盲区。

量化评估粒度（chunk\_size）的鲁棒性分析
见附录~\ref{sec:app-chunksize}。

\paragraph{证据边界声明。}
本章实验结果的证据成熟度分为两个层次：
（1）\emph{完整验证}：INT8 行为对齐量化经过 5 个种子 × 4 个模型 × 4 项基准的系统评测，
统计检验（Bootstrap CI + 排列检验 + BH-FDR 校正）全部完成，
构成本文的规范主证据；
（2）\emph{扩展验证}：INT4-RoleAlign 和 MixedKV 的跨模型结果基于 3 个种子 × 3 个模型，
PPL/Needle/LongBench/RULER 四项评测已完成聚合。
RULER 数据来自修正 \texttt{seq\_len} 参数后的补跑
（原始脚本因缺少 \texttt{--seq\_len} 参数导致 ctx$>$4K 被截断至 4096），
s\_niah $\geq$99\%、mk\_niah $\geq$93\%。
后续引用时将明确区分完整验证与扩展验证的证据等级。

\subsection{主要发现}
\label{subsec:exp-findings}
% ------------------------------------------------------------

\textbf{发现一：attention-KL 是正确校准对象，INT8 给出 canonical validation。}
INT8-ours 在四个模型上保持质量不劣于 FP16
（RULER $-$2.64\% 集中于检索子任务），
KV 显存降低 44\%、延迟降低 8--38\%。
Triton 融合核函数是效率收益的关键。

\textbf{发现二：低比特退化是 Key 主导、GQA 架构依赖的结构性问题。}
受控 K/V 消融一致确认 Key 量化主导退化，Value 对 INT4 具有显著鲁棒性。
$H_{kv}$ 越大，量化噪声稀释越强——
GQA 头数应作为比模型参数量更重要的量化策略考量因素。

\textbf{发现三：INT4-RoleAlign 恢复低比特检索——诊断驱动的设计闭环。}
Needle 全部恢复至 100\%（RULER 检索提供支持性证据，
s\_niah $\geq$99\%，4K--32K）。
PPL 退化因模型规模而异（8B: 2.4\%，7B: 6.1\%，1.5B: 13.7\%），
\emph{局部检索能力对 INT4 量化噪声的鲁棒性远高于全局语言建模质量}。
KV Cache 显存节省 73\%，但 TPOT 增加约 2--2.5$\times$，
构成检索恢复与系统效率之间的明确能力边界
（第~\ref{subsec:exp-int4-boundary}~节）。

\textbf{发现四：跨基准差异化敏感度。}
Needle（单点检索）在 INT8 下完美保持，
RULER 多目标检索处于 INT8 边缘（$-$2.64\%），
INT4 的 15 级对几乎所有长上下文任务构成挑战。

\textbf{发现五：$\tau^{-1}$ 格式依赖。}
$\tau^{-1}$ 在非对称 INT4 上无收益甚至有害，
增强机制不可跨格式迁移。

\subsection{威胁效度分析}
\label{subsec:exp-threats}
% ------------------------------------------------------------

本文的实验设计可能面临以下效度威胁：

\textbf{内部效度。}
（a）种子数 5 个，通过固定贪婪解码消除采样随机性。
（b）PPL 采用 chunk\_size=128 协议，可能低估量化影响；
chunk\_size=1 压力测试仅覆盖 1.5B 单模型。
（c）校准使用 WikiText-103 的 128 条样本，
B10 消融（第~\ref{subsec:exp-ablation-b10}~节）表明样本数 16--256 影响极小，
但未考察不同校准数据集的影响。
（d）所有评测在贪婪解码下进行，未验证采样解码场景。
（e）非劣性判定基于预设阈值 + Bootstrap CI，
而非正式 TOST 检验~\cite{schuirmann1987tost}，
结论应理解为实证判断而非严格统计推断。

\textbf{外部效度。}
（a）模型覆盖 4 个（1.5B--8B），未涉及 70B+ 规模。
（b）评测以英文/中文为主，未覆盖多语言、数学推理等场景。
（c）单 GPU 实验，未验证多 GPU 张量并行场景。

\textbf{构造效度。}
（a）KIVI-style 为简化实现（见第~\ref{sec:exp-kivi}~节差异声明），
对比结论仅适用于本文版本。
（b）RULER CWE 使用自实现生成器，干扰词频约 90 倍于目标，
影响绝对值与官方可比性。
（c）LongBench 使用合成数据源，测量量化后信息保持能力，
绝对分数不可与官方排行榜对比；
Mistral-7B 出现退化 artifact（详见附录~\ref{sec:app-kv-ablation-full}）。
（d）K/V 消融仅覆盖 RULER 和 LongBench，未纳入 PPL。
