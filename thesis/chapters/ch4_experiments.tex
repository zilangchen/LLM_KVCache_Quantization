\chapter{实验与结果分析}
\label{chap:experiments}

第三章给出了 behavior-guided 量化框架的两层设计：校准层（INT8 canonical path 与 INT4-RoleAlign）
与预算分配层（layer-wise allocator 与 AutoK proposer）。
本章按\textbf{三个研究问题}（RQ1-3）与\textbf{三个贡献点}（C1-C3）对应的证据组织实验：
\begin{itemize}[leftmargin=2em, itemsep=1pt, topsep=2pt]
  \item \textbf{第一层的闭环验证（RQ1 / C2）}：
    先在 Qwen2.5-1.5B-Instruct 上建立 INT8 canonical path 的保真度基线
    （第~\ref{sec:exp-setup}~节，表~\ref{tab:t1-int8-canonical}），
    再把 behavior 原则推进到 INT4 非对称格式（第~\ref{sec:exp-int4-kivi}~节），
    以 INT4-RoleAlign 对比 KIVI-style 的 calibration philosophy 差异
    （表~\ref{tab:t2-int4-kivi} 与表~\ref{tab:s3-rolealign-vs-kivi}）；
  \item \textbf{第二层的跨模型呈现（RQ2 / C3）}：
    在预算分配维度上跨 4 模型 $\times$ 4 policy $\times$ 3 task 对比
    （第~\ref{sec:exp-cross-model}~节，表~\ref{tab:t3-cross-model-main}），
    揭示一张由 \textbf{(family, scale, task)} 共同决定的 operating regime 地图；
  \item \textbf{机制桥梁与单模型剖面（RQ3 / C3 补充）}：
    第~\ref{sec:exp-kv-sensitivity}~节通过 K/V 独立消融给出图~\ref{fig:kv-ablation-summary-ruler}
    作为架构选择的 empirical 桥梁；
    第~\ref{sec:exp-per-model}~节对 Mistral / 3B / 14B 三个 per-model regime 展开剖面分析。
\end{itemize}
第~\ref{sec:exp-deployment}~节给出部署效率的补充证据；
第~\ref{sec:exp-discussion}~节把本章的经验结论收束到第五章的讨论起点。

\paragraph{Clean-provenance 数据基线。}
本章所有正文数字来自 \textbf{clean-provenance pin \code{ddada19}} 下重跑的 canonical 数据
（\code{results/clean\_rerun\_20260419T09/}），
runtime 文件以 md5-locked manifest 固化，校准产物统一重建，

\subsection{评测基准}
\label{subsec:exp-benchmarks}
% ------------------------------------------------------------

评测体系覆盖语言建模质量、长上下文检索、长文档理解、合成推理压测和系统效率五个维度：

\textbf{（1）困惑度（PPL）。}
基于 WikiText-2\cite{merity2016wikitext} 数据集评测语言建模质量。
采用分块评估策略：\code{chunk\_size=128}，
\code{stride=512}，
\code{max\_length=1024}，累计评测至少 100 万 token。
此处需要说明，chunk size 为 128 意味着每个 chunk 内新生成的 KV 采用 FP16 精度（未量化），
量化仅作用于跨 chunk 继承的历史 KV Cache。

\textbf{（2）Needle-in-a-Haystack~\cite{kamradt2023needle}。}
测试量化后的精确检索保真度。在 4K、8K、16K、32K 四个上下文长度点，
每个长度均匀设置 20 个深度级别（depth level），在干扰文本中插入唯一的``金针''信息，
要求模型在生成时准确检索并复述该信息。
通过率（pass rate）定义为通过精确匹配（exact match）的测试用例占比。

\textbf{（3）LongBench-style 合成评测。}
借鉴 LongBench~\cite{bai2024longbench} 的任务分类体系，
本文自行实现了确定性合成 QA 生成器，
覆盖 7 个代表性任务类别：
单文档问答（narrativeqa、dureader）、
多文档问答（hotpotqa）、
摘要生成（gov\_report、vcsum）、
少样本分类（trec）和
代码补全（lcc）。
合成样本通过在固定噪声上下文中植入锚点答案来构造，
确保完全离线可复现且与自定义推理管线集成。
各任务沿用 LongBench 对应指标类型：QA 类为 token-F1，
摘要类和中文 QA 为 Rouge-L，分类类为 Accuracy，代码类为 Edit Similarity。
主表汇总分数（\code{longbench\_score}）定义为 7 个任务指标的宏平均。
\textbf{注意}：由于使用合成数据而非 LongBench 原始数据集，
绝对分数不可与 LongBench 官方排行榜直接对比，
但量化方法之间的相对差异仍可反映长上下文处理能力的保持程度。
附录~\ref{sec:app-longbench-official} 使用官方 LongBench 数据集
的 3 个子任务验证了该结论在真实数据上的泛化性。

\textbf{（4）RULER~\cite{hsieh2024ruler}。}
合成检索与推理能力的压力测试，包含 4 个子任务：
S-NIAH（单 passkey 检索）、MK-NIAH（多 key-value 并发检索，4 keys）、
VT（Variable Tracking，链式变量赋值，4 hops）和
CWE（Common Words Extraction，频率聚合，top-10）。
每个子任务在 4K、8K、16K、32K 四个长度点上评测，
总分为 4 个子任务准确率的宏平均。
本文自行实现了确定性合成任务生成器（task generator），
集成到自定义推理管线中，独立于 RULER 官方运行框架。
\textbf{评分说明}：RULER 的 S-NIAH 采用精确匹配（生成必须与期望完全一致），
Needle-in-a-Haystack 采用包含匹配（生成含目标 UUID 即通过）；
同一模型在 S-NIAH 上的通过率通常低于 Needle，差异源于评分粒度而非量化方法。

\textbf{（5）推理效率。}
包括三项系统指标：
Time-Per-Output-Token（TPOT，解码阶段每 token 延迟，单位 ms），
KV Cache 显存占用（单位 MB）以及
GPU 峰值显存（单位 MB）。
所有时间测量通过 CUDA 事件同步保证精确性。

\subsection{基线方法}

三类模式的关键差异在三个维度：
（a）\textbf{Calibration philosophy}：percentile 裁剪仅利用数值分布的百分位截断；
KL 行为对齐以 attention behavior（softmax 输出分布）的 KL 散度为目标函数；
KIVI-style 的 absmax/min 在 prefill/decode 阶段由运行时统计量决定，不需要离线搜索。
本文立场（第~\ref{sec:exp-int4-kivi}~节）是：calibration philosophy 差异在同 format 下
对 quality / PPL / Needle 的影响是\textbf{小而有方向性的}，不是 winner-take-all 的缺口。
（b）\textbf{量化轴}：对称 per-group 与非对称 per-channel/per-token 的差异是\textbf{架构层面}的，
与 calibration 正交；第~\ref{sec:exp-kv-sensitivity}~节通过 K/V 独立消融（图~\ref{fig:kv-ablation-summary-ruler}）
为非对称格式提供 empirical bridge。
（c）\textbf{Scale 时间语义}：
离线 KL 校准（INT8-ours / INT4 对称 / INT4-RoleAlign）推理时 scale 固定，
可以跟随模型权重一起审计与冻结；
runtime percentile（INT8/INT4-Baseline）逐 token 动态重算；
KIVI-style 在 K 上 prefill 一次、decode 复用，V 上逐 token 重算——该混合策略使 KIVI 的 scale 不需要校准产物但也无法被离线固化。

\subsection{统计框架}
\label{subsec:exp-statistics}
% ------------------------------------------------------------

实验结论的统计可靠性由以下框架保障：

质量评测使用 5 个固定种子（1234--1238），吞吐量测试使用 8 个固定种子（1234--1241），
统一采用贪婪解码消除采样随机性。
对每项指标计算基于 10\,000 次重采样的 95\% Bootstrap 置信区间（percentile 方法，非 BCa）~\cite{efron1994introduction}，
配对比较采用符号翻转置换检验（适用于 $n=5$ 小样本）。
\cite{madaan2024variance}~在 10 个从头训练的 Llama-2-7B 上经验证实 Bootstrap CI 相对真实种子方差偏保守，
本文 $n{=}5$ 下的 CI 宽度倾向于高估不确定性。
\cite{fogliato2024precise}~另指出在 benchmark 评测中，
即使观测样本量较小，只要显著性约束设计合理，效应量估计仍可保持较高精度；
这与本文在 $n{=}5$ seed 下以 Bootstrap CI + BH-FDR 做质量差异判读的策略一致。
\textbf{Sign-flip $p$ 值下界}：在 $n=5$ 配对样本 + Phipson-Smyth $+1$ 校正下，
两尾 sign-flip 检验的最小可能 $p$ 值为 $3/33\approx 0.091$；
因此表中标注 $p\approx 0.091$ 的行已达当前样本量下的统计极限，
应读作\emph{在 $n=5$ 下不可分辨}（inconclusive at this power），
而非``方法之间无差异''的正面结论。

多重比较通过 Benjamini-Hochberg 方法~\cite{benjamini1995controlling}
控制假发现率（$\alpha=0.05$），每张主表定义一个比较族——
具体地，同表内同类指标（如所有 PPL 测试或所有 TPOT 测试）共享同一 FDR 族，
跨不同指标类型的 $q$ 值不直接合并。

\textbf{非劣性声明的实际操作}：
本文所述 ``non-inferiority''（如 PPL $<0.5\%$、Needle/LongBench $\geq-1$pp）
通过比较 Bootstrap 95\% CI 的相应单侧边界与预先设定的 margin 实现
（informal non-inferiority via CI bound vs margin），
并未严格执行 Schuirmann 两单侧检验 TOST 的形式化步骤~\cite{schuirmann1987tost}；
读者应将``non-inferiority''理解为``CI 在预设阈值内''而非 TOST 下的 $p$ 值显著。

\textbf{PPL 确定性说明}：
贪婪解码下 PPL 在\textbf{相同硬件、相同软件栈}的相同模型和测试集上是确定性的，
种子仅影响校准阶段的样本选择，校准完成后 PPL 不随种子变化
（$\pm$0.00，CI 退化为点估计）。
因此，PPL 指标上的 Bootstrap CI 和符号翻转检验不具统计意义；
PPL 差异的可靠性依赖于效应量本身（如 $<$0.3\% 退化）而非 $p$ 值。
追加 5 个额外种子（1239--1243，共 $n=10$）验证了该确定性：
三模型的 PPL 值与前 5 个种子完全一致（例如 Qwen2.5-1.5B 上
FP16 PPL=9.31、INT8-Canonical=9.34、INT4-RoleAlign=10.58，
对所有 10 个种子逐位相同），
确认了贪婪解码下 PPL 的种子独立性。
\cite{song2025nondeterminism}~指出跨硬件浮点非结合律可引入至多 9\% 的准确率漂移；
本文 PPL 确定性声明严格限定同一 H20 + CUDA 配置。
统计框架主要服务于具有种子变异性的指标：
Needle（depth 采样变化）、RULER 与 LongBench（合成样本生成的随机性）。

\section{INT4 推进：从对称 INT4 失效到 RoleAlign vs KIVI 对比}
\label{sec:exp-int4-kivi}
\label{sec:exp-main}% backward-compat 兼容旧 label
% ============================================================

\noindent\textbf{从 canonical 到低比特（对应 §2.2）。}
第~\ref{subsec:exp-int8-canonical}~节已确认 behavior-guided framework 在 INT8 canonical path
下保持 FP16 可比保真度（mean $\Delta{=}+0.02$）。
本节把同一原则推进到更激进的 INT4 位宽：
从\textbf{对称 INT4 的系统性失效}出发，
通过 K/V 独立消融定位\textbf{Key 主导退化}这一 empirical 事实（§\ref{sec:exp-kv-sensitivity}），
然后在 KIVI-style 的 per-channel K + per-token V 非对称格式上落成
\textbf{INT4-RoleAlign}——behavior-guided framework 在低比特的第二个实例化。
§\ref{subsec:exp-int4-cross-model} 以 Table~\ref{tab:t2-int4-kivi} 给出跨 4 模型的 RoleAlign vs KIVI 对比；
§\ref{subsec:exp-int4-honest} 给出本节的\textbf{三层诚实分析}
（L1 事实 / L2 suggests / L3 open question），
并为 §2.5 预留的 Allocator-vs-KIVI matched-budget 正式对比留出 Hook 位置。

\paragraph{对称 INT4 的阶跃崩塌（motivation）。}
在 INT8 下可行的 behavior-guided 对称量化在 INT4 下遇到实证瓶颈：
从 256 个量化级别降到 15 个后，
Qwen2.5-1.5B / 7B 上对称 INT4 的 Needle 通过率从 100\% 直接降至 0\%，
PPL 退化 $>$15\%。
这一阶跃崩塌来自 softmax 对 $QK^\top$ 分数的指数敏感性——
一旦量化噪声把目标 token 的分数挤出 top-k，
softmax 会指数级地把概率质量转移到错误位置，检索完全失败。
LLaMA-3.1-8B（$H_{kv}{=}8$）是唯一例外，其 INT4 Needle 仍能维持 98\%，
提示\textbf{架构 $H_{kv}$ 层面}存在噪声稀释效应。
这一观察引向下一节的 K/V 独立消融。

\subsection{K/V Role Mechanism：Key 主导退化的 empirical bridge}
\label{sec:exp-kv-sensitivity}% 保留原 label 兼容外部引用
% ============================================================

\noindent\textbf{误差分解公式~\eqref{eq:ch3-error-decomp} 的实证对照（支撑图~\ref{fig:kv-ablation-summary-ruler}）。}
第三章误差分解指出两条传播路径：
K 误差经 softmax 扭曲 attention 分布，V 误差经加权聚合偏移 output。
本节通过 K/V 独立消融分离二者的 empirical 效应——
Key 量化主导质量退化，Value 对 INT4 相对鲁棒。
这一实证事实为下一小节采用 \textbf{per-channel K + per-token V} 的非对称格式
提供 empirical bridge（RoleAlign 的架构选择来自 framework 的诊断，
不是 ad hoc 工程决策）。

% ------------------------------------------------------------
\paragraph{消融实验设计}
\label{subsec:exp-kv-ablation-design}
% ------------------------------------------------------------

为分离 Key 和 Value 量化对质量的独立影响，设计以下三种消融配置：

\begin{itemize}[leftmargin=2em, itemsep=2pt]
  \item \textbf{K-only}（K@INT8, V@FP16）：仅对 Key 进行 INT8 量化，
    Value 保持 FP16 全精度，隔离 Key 量化的影响。
  \item \textbf{V-only}（K@FP16, V@INT4）：仅对 Value 进行 INT4 量化，
    Key 保持 FP16 全精度，隔离 Value 量化的影响。
  \item \textbf{K4V8}（K@INT4, V@INT8）：Key 采用 INT4 量化，
    Value 采用 INT8 量化，测试 Key 在极低精度下的退化程度。
\end{itemize}

三种配置分别隔离 K/V 的独立效应及极端组合，
在三个主评测模型（Qwen2.5-1.5B/7B、LLaMA-3.1-8B）上使用 3 个种子运行 RULER 和 LongBench 评测。

（1.5B 见表~\ref{tab:kv-ablation-ppl}；14B 扩展见表~\ref{tab:14b-kv-ablation}）。

\paragraph{核心发现：K 量化是长上下文质量退化的主导因素}
\label{subsec:exp-kv-key-bottleneck}
% ------------------------------------------------------------

表~\ref{tab:kv-ablation-ruler} 和表~\ref{tab:kv-ablation-longbench} 的结果揭示了
一个清晰而一致的模式：

\textbf{K@INT8 退化极小。}
K-only（K@INT8, V@FP16）在三个主评测模型上 RULER 均接近 FP16 基线，
即 INT8 精度对 Key 矩阵的表达能力充足。

\textbf{V@INT4 影响有限。}
V-only（K@FP16, V@INT4）同样保持接近 FP16 质量。
Value 仅作为 softmax 加权求和目标，
其量化噪声不影响 attention logits 分布，
且被多头平均进一步稀释。

\textbf{K@INT4 在 Qwen 系列上完全崩溃。}
K4V8 在 Qwen2.5-1.5B/7B 上 RULER 崩溃至 0\%，
产生典型重复退化（如 ``er er er er\ldots''），
印证第~\ref{subsec:exp-int4-honest}~节分析：
Key 的 INT4 量化破坏 attention logits 排序结构。
对称 INT4 的失效是 100\%$\to$0\% 的阶跃崩塌而非渐进退化：
Needle 与 RULER 检索依赖 softmax 对目标 token 的近乎确定性聚焦，
而 softmax 对 $QK^T$ 分数的指数敏感性意味着，
一旦 Key 量化噪声把目标 token 的分数挤出 top-k，
softmax 会指数级地把概率质量转移到错误位置，
检索即完全失败——不会出现 50\%/30\%/10\% 的平滑衰减过程。
LLaMA-3.1-8B（$H_{kv}=8$）上 K4V8 RULER 仍为 31.12\%，
说明 GQA 噪声稀释在 $H_{kv}=8$ 时把噪声幅度压到了崩塌阈值以下。

综上，\textbf{Key 决定``看哪里''、Value 决定``看到什么''}，
前者对量化噪声的敏感性远高于后者。

\subsection{INT4 vs KIVI 的三层诚实分析}
\label{subsec:exp-int4-honest}
\label{subsec:exp-rolealign-boundary}% backward-compat
% ============================================================

\noindent\textbf{L1（事实层）。}
在 clean-provenance pin=\code{ddada19} 与 4 模型 × 3 task 的范围内，
INT4-RoleAlign 与 KIVI-style 的 quality / PPL / Needle 差距\textbf{不显著}
（$|\Delta \text{PPL}| \leq 0.15$；Needle 均 100\%；LongBench mean 差距
在 $[{-}0.05, +0.12]$ 之间）。
两种 calibration philosophy 在此范围内对\emph{质量指标}的实际影响\textbf{小而有方向性}，
不是 winner-take-all 的结构性缺口。

\noindent\textbf{L2（经验推测层）。}
数据\emph{suggests} 三条尚未定论的规律：
(i)~两者质量差距随 $H_{kv}$ 减小略微放大，提示 calibration
在 GQA 稀释较弱时可能有边际价值；
(ii)~RoleAlign 的"离线可审计产物"是一个\emph{工程维度的优势}，
不是 quality 维度的胜出；
(iii)~allocator 扩展（§\ref{sec:exp-cross-model}）
在 KIVI 的 calibration-free 格式上\emph{难以对标}，
这构成 RoleAlign 的\emph{方法结构优势}而非数值优势。

\noindent\textbf{L3（开放问题层）。}
以下两个问题\emph{本论文实验范围不足以回答}：
(a)~\textbf{format vs calibration 的耦合未完全解耦}——
因 RoleAlign 与 KIVI 共享 format，
本节不能 isolate 出"同 format 下离线校准相对运行时校准的独立增益"；
需要 pure-format ablation（同 calibration、不同 format）
与 pure-calibration ablation（同 format、不同 calibration）的正交实验，
本论文暂不包括。
(b)~\textbf{matched-budget 正式对比缺口}——
本节对比是\emph{每个方法独立最优配置}，
不是 \emph{matched memory budget 下}的受控对照；
正式的 Allocator-vs-KIVI matched-budget 对比构成一个独立的实验包，
见第五章 §5.3 open questions。

\section{跨模型 Regime 地图：behavior-guided allocator 的跨模型呈现}
\label{sec:exp-cross-model}
% ============================================================

\noindent\textbf{从 calibration 到 allocation（对应 §3）。}
第~\ref{sec:exp-int4-kivi}~节在 calibration 层面（INT4-RoleAlign 对标 KIVI-style）呈现了
\textbf{behavior 原则在 format 层的落点}。
本节把该原则推广到 \textbf{allocation 层面}：
对每个模型，用 behavior sensitivity profile 指导逐层预算分配，
跨 4 模型 × 4 policy × 3 core task 形成对比矩阵。
本节的对比采用\textbf{近似对齐的 INT4 budget band}——
各 allocator policy 与 \code{uniform\_int4\_k4v4} 基线同量级但\emph{非严格 matched memory}
（详细说明见下文"Budget band 的定义"段），在此前提下呈现 4 种 policy 的跨模型\textbf{regime signature}。

\paragraph{Budget band 的定义（非严格 matched-budget）。}
\label{para:ch4-budget-band}
记第 $l$ 层的 K / V 位宽分别为 $b_K^{(l)}$、$b_V^{(l)}$，
单次推理 KV cache 占用可近似写作
\begin{equation}
  \text{mem}(\pi) = \sum_{l=1}^{L} \left( b_K^{(l)} + b_V^{(l)} \right) \cdot H_{kv} \cdot T \cdot d_{\text{head}} / 8,
  \label{eq:ch4-matched-budget}
\end{equation}
其中 $L$ 为层数、$T$ 为上下文长度、$H_{kv}$ 为 KV head 数、$d_{\text{head}}{=}128$。
本节所有 allocator policy 的\emph{平均位宽} $\bar{b}{=}\sum_l (b_K^{(l)} + b_V^{(l)}) / (2L)$
经 AutoK 后落在 $[4.5, 5.0]$ bit 带内，而 \code{uniform\_int4\_k4v4} 为 $\bar{b}{=}4.0$ bit——
按公式~\eqref{eq:ch4-matched-budget}，allocator policy 相对 uniform 的
KV memory 差距大约在 $+12.5\%$ 至 $+25\%$ 之间
（\textbf{不是}严格 matched memory 的 $\pm 3\%$ 带）。
本文因此把本节定位为\textbf{regime signature readout}而非\emph{形式化} matched-budget comparison：
T3 报告的是"在同一数量级预算内不同 allocation 风格跑出什么 quality"，
图~\ref{fig:t7-pareto} Pareto 前沿直接可视化 quality--memory 权衡。
严格 matched-budget 的 Allocator-vs-KIVI formal compare 作为条件 Future Work
（story §13 Hook、第~\ref{sec:conclusion-future}~节）。

\paragraph{Regime 地图的三条读数。}
把 T3 / 图④ / 图⑦ / 图⑧ 的证据汇总，得到以下\textbf{跨模型经验规律}（C3）：
\begin{enumerate}[leftmargin=2em, itemsep=1pt, topsep=2pt]
  \item \textbf{Scale-dependent regime}：小尺度模型（3B）的 sensitivity 早层集中，
    allocator 在早层少数位置上就能"回救"大部分 quality；
    大尺度模型（14B）的 sensitivity 弥散化，allocator 收益衰减至 margin 级别。
  \item \textbf{Family-dependent regime}：Mistral-7B 是跨模型里 AutoK
    最清晰的赢家，而同参数量的 Qwen2.5-7B / Llama-3.1-8B 上 AutoK 仅与 BA-$k_\text{best}$ 持平。
  \item \textbf{Task-dependent regime}：同模型下不同 task（NarrativeQA / HotpotQA / GovReport）
    的 winner 也并非一致——Llama-3.1-8B 上 GovReport 偏好 AutoK，
    其余两个 task 偏好 BA-$k_{11}$。
\end{enumerate}
综上，本章主张 behavior-guided allocator 的真实贡献\textbf{不是"击败均匀分配"},
而是\textbf{揭示一张 (family, scale, task) 共同决定的 operating regime 地图}。
这也为第五章 §\ref{sec:discussion-heuristic} 正面承认 \textbf{heuristic 是强基线}
提供了经验基础——在某些 regime 区间里，保护哪些 $k$ 个位置本就是"决定论式"选择，
无论启发式还是 behavior-guided 都会落在相似集合。

\subsection{Qwen2.5-3B：Early-Layer Rescue Regime}
\label{subsec:exp-per-model-3b}
% ------------------------------------------------------------

Qwen2.5-3B 揭示了一个在其它模型上未出现的\textbf{异质 regime}：
单层 behavior-guided 保护相对单层启发式保护呈现 catastrophic $\Delta$。
记 $\Delta(\pi_{\text{BA}}, \pi_{\text{Heur}}, t)$ 为同 $k{=}1$ 预算下
behavior-guided 相对 heuristic 在 task $t$ 上的分差：
\begin{equation}
  \Delta(t) = \text{score}_\pi(\text{BA-}k_1, t) - \text{score}_\pi(\text{Heur-}k_1, t).
  \label{eq:ch4-rescue-delta}
\end{equation}
表~\ref{tab:t5-3b-early-layer} 给出 3B 上各 task 的 $\Delta$ 分布。

\input{tables/table_t5_3b_early_layer}

关键观察：
\begin{itemize}[leftmargin=2em, itemsep=1pt, topsep=2pt]
  \item \textbf{Heuristic 在 NarrativeQA / HotpotQA 上近乎完全失效}
    （Heur-$k_1$ 分别为 $3.08$ 与 $1.39$），
    原因是 heuristic 按等距位置保护中层（$l \approx L/2 = 18$），
    而 3B 的 behavior sensitivity profile 是\textbf{早层集中型}
    （图~\ref{fig:sensitivity-heatmap} 中 3B top-3 protected = $\{0, 1, 2\}$）；
  \item \textbf{BA-$k_1$ 把 NarrativeQA 从 $3.08$ 回救到 $7.17$}（$\Delta \approx +4$ 分），
    说明 behavior profile 给出的"哪一层需要保护"是\emph{模型特异性}而非通用规则——
    中层 heuristic 在 3B 上没有合适的 empirical 基础。
\end{itemize}
这一事实使 3B 成为论文 C3 regime map 的\textbf{最强方法差异化证据}：
如果 behavior-guided allocator 在\emph{所有}模型上都仅与 heuristic 持平，
则它对 pipeline 的独立贡献难以声明；
3B 上的 $\Delta \gg 0$ 表明 behavior 信号在\emph{某些 regime 区间内}
提供了 heuristic 结构性缺失的信息。

\textbf{1.5B 的补充趋势。}
Qwen2.5-1.5B 作为同族更小模型，
在 canonical step1（§\ref{subsec:exp-int8-canonical}）只参与 INT8 保真度验证
不纳入 allocator sweep；
但从早层 calibration profile 判读（artifact \code{kv\_calib\_kl\_selected\_v3\_quick.json}），
其 top-$k$ sensitivity 同样呈早层集中模式。
这一趋势\textbf{与 3B 同向}，作为 supporting 证据出现，不独立成节。

\begin{equation}
  \text{gap}_{\text{rel}}(t) = \frac{\text{top}_1(t) - \text{top}_3(t)}{\text{top}_1(t)} \times 100\%.

这一 regime 正好解释 §\ref{sec:exp-cross-model} 的表~\ref{tab:t3-cross-model-main}

在 Phase 1（4K）下两者的 交叉点 幅度高度一致
（8B: $\Delta=-0.39$~ms，14B: $\Delta=-0.44$~ms），
为 $H_{kv}$ 而非模型参数量主导 Phase Boundary 提供支持证据。
但由于 $H_{kv}$ 与模型规模在当前模型集合中部分共变，
该结论应理解为结构性相关而非严格因果识别。

\subsection{部署建议}
\label{subsec:exp-deployment-rec}

基于上述分析，针对 NVIDIA H20 平台给出以下部署建议：
\begin{itemize}[leftmargin=2em, itemsep=2pt]
  \item $H_{kv}\geq 4$ 且序列长度 $\geq$8K：
    使用 Triton 融合核（triton\_ra），
    在 14B / 32K 下可实现约 40\% 的 TPOT 降低。
  \item $H_{kv}=2$ 或序列长度 $<$8K：
    使用 naive dequant+SDPA 路径（torch\_ref）。
  \item 上述建议严格限于 NVIDIA H20（sm\_90, 130 SM, 98~GB HBM）。
    在 SM 数量或带宽不同的硬件上，Phase Boundary 可能移位。
\end{itemize}

% ------------------------------------------------------------
\subsection{主要发现}
\label{subsec:exp-findings}
% ------------------------------------------------------------

综合各节实验证据，本章在 $H_{kv}$ 组织下得到三条主发现：
(i) 校准目标的 KL-vs-MSE 分歧呈 bit-width $\times$ 规模双重依赖
（INT8 下趋同，INT4 下在 $H_{kv}=2$ 分化而在 $H_{kv}=4$ 趋同）；
(ii) Key 主导退化在四模型 K/V 消融上一致
（14B 保留 Key 为 FP16 恢复 93\% PPL 退化），
诊断导出的 per-channel K + per-token V 格式（INT4-RoleAlign）
保持四模型 Needle 100\% 与 KV 压缩约 73\%；
(iii) 融合核效率在 $(H_{kv}, \text{seq\_len})$ 空间中分化
（$H_{kv}=8$ 在 32K 下延迟减少约 40\%，$H_{kv}=2$ 下始终慢于 SDPA）。
完整表述与部署建议见第~\ref{chap:conclusion}~章。

\paragraph{PPL 非单调现象}
PPL 退化随模型规模总体减弱（1.5B: 13.7\%，7B: 6.1\%，8B: 2.4\%），
但 14B 的退化（7.6\%）高于 8B（2.4\%），打破了单调趋势。
可能的解释包括：
（a）校准数据量（128 条 WikiText-2）对 14B 可能不足；
（b）Qwen 与 LLaMA 模型族在 outlier 分布上的差异；
（c）14B 的层数（40 层 vs 8B 的 32 层）导致量化误差更多累积路径；
（d）评测窗口差异——14B 因显存限制仅在 WikiText-2 test split 前 32767 tokens
的固定 prefix 上评测，其他模型在完整 test split（$\sim$302K tokens）上评测，
PPL 测量窗口的差异无法完全分离，14B 7.6\% 应视为外部效度锚点的补充观察
（详见表~\ref{tab:t2-int4-kivi} tnote[b]）。
该非单调现象不影响核心结论（14B Needle 仍为 100\%），
但提示校准策略可能需要针对大模型专门调优。

% ------------------------------------------------------------
\subsection{威胁效度分析}
\label{subsec:exp-threats}
% ------------------------------------------------------------

本文的实验结论应在以下六条效度威胁声明的边界内理解：

\textbf{威胁一：$H_{kv}$ 与规模共变。}
$H_{kv}$ 与模型规模部分共变（1.5B/$H_{kv}$=2, 7B/4, 8B/8, 14B/8）。
8B 与 14B 共享 $H_{kv}=8$ 但参数量差 1.75$\times$，
短序列 交叉点 幅度一致（$\Delta\approx-0.4$~ms），
为 $H_{kv}$ 结构性相关提供支持。
但相关结论应理解为结构性证据而非严格受控因果识别。

\textbf{威胁二：解码策略依赖。}
所有实验使用 greedy decoding（temp=0）。
量化误差在 sampling（temp$>$0）下的表现未知。
部署建议限定于 greedy/near-greedy 场景。

\textbf{威胁三：硬件依赖。}
所有 TPOT 数据来自单张 NVIDIA H20（98~GB, sm\_90, 130 SM）。
Phase Boundary 在 A100/H100 上可能因 SM 数量和带宽比不同而移位。

\textbf{威胁四：校准数据分布依赖。}
仅用 WikiText-2（128 条样本）校准。
代码、多语言等分布外数据的适用性未验证。

\textbf{威胁五：单实现威胁。}
KIVI baseline 为自实现（核心量化轴策略一致，
未复现全部工程优化）。
RoleAlign 与 KIVI 的差异可能混入实现差异。

\textbf{威胁六：评测协议依赖。}
RULER 与 LongBench 使用本文自行实现的合成任务生成器，
所有 benchmark 结论均在本文协议实现下的相对对比中成立。
