\chapter{行为对齐量化框架设计}

本章按三个层级组织行为对齐量化框架。
\emph{层一（视角与度量）}：
以量化前后注意力分布之间的 KL 散度（attention-KL）刻画行为偏移，
作为本章后续校准与诊断共享的基础度量
（第~\ref{sec:ch3-overview}--\ref{sec:ch3-calibration}~节）。
\emph{层二（选择协议）}：
在 attention-KL 度量下建立 robust selection 协议，
既用于校准参数搜索，也用作分析不同量化配置的诊断工具；
INT8 对称路径在该协议下给出具体的校准流程与自适应保护机制
（第~\ref{sec:ch3-static-scale}~节）。
\emph{层三（实例化）}：
在 KIVI~\cite{liu2024kivi} 提出的
per-channel Key + per-token Value 非对称格式上，
以离线 attention-KL 搜索确定每层的 BA percentile，
得到行为引导的实例化 RoleAlign
（第~\ref{sec:ch3-rolealign}~节），
并由 Triton 融合量化解码核函数提供系统实现支撑
（第~\ref{sec:ch3-triton}--\ref{sec:ch3-system}~节）。
逐头温度校正 $\tau^{-1}$ 作为诊断过程中观察到的启发式呈现
（第~\ref{sec:ch3-invtau}~节），
第~\ref{sec:ch3-complexity}~节给出复杂度与资源分析。

\section{框架总体设计}
\label{sec:ch3-overview}

\subsection{系统架构概述}

行为对齐量化框架分两个阶段。
离线阶段用少量校准数据搜索量化参数，目标是尽量保持注意力行为不变；
在线阶段直接加载这些参数做量化推理——
校准参数加载本身不引入额外开销，
量化本身引入的反量化计算代价则由 Triton 融合核函数承接，
其具体延迟 profile 详见第~\ref{sec:exp-main}~节。
校准时为每层 Key 和 Value 分别独立计算 KL 散度
并搜索最优 Scale（第~\ref{subsec:ch3-two-stage}~节），
为 K/V 量化敏感性提供了统一的度量基础。
当 Key 和 Value 设置不同量化位宽时（如 K@INT8+V@INT4），
校准参数仍可通过同一流程独立获取。
第~\ref{sec:exp-kv-sensitivity}~节的消融实验利用该特性
定量分析了 K/V 精度的独立敏感性。

框架的设计空间涉及两个正交维度：
\textbf{量化格式}（对称 vs 非对称）与\textbf{校准目标}（MSE vs KL）。
第~\ref{sec:ch3-static-scale}~节基于对称格式建立 INT8 路径，
第~\ref{sec:ch3-rolealign}~节的 INT4-RoleAlign 在此基础上
同时升级量化格式为非对称，实现两维协同改进
（详见第~\ref{para:ch3-orthogonality}~段）。
该设计将计算密集的参数搜索过程从推理关键路径中分离出来，
使在线阶段的推理延迟不因行为对齐校准而增加。
图~\ref{fig:ch3-framework} 展示了框架的总体架构。

\subsection{离线校准阶段}

离线校准阶段的目标是在少量代表性校准数据上确定关键量化参数：
（1）逐层逐组的静态量化 Scale $s$，用于将浮点键值张量映射到整数表示；
（2）自适应保护参数，防止推理时超出校准范围的激活值被过度裁剪；
（3）逐头温度校正因子 $\tau^{-1}$（inverse temperature），
作为诊断框架的结构性产出，用于揭示量化噪声在 GQA 尺度下的行为特性
（详见第~\ref{sec:ch3-invtau}~节）。

校准流程依次经历四个步骤。
首先，\textbf{校准数据采集}使用 FP16 精度的参考模型在校准数据集
（如 WikiText-2~\cite{merity2016wikitext} 的子集）上执行前向传播，
逐层提取 Query 向量 $\bQ$、Key 矩阵 $\bK$ 和 Value 矩阵 $\bV$；
Query 向量须经过输入层归一化与旋转位置编码（RoPE~\cite{su2024rope}）处理，
使校准时的 $\bQ \bK^T$ 点积与实际推理时一致。
其次，\textbf{Scale 搜索}在候选 clip percentile 和 group size 的网格上，
通过量化-反量化-计算 KL 散度的流程评估每组参数的行为偏差，
选择使 KL 散度最小的参数组合。
随后，\textbf{逐头 $\tau^{-1}$ 优化}在确定最优 Scale 的基础上，
为每个注意力头独立搜索最优的温度校正因子，
进一步降低量化后注意力分布与参考分布之间的 KL 散度。
最后，\textbf{校准产物输出}将搜索结果序列化为 JSON 文件，
包含模型配置、逐层 Key/Value 静态 Scale、逐层逐头 $\tau^{-1}$ 值以及搜索超参数记录。

\paragraph{校准产物版本说明}
    本文 1.5B 主模型的实验使用校准产物 v3\_quick，
    该版本在生成时 Q 向量的预处理流程尚未包含 input\_layernorm 和 RoPE 变换，
    后续版本已修正。
    实验结果（第四章）显示该产物对 INT8 量化的影响有限
    （PPL 退化 $-$0.20\%，Needle 100\%），
    但完整预处理后的重校准对比验证尚未完成（特别是 RULER / LongBench 上的验证），列为未来工作。
    因此本文温度校正消融的定量数字基于 v3\_quick 校准产物获得，
    完整重校准后的数值可能存在小幅偏移。
    校准产物使用 128 条 WikiText-2 样本离线生成，各模型独立校准。

    \paragraph{校准-评测的 task transfer 假设}
    本文的校准目标采用 WikiText-2 上的 attention-KL 散度作为代理损失函数，
    而评测则跨越 WikiText-2 PPL、合成 Needle-in-a-Haystack、
    RULER 13 子任务与 LongBench 16 真实任务等多个 domain 与任务类型。
    两者之间存在三重 shift：
    (i) domain shift（general English → synthetic long-context），
    (ii) 任务类型 shift（token-level 似然 → retrieval/reasoning），
    (iii) 序列长度 shift（校准 $<$4K → 评测 32K）。
    本文采用 WikiText 校准的核心假设是：
    \emph{最小化 WikiText attention-KL 获取的量化参数对下游多任务泛化具有 approximately task-agnostic 的迁移能力}。
    该假设未经形式化证明，仅通过第四章的 4-metric 联合评测进行间接经验支撑，
    其严格验证留作未来工作。
    同时，校准与 PPL 评测均从 WikiText-2 raw \texttt{test} split 加载数据，但使用方式不同：
    校准仅取前 $n=128$ 段长度 $\leq 512$ token 的样本（累计约 1.5K\,token，
    占 test split 总量约 0.03\%），用于离线提取 attention 分布 KL 作为代理损失；
    PPL 评测则使用 test split 全量（约 302K\,token）以滑动窗口方式计算困惑度。
    由于校准不涉及 token-level 预测目标，且使用量远小于评测样本，
    此共用 split 带来的 potential contamination 影响限于 attention 分布的统计估计，
    不直接泄漏 PPL 评测的 teacher-forcing 目标。
    本文未对此作严格消融验证，是已披露的方法学局限。

\subsection{在线推理阶段}

在线推理阶段加载离线校准产物以极低的额外计算开销执行量化推理。
启动时一次性从 JSON 文件读取静态 Scale 和 $\tau^{-1}$ 预加载到 GPU 显存。
Prefill 阶段对输入序列执行完整前向传播，在每一层注意力计算前使用逐头 $\tau^{-1}$
对 Query 向量进行预缩放（Q pre-scaling），然后将 Key/Value 使用静态 Scale
量化为 INT8/INT4 格式存入缓存。Decode 阶段每步生成一个新 token，
新的 Query 向量同样经过 $\tau^{-1}$ 预缩放后，
通过 Triton 融合核函数直接从 INT8/INT4 缓存中读取并反量化 Key/Value，
在片上完成注意力计算，避免将反量化结果写回全局显存。

\section{行为对齐校准方法}
\label{sec:ch3-calibration}

\subsection{KL 散度目标函数}

本文选择 Kullback-Leibler（KL）散度衡量量化前后注意力行为的一致性。
对于给定的 Query 向量 $\bq \in \mathbb{R}^{d_k}$ 和 Key 矩阵 $\bK \in \mathbb{R}^{n \times d_k}$，
FP16 精度下的注意力权重分布定义为：
\begin{equation}
  \bm{p}_{\text{ref}} = \softmax\!\left(\frac{\bq \bK^T}{\sqrt{d_k}}\right)
  \label{eq:ch3-pref}
\end{equation}
其中 $n$ 为上下文长度，$d_k$ 为注意力头维度。
设 $\hat{\bK}$ 为经过量化——反量化（quantize-dequantize）后的 Key 矩阵，
量化后的注意力权重分布为：
\begin{equation}
  \bm{p}_{\text{quant}} = \softmax\!\left(\frac{\bq \hat{\bK}^T}{\sqrt{d_k}}\right)
  \label{eq:ch3-pquant}
\end{equation}
行为对齐校准的目标是最小化两者之间的 KL 散度：
\begin{equation}
  D_{\KL}\!\left(\bm{p}_{\text{ref}} \,\|\, \bm{p}_{\text{quant}}\right)
  = \sum_{i=1}^{n} p_{\text{ref},i} \cdot \ln\frac{p_{\text{ref},i}}{p_{\text{quant},i}}
  \label{eq:ch3-kl}
\end{equation}

选择 KL 散度而非均方误差（MSE）作为目标函数基于以下考量：
\begin{enumerate}[label=(\arabic*)]
  \item \textbf{分布层面的度量}：
    KL 散度直接衡量两个概率分布之间的差异，而 MSE 度量的是逐元素数值误差。
    注意力权重经过 softmax 归一化后形成概率分布，
    KL 散度能够更准确地捕捉分布形态（如峰值位置、尾部衰减）的变化。
  \item \textbf{对尾部分布的敏感性}：
    当 $p_{\text{ref},i}$ 较大而 $p_{\text{quant},i}$ 偏小时，
    KL 散度中的 $\ln(p_{\text{ref},i}/p_{\text{quant},i})$ 项会产生较大的惩罚，
    从而对注意力集中位置的偏移施加更强的约束。
    这一特性对于保持模型的长距离依赖建模能力尤为重要。
  \item \textbf{信息论解释}：
    KL 散度等价于使用量化分布 $\bm{p}_{\text{quant}}$ 编码来自参考分布 $\bm{p}_{\text{ref}}$ 的样本时，
    相对于最优编码所额外消耗的信息量（单位：nat）。这一解释为量化误差提供了清晰的信息论含义。
  \item \textbf{softmax 非线性下 MSE 的局限性}：
    设 Key 矩阵的量化误差为 $\bm{\delta} = \hat{\bK} - \bK$，
    则量化后的注意力 logits 变为
    $\bm{l}_{\text{ref}} + \bq\bm{\delta}^T / \sqrt{d_k}$。
    由于 softmax 函数对 logits 差值的指数放大特性，
    即使 $\|\bm{\delta}\|_2^2$ 较小（MSE 优化目标），
    若误差 $\bm{\delta}$ 恰好增大了某些位置间的 logit 差异，
    经 softmax 映射后可能导致注意力权重的显著重新分配。
    反之，MSE 较大的量化方案可能因误差方向与注意力梯度正交
    而对最终分布影响甚微。
    KL 散度直接在 softmax 输出空间上度量，
    天然规避了这一"数值精度与行为保真度不一致"的问题。
\end{enumerate}

\paragraph{KL 散度方向的选择}
式~\eqref{eq:ch3-kl} 采用 forward KL（$D_{\KL}(p_{\text{ref}} \| p_{\text{quant}})$），
即以 FP16 参考分布为权重的期望对数比。
该方向具有 \emph{zero-avoiding} 特性：
当参考分布在某个 token 位置有显著注意力权重
而量化分布在该位置权重过低时，对数比趋正无穷，产生强惩罚。
这恰好匹配 KV Cache 量化的核心关切：
防止量化导致模型\emph{遗漏}关键 token 的注意力。
相比之下，reverse KL（$D_{\KL}(p_{\text{quant}} \| p_{\text{ref}})$）
具有 \emph{zero-forcing} 特性，
主要惩罚量化分布在参考分布低权重位置上的虚假注意力，
对遗漏关键 token 的敏感度较低。
Jensen-Shannon 散度（对称化的均值分布 KL）在理论上同样可行，
但引入额外的混合分布计算开销且不保留 zero-avoiding 的强惩罚特性。
Forward KL 方向消融的实验验证列入后续工作
（见第~\ref{sec:conclusion-future}~节）。

在实际计算中，为防止 $\ln(0)$ 导致的数值不稳定，
对概率值施加小量截断 $\epsilon = 10^{-6}$：
\begin{equation}
  D_{\KL}\!\left(\bm{p}_{\text{ref}} \,\|\, \bm{p}_{\text{quant}}\right)
  \approx \sum_{i=1}^{n}
    \tilde{p}_{\text{ref},i} \cdot
    \left(\ln\tilde{p}_{\text{ref},i} - \ln\tilde{p}_{\text{quant},i}\right)
  \label{eq:ch3-kl-approx}
\end{equation}
其中 $\tilde{p} = \max(p, \epsilon)$。
由于 softmax 输出的主要概率质量远大于 $\epsilon$
（对于 32K 序列，均匀分布下的单 token 概率约 $3 \times 10^{-5}$，
远高于 $\epsilon = 10^{-6}$），
截断仅在极端尾部生效，对 KL 值的相对影响 $<$0.1\%，
在实际中可以忽略不计。

\paragraph{校准目标的规模依赖性}

上述 KL 目标的必要性并非在所有设置下均等。
后续实验（第四章）将表明：
在 INT8 精度下，KL 与 MSE 对参数的选择完全趋同，
数值代理足以替代行为度量；
而在 INT4 精度下，两种目标是否趋同取决于模型规模——
较小模型（$H_{kv}=2$）上二者选出不同参数，
较大模型（$H_{kv}=4$）上再次趋同。
这一 bit-width $\times$ 规模的交互效应
构成了本文以 $H_{kv}$ 为结构性关联变量的实验起点。

\subsection{两阶段搜索策略}
\label{subsec:ch3-two-stage}

校准采用两阶段分离搜索策略，
其核心动机在于避免 Scale 搜索与温度校正之间的循环依赖：
$\tau^{-1}$ 的最优值依赖于已确定的 Scale（因为反量化结果受 Scale 影响），
而 Scale 的评估则不应包含 $\tau^{-1}$（否则温度校正会掩盖不同 Scale 之间的实际量化误差差异）。

\paragraph{第一阶段：Scale 搜索}

在第一阶段中，框架不施加任何温度校正（即 $\tau^{-1} = 1$），
在候选 clip percentile $p_c$ 和 group size $g$ 的笛卡尔积上进行网格搜索（具体候选值集合见附录表~\ref{tab:app-search-space}）。
对于每组候选参数，先从校准样本的 Key/Value 激活值中计算逐组绝对值最大值（absmax）
并取 $p_c$ 百分位数作为裁剪阈值，再计算静态 Scale $s = \text{absmax}_{p_c} / q_{\max}$
（$q_{\max} = 127$ 对应 INT8，$q_{\max} = 7$ 对应 INT4）；
然后对每个校准样本执行量化-反量化并计算量化后注意力分布与参考分布之间的 KL 散度；
最后汇总所有层、所有头、所有样本的 KL 散度统计量（均值、P95、最大值）和裁剪率。

候选参数的选择采用鲁棒性优先策略（robust selection）：
优先在裁剪率满足约束（Key 裁剪率 $\leq 5\%$、Value 裁剪率 $\leq 5\%$）的可行集中，
按 P95 KL 散度升序排列选择最优参数。
若所有候选均不满足裁剪率约束，则回退至按裁剪率总和升序排列的策略。
Key 和 Value 分别独立搜索最优的 clip percentile，
Group size 则保持 Key 和 Value 一致。

\paragraph{第二阶段：逐头 $\tau^{-1}$ 优化}

在第一阶段确定最优 Scale 之后，第二阶段为每个注意力头独立搜索最优的温度校正因子 $\tau^{-1}$。
候选值集合涵盖了从平滑化（$\tau^{-1} < 1$）到锐化（$\tau^{-1} > 1$）的范围：
\begin{equation}
  \mathcal{T} = \{0.5,\; 0.7,\; 0.85,\; 1.0,\; 1.2,\; 1.5,\; 2.0\}
  \label{eq:tau-candidates}
\end{equation}

对于第 $l$ 层第 $h$ 个注意力头，最优 $\tau^{-1}$ 的选择为：
\begin{equation}
  \tau^{-1}_{l,h} = \argmin_{\tau^{-1} \in \mathcal{T}}
  \;\frac{1}{|\mathcal{D}|} \sum_{(\bq, \bK) \in \mathcal{D}}
  D_{\KL}\!\bigl(
    \bm{p}_{\text{ref}} \;\big\|\;
    \bm{p}_{\tau}\bigr),\quad
  \bm{p}_{\tau} = \softmax\!\Bigl(\frac{\bq \hat{\bK}^T \cdot \tau^{-1}}{\sqrt{d_k}}\Bigr)
  \label{eq:ch3-invtau-opt}
\end{equation}
其中 $\mathcal{D}$ 为校准样本集合，
$\hat{\bK}$ 为使用第一阶段确定的 Scale
进行量化与反量化后的 Key 矩阵。

分组查询注意力（GQA）~\cite{ainslie2023gqa}机制下，
多个 Query 头共享同一个 KV 头。
设 Query 头数为 $H_q$、KV 头数为 $H_{kv}$，
则第 $h$ 个 Query 头对应的 KV 头索引为
$\lfloor h / (H_q / H_{kv}) \rfloor$。
每个 Query 头仍然独立搜索 $\tau^{-1}$，但使用相同的量化 Key 矩阵。

\subsection{算法伪代码}

算法~\ref{alg:ch3-calibration} 总结了行为对齐校准的完整流程。

\begin{algorithm}[H]
  \caption{行为对齐校准算法}
  \label{alg:ch3-calibration}
  \begin{algorithmic}[1]
    \Require 参考模型 $\mathcal{M}$；校准数据集 $\mathcal{D}$；
             候选 clip percentile 集合 $\mathcal{P}_c$；
             候选 group size 集合 $\mathcal{G}$；
             候选温度因子集合 $\mathcal{T}$；
             量化位宽 $b$（$q_{\max} = 2^{b-1} - 1$）
    \Ensure 逐层 Key/Value Scale $\{s^K_l, s^V_l\}$；逐头温度因子 $\{\tau^{-1}_{l,h}\}$

    \Statex \textbf{// 阶段 0：采集校准样本}
    \For{每个校准样本 $x \in \mathcal{D}$}
      \State 执行 FP16 前向传播，提取每层的 $\bQ$（含 LayerNorm + RoPE）、$\bK$、$\bV$
    \EndFor

    \Statex
    \Statex \textbf{// 阶段 1：Scale 搜索（无温度校正）}
    \For{每组候选参数 $(p_c, g) \in \mathcal{P}_c \times \mathcal{G}$}
      \State 计算 Scale：$s = \text{percentile}(|\bK|, p_c) / q_{\max}$（逐层逐组）
      \State 量化——反量化：$\hat{\bK} = \text{round}(\bK / s) \cdot s$
      \State 计算 KL 散度统计量：$\text{mean\_kl}, \text{p95\_kl}, \text{clip\_rate}$
    \EndFor
    \State 按鲁棒性策略选择最优 $(p_c^*, g^*)$，确定 $\{s^K_l\}$ 和 $\{s^V_l\}$

    \Statex
    \Statex \textbf{// 阶段 2：逐头 $\tau^{-1}$ 优化}
    \For{每层 $l = 1, \dots, L$}
      \For{每个注意力头 $h = 1, \dots, H_q$}
        \For{每个候选 $\tau^{-1} \in \mathcal{T}$}
          \State $\text{loss}(\tau^{-1}) \gets \frac{1}{|\mathcal{D}|}\sum_{(\bq,\bK) \in \mathcal{D}}
            D_{\KL}\!\left(\bm{p}_{\text{ref}} \,\|\, \softmax\!\left(\frac{\bq\hat{\bK}^T \cdot \tau^{-1}}{\sqrt{d_k}}\right)\right)$
        \EndFor
        \State $\tau^{-1}_{l,h} \gets \argmin_{\tau^{-1} \in \mathcal{T}} \text{loss}(\tau^{-1})$
      \EndFor
    \EndFor

    \Statex
    \State \Return $\{s^K_l, s^V_l\}$, $\{\tau^{-1}_{l,h}\}$
  \end{algorithmic}
\end{algorithm}

\section{温度校正的格式-尺度双依赖性}
\label{sec:ch3-invtau}

在 Qwen2.5-1.5B/7B/8B 三个模型规模上执行诊断流程时，
本文观察到逐头温度校正（$\tau^{-1}$）的有效性
对量化格式和 GQA 头数同时敏感；
2026-04-09 补充的 Qwen2.5-14B ($H_{kv}=8$) 数据进一步支持这一模式。
本节把 $\tau^{-1}$ 作为一个启发式保留在校准管线中，
并在后文给出其有效性边界，不将其作为本文的主贡献列入。
$\tau^{-1}$ 最初作为一个补偿工具被引入，用于对冲量化误差在注意力 logits 上叠加的噪声
（在此假设量化误差近似独立同分布，本段随后讨论该假设的边界）。
该噪声使 softmax 输出熵增、分布趋于平坦化，对长距离聚焦任务尤为敏感。
本文用校正因子 $\tau^{-1}_{l,h}$ 对每个注意力头独立补偿该效应，
校正后的注意力计算为：
\begin{equation}
  \bm{p}_{\text{corrected}} = \softmax\!\left(\frac{\bq\hat{\bK}^T \cdot \tau^{-1}_{l,h}}{\sqrt{d_k}}\right)
  \label{eq:ch3-corrected-attn}
\end{equation}
其中 $\tau^{-1} > 1$ 锐化分布以补偿平坦化，$\tau^{-1} = 1$ 退化为标准基线。
实现上，$\tau^{-1}$ 通过 Q 预缩放（$\bQ' = \bQ \cdot \tau^{-1}_{l,h}$）等价实现，
\emph{逐头 $\tau^{-1}_{l,h}$ 沿 $\bQ$ 的 head 维独立作用于对应切片}，
无需修改注意力核函数签名（完整等价性推导见附录~\ref{sec:app-invtau-detail}）。

图~\ref{fig:ch3-invtau-heatmap} 展示了校准后的逐层逐头 $\tau^{-1}$ 分布，
绝大多数头的 $\tau^{-1} = 1.0$（无需校正），校准具有高度稀疏性。

\paragraph{GQA 尺度依赖性的诊断发现}
\label{para:ch3-invtau-role}

\textbf{方法论说明}. 本段呈现的 GQA 尺度依赖性是\emph{诊断发现}而非理论推导。
其机制归因采用直觉论证（intuitive argument）呈现，
与第~\ref{subsec:exp-statistics}~节对 deterministic PPL effect size
的处理保持一致：所报告的跨模型百分比差异为 \emph{effect size on greedy-decoded PPL}，
因贪婪解码下 PPL 为 bit-identical 确定量（第~\ref{subsec:exp-statistics}~节 L180--207），
统计意义上的 $p$-value 不适用，跨模型比较仅基于 effect size 方向与量级。
形式化证明与闭式误差上界留作未来工作。

$\tau^{-1}$ 的收益具有显著的格式依赖性，
并在非对称格式下进一步呈现 GQA 尺度的强相关性。
在 INT8 对称格式下（256 个量化级别），量化噪声引起的 attention 平坦化幅度有限，
自适应保护机制已是 Needle 通过率的决定性因素，
$\tau^{-1}$ 仅提供边际改善（第~\ref{subsec:exp-ablation-temperature}~节）。
在 INT4-RoleAlign 的非对称格式下（16 个量化级别），
4-bit 粗量化导致更显著的 attention 平坦化，
$\tau^{-1}$ 温度校正的效果预期与模型 GQA 尺度相关——
这是一个由格式粒度和头共享结构共同决定的启发式假设，
完整实验证据与定量分析见第~\ref{subsec:exp-rolealign-invtau}~节。

\textbf{机制直觉与独立性假设的边界}.
上述 "$\tau^{-1}$ 有效性随 $H_{kv}$ 衰减" 的机制归因可概括为
$\sigma_{\text{eff}} \propto \sigma / \sqrt{N_{\text{rep}}}$，
即 GQA 对 KV 头的共享使得 query 头平均下"有效噪声规模"被稀释。
该 $1/\sqrt{N}$ 规律依赖一个关键简化假设：\emph{不同 query 头所见的量化噪声近似独立}。
事实上，所有 $N_{\text{rep}}$ 个 query 头在同一 KV 头上共享\emph{同一个}量化后 K 张量，
噪声实际存在 pairwise 结构性相关，会削弱 $1/\sqrt{N_{\text{rep}}}$ 稀释规律。
因此本段报告的规律应理解为 \emph{在 query 交互独立性假设下的启发式解释}，
而非严格定理。完整的相关噪声模型与形式化误差上界
留作未来工作；现有证据来源于 3 个模型规模（$H_{kv} \in \{2, 4, 8\}$）
的经验观察而非 closed-form 推导。

据文献调研（详见第~\ref{sec:ch2-quant-softmax}~节），
KV Cache 量化中温度校正的 GQA 尺度依赖性此前未被系统报道；
最相近的 principled 工作 KVTuner~\cite{xu2025kvtuner} Lemma 1
给出 Key error 13.9$\times$ INT8$\to$INT4 的闭式放大因子，
但停留在 layer 粒度而不涉及 GQA 架构的跨头稀释结构。
本章节的诊断发现与 KVTuner 的层级误差分配互为正交。
该机制的完整理论推导见附录~\ref{sec:app-invtau-detail}。

\section{自适应保护与 INT8 对称量化配置}
\label{sec:ch3-static-scale}

\subsection{静态 Scale 的设计}

本框架采用离线校准得到的静态 Scale 而非每 token 步动态计算。
静态 Scale 将运行时 absmax 扫描移至离线阶段（推理时仅需一次内存读取），
保证不同 token 步使用相同的量化网格避免早期/晚期 token 精度不一致，
并将推理结果完全由模型权重与 JSON 校准产物决定以便精确复现。

静态 Scale 的计算遵循对称量化（symmetric quantization）公式。
对于输入张量 $\bx \in \mathbb{R}^{B \times H \times S \times D}$，
按 group size $g$ 将最后一维划分为 $G = D / g$ 个组：
\begin{equation}
  s_{b,h,s,j} = \frac{\text{percentile}\!\left(|x_{b,h,s,jg:(j+1)g}|,\; p_c\right)}{q_{\max}}
  \label{eq:ch3-static-scale}
\end{equation}
其中 $j = 0, \dots, G-1$ 为组索引，$p_c$ 为 clip percentile（典型值为 99.5），
$q_{\max} = 127$（INT8）或 $q_{\max} = 7$（INT4）。

量化操作为：
\begin{equation}
  \hat{x} = \text{clamp}\!\left(\text{round}\!\left(\frac{x}{s}\right),\; -q_{\max},\; q_{\max}\right) \cdot s
  \label{eq:ch3-quant-deq}
\end{equation}

Scale 张量始终以 float32 精度存储，不随输入张量的 dtype 变化，
以避免 float16 精度下极小 Scale 值退化为零导致的数值不稳定。

\paragraph{量化时间语义}
\label{para:ch3-quant-timing}

在推理过程中，量化操作发生在 KV Cache 的 \code{append} 步骤：
每当新的 Key/Value 张量被追加至缓存时，
使用对应的静态 Scale 即时执行量化，
将浮点张量转换为 INT8/INT4 格式后写入缓存。
一旦写入，已存储的量化值\emph{不再更新}。
历史 token 的量化表示在后续 decode 步骤中保持不变，
反量化时使用各位置自身的 Scale 独立还原：
\begin{equation}
  \hat{x}_{b,h,t,j} = x^q_{b,h,t,j} \cdot s_{b,h,t,j}
  \quad \forall\, t = 1, \dots, T
  \label{eq:ch3-per-position-deq}
\end{equation}
其中 $t$ 为序列位置索引，$j$ 为组索引。
这一"写入即冻结"语义保证缓存的流式积累不引入跨 token 的量化参数耦合。

然而，静态 Scale 的有效性前提是校准数据能够充分代表推理时的输入分布。
当实际推理中出现校准未覆盖的激活值范围时，
静态 Scale 可能导致过度裁剪（clipping），引起显著的精度退化。
下文的自适应保护机制正是为解决这一边界情况而设计。

\subsection{自适应保护机制}

静态 Scale 的一个潜在风险是：当推理时的输入分布超出校准数据的覆盖范围时，
某些激活值可能超出静态 Scale 对应的量化区间，
导致大量值被裁剪至 $\pm q_{\max}$，引起显著的精度退化。

为应对这一风险，本框架设计了自适应保护机制（adaptive static scale protection）。
其核心思想是：在每次量化时，
比较静态 Scale 与当前输入的动态 Scale，取两者的较大值作为最终 Scale：
\begin{equation}
  s_{\text{final}} = \max\!\left(s_{\text{static}} \cdot m,\; s_{\text{dynamic}}\right)
  \label{eq:ch3-adaptive}
\end{equation}
其中 $s_{\text{static}}$ 为离线校准得到的静态 Scale，
$s_{\text{dynamic}} = |x|_{\max} / q_{\max}$ 为当前输入的动态 Scale，
$m \geq 1$ 为保护裕度（protection margin），默认值为 $1.0$。

其设计逻辑为：当输入分布在校准覆盖范围内时
$s_{\text{dynamic}} \leq s_{\text{static}}$，最终使用静态 Scale 保持校准一致性；
当出现异常大的激活值时 $s_{\text{dynamic}} > s_{\text{static}}$，
自动切换为动态 Scale 避免大量裁剪。
保护裕度 $m$ 可根据场景调节（$m > 1$ 为更宽松保护，$m = 1$ 为最严格模式）。

\paragraph{自适应保护对历史缓存的影响}
当自适应机制在某个 token 步触发
（$s_{\text{dynamic}} > s_{\text{static}}$）时，
\emph{仅当前 token 的 Scale 受影响}，
已存储在缓存中的历史 token 保持其原始量化值和 Scale 不变。
这是因为量化操作遵循"写入即冻结"语义
（第~\ref{para:ch3-quant-timing}~段）：
每个位置的 Scale 在 \code{append} 时确定并永久存储，
后续 token 的自适应 Scale 调整不会回溯修改历史缓存。
因此，自适应保护是一种\emph{前向保护}机制，
其局部性保证缓存的流式积累不引入跨 token 的 Scale 耦合。

此外，量化后对裁剪率进行实时监控：
当某层的裁剪率超过预设阈值（默认 5\%）时，发出运行时警告，
提示用户可能需要重新校准或启用自适应保护。

\paragraph{向非对称格式的扩展}
上述 Scale 搜索框架在设计上是格式无关的：
当量化格式从对称 per-group 升级为非对称 per-channel/per-token 时
（第~\ref{sec:ch3-rolealign}~节），
同一 KL 散度目标函数可直接用于搜索最优的 K percentile 和 V percentile 参数，
无需修改搜索框架的核心逻辑。

\section{KIVI-style 格式上的行为引导实例化 RoleAlign}
\label{sec:ch3-rolealign}

INT4-RoleAlign 是在 KIVI~\cite{liu2024kivi} 提出的
per-channel Key + per-token Value 非对称格式上，
以 attention-KL 为目标的行为引导实例化。
该方案受第~\ref{sec:exp-kv-sensitivity}~节的 K/V 消融诊断结果启发：
INT4 退化呈 Key 主导、且与 GQA 头数 $H_{kv}$ 密切相关。
以下首先阐明量化格式与校准目标的正交性，再详述具体设计。

\paragraph{量化格式与校准目标的正交性}
\label{para:ch3-orthogonality}
理解 INT4-RoleAlign 的设计逻辑需要区分两个正交的设计维度：
\textbf{量化格式}（对称 per-group vs 非对称 per-channel/per-token）
与\textbf{校准目标}（MSE/percentile vs KL 行为对齐）。
前述各节的 INT8 路径在校准目标维度实现了核心改进，
以 KL 散度替代 MSE/percentile，
结合自适应保护机制保障长上下文鲁棒性。
当校准目标维度的改进已到位后，若低比特仍出现系统性失效，
格式维度的升级成为自然的设计方向。
INT4-RoleAlign 正是这一正交扩展的实证：
在保持 KL 校准目标不变的前提下，将量化格式从对称 per-group
升级为 per-channel K + per-token V 非对称，
实现校准目标与量化格式两个维度的协同改进。

前述各节证明行为对齐校准在 INT8 对称格式下的有效性。
当位宽降至 4~bit 时，量化级别从对称 INT8 的 255 级（值域 $[-127, 127]$）
骤降至对称 INT4 的 15 级（值域 $[-7, 7]$），
Key 张量的通道间数值异质性可能超出对称格式的表示能力。
本文后续使用的非对称 INT4 路径（§\ref{sec:ch3-rolealign}）
采用完整 $[-8, 7]$ 共 16 个离散值，通过零点偏移获得一个额外的表示量级。
第~\ref{sec:exp-int4}~节的诊断实验将证实这一判断：
受控 K/V 消融在三个模型族上一致确认 Key 量化主导质量损失，
且退化程度呈显著的 GQA 架构依赖特征。
Key 决定了注意力的``看哪里''（attention logits 的排序结构），
其通道间数值异质性在低比特下无法被对称格式有效捕捉。
上述诊断结果提示采用角色差异化量化轴是一个合理的设计方向：
对 Key 和 Value 采用差异化的量化轴，
以适配两者不同的数值分布特性与功能角色。

\subsection{从对称到非对称的格式升级}

基于上述诊断发现，本节提出 \textbf{INT4-RoleAlign}
（Role-Aware Asymmetric Quantization with BA-guided Calibration），
后文统一简称为 INT4-RoleAlign（不省略 INT4- 前缀以明确比特宽度约束）。
该方法吸收 KIVI~\cite{liu2024kivi} 的非对称量化轴设计思想，
将 Key 和 Value 的量化策略从统一的 per-group 对称
升级为角色感知的非对称格式：

\begin{itemize}[leftmargin=2em, itemsep=2pt]
  \item \textbf{Key：per-channel 非对称量化}。
    沿序列维度（across tokens）为每个通道独立计算 Scale 与零点偏移，
    捕捉 Key 矩阵中通道间的数值异质性。
    形式化地，对于第 $l$ 层的 Key 矩阵 $\bK \in \mathbb{R}^{S \times d_k}$，
    记 $p_K$ 为 Key 的双端截断 percentile（由离线 BA 校准搜索确定），
    per-channel 量化参数为：
    \begin{equation}
      \begin{aligned}
        m^K_{l,j} &= \mathrm{percentile}\!\left(K_{:,j},\; 100 - p_K\right), \;\;
        M^K_{l,j}  = \mathrm{percentile}\!\left(K_{:,j},\; p_K\right), \\
        s^K_{l,j} &= \frac{M^K_{l,j} - m^K_{l,j}}{q_{\max} - q_{\min}}, \;\;
        z^K_{l,j}  = m^K_{l,j} - q_{\min}\cdot s^K_{l,j},
      \end{aligned}
      \;\; j = 1, \dots, d_k
      \label{eq:ch3-perchannel-k}
    \end{equation}
    其中 $q_{\min}, q_{\max}$ 为量化比特宽度下的最小/最大整数值
    （INT4：$q_{\min}=-8, q_{\max}=7$；INT8：$q_{\min}=-128, q_{\max}=127$）。
    量化与反量化规则为
    $q = \mathrm{clamp}(\mathrm{round}((x - z)/s),\, q_{\min},\, q_{\max})$，
    $\hat{x} = q\cdot s + z$。
  \item \textbf{Value：per-token 非对称量化}。
    沿特征维度（across channels）为每个 token 独立计算 Scale 与零点，
    适配 Value 矩阵中 token 间的动态范围差异：
    \begin{equation}
      \begin{aligned}
        m^V_{l,t} &= \mathrm{percentile}\!\left(V_{t,:},\; 100 - p_V\right), \;\;
        M^V_{l,t}  = \mathrm{percentile}\!\left(V_{t,:},\; p_V\right), \\
        s^V_{l,t} &= \frac{M^V_{l,t} - m^V_{l,t}}{q_{\max} - q_{\min}}, \;\;
        z^V_{l,t}  = m^V_{l,t} - q_{\min}\cdot s^V_{l,t},
      \end{aligned}
      \;\; t = 1, \dots, S
      \label{eq:ch3-pertoken-v}
    \end{equation}
    其中 $p_V$ 为 Value percentile 参数。
    特别地，$(p_K, p_V) = (100, 100)$ 时退化为标准 min-max 非对称量化
    （即 KIVI-style absmin/absmax 配置）；$p_K, p_V < 100$ 时以对称
    双端截断缓解极端 outlier 对 scale 的放大。
\end{itemize}

\subsection{BA-guided Percentile 校准}
\label{subsec:ch3-ba-percentile}

INT4-RoleAlign 的核心设计区别在于参数确定范式。
KIVI-style 完全依赖运行时统计量（absmax/min）无需离线校准，
INT4-RoleAlign 通过行为对齐框架的离线搜索确定最优的 $p_K$ 和 $p_V$ percentile 参数。

\paragraph{参数化动机}
使用 $(p_K, p_V)$ 离散 percentile 参数化而非直接在 per-channel scale 向量上做连续优化，
有两个理由：(1) percentile 参数化将搜索空间约束为低维离散网格，
可穷举枚举并且在 outlier 干扰下保持鲁棒；
(2) 离散网格为第~\ref{subsec:exp-statistics}~节的 BH-FDR
统计框架提供可枚举的比较族。
代价是将原始 KL 目标约束到一个 6$\times$6 离散网格上；
相比 KVTuner~\cite{xu2025kvtuner} Lemma 1 的 closed-form layer-wise 误差分配，
本文的 heuristic search 选择是为了与第~\ref{sec:ch3-static-scale}~节
对称路径的 $(p_c, g)$ 搜索框架保持接口一致，代价换来部署简洁性。

\paragraph{搜索空间}
$\mathcal{P}_K = \mathcal{P}_V = \{99.0,\, 99.5,\, 99.9,\, 99.95,\, 99.99,\, 100.0\}$，
$|\mathcal{P}_K| = |\mathcal{P}_V| = 6$，搜索采用完整笛卡尔积枚举 $|\mathcal{P}_K \times \mathcal{P}_V| = 36$ 个配置。
特别值得指出：$(p_K, p_V) = (100.0,\, 100.0)$ 恰好对应 KIVI-style 的 absmax/min 配置，
因此 RoleAlign 与 KIVI-style 的比较可理解为同一非对称格式空间内的参数化比较，
而非跨格式比较。
当前证据表明该搜索空间允许找到与行为诊断一致的非端点配置，
但尚不足以据此宣称 RoleAlign 在方法论上优于 KIVI-style
（详见第~\ref{subsec:exp-rolealign-results}~节的实证讨论）。

搜索目标仍为 KL 散度（公式~\eqref{eq:ch3-kl}），
但搜索空间从对称格式的 $(p_c, g)$ 切换为非对称格式的 $(p_K, p_V)$：
\begin{equation}
  (p_K^*, p_V^*) = \argmin_{(p_K, p_V) \in \mathcal{P}_K \times \mathcal{P}_V}
  \frac{1}{|\mathcal{D}|} \sum_{(\bq, \bK, \bV) \in \mathcal{D}}
  D_{\KL}\!\left(\bm{p}_{\text{ref}} \,\|\, \bm{p}_{\text{asym}}\right)
  \label{eq:ch3-ba-percentile}
\end{equation}
其中 $\bm{p}_{\text{asym}}$ 为使用 per-channel K Scale 和 per-token V Scale
量化后的注意力分布。

\section{Triton 融合量化解码注意力}
\label{sec:ch3-triton}

本节给出上述量化方案对应的 Triton 融合核函数实现。
在自回归解码阶段，每步仅生成一个 token（$q_{\text{len}} = 1$），
但需要与整个上下文的 Key/Value 缓存进行注意力计算。
若采用"先反量化全部缓存至 FP16，再执行标准注意力"的朴素方案，
将产生大量不必要的全局显存读写，抵消量化带来的存储收益。

本框架采用 Triton~\cite{tillet2019triton} 编程语言实现了融合量化解码注意力核函数，
将 INT8/INT4 反量化与注意力计算在单个 GPU kernel 内完成。
该核函数借鉴了 FlashAttention~\cite{dao2022flashattention} 的 online softmax 技术，
避免材化（materialize）完整的注意力得分矩阵。

\subsection{INT8 核函数设计}
\label{subsec:ch3-triton-int8}

INT8 融合核函数沿用 FlashAttention~\cite{dao2022flashattention} 的 online softmax 技术
将反量化与注意力计算融合在单个 GPU kernel 内，
避免材化完整的注意力得分矩阵与 FP16 反量化中间张量。
Grid 按 $(B, H_q)$ 启动；每个 program 实例沿序列维度分块迭代，
在每个分块内片上反量化 Key/Value（FP32 精度，避免 FP16 乘法精度损失），
按式~\eqref{eq:ch3-online-softmax} 更新全局最大值 $m$ 与指数和 $\ell$，
最终归一化得到 FP16 输出。
\begin{equation}
\begin{aligned}
  m' &= \max(m,\; \max_i \text{score}_i),\quad
  \alpha = e^{m - m'},\quad \beta_i = e^{\text{score}_i - m'} \\
  \ell' &= \ell \cdot \alpha + \sum_i \beta_i,\quad
  \text{acc} \gets \text{acc} \cdot \alpha + \sum_i \beta_i \cdot \bV_{\text{fp32}}[i, :]
\end{aligned}
\label{eq:ch3-online-softmax}
\end{equation}
分块大小 $B_s$ 采用启发式：上下文 $\geq 8192$ 时用 $B_s=128$，否则 $B_s=64$。
得分计算采用逐元素乘法累加而非 Tensor Core（$\bq$ 为 $q_{\text{len}}=1$ 向量）。
整体复杂度为 $O(B \cdot H_q \cdot S \cdot d_k)$，
而访存较 naive dequant+SDPA 降低 $O(S \cdot d_k)$ 字节的中间张量物化。


\subsection{INT4 核函数设计}
\label{subsec:ch3-triton-int4}

INT4 量化采用 bit-packing 存储：两个 INT4 值（$[-8, 7]$）打包为一个 INT8 字节，
存储维度从 $d_k$ 压缩为 $d_k / 2$。INT4 核函数的设计复用了 INT8 核函数的计算流程，
仅在输入阶段增加一个解包（unpack）步骤。

解包操作将 packed INT8 张量（形状 $[B, H_{kv}, S, d_k/2]$）还原为
signed INT8 张量（形状 $[B, H_{kv}, S, d_k]$，值域 $[-8, 7]$）：
\begin{equation}
\begin{aligned}
  \text{high} &= (x_{\text{packed}} \gg 4) \quad \text{（算术右移，保留符号位）} \\
  \text{low}  &= (x_{\text{packed}} \ll 4) \gg 4 \quad \text{（先左移再算术右移，实现符号扩展）}
\end{aligned}
\label{eq:ch3-unpack-int4}
\end{equation}
解包后的 INT8 张量直接送入 INT8 核函数执行后续的反量化和注意力计算。
这一设计避免了为 INT4 单独编写一套 Triton 核函数，降低了代码维护成本。

\subsection{INT4 非对称融合核函数}
\label{subsec:ch3-triton-int4-asym}

上述 INT4 核函数针对的是对称量化（per-group Scale，无 zero-point）。
INT4-RoleAlign 和 KIVI-style 采用非对称量化，具有不同的 Scale 维度：
Key 使用 per-channel Scale（对所有 token 共享），
Value 使用 per-token Scale（对所有 channel 共享），
且两者均包含 zero-point 偏移。
这一格式差异使得上述``物化→调用 INT8 核函数''的方案不再适用，
需要设计专用的非对称融合核函数。

本文采用\textbf{分通道（split-channel）}设计，
在 kernel 内部直接对 packed INT4 字节进行 nibble 解析和非对称反量化，
避免任何中间张量的全局显存物化。
核心思路是将 Query、K Scale/ZP 按奇/偶通道分别加载（各 $d_k/2$ 个值），
$\text{QK}^T$ 分解为偶数与奇数通道两半之和
$s_i = \sum_j q_{2j} k^{(hi)}_{i,j} + \sum_j q_{2j+1} k^{(lo)}_{i,j}$，
online softmax 在合并后的 $s_i$ 上正常执行；
V 的加权求和同样按奇/偶通道分别累积后合并写回。
非对称反量化中 K 的 per-channel Scale/ZP 在循环外加载一次（$O(d_k)$），
V 的 per-token Scale/ZP 按 block 加载（$O(\text{BLOCK\_SIZE})$），均在 FP32 精度下完成。
该设计在数值精度上与 PyTorch 参考实现的 max abs diff $<10^{-2}$，
并通过端到端 Needle 评测验证（10/10 全 PASS，完整数据见
第~\ref{subsec:exp-rolealign-results}~节与第~\ref{subsec:exp-rolealign-boundary}~节）。
\emph{与 Tensor Core 路径的关系。}本节的 split-channel 方案基于 CUDA Core，
通过奇偶 lane 分裂规避 per-channel 反量化中的 scale 广播冲突，验证了算法路径的数值正确性。
其 Tensor Core 并行潜力与 BitDecoding~\cite{du2026bitdecoding} 的 NVFP4 设计互为补充；
后者在硬件资源利用维度展示了进一步优化空间，但依赖 NVFP4 数值格式与 Blackwell 架构支持。

\emph{与 KIVI Residual Buffer 的对比}：
KIVI~\cite{liu2024kivi} 推荐将最近 $R$ 个 token 保持 FP16（Residual Buffer），
仅对更早的 token 执行 INT4 量化，由此在缓存中形成 INT4/FP16 混合格式。
融合核函数若要处理该混合布局，需要对量化区和 FP16 区分别加载、
分别计算注意力后再合并，增加了 kernel 设计的复杂度。
本文的 RoleAlign 采用离线校准确定静态 Scale，
所有 token（含 decode 阶段新增 token）均以统一 INT4 格式存储，
无需 Residual Buffer，使得单一融合核函数即可处理整个 KV Cache。

\paragraph{核内近似 Percentile}
\label{para:ch3-in-kernel-pct}

KIVI-style 非对称量化中，Value 的 per-token Scale/zero-point
依赖每个 token 位置的极值统计量；标准做法是在 \code{append} 时
计算精确的 absmax/min 并预存，融合核函数直接读取。
本文实现了一种核内近似方案：在 tile 加载过程中对每个 token
的 $d_k$-维 Value 向量独立排序后取前 2 大/前 2 小的均值
作为 percentile 剪裁边界的在线估计，近似 $p{\approx}99.9$ 双端裁剪（式~\eqref{eq:ch3-in-kernel-pct}）。
\begin{equation}
  \hat{v}_{\max,i} \approx \mathrm{mean}\bigl(\mathrm{sort}(\bv_i,\text{desc})[1{:}2]\bigr),\quad
  \hat{v}_{\min,i} \approx \mathrm{mean}\bigl(\mathrm{sort}(\bv_i,\text{asc})[1{:}2]\bigr)
\label{eq:ch3-in-kernel-pct}
\end{equation}
该近似使非对称融合核函数无需依赖外部预计算统计量，
避免校准产物中 percentile 参数与 kernel 路径的耦合；
延迟影响在第四章实验中量化。

\subsection{GQA 支持机制}
\label{subsec:ch3-gqa}

分组查询注意力（GQA）~\cite{ainslie2023gqa} 中，
$H_q$ 个 Query 头被均匀分为 $H_{kv}$ 组，每组共享同一对 KV 头。
定义重复因子 $N_{\text{rep}} = H_q / H_{kv}$，
核函数通过以下映射实现 GQA 支持：
\begin{equation}
  h_{kv} = \lfloor h_q / N_{\text{rep}} \rfloor
  \label{eq:ch3-gqa-mapping}
\end{equation}
其中 $h_q$ 为 Query 头索引。
在核函数内部，Grid 按 $(B, H_q)$ 启动，
每个程序实例根据 $h_{kv}$ 索引读取对应的 KV 缓存，
无需在显存中复制 KV 头，从而节省了 $N_{\text{rep}}$ 倍的显存带宽。

\subsection{Phase Boundary 效率分析框架}
\label{subsec:ch3-phase-boundary}

融合量化解码核函数的效率增益并非在所有配置下均存在。
当 KV 头数 $H_{kv}$ 较小时，Triton Grid 中的 program 实例数量有限，
GPU SM（Streaming Multiprocessor）利用率较低；
当序列长度较短时，单次 decode 的访存量不足以使带宽节省超过
额外的 kernel 启动与反量化计算开销。
因此，融合核相对于 naive dequant+SDPA 路径的
效率优势依赖于 $(H_{kv}, \text{seq\_len})$ 的组合。

本文将这一二维空间中融合核开始产生显著延迟优势的边界
称为 \textbf{Phase Boundary}。
定义融合核的 TPOT（Time Per Output Token）为 $T_{\text{fused}}$，
naive dequant+SDPA 路径的 TPOT 为 $T_{\text{naive}}$，
则 Phase Boundary 可表示为满足以下条件的参数组合：
\begin{equation}
  \Delta T(H_{kv}, S) = T_{\text{naive}}(H_{kv}, S) - T_{\text{fused}}(H_{kv}, S) \approx 0
  \label{eq:ch3-phase-boundary}
\end{equation}
在 $\Delta T > 0$ 的区域（长序列、高 $H_{kv}$），融合核提供正向加速；
在 $\Delta T \leq 0$ 的区域（短序列、低 $H_{kv}$），naive 路径更优。

影响 Phase Boundary 位置的两个主要机制为：
\textbf{SM 利用率}方面，Triton 核函数的 Grid 按 $(B, H_q)$ 启动，
当 $H_{kv}$ 较小（如 $H_{kv}=2$，$H_q=12$）时实例总数可能不足以填满所有 SM
而导致硬件利用率受限；相反 $H_{kv}=8$（$H_q=32$，如 LLaMA-3.1-8B）时 Grid 更密，SM 占用率更高。
\textbf{带宽节省与反量化开销的权衡}方面，融合核通过在 SRAM 中完成反量化
避免了全局显存的中间张量物化，带宽节省量与序列长度成正比（$O(S \cdot d_k)$），
但反量化本身引入额外的 ALU 指令开销，当序列较短时带宽节省不足以抵消该开销。

上述分析给出定性的 cost 直觉。
Phase Boundary 的具体位置依赖于硬件参数（SM 数量、显存带宽、L2 缓存大小）
和 kernel 实现细节，第四章将通过跨模型的实测数据给出经验刻画。

\section{系统实现}
\label{sec:ch3-system}

\subsection{KV Cache 管理架构}

本框架实现了统一的 KV Cache 管理接口，支持多种量化模式下的透明切换。
所有 Cache 类型均实现 \code{append(layer\_id, k, v)}
和 \code{get\_kv(layer\_id)} 两个核心方法，对上层生成循环提供一致的抽象。
\textbf{FP16Cache} 直接以 FP16 精度存储 Key/Value 张量作为全精度参考基线。
\textbf{INT8Cache} 在 \code{append} 时执行对称量化（公式~\eqref{eq:ch3-quant-deq}），
Key/Value 存储为 INT8 格式并保存 FP32 精度 Scale 张量，支持动态/静态 Scale
两种模式与自适应保护机制（第~\ref{sec:ch3-static-scale}~节），
decode 阶段可选 Triton 融合路径或 PyTorch 参考实现。
\textbf{INT4Cache} 在 INT8Cache 基础上增加 bit-packing 层，
\code{append} 时先量化为 INT4 值域 $[-8, 7]$ 再将相邻两值打包为一个 INT8 字节，
\code{get\_kv} 执行解包和反量化，存储维度从 $[B, H, S, d_k]$ 压缩为 $[B, H, S, d_k/2]$。
\textbf{KIVIStyleCache} 实现 KIVI~\cite{liu2024kivi} 风格的非对称量化策略
（Key per-channel、Value per-token），不使用本文行为对齐校准，作为对比基线。

\subsection{生成循环集成}

生成循环（generation loop）是连接校准产物、Cache 管理和 Triton 核函数的枢纽：
启动时从 JSON 文件加载静态 Scale 和 $\tau^{-1}$ 转换为 GPU 张量注入 Cache 引擎；
decode 阶段（$q_{\text{len}}=1$）通过替换注意力层前向函数拦截原始计算路径，
对 Query 施加 $\tau^{-1}$ 预缩放后调用 Triton 融合核函数；
并根据配置中的 \code{kv\_mode} 参数自动选择对应的 Cache 类型和 decode 路径。

\subsection{量化模式总览}

表~\ref{tab:ch3-kv-modes} 总结了本框架支持的 9 种量化模式及其核心特征。

\section{复杂度与资源分析}
\label{sec:ch3-complexity}

本节从时间复杂度、显存占用和访存效率三个维度对框架进行定量分析，
为第四章的实验结果提供理论预期。

\subsection{离线校准时间复杂度}

离线校准的计算开销主要来自两个搜索阶段。
设校准样本数为 $N$，序列长度 $n$，模型层数 $L$，
Query 头数 $H_q$，头维度 $d_k$。

\paragraph{第一阶段（Scale 搜索）}
在 clip percentile 集合 $\mathcal{P}_c$（$|\mathcal{P}_c| = 4$）与
group size 集合 $\mathcal{G}$（$|\mathcal{G}| = 4$）的笛卡尔积上搜索最优 Scale，
时间复杂度为：
\begin{equation}
  T_1 = O\!\left(|\mathcal{P}_c| \cdot |\mathcal{G}| \cdot N \cdot L \cdot H_q \cdot n \cdot d_k\right)
  \label{eq:ch3-complexity-stage1}
\end{equation}

\paragraph{第二阶段（$\tau^{-1}$ 搜索）}
为每个注意力头遍历 $|\mathcal{T}| = 7$ 个候选温度因子：
\begin{equation}
  T_2 = O\!\left(|\mathcal{T}| \cdot N \cdot L \cdot H_q \cdot n \cdot d_k\right)
  \label{eq:ch3-complexity-stage2}
\end{equation}

由于 $|\mathcal{P}_c| \cdot |\mathcal{G}| > |\mathcal{T}|$，第一阶段占主要开销。
在默认配置下，Qwen2.5-1.5B 的完整校准约需 15--20 分钟，
7B 模型约需 40--50 分钟（单张 H20 GPU）。

\subsection{KV Cache 显存精确公式}

对于具有 $L$ 层、$H_{kv}$ 个 KV 头、头维度 $d_k$、
当前序列长度 $S$ 的模型，KV Cache 的显存占用如下。

\paragraph{FP16 模式}
Key 和 Value 各以 FP16 格式存储：
\begin{equation}
  M_{\text{fp16}} = 2 \cdot L \cdot H_{kv} \cdot S \cdot d_k \cdot 2 \;\text{bytes}
  = 4 L H_{kv} S d_k
  \label{eq:ch3-mem-fp16}
\end{equation}

\paragraph{INT8 模式}
Key/Value 以 INT8 存储，每组 $g$ 个元素共享一个 FP32 Scale：
\begin{equation}
  M_{\text{int8}} = 2 L H_{kv} S \left(d_k + \frac{d_k}{g} \cdot 4\right)
  = 2 L H_{kv} S \cdot d_k \!\left(1 + \frac{4}{g}\right)
  \label{eq:ch3-mem-int8}
\end{equation}

\paragraph{INT4 模式}
两个 INT4 值打包为一个 INT8 字节，存储维度减半：
\begin{equation}
  M_{\text{int4}} = 2 L H_{kv} S \left(\frac{d_k}{2} + \frac{d_k}{g} \cdot 4\right)
  = 2 L H_{kv} S \cdot d_k \!\left(\frac{1}{2} + \frac{4}{g}\right)
  \label{eq:ch3-mem-int4}
\end{equation}

在默认 $g = 16$ 下，INT8 的理论显存比值为 $(1 + 4/g)/2 = 62.5\%$（节省 37.5\%），
实验中观测到约 44\% 的节省（第四章表~\ref{tab:kv-modes}），
差异来自 Scale 张量的紧凑存储优化。
INT4 在 $g=16$ 下理论比值为 37.5\%，节省可达 62.5\%，
但伴随第四章所示的质量退化。

\subsection{Triton 核函数访存分析}

decode 阶段的注意力计算属于典型的访存密集型操作：
每个 token 步仅处理一个 Query 向量（$q_{\text{len}} = 1$），但需遍历整个 KV 缓存。
对于 INT8 融合核函数，每个分块（大小 $B_s$）的访存量为
$2 B_s \cdot d_k (1 + 4/g)$ 字节（Key/Value 数据与 Scale），
计算量约 $4 B_s \cdot d_k$ 次浮点运算（点积、加权求和、反量化）。
算术强度为：
\begin{equation}
  \text{AI}_{\text{int8}} = \frac{4 B_s \cdot d_k}{2 B_s \cdot d_k (1 + 4/g)}
  = \frac{2}{1 + 4/g}
  \;\text{FLOP/byte}
  \label{eq:ch3-ai-int8}
\end{equation}
$g=16$ 时 $\text{AI}_{\text{int8}} = 1.6$ FLOP/byte，
为 FP16 朴素实现（1.0 FLOP/byte）的 1.6 倍。
这解释了量化在 decode 阶段实现加速的根本原因：
相同计算量对应更少的数据搬运。

\subsection{校准产物存储开销}

校准产物以 JSON 格式存储逐层 Key/Value 的静态 Scale（$2L \cdot d_k/g$ 个 FP32 值）
和逐层逐头 $\tau^{-1}$（$L \cdot H_q$ 个 FP32 值）。
以 Qwen2.5-7B 为例，参数总量约 1{,}232 个浮点数，
JSON 文件约 30--50~KB，相对于模型权重（$\approx$14~GB FP16）完全可以忽略不计。

\section{本章小结}
\label{sec:ch3-summary}

行为对齐量化框架由三个层级构成：
attention-KL 作为行为偏移度量提供基础，
该度量下的 robust selection 协议同时承担校准搜索与诊断分析两种角色，
而最终的低比特实例化 RoleAlign 在 KIVI~\cite{liu2024kivi}
提出的 per-channel Key $+$ per-token Value 非对称格式上
以离线 attention-KL 搜索 BA percentile，
由 Triton 融合量化解码核函数提供系统实现支撑。
逐头温度校正 $\tau^{-1}$ 作为诊断过程中观察到的启发式呈现，
其效果同时依赖量化格式和 GQA 头数 $H_{kv}$。
下一章围绕 bit-width 依赖的校准目标对比、
Key 主导退化与 retrieval/PPL 解耦的受控诊断、
以及 RoleAlign 实例化的验证与能力边界展开实验证据。
