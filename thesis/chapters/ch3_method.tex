\section{注意力近似误差分析}
\label{sec:ch3-problem}

设标准 attention 的单头解码过程为：
\begin{equation}
  z_i = \frac{\bq^\top \bk_i}{\sqrt{d_k}},\quad
  a_i = \mathrm{softmax}(z)_i,\quad
  o = \sum_i a_i\, v_i
  \label{eq:ch3-attn}
\end{equation}
其中 $\bq \in \mathbb{R}^{d_k}$ 为当前 query、
$\bk_i, v_i$ 为第 $i$ 个历史 token 对应的 Key / Value。
量化将 $\bk_i, v_i$ 替换为 $\hat{\bk}_i, \hat{v}_i$，
进而得到量化版本的 logits $\hat z_i$ 与分布 $\hat a_i = \mathrm{softmax}(\hat z)_i$；
量化后的 attention 输出为
\begin{equation*}
  \hat o = \sum_i \hat a_i \, \hat v_i.
\end{equation*}
输出端的误差可严格分解为以下 telescoping 形式：
\begin{equation}
  \hat{o} - o
  = \underbrace{\sum_i (\hat a_i - a_i)\, v_i}_{\text{分布侧贡献项}}
  + \underbrace{\sum_i \hat a_i\, (\hat v_i - v_i)}_{\text{聚合侧贡献项}}
  \label{eq:ch3-error-decomp}
\end{equation}
该分解是精确的代数 telescoping 展开（非 K/V 因果分离）：
第一项源自 K 扰动经 softmax 非线性传播引起的分布偏移 $(\hat a_i - a_i)$，
在 FP16 的 V 下加权求和；
第二项源自 V 扰动 $(\hat v_i - v_i)$，
在量化后 attention $\hat a_i$（已受 K 扰动影响）下的直接聚合贡献。
两项共同刻画量化对注意力行为（分布 $\bm{a}$ 与输出 $\bm{o}$）的完整损伤；
真正被下游任务感知的，是这两条路径的最终合成，
而非张量数值距离 $\|\hat{\bk} - \bk\|$ 或 $\|\hat{v} - v\|$ 本身。

基于该分解，本文采用引言中提出的假设 H 作为设计原则：
以注意力行为——特别是分布 $\bm{a}$——的保持度作为直接优化对象。
形式上，若以 $\bm{p}$ 记 FP16 下的注意力权重分布、$\bm{q}$ 记量化后分布，
行为偏移由 Kullback--Leibler 散度衡量：
\begin{equation}
  D_{\KL}(\bm{p} \| \bm{q}) = \sum_i p_i \log \frac{p_i}{q_i}
  \label{eq:ch3-kl-general}
\end{equation}
$D_{\KL}$ 仅直接刻画\emph{分布侧}偏移；
聚合侧 $\bm{o}$ 的保真度由校准后模型在端到端任务指标
（PPL、Needle、LongBench 等）上的表现间接验证。
本章后续以 $D_{\KL}$ 作为两层设计的共同度量：
在校准层（§\ref{sec:ch3-calibration}）作为目标函数，
在分配层（§\ref{sec:ch3-allocator}）作为跨层敏感度度量，
使行为引导原则贯通校准与预算分配两个抽象层次。

图~\ref{fig:error-decomposition} 可视化了上述两条误差传播路径。

\section{方法框架总览}
\label{sec:ch3-overview}
\label{sec:ch3-system}

行为引导量化框架分离线与在线两个阶段。离线阶段在少量 WikiText-2 校准样本上独立地
为每层的 Key 与 Value 搜索最优的 per-layer per-group 静态 Scale 常量，
目标是最小化量化前后注意力分布的 KL 散度（§\ref{sec:ch3-calibration}），
同时产出自适应保护超参用于应对推理时偶发的分布外激活值。
搜索空间由两个正交维度展开——量化格式（对称 per-group 与非对称
per-channel + per-token）与校准目标（KL 行为引导），正交性的详细推导见
§\ref{para:ch3-orthogonality}。校准产物以 JSON 形式输出，仅包含离线求得的
per-layer per-group Scale 常量、保护超参与量化模式元数据（推理时 adaptive 保护
触发的逐 token Scale 覆盖作为 cache 运行态不落盘）；该 JSON 构成推理路径的唯一
离线输入，框架的总体架构图由第~\ref{sec:ch1-structure}~节的
图~\ref{fig:framework-overview} 承担。

在线推理阶段以极低的额外开销执行量化推理。启动时一次性加载校准 JSON 到 GPU
显存；Prefill 与 Decode 阶段按 \code{kv\_mode} 路由到对应的 KV Cache 类型与
解码核——FP16 全精度基线、INT8 对称（配合自适应保护）、INT4 对称（bit-packing
将存储从 $d_k$ 压缩至 $d_k/2$），以及 KIVI-style 的 per-channel K + per-token V
非对称格式。Triton 融合核函数直接从 INT8/INT4 缓存中读取并反量化 Key/Value，
在片上完成注意力计算，避免将中间张量回写到全局显存。整条在线路径由校准产物
完全决定，不引入逐步统计更新，推理结果精确复现。

表~\ref{tab:ch3-kv-modes} 汇总本框架支持的 9 种量化模式及其核心特征。其中
\code{int8\_ours} 与 \code{int4\_ours} 是本文的行为引导量化方法，使用离线
校准产生的静态 Scale；\code{int8\_ours} 进一步配合 Triton 融合核函数以获得

对比基线；\code{int4\_ours\_asym}（INT4-RoleAlign）在同一非对称格式下引入
行为引导校准，构成本文在低比特下的核心实例化。

\section{行为引导校准方法}
\label{sec:ch3-calibration}

\subsection{KL 散度目标函数}

本文选择 Kullback--Leibler（KL）散度衡量量化前后注意力行为的一致性。
对于给定的 Query 向量 $\bq$ 和上下文内的 Key 矩阵 $\bK$（列维度为头维度 $d_k$），
FP16 精度下的注意力权重分布定义为：
\begin{equation}
  \bm{p}_{\text{ref}} = \softmax\!\left(\frac{\bq \bK^T}{\sqrt{d_k}}\right)
  \label{eq:ch3-pref}
\end{equation}
设 $\hat{\bK}$ 为经过量化——反量化（quantize-dequantize）后的 Key 矩阵，
量化后的注意力权重分布为：
\begin{equation}
  \bm{p}_{\text{quant}} = \softmax\!\left(\frac{\bq \hat{\bK}^T}{\sqrt{d_k}}\right)
  \label{eq:ch3-pquant}
\end{equation}
行为引导校准的目标是最小化两者之间的 KL 散度：
\begin{equation}
  D_{\KL}\!\left(\bm{p}_{\text{ref}} \,\|\, \bm{p}_{\text{quant}}\right)
  = \sum_{i=1}^{n} p_{\text{ref},i} \cdot \ln\frac{p_{\text{ref},i}}{p_{\text{quant},i}}
  \label{eq:ch3-kl}
\end{equation}

图~\ref{fig:calib-pipeline} 把这一目标函数在离线校准阶段可视化：
WikiText-2 校准数据 $\to$ FP16 前向提取 $(Q, K, V)$ $\to$ 共享 KL 目标 $\to$ 分别在 INT8 canonical 与 INT4-RoleAlign 两条路径上做参数搜索 $\to$ 输出可审计的校准产物 JSON。
两条路径的唯一差异是搜索空间（对称 per-group vs 非对称 per-channel/per-token），
目标函数与产物接口共享，$\tau^{-1}$ 温度校正路径已降级为附录诊断说明（见附录~\ref{sec:app-invtau-diagnostic}）。

\input{figures/fig3_calib_pipeline}

选择 KL 散度而非均方误差（MSE）作为目标函数基于三点考量。
\textbf{第一，分布层面的度量。}
注意力权重经过 softmax 归一化后形成概率分布，
KL 散度直接衡量两个分布之间的差异，
相比 MSE 的逐元素数值误差度量能更准确地捕捉分布形态
（如峰值位置、尾部衰减）的变化。
\textbf{第二，对尾部分布的敏感性。}
当 $p_{\text{ref},i}$ 较大而 $p_{\text{quant},i}$ 偏小时，
KL 散度中的 $\ln(p_{\text{ref},i}/p_{\text{quant},i})$ 项会产生较大惩罚，
对注意力集中位置的偏移施加更强约束，
这对保持模型的长距离依赖建模能力尤为重要。
\textbf{第三，softmax 非线性下 MSE 的局限性。}
设 Key 矩阵的量化误差为 $\bm{\delta} = \hat{\bK} - \bK$，
量化后的注意力 logits 满足
\begin{equation*}
  \bm{l}_{\text{quant}} = \bm{l}_{\text{ref}} + \frac{\bq\,\bm{\delta}^T}{\sqrt{d_k}},
\end{equation*}
经 softmax 映射后注意力权重的重新分配与 $\bm{\delta}$ 方向相关而非仅与 $\|\bm{\delta}\|_2^2$ 相关，
因此 MSE 较小的量化方案未必对应注意力行为的保真。
KL 散度直接在 softmax 输出空间度量，天然规避这一数值精度与行为保真度不一致的问题。

本文采用 forward KL（$D_{\KL}(p_{\text{ref}} \| p_{\text{quant}})$）
而非 reverse KL 或 Jensen-Shannon 散度：
其 \emph{mass-covering}（zero-avoiding）特性在参考分布权重较大而量化分布权重过低时产生强惩罚——
当 $p_{\text{ref},i}$ 显著而 $p_{\text{quant},i}\to 0$ 时，被乘项 $\ln(p_{\text{ref},i}/p_{\text{quant},i})\to\infty$，
从而迫使量化分布对参考高概率位置给予足够权重，
恰好匹配 KV Cache 量化的核心关切——防止量化导致模型遗漏关键 token 的注意力。

在实际计算中，为防止 $\ln(0)$ 导致的数值不稳定，
对概率值施加小量截断 $\epsilon = 10^{-6}$：
\begin{equation}
  D_{\KL}\!\left(\bm{p}_{\text{ref}} \,\|\, \bm{p}_{\text{quant}}\right)
  \approx \sum_{i=1}^{n}
    \tilde{p}_{\text{ref},i} \cdot
    \left(\ln\tilde{p}_{\text{ref},i} - \ln\tilde{p}_{\text{quant},i}\right)
  \label{eq:ch3-kl-approx}
\end{equation}
其中 $\tilde{p} = \max(p, \epsilon)$。
由于 softmax 输出的主要概率质量远大于 $\epsilon$
（对于 32K 序列，均匀分布下的单 token 概率约 $3 \times 10^{-5}$，
远高于 $\epsilon = 10^{-6}$），
截断仅在极端尾部生效。
理论最坏情况下（完全均匀分布 + 全部尾部被截断），
截断误差上界为 $\epsilon\cdot n\cdot|\ln\epsilon|\approx 0.45$（$n=32768$，$\epsilon=10^{-6}$），
但实际 softmax 输出的尾部概率质量远低于 $\epsilon$ 阈值——
我们在校准集上经验测量此截断对 KL 值的相对影响 $<$0.1\%，
在实际中可以忽略不计。

\paragraph{校准目标的规模依赖性}

上述 KL 目标的必要性并非在所有设置下均等。
后续实验（第四章）将表明：
在 INT8 精度下，KL 与 MSE 对参数的选择完全趋同，
数值代理足以替代行为度量；
而在 INT4 精度下，两种目标是否趋同取决于模型规模——
较小模型（$H_{kv}=2$）上二者选出不同参数，
较大模型（$H_{kv}=4$）上再次趋同。
这一位宽与模型规模的交互效应为第四章的跨模型受控实验提供了行为引导校准目标的实证起点。

\subsection{Scale 搜索策略}
\label{subsec:ch3-two-stage}

INT8 canonical 路径与 INT4 对称路径共享同一搜索空间：
Key 与 Value 各自独立的 clip percentile $p_c \in \mathcal{P}_c$，
以及共享的 group size $g \in \mathcal{G}$
（具体候选值集合见附录表~\ref{tab:app-search-space}）。
对于每组候选参数，先从校准样本的 Key/Value 激活值中计算逐组绝对值最大值（absmax）
并取 $p_c$ 百分位数作为裁剪阈值，
再由该阈值导出静态 Scale
\begin{equation*}
  s = \text{absmax}_{p_c} \,\big/\, q_{\max},
  \qquad q_{\max} = 127\ (\text{INT8})\ /\ 7\ (\text{INT4}).
\end{equation*}
对每个校准样本执行量化--反量化后，
计算量化后注意力分布 $\bm{p}_{\text{quant}}$ 与参考分布 $\bm{p}_{\text{ref}}$ 之间的 KL 散度，
并汇总所有层、所有头、所有样本的 KL 散度统计量（均值、P95、最大值）以及裁剪率。

候选参数的选择采用稳健优先策略：
优先在裁剪率满足约束
（$\max\{\text{Key 裁剪率},\,\text{Value 裁剪率}\} \leq 1\%$）的可行集中，
按 P95 KL 散度升序排列选择最优参数；
若所有候选均不满足裁剪率约束，则退而按裁剪率总和升序排列。
Key 与 Value 各自独立搜索最优的 clip percentile，
group size 则对 Key 与 Value 共享以保证缓存布局一致。
以上策略构成本文主线的完整校准过程：
$p_c$ 与 $g$ 经一次搜索后产生静态 Scale，推理阶段直接读取。

\paragraph{温度校正的历史定位。}
在早期探索阶段，框架曾考察过一个额外的逐头温度校正因子
$\tau^{-1}_{l, h}$ 作为分布尾部重加权的可选组件；
在跨规模评测中该组件的收益呈现显著的格式 $\times$ 模型族异质性，
未能形成跨模型稳定的方法贡献。
本文因此将 $\tau^{-1}$ 作为诊断观察保留于
附录~\ref{sec:app-invtau-diagnostic}（含完整的候选空间、搜索目标与跨规模实证），
主线在校准与推理阶段均以上述单一 Scale 搜索为准。

\section{行为引导校准的 INT8 和 INT4 实现}
\label{sec:ch3-static-scale}

\subsection{静态 Scale 的设计}

本框架采用离线校准得到的静态 Scale 而非每 token 步动态计算。
静态 Scale 将运行时 absmax 扫描移至离线阶段（推理时仅需一次内存读取），
保证不同 token 步使用相同的量化网格避免早期/晚期 token 精度不一致，
并将推理结果完全由模型权重与 JSON 校准产物决定以便精确复现。

静态 Scale 的计算遵循对称量化（symmetric quantization）公式。
对于第 $l$ 层 Key 或 Value 在校准集 $\mathcal{D}$ 上收集的激活张量，
按 group size $g$ 将通道维度 $d_k$ 划分为 $G = d_k / g$ 个组，
逐组汇总 $\mathcal{D}$ 中所有 batch 与 token 的绝对值后取 $p_c$ 百分位数作为 clip 阈值：
\begin{equation}
  s^{(l)}_j = \frac{\text{percentile}_{\mathcal{D}}\!\left(|x^{(l)}_{:,\,:,\,:,\,jg:(j+1)g}|,\; p_c\right)}{q_{\max}},
  \qquad j = 0, \dots, G{-}1
  \label{eq:ch3-static-scale}
\end{equation}
其中 $p_c$ 为 clip percentile（典型值为 99.5），
$q_{\max} = 127$（INT8）或 $q_{\max} = 7$（INT4）。
$s^{(l)}_j$ 是 per-layer per-group 常量，与 batch/位置无关，离线计算一次后写入 JSON。

量化操作为：
\begin{equation}
  \hat{x} = \text{clamp}\!\left(\text{round}\!\left(\frac{x}{s}\right),\; -q_{\max},\; q_{\max}\right) \cdot s
  \label{eq:ch3-quant-deq}
\end{equation}

Scale 张量始终以 float32 精度存储，不随输入张量的 dtype 变化，
以避免 float16 精度下极小 Scale 值退化为零导致的数值不稳定。

对称 per-group 量化路径的完整数据结构与校准搜索空间
（位宽、量化范围、Scale 精度/类型/形状、量化轴、候选 clip percentile 与 group size 集合）
汇总于附录表~\ref{tab:app-search-space}。

\paragraph{量化时间语义}
\label{para:ch3-quant-timing}

在推理过程中，量化操作发生在 KV Cache 的 \code{append} 步骤：
每当新的 Key/Value 张量被追加至缓存时，以校准产物中的 $s^{(l)}_j$ 即时量化
（自适应保护触发时由动态计算的 $s_{\text{final}}$ 覆盖，详见下一小节），
将浮点张量转换为 INT8/INT4 格式后连同该 token 使用的 Scale 快照一起写入缓存。
一旦写入，已存储的量化值与 Scale 快照\textbf{不再更新}。
反量化时使用各位置自身的 Scale 快照独立还原：
\begin{equation}
  \hat{x}_{b,h,t,j} = x^q_{b,h,t,j} \cdot s^{\text{cache}}_{b,h,t,j}
  \quad \forall\, t = 1, \dots, T
  \label{eq:ch3-per-position-deq}
\end{equation}
其中 $s^{\text{cache}}_{b,h,t,j}$ 为第 $t$ 个 token 在 append 时记录的 per-group Scale
快照：纯静态路径下 $s^{\text{cache}}_{b,h,t,j} = s^{(l)}_j$；
自适应保护触发时等于 $s_{\text{final}}$。
这一``写入即冻结''语义保证缓存的流式积累不引入跨 token 的量化参数耦合。

然而，静态 Scale 的有效性前提是校准数据能够充分代表推理时的输入分布。
当实际推理中出现校准未覆盖的激活值范围时，
静态 Scale 可能导致过度裁剪（clipping），引起显著的精度退化。
下文的自适应保护机制正是为解决这一边界情况而设计。

\subsection{自适应保护机制}

静态 Scale 的一个潜在风险是：当推理时的输入分布超出校准数据的覆盖范围时，
某些激活值可能超出静态 Scale 对应的量化区间，
导致大量值被裁剪至 $\pm q_{\max}$，引起显著的精度退化。

为应对这一风险，本框架设计了自适应保护机制（adaptive static scale protection）。
其核心思想是：在每次量化时，
比较静态 Scale 与当前输入的动态 Scale，取两者的较大值作为最终 Scale：
\begin{equation}
  s_{\text{final}} = \max\!\left(s_{\text{static}} \cdot m,\; s_{\text{dynamic}}\right)
  \label{eq:ch3-adaptive}
\end{equation}
其中 $s_{\text{static}}$ 为离线校准得到的静态 Scale，
$s_{\text{dynamic}} = |x|_{\max} / q_{\max}$ 为当前输入的动态 Scale，
$m \geq 1$ 为保护裕度（protection margin），默认值为 $1.0$。

其设计逻辑为：当输入分布在校准覆盖范围内时
$s_{\text{dynamic}} \leq s_{\text{static}}$，最终使用静态 Scale 保持校准一致性；
当出现异常大的激活值时 $s_{\text{dynamic}} > s_{\text{static}}$，
自动切换为动态 Scale 避免大量裁剪。
保护裕度 $m$ 可根据场景调节（$m > 1$ 为更宽松保护，$m = 1$ 为最严格模式）。

\paragraph{自适应保护对历史缓存的影响}
当自适应机制在某个 token 步触发
（$s_{\text{dynamic}} > s_{\text{static}}$）时，
\textbf{仅当前 token 的 Scale 受影响}，
已存储在缓存中的历史 token 保持其原始量化值和 Scale 不变。
这是因为量化操作遵循``写入即冻结''语义
（第~\ref{para:ch3-quant-timing}~段）：
每个位置的 Scale 在 \code{append} 时确定并永久存储，
后续 token 的自适应 Scale 调整不会回溯修改历史缓存。
因此，自适应保护是一种\textbf{前向保护}机制，
其局部性保证缓存的流式积累不引入跨 token 的 Scale 耦合。

\paragraph{向非对称格式的扩展}
上述 Scale 搜索框架在设计上是格式无关的：
当量化格式从对称 per-group 升级为非对称 per-channel/per-token 时
（第~\ref{sec:ch3-rolealign}~节），
同一 KL 散度目标函数可直接用于搜索最优的 K percentile 和 V percentile 参数，
无需修改搜索框架的核心逻辑。


INT4-RoleAlign（INT4 Role-Aware 量化）是在 KIVI~\cite{liu2024kivi} 提出的
per-channel Key + per-token Value 非对称格式上，
以 attention-KL 为目标的行为引导实例化。
该方案的设计假设——Key 量化对长上下文检索质量的敏感性高于 Value、
且退化幅度与 GQA 头数 $H_{kv}$ 呈结构性相关——
源自 KIVI~\cite{liu2024kivi} 对 Key 异常值分布的观察与本文早期消融，
并由第~\ref{sec:exp-kv-sensitivity}~节的跨模型 K/V 消融系统验证。
以下首先阐明量化格式与校准目标的正交性，再详述具体设计。

\paragraph{量化格式与校准目标的正交性}
\label{para:ch3-orthogonality}
理解 INT4-RoleAlign 的设计逻辑需要区分两个正交的设计维度：
\textbf{量化格式}（对称 per-group vs 非对称 per-channel/per-token）
与\textbf{校准目标}（MSE/percentile vs KL 行为引导）。
前述各节的 INT8 路径在校准目标维度实现了核心改进，
以 KL 散度替代 MSE/percentile，
结合自适应保护机制保障长上下文鲁棒性。
当校准目标维度的改进已到位后，若低比特仍出现系统性失效，
格式维度的升级成为自然的设计方向。
INT4-RoleAlign 正是这一正交扩展的实证：
在保持 KL 校准目标不变的前提下，将量化格式从对称 per-group
升级为 per-channel K + per-token V 非对称，
实现校准目标与量化格式两个维度的协同改进。

前述各节证明行为引导校准在 INT8 对称格式下的有效性。
当位宽降至 4~bit 时，量化级别从对称 INT8 的 255 级（值域 $[-127, 127]$）
骤降至对称 INT4 的 15 级（值域 $[-7, 7]$），
Key 张量的通道间数值异质性可能超出对称格式的表示能力。
本节介绍的非对称 INT4 路径采用完整 $[-8, 7]$ 共 16 个离散值，
通过零点偏移获得一个额外的表示量级。
第~\ref{sec:exp-int4-kivi}~节的诊断实验提供独立证据支持这一设计判断：
受控 K/V 消融在三个模型族上一致确认 Key 量化主导质量损失，
退化程度与 GQA 头数 $H_{kv}$ 呈结构性相关。
Key 决定了注意力的``看哪里''（attention logits 的排序结构），
其通道间数值异质性在低比特下无法被对称格式有效捕捉。
上述诊断结果提示采用角色差异化量化轴是一个合理的设计方向：
对 Key 和 Value 采用差异化的量化轴，
以适配两者不同的数值分布特性与功能角色。

\subsection{从对称到非对称的格式升级}
\label{sec:ch3-rolealign}

基于上述诊断发现，本节提出 \textbf{INT4-RoleAlign}
（Role-Aware Asymmetric Quantization with Behavior-Guided Calibration），
后文统一简称为 INT4-RoleAlign（不省略 INT4- 前缀以明确比特宽度约束）。
该方法吸收 KIVI~\cite{liu2024kivi} 的非对称量化轴设计思想，
将 Key 和 Value 的量化策略从统一的 per-group 对称
升级为 Role-Aware 的非对称格式：

\begin{itemize}[leftmargin=2em, itemsep=2pt]
  \item \textbf{Key：per-channel 非对称量化}。
    沿序列维度（across tokens）为每个通道独立计算 Scale 与零点偏移，
    捕捉 Key 矩阵中通道间的数值异质性。
    形式化地，对于第 $l$ 层的 Key 矩阵 $\bK \in \mathbb{R}^{S \times d_k}$，
    记 $p_K$ 为 Key 的双端截断 percentile（由离线行为引导校准搜索确定），
    per-channel 量化参数为：
    \begin{equation}
      \begin{aligned}
        m^K_{l,j} &= \mathrm{percentile}\!\left(K_{:,j},\; 100 - p_K\right), \;\;
        M^K_{l,j}  = \mathrm{percentile}\!\left(K_{:,j},\; p_K\right), \\
        s^K_{l,j} &= \frac{M^K_{l,j} - m^K_{l,j}}{q_{\max} - q_{\min}}, \;\;
        z^K_{l,j}  = m^K_{l,j} - q_{\min}\cdot s^K_{l,j},
      \end{aligned}
      \;\; j = 1, \dots, d_k
      \label{eq:ch3-perchannel-k}
    \end{equation}
    其中 $q_{\min}, q_{\max}$ 为量化比特宽度下的最小/最大整数值
    （INT4：$q_{\min}=-8, q_{\max}=7$；INT8：$q_{\min}=-128, q_{\max}=127$）。
    量化与反量化规则为
    $q = \mathrm{clamp}(\mathrm{round}((x - z)/s),\, q_{\min},\, q_{\max})$，
    $\hat{x} = q\cdot s + z$。
  \item \textbf{Value：per-token 非对称量化}。
    沿特征维度（across channels）为每个 token 独立计算 Scale 与零点，
    适配 Value 矩阵中 token 间的动态范围差异：
    \begin{equation}
      \begin{aligned}
        m^V_{l,t} &= \mathrm{percentile}\!\left(V_{t,:},\; 100 - p_V\right), \;\;
        M^V_{l,t}  = \mathrm{percentile}\!\left(V_{t,:},\; p_V\right), \\
        s^V_{l,t} &= \frac{M^V_{l,t} - m^V_{l,t}}{q_{\max} - q_{\min}}, \;\;
        z^V_{l,t}  = m^V_{l,t} - q_{\min}\cdot s^V_{l,t},
      \end{aligned}
      \;\; t = 1, \dots, S
      \label{eq:ch3-pertoken-v}
    \end{equation}
    其中 $p_V$ 为 Value percentile 参数。
    特别地，$(p_K, p_V) = (100, 100)$ 时退化为标准 min-max 非对称量化
    （即 KIVI-style absmin/absmax 配置）；$p_K, p_V < 100$ 时以对称
    双端截断缓解极端 outlier 对 scale 的放大。
\end{itemize}

\subsection{Behavior-Guided Percentile 校准}
\label{subsec:ch3-ba-percentile}

INT4-RoleAlign 的核心设计区别在于参数确定范式。
KIVI-style 完全依赖运行时统计量（absmax/min）无需离线校准，
INT4-RoleAlign 通过行为引导框架的离线搜索确定最优的 $p_K$ 和 $p_V$ percentile 参数。

\paragraph{参数化动机}
采用 $(p_K, p_V)$ 离散 percentile 参数化而非直接在 per-channel scale 向量上连续优化，
有两个理由：
percentile 参数化将搜索空间约束为低维离散网格，
可穷举枚举且在 outlier 干扰下保持鲁棒；
离散网格与第~\ref{sec:ch3-static-scale}~节对称路径的 $(p_c, g)$ 搜索框架保持接口一致。

\paragraph{搜索空间}
$\mathcal{P}_K = \mathcal{P}_V = \{95.0,\, 97.0,\, 99.0,\, 99.5,\, 99.9,\, 100.0\}$，
$|\mathcal{P}_K| = |\mathcal{P}_V| = 6$，搜索采用完整笛卡尔积枚举 $|\mathcal{P}_K \times \mathcal{P}_V| = 36$ 个配置。
特别值得指出：$(p_K, p_V) = (100.0,\, 100.0)$ 恰好对应 KIVI-style 的 absmax/min 配置，
因此 RoleAlign 与 KIVI-style 的比较可理解为同一非对称格式空间内的参数化比较，
而非跨格式比较。

搜索目标仍为 KL 散度（公式~\eqref{eq:ch3-kl}），
但搜索空间从对称格式的 $(p_c, g)$ 切换为非对称格式的 $(p_K, p_V)$：
\begin{equation}
  (p_K^*, p_V^*) = \argmin_{(p_K, p_V) \in \mathcal{P}_K \times \mathcal{P}_V}
  \frac{1}{|\mathcal{D}|} \sum_{(\bq, \bK, \bV) \in \mathcal{D}}
  D_{\KL}\!\left(\bm{p}_{\text{ref}} \,\|\, \bm{p}_{\text{asym}}\right)
  \label{eq:ch3-ba-percentile}
\end{equation}
其中 $\bm{p}_{\text{asym}}$ 为使用 per-channel K Scale 和 per-token V Scale
量化后的注意力分布。

表~\ref{tab:ch3-rolealign-vs-kivi} 从四个维度对比 INT4-RoleAlign 与 KIVI-style 的设计差异。

两者共享相同的量化格式但在参数确定范式上存在本质差异：
KIVI-style 的无校准设计使其部署简便，
而 INT4-RoleAlign 通过离线 KL 搜索获取更优的 percentile 参数，
以行为引导校准实现更稳定的检索能力保持
（PPL 退化因模型规模而异，详见第~\ref{subsec:exp-int4-cross-model}~节）。
如第~\ref{subsec:ch3-ba-percentile}~节所述，
KIVI-style 的 absmax/min 配置对应 RoleAlign 搜索空间的端点 $(p_K, p_V) = (100, 100)$，
因此二者的比较建立在``同一搜索空间的两个点''而非``两种不同方法''的基础上。


本节给出行为引导校准方案对应的 Triton 融合核函数实现，
作为 INT8 canonical path 从校准产物到可部署推理路径的系统桥梁。
在自回归解码阶段，每步仅生成一个 token（$q_{\text{len}} = 1$），
但需要与整个上下文的 Key/Value 缓存进行注意力计算。
若采用``先反量化全部缓存至 FP16，再执行标准注意力''的朴素方案，
将产生大量不必要的全局显存读写，抵消量化带来的存储收益。

本框架采用 Triton~\cite{tillet2019triton} 编程语言实现了融合量化解码注意力核函数，
将 INT8/INT4 反量化与注意力计算在单个 GPU kernel 内完成。
该核函数借鉴了 FlashAttention~\cite{dao2022flashattention} 的 online softmax 技术，
避免材化（materialize）完整的注意力得分矩阵。

\subsection{INT8 核函数设计}
\label{subsec:ch3-triton-int8}
\label{sec:ch3-triton}

INT8 融合核函数沿用 FlashAttention~\cite{dao2022flashattention} 的 online softmax 思想
将反量化与注意力计算融合在单个 GPU kernel 内，
在片上按 FP32 精度反量化 Key/Value（避免 FP16 乘法精度损失）
并直接累加注意力输出，
避免材化完整的注意力得分矩阵与 FP16 反量化中间张量。
整体复杂度 $O(B \cdot H_q \cdot S \cdot d_k)$ 与朴素实现相同，
但访存较未融合的 dequant+SDPA 路径降低 $O(S \cdot d_k)$ 字节的中间张量物化。


\subsection{INT4 核函数设计}
\label{subsec:ch3-triton-int4}

INT4 量化采用 bit-packing 存储：两个 INT4 值（$[-8, 7]$）打包为一个 INT8 字节，
存储维度从 $d_k$ 压缩为 $d_k / 2$。核函数在片上通过算术右移完成符号扩展解包，
将 packed 张量还原为 signed INT8 后复用 INT8 核函数的反量化与注意力计算流程。

\subsection{INT4 非对称融合核函数}
\label{subsec:ch3-triton-int4-asym}

上述 INT4 核函数针对的是对称量化（per-group Scale，无 zero-point）。
INT4-RoleAlign 和 KIVI-style 采用非对称量化，具有不同的 Scale 维度：
Key 使用 per-channel Scale（对所有 token 共享），
Value 使用 per-token Scale（对所有 channel 共享），
且两者均包含 zero-point 偏移。
这一格式差异使得上述``物化→调用 INT8 核函数''的方案不再适用，
需要设计专用的非对称融合核函数。

本文采用\textbf{分通道（split-channel）}设计，
在 kernel 内部直接对 packed INT4 字节完成 nibble 解包与非对称反量化，
避免任何中间张量的全局显存物化。
K 的 per-channel Scale/ZP 在循环外一次性加载，
V 的 per-token Scale/ZP 按 block 加载，均在 FP32 精度下完成累加，
正确性由端到端 Needle 评测验证（详见第~\ref{subsec:exp-int4-cross-model}~节）。
该设计基于 CUDA Core，
与 BitDecoding~\cite{du2026bitdecoding} 的 NVFP4 + Tensor Core 设计互为补充。

\textbf{与 KIVI Residual Buffer 的实现差异}：
KIVI~\cite{liu2024kivi} 原论文通过 FP16 Residual Buffer（$R=128$）
为最近 $R$ 个 token 保留全精度存储，
该组件在 \emph{streaming decode}（chunk\_size$=1$）设置下是关键缓冲，
能缓解 per-channel K 在极小解码粒度下 scale range 失配的问题。
本文的 KIVI-style 基线复现了 per-channel K + per-token V 的非对称\emph{格式}，
但未实现 Residual Buffer 组件，
且本文主线评测协议为 chunk\_size$=128$，
在该设置下 Residual Buffer 的作用已由 chunk 边界 FP16 累积计算近似吸收。
本文 RoleAlign 的 Triton 融合核针对 chunk\_size$=128$ 的非 streaming 评测设计，
统一以 INT4 格式处理整个 KV Cache。
第~\ref{subsec:exp-int4-honest}~节 cs$=1$ 的失效实验划定了
per-channel K 在 streaming decode 下本文 KIVI-style 基线与 RoleAlign 的共同边界；
KIVI 原论文的 Residual Buffer 在该场景下有效，
完整 streaming kernel 的适配（Residual Buffer + flash-decoding 分块）
留待未来工作。

\paragraph{RoleAlign 与融合核的路径分工}
\label{para:ch3-in-kernel-pct}

RoleAlign 的默认推理路径为 \code{torch\_ref} 参考实现：
离线校准产物中的 $(p_K, p_V)$ 直接导出 per-channel K Scale 与 per-token V Scale，
在 PyTorch 层执行非对称量化与反量化。
上述 INT4 非对称融合核作为该格式的工程可行性验证：
kernel 内部对每 token 在线估计 percentile 裁剪边界以计算 Value 的 per-token Scale/zero-point，
无需外部预计算统计量。
由于 decode 阶段 $M=1$ 不满足 Tensor Core 使用条件
（经验交叉点详见第~\ref{subsec:ch3-phase-boundary}~节），
端到端评测默认使用 torch\_ref 路径，融合核作为加速潜力的验证留存。

\subsection{GQA 支持机制}
\label{subsec:ch3-gqa}

分组查询注意力（GQA）~\cite{ainslie2023gqa} 中，
$H_q$ 个 Query 头被均匀分为 $H_{kv}$ 组，每组共享同一对 KV 头。
定义重复因子 $N_{\text{rep}} = H_q / H_{kv}$，
核函数通过以下映射实现 GQA 支持：
\begin{equation}
  h_{kv} = \lfloor h_q / N_{\text{rep}} \rfloor
  \label{eq:ch3-gqa-mapping}
\end{equation}
其中 $h_q$ 为 Query 头索引。
在核函数内部，Grid 按 $(B, H_q)$ 启动，
每个程序实例根据 $h_{kv}$ 索引读取对应的 KV 缓存，
无需在显存中复制 KV 头，从而节省了 $N_{\text{rep}}$ 倍的显存带宽。

\subsection{\texorpdfstring{经验交叉点：融合核延迟收益的 $(H_{kv}, \text{seq\_len})$ 空间特征}{经验交叉点：融合核延迟收益的 (Hkv, seq\_len) 空间特征}}
\label{subsec:ch3-phase-boundary}

融合量化解码核函数的效率增益并非在所有配置下均存在。
当 KV 头数 $H_{kv}$ 较小时，Triton Grid 中的 program 实例数量有限，
GPU SM（Streaming Multiprocessor）利用率较低；
当序列长度较短时，单次 decode 的访存量不足以使带宽节省超过
额外的 kernel 启动与反量化计算开销。
因此，融合核相对于本文的 \code{torch\_ref}
（手写 einsum 参考实现，而非 PyTorch 原生 SDPA）的
相对延迟优势依赖于 $(H_{kv}, \text{seq\_len})$ 的组合。

本文将这一二维空间中融合核开始产生显著延迟优势的边界
称为\textbf{经验交叉点}（empirical crossover）。
定义融合核的 TPOT（Time Per Output Token）为 $T_{\text{fused}}$，
naive 路径 TPOT 为 $T_{\text{naive}}$，则\emph{经验交叉点}定义为满足如下条件的参数组合：
\begin{equation}
  \Delta T(H_{kv}, S) = T_{\text{naive}}(H_{kv}, S) - T_{\text{fused}}(H_{kv}, S) \approx 0
  \label{eq:ch3-phase-boundary}
\end{equation}
公式 (\ref{eq:ch3-phase-boundary}) 是交叉点的\emph{定义式}而非预测式，
本文并未给出基于 roofline / 带宽 / 启动开销的闭式预测模型；
交叉点的具体位置由第四章的实测数据经验刻画。
在 $\Delta T > 0$ 的区域（长序列、高 $H_{kv}$），融合核提供正向加速；
在 $\Delta T \leq 0$ 的区域（短序列、低 $H_{kv}$），naive 路径更优。

影响经验交叉点位置的两个主要机制为：
\textbf{SM 利用率}方面，Triton 核函数的 Grid 按 $(B, H_q)$ 启动，
当 $H_{kv}$ 较小（如 $H_{kv}=2$，$H_q=12$）时实例总数可能不足以填满所有 SM
而导致硬件利用率受限；相反 $H_{kv}=8$（$H_q=32$，如 LLaMA-3.1-8B）时 Grid 更密，SM 占用率更高。
本文当前实现聚焦于 $H_{kv}\geq 4$ 的主流 GQA 配置；
对于 $H_{kv}=2$ 场景，Split-K / Flash-Decoding~\cite{dao2023flashdecoding}
沿序列维度切分 program 实例可提升 SM 占用率，
作为 kernel 层的扩展方向留待未来工作。
\textbf{带宽节省与反量化开销的权衡}方面，融合核通过在 SRAM 中完成反量化
避免了全局显存的中间张量物化，带宽节省量与序列长度成正比（$O(S \cdot d_k)$），
但反量化本身引入额外的 ALU 指令开销，当序列较短时带宽节省不足以抵消该开销。

\textbf{加速口径与 Tensor Core 的关系}：
本文融合核的 40\% 加速口径相对于``未融合 INT4 参考实现''成立；
与 tensor-core-enabled FP16 FlashAttention-2 基线的对比属于另一维度的绝对加速测度。
由于 decode 阶段每个 program 仅处理单个 query token（$M=1$），
不满足 Triton \code{tl.dot} 的 $M\geq 16$ Tensor Core（HMMA 指令）使用条件，
本文 INT8/INT4 融合 decode 核通过 \code{tl.sum(a*b)} 在 CUDA Core 上实现点积。
未来 tensor-core-enabled decode kernel（如 BitDecoding 的 NVFP4 路径~\cite{du2026bitdecoding}）
为绝对加速维度的扩展方向。
经验交叉点的具体位置依赖于硬件参数（SM 数量、显存带宽、L2 缓存大小）
和 kernel 实现细节，第四章将通过跨模型的实测数据给出经验分析。

\section{行为引导的层间预算分配器}
\label{sec:ch3-allocator}

校准层为每个模型产出一份逐层敏感度画像——
记为 $\{D_{\KL}^{(l)}\}_{l=1}^L$，
其中 $D_{\KL}^{(l)}$ 是第 $l$ 层在校准数据上的 attention-KL 散度
（对多头做 max 或 mean 聚合，详见第~\ref{subsec:ch3-alloc-sensitivity}~节）。
分配层以该 profile 作为跨层预算分配的输入，
将同一行为引导原则从"怎么量化"推进到"哪里该多保护"，
实现框架在两个抽象层次上的贯通。

\subsection{Layer-wise Budget Allocation}
\label{subsec:ch3-alloc-layerwise}

设总预算为固定平均 bit $\bar b$（对应相同的 KV memory 约束），
每层独立决定位宽 $b_l \in \{4, 8, 16\}$ 满足平均位宽约束
\begin{equation*}
  \frac{1}{L}\sum_{l=1}^{L} b_l \approx \bar b.
\end{equation*}
本文实现的 top-$k$ 选择器保护敏感度最高的 $k$ 层使用较高位宽 $b_{\text{hi}}$，
其余层使用较低位宽 $b_{\text{lo}}$，形式化为：
\begin{equation}
  \mathcal{P}_{\text{top-}k} = \{\, l : D_{\KL}^{(l)} \in \text{top-}k(\{D_{\KL}^{(l')}\}_{l'=1}^L)\,\}
  \label{eq:ch3-alloc-topk}
\end{equation}
对 $l \in \mathcal{P}_{\text{top-}k}$ 使用 $b_{\text{hi}}$，其余层使用 $b_{\text{lo}}$，
整体 bit 预算与 $\bar b$ 对齐（可由 $b_{\text{hi}}, b_{\text{lo}}, k$ 三者联合决定）。

对照组是启发式选择器：
用等距位置（如 $\{0, L-1\}$、$\{0, L/2, L-1\}$ 或
$\mathrm{linspace}(0, L-1, k)$）
保护固定 $k$ 层，不使用敏感度信息。
启发式选择器在实验章节中作为强基线与行为引导策略并列比较
（详见第~\ref{sec:exp-cross-model}~节的跨模型对比）。

\subsection{Role-Aware 扩展：K/V 非对称预算}
\label{subsec:ch3-alloc-role}

进一步的颗粒度是对 Key 与 Value 分别分配预算。
K 与 V 在误差分解（方程~\eqref{eq:ch3-error-decomp}）中承担不同角色，
其敏感度画像也不必相同。
Role-Aware Allocator 为每层独立决定 $(b_l^K, b_l^V)$ 对，
总预算约束变为
\begin{equation*}
  \frac{1}{2L}\sum_{l=1}^{L} (b_l^K + b_l^V) \approx \bar b.
\end{equation*}
这一扩展在当前实现中可行，
但在本文跨模型实验（第~\ref{sec:exp-cross-model}~节）中
尚未系统性地超越同预算下的对称 top-$k$ 策略，
作为框架下一步研究方向在第~\ref{sec:conclusion-future}~节讨论。

\subsection{Sensitivity 聚合与鲁棒性}
\label{subsec:ch3-alloc-sensitivity}

逐层敏感度的计算涉及两个聚合选择：
\textbf{head 间聚合}（max / mean）与
\textbf{token 间聚合}（数据分布上的平均 KL）。
本文默认采用 max over heads + mean over tokens，
对应"最敏感头主导"的鲁棒设计。
替代聚合方式（如 mean over heads）在跨模型实验中展现出与 max 不同的
适用区间特征；具体 7B 上的佐证案例见
第~\ref{sec:exp-cross-model}~节。


Fixed-$k$ allocator 的 $k$ 值在不同模型上并不稳定
——跨模型实验（第~\ref{sec:exp-cross-model}~节）显示不同模型家族、规模、
任务上 best-$k$ 差异明显。
这使得分配器调参在每个新模型上都需要重新扫描 $k$，
引入显著的实验开销。
AutoK 以校准产物的敏感度画像本身作为
"合适 $k$ 值"的自动建议器，避免人工扫描，
从而把分配器的跨模型可迁移性提升一层。

\subsection{\texorpdfstring{Coverage-Based $k$ 选择}{Coverage-Based k 选择}}
\label{subsec:ch3-autok-cov}
\label{sec:ch3-autok}

给定一个覆盖度阈值 $\rho \in (0, 1)$
（本文默认 $\rho = 0.80$，记为 \texttt{cov80}），
AutoK 选择最小的 $k^*$，使得 top-$k$ 敏感度之和占总敏感度质量的 $\rho$ 以上：
\begin{equation}
  k^* = \min \left\{\, k :
    \frac{\sum_{l \in \text{top-}k(\mathbf{D})} D_{\KL}^{(l)}}
         {\sum_{l=1}^L D_{\KL}^{(l)}} \geq \rho \,\right\}
  \label{eq:ch3-autok-cov}
\end{equation}
其中 $\mathbf{D} = \{D_{\KL}^{(l)}\}_{l=1}^L$ 为 sensitivity 向量。
直觉上，$\rho$ 越大 $k^*$ 越大（保护更多层），
且 $k^*$ 会根据模型 sensitivity 分布形状自适应：
分布越集中的模型 $k^*$ 越小，分布越分散则 $k^*$ 越大。
本文在 \texttt{cov80}、\texttt{cov85}、\texttt{cov90} 三个阈值下分别给出结果。

\subsection{AutoK 的定位}
\label{subsec:ch3-autok-position}

AutoK 作为行为引导框架在预算分配层的自然延伸，
针对 fixed-$k$ allocator 的 $k$ 值在跨模型上不稳定这一问题提供自动建议，
把分配器的跨模型可迁移性提升到无需人工 $k$ 扫描的层次。
实验章节（第~\ref{subsec:exp-per-model-mistral}~节）显示
AutoK 在 Mistral-7B-Instruct-v0.3 上给出框架最清晰的正面案例；
在其它模型上 AutoK 与 fixed-$k$ 呈现混合信号。
AutoK 的价值因此定位于``适用区间存在且 fixed-$k$ 不稳定时的
敏感度引导预算建议''这一具体场景。

\section{复杂度与资源分析}
\label{sec:ch3-complexity}

本节从时间复杂度、显存占用和访存效率三个维度对框架进行定量分析，
为第四章的实验结果提供理论预期。

\subsection{离线校准时间复杂度}

离线校准的计算开销集中于 Scale 搜索。
设校准样本数为 $N$，序列长度 $n$，模型层数 $L$，
Query 头数 $H_q$，头维度 $d_k$。
搜索在 clip percentile 集合 $\mathcal{P}_c$（$|\mathcal{P}_c| = 4$）与
group size 集合 $\mathcal{G}$（$|\mathcal{G}| = 4$）的笛卡尔积上枚举最优 Scale，
时间复杂度为
\begin{equation}
  T_{\text{calib}} = O\!\left(|\mathcal{P}_c| \cdot |\mathcal{G}| \cdot N \cdot L \cdot H_q \cdot n \cdot d_k\right).
  \label{eq:ch3-complexity-stage1}
\end{equation}
在默认配置下，Qwen2.5-1.5B 的完整校准约需 15--20 分钟，
7B 模型约需 40--50 分钟（单张 H20 GPU）。

\subsection{KV Cache 显存精确公式}

对于具有 $L$ 层、$H_{kv}$ 个 KV 头、头维度 $d_k$、
当前序列长度 $S$ 的模型，KV Cache 的显存占用如下。
为支持``写入即冻结''语义下的自适应保护覆盖，缓存为每个 token 各组保存一份
FP32 Scale 快照（静态路径下等于离线常量 $s^{(l)}_j$，adaptive 触发时记录当次
动态值 $s_{\text{final}}$），计入下方 INT8/INT4 公式的 Scale 项。

\paragraph{FP16 模式}
Key 和 Value 各以 FP16 格式存储：
\begin{equation}
  M_{\text{fp16}} = 2 \cdot L \cdot H_{kv} \cdot S \cdot d_k \cdot 2 \;\text{bytes}
  = 4 L H_{kv} S d_k
  \label{eq:ch3-mem-fp16}
\end{equation}

\paragraph{INT8 模式}
Key/Value 以 INT8 存储，每组 $g$ 个元素共享一个 FP32 Scale：
\begin{equation}
  M_{\text{int8}} = 2 L H_{kv} S \left(d_k + \frac{d_k}{g} \cdot 4\right)
  = 2 L H_{kv} S \cdot d_k \!\left(1 + \frac{4}{g}\right)
  \label{eq:ch3-mem-int8}
\end{equation}

\paragraph{INT4 模式}
两个 INT4 值打包为一个 INT8 字节，存储维度减半：
\begin{equation}
  M_{\text{int4}} = 2 L H_{kv} S \left(\frac{d_k}{2} + \frac{d_k}{g} \cdot 4\right)
  = 2 L H_{kv} S \cdot d_k \!\left(\frac{1}{2} + \frac{4}{g}\right)
  \label{eq:ch3-mem-int4}
\end{equation}

在默认 $g = 16$ 下，INT8 的理论显存比值为 $(1 + 4/g)/2 = 62.5\%$（节省 37.5\%），
实验中观测到约 44\% 的节省（第四章表~\ref{tab:kv-modes}），
差异来自 Scale 张量的紧凑存储优化。
INT4 在 $g=16$ 下理论比值为 37.5\%，节省可达 62.5\%，
但伴随第四章所示的质量退化。

\subsection{Triton 核函数访存分析}

decode 阶段的注意力计算属于典型的访存密集型操作：
每个 token 步仅处理一个 Query 向量（$q_{\text{len}} = 1$），但需遍历整个 KV 缓存。
对于 INT8 融合核函数，每个分块（大小 $B_s$）的访存量为
$2 B_s \cdot d_k (1 + 4/g)$ 字节（Key/Value 数据与 Scale），
计算量约 $4 B_s \cdot d_k$ 次浮点运算（点积、加权求和、反量化）。
算术强度为：
\begin{equation}
  \text{AI}_{\text{int8}} = \frac{4 B_s \cdot d_k}{2 B_s \cdot d_k (1 + 4/g)}
  = \frac{2}{1 + 4/g}
  \;\text{FLOP/byte}
  \label{eq:ch3-ai-int8}
\end{equation}
$g=16$ 时 $\text{AI}_{\text{int8}} = 1.6$ FLOP/byte，
为 FP16 朴素实现（1.0 FLOP/byte）的 1.6 倍。
这解释了量化在 decode 阶段实现加速的根本原因：
相同计算量对应更少的数据搬运。

\subsection{校准产物存储开销}

校准产物以 JSON 格式存储每层 Key/Value 的 per-layer per-group 静态 Scale 常量
$\{s^{(l)}_j\}$，共 $2L \cdot d_k / g$ 个 FP32 值，以及 clip percentile 与
group size 两个超参数；推理时 adaptive 保护产生的逐 token Scale 快照作为
cache 运行态不落盘。以 Qwen2.5-7B 为例，JSON 文件约 10--30~KB，
相对于模型权重（$\approx$14~GB FP16）完全可以忽略不计。

\section{本章小结}
\label{sec:ch3-summary}

本章以假设 H 为出发点，
给出了行为引导量化框架的两层实例化。

\textbf{第一层（校准）}：
以量化前后 attention 分布的 Kullback--Leibler 散度为离线搜索目标，
分别在 INT8 对称格式（配合静态 scale 与自适应保护机制，
以及 Triton 融合量化解码核作为系统落地）
与 INT4 非对称格式（KIVI-style per-channel Key + per-token Value，
配合 Behavior-Guided percentile 离线搜索）两个量化位宽上落成。

\textbf{第二层（分配）}：
将校准产物的逐层敏感度画像
作为跨层预算分配的输入，提出行为引导的层间预算分配器；
进一步引入 AutoK 作为敏感度引导的预算建议机制，
以覆盖度阈值 $\rho$ 自适应地为不同模型选择合适的保护层数。
两层共享同一行为度量 $D_{\KL}$
（方程~\eqref{eq:ch3-kl}，方程~\eqref{eq:ch3-autok-cov}），
使行为引导原则在框架内部贯通校准与分配两个抽象层次。

Triton 融合量化解码核作为 INT8 canonical path 的系统落地实例，
承担校准方案到可部署推理路径的桥梁角色。

下一章以溯源完整实验协议在六个主流 LLM 与五个 LongBench 任务上给出实证。
所覆盖的模型为：
\begin{itemize}[leftmargin=2em, itemsep=1pt, topsep=2pt]
  \item Qwen2.5-1.5B / 3B / 7B / 14B；
  \item LLaMA-3.1-8B；
  \item Mistral-7B-Instruct-v0.3。
\end{itemize}
实证重点包括：
\begin{itemize}[leftmargin=2em, itemsep=1pt, topsep=2pt]
  \item INT8 canonical path 的保真度（第~\ref{sec:exp-main}~节）；
  \item INT4 RoleAlign 与 KIVI 的跨模型比较；
  \item 行为引导分配器的跨模型适用区间地图（主结果）；
\end{itemize}
以及 Mistral、3B、14B 的逐模型案例分析。
