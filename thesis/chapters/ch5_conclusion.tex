\section{研究结论}
\label{sec:conclusion-answers}% backward-compat with earlier \ref
% ============================================================

围绕第一章提出的四个研究问题，本文得到的核心结论可以归结为一条主线。KV Cache 量化同时涉及缓存元素的 bit 存储方式，以及量化后的 Key 与 Value 如何进入注意力计算，并进一步影响分布、聚合输出和任务行为。第三章给出这一问题的形式化表达，第四章则从基准保真、低比特恢复、跨模型预算和系统读数四个层面完成验证。

注意力行为可以作为 KV Cache 量化的统一观察对象。第三章的误差传播分析表明，缓存扰动经过 logits、softmax 和 Value 聚合后才真正进入模型行为。第四章的 K/V 角色诊断进一步显示，同等 bit-width 下 Key 与 Value 的扰动后果并不对称，Key 侧低比特噪声更容易触发注意力排序偏移。基准校准、低比特恢复和预算分配因此可以围绕同一组行为量展开。研究问题 1 得到肯定回答，注意力分布、聚合输出和任务行为构成本文需要保持的共同对象。

在 \texttt{INT8} 基准路径中，行为引导校准形成了保守位宽下的保真锚点。第四章显示，在 Qwen2.5-1.5B 的 task-core 协议内，\texttt{INT8-Canonical} 相对 \texttt{FP16} 的三任务平均差异为 mean $\Delta=+0.02$，三个官方 LongBench 子任务的小范围对照也延续同一方向。这一结果回答了研究问题 2。行为引导校准可以在当前基准位宽和评测协议下建立稳定参照，为后续低比特路径和系统实现提供起点。更广真实应用分布仍需要更大范围的官方数据与任务覆盖继续验证。

跨模型预算结果表明，行为敏感度画像可以作为预算分配的组织依据。第四章在同量级 \texttt{INT4} 预算带内读出了清晰的模型条件差异。Qwen2.5-3B 的三任务 core mean 中，BA-k1 为 6.90，中层 Heuristic-k1 为 3.48，说明 early-layer bottleneck 下位置先验可能错过关键层。LLaMA-3.1-8B 表现为中等规模共识区间。Qwen2.5-14B 上 Uniform 的 7.23 与 \texttt{BA-AutoK} 90\% 覆盖率的 7.15 接近，呈现高性能簇共存而无稳定最优。heuristic 在多个区间仍是强基线，说明部分模型的可行层集本来就很窄。研究问题 3 的结论是，敏感度画像能够组织和命名跨模型适用区间，并揭示策略有效性随模型族、规模和任务条件变化的结构。

固定预算选择进一步可以由敏感度画像生成模型级候选。\texttt{AutoK} 将离线行为敏感度排序转写为覆盖率预算建议，在 Mistral-7B 上给出最清晰的单模型支持。\texttt{BA-AutoK} 80\% 覆盖率的 core mean 为 14.76，extend mean 为 15.69，五任务均值为 15.14。Qwen2.5-14B 这类高性能簇接近的设置则说明，\texttt{AutoK} 更适合作为候选区间生成器，与模型画像和预算带共同解释。研究问题 4 的结论是，画像驱动建议能够把手工 sweep 中的固定 $k$ 选择转化为模型级预算候选，其有效性由模型家族、规模、任务和预算带共同塑造。

这四组结论共同指向本文的核心判断。注意力行为可以贯通校准层与分配层，先在 \texttt{INT8} 基准路径中建立保真参照，再在低比特场景中落实为 K/V 角色感知恢复路径，并进一步扩展为跨模型策略适用区间和 \texttt{AutoK} 预算候选生成机制。后文将在这一基础上归纳本文的创新点、学术价值与适用边界。

\section{创新点与学术价值}
\label{sec:ch5-innovation-value}
% ============================================================

从问题建模看，本文把 KV Cache 量化的对象从张量近似推进到注意力行为保持。第三章的误差传播分析说明，缓存量化误差进入下游时会经注意力分布和聚合输出表现为行为扰动。基准验证、低比特实例和预算扩展因此可以在同一组行为变量下讨论，校准目标与分配目标也被放到同一条分析主线中。

从方法链条看，本文把这一原则落实为三个相互衔接的环节。\texttt{INT8} 基准用于建立保守位宽下的保真参照，\texttt{INT4-RoleAlign} 在与 \texttt{KIVI-style} 共享非对称格式的条件下分析低比特失稳中的 K/V 角色差异，行为敏感度画像与 \texttt{AutoK} 则将离线行为读数转化为预算候选。这样，校准参数选择、低比特恢复和层级预算分配可以沿同一套产物继续传递。

系统实现提供了这条方法链的复核入口。离线校准产物、低比特缓存格式和 Triton Decode kernel 被接入同一条在线推理路径，使容量压缩、反量化开销、nibble unpack、GQA head mapping 与 TPOT 读数能够在真实 Decode 语义下被观察。系统层贡献不单独写成通用加速宣称，而是为行为校准和低比特恢复提供可执行、可复核的实验接口。

从经验结果看，跨模型实验把预算策略的有效性落到模型族、规模和任务条件上。Mistral-7B、Qwen2.5-3B、LLaMA-3.1-8B 与 Qwen2.5-14B 分别呈现 \texttt{AutoK} 单模型支持、早层保护、中等规模共识区间和高性能簇共存等结构。heuristic 多次作为强基线出现，提示某些模型条件下可行层集天然较窄。行为引导框架的经验价值，在于把这些现象从单点跑分比较推进为可复核的敏感度画像图谱。

这些结果共同支持本文的核心判断。注意力行为可以把基准校准、低比特诊断、跨模型预算分析和系统复核纳入同一解释框架。本文回答的是压缩后的缓存如何被观测、诊断、恢复和分配预算，也为后续实验扩展与部署复核提供了可沿用的组织方式。

\section{研究局限性}
\label{sec:conclusion-limitations}
% ============================================================

上述结论有明确的适用域。它们来自本文固定的评测协议、比较口径、模型集合和系统环境，后续扩展需要沿这些维度继续补充证据。

评测域方面，正文主矩阵使用统一生成规则下的 LongBench 风格合成基准任务。官方 LongBench 真实数据对照覆盖 Qwen2.5-1.5B、NarrativeQA/HotpotQA/GovReport 三个任务、每任务最多 50 个样本和单一随机种子。这一设置用于检查官方任务是否延续主协议方向。更开放的真实应用分布仍需要覆盖多轮问答、工具调用、代码仓库级检索、超长报告摘要和领域文档问答。PPL、Needle、RULER 与任务级指标共同降低了单一度量偏差，长上下文质量还可以继续引入更多观察维度。

比较口径方面，逐层预算分配结果来自同量级 \texttt{INT4} 预算带。第 4.4 节已经读出不同模型的结构落点，说明模型族、规模和任务共同塑造高性能区间，heuristic 在部分区间中是强基线。严格 matched-budget 条件下的形式化胜负判定，以及分配器相对 \texttt{KIVI-style} 的预算匹配比较，仍属于后续比较协议需要补齐的内容。因此，分配器方向的当前贡献落在跨模型适用区间的识别、组织和解释上。

模型与机制方面，本文模型集合以 decoder-only GQA 模型为主。MHA、MQA、滑动窗口注意力、MoE 和新的长上下文注意力变体还需要单独验证。\texttt{INT4-RoleAlign} 与 \texttt{KIVI-style} 的比较限定在 \texttt{per-channel K + per-token V} 同格式条件内，有助于减少格式差异干扰。格式贡献、校准贡献与架构因素还可以通过更正交的实验继续拆分。当前证据支持一条已验证设定中的低比特恢复路径，以及一组受模型族、规模和任务共同影响的结构读数。

系统环境方面，部署效率、性能交叉边界和显存读数绑定于当前 \texttt{NVIDIA H20}、固定软件栈、\texttt{batch=1}、贪婪解码以及当前 Triton/FlashInfer 后端组合。A100/H100、消费级 GPU、多 batch serving、speculative decoding、连续批处理调度或 streaming decode 都可能移动交叉点。\texttt{AutoK} 目前主要承担模型级静态候选生成器的角色，prompt 级自适应路由仍处于探索阶段。结果溯源链保证当前版本可追溯，未来架构或部署栈变化后还需要重新复核。

这些边界共同说明，当前关于基准路径保真、低比特恢复与跨模型策略适用区间的结论，已经被限定在清楚的评测域和系统域内。下一节沿这些边界继续整理可推进的后续方向。

\section{未来工作展望}
\label{sec:conclusion-future}
% ============================================================

基于上述适用边界，后续工作可以继续围绕调用粒度、预算表达力和比较口径扩展行为引导框架。

调用粒度可以从模型级建议推进到 prompt 级条件化策略选择。现阶段的 \texttt{AutoK} 根据离线行为敏感度画像给出模型级预算候选。下一步需要检验输入提示的结构表征能否与高性能区间稳定对应，并据此把策略调用从“按模型给建议”推进到“按输入做选择”。如果 prompt 结构能够稳定预测敏感层分布，预算建议就可以从静态模型画像进一步扩展到输入条件化路由。

预算表达力可以通过角色感知分配机制继续增强。当前框架已经把 Key 与 Value 的误差传播差异纳入低比特恢复路径，但分配层仍主要使用层级敏感度画像。后续可以进一步整合 Key 与 Value 两侧的预算描述、耦合关系和画像读数，形成同时考虑层、角色和预算带的联合分配机制。这样，K/V 角色差异就能从格式设计继续进入分配器的核心决策。

比较口径也可以继续收紧。第四章已经显示，不同模型在同量级 \texttt{INT4} 预算带内会落入不同策略适用区间。更严格的 matched-budget 协议可以进一步比较 behavior-guided allocator、位置启发式、uniform 策略和现有 KV Cache 压缩基线之间的相对优势。比较口径收紧以后，适用区间的判读可以从经验组织推进到更可比较的结构主张。

这些方向共同指向同一目标，即在调用粒度上引入 prompt 条件，在预算机制上发展角色感知分配器，在比较口径上补齐严格同预算实验，使行为引导框架的可解释性、可比较性和适用域继续扩大。

\section{结语}
\label{sec:conclusion-closing}
% ============================================================

长上下文高效推理让 KV Cache 量化同时面对容量、行为和系统三类问题。本文以注意力行为作为校准层与分配层共用的观察对象，在 \texttt{INT8} 基准实例、\texttt{INT4} 恢复路径、跨模型预算扩展和 Triton Decode 复核之间建立了可相互校验的证据链。

跨模型实验进一步说明，预算策略需要结合模型家族、参数规模与任务条件读取。敏感度画像将这些差异整理为可命名的适用区间，\texttt{AutoK} 则把离线行为读数转化为模型级预算候选。在长上下文推理的部署需求下，本文提供了一条以注意力行为保持为中心，连接低比特恢复、预算候选生成与系统复核的 KV Cache 行为对齐量化路径。
