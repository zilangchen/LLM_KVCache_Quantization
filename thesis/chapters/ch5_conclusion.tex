\section{主要发现}
\label{sec:conclusion-findings}
\label{sec:conclusion-answers}% backward-compat with earlier \ref
% ============================================================

本文以 behavior-guided 原则组织 KV Cache 量化的 calibration 与 allocation 两层，
跨 Qwen / LLaMA / Mistral 三个模型家族、1.5B--14B 五个规模与 5 个 LongBench 任务
给出 5 条 final-ready 的\textbf{经验规律}（clean-provenance pin=\code{ddada19}）。

\paragraph{发现一：Heuristic 是强基线——结构观察而非方法失败。}
\label{sec:discussion-heuristic}
我们的跨模型实验反复呈现同一结果：
等距位置保护启发式（heuristic-$k$）在多个模型的多个 task 上与 behavior-guided allocator 接近，
在 Qwen2.5-14B 上甚至反超 BA-$k$（第~\ref{subsec:exp-per-model-14b}~节）。
我们认为这不是需要 explain away 的现象，而是 framework 自身意义的体现：
如果 behavior-guided allocator 只靠"击败弱 baseline"成立，它的贡献会非常脆弱；
而 heuristic 作为强 baseline 的事实，把论文贡献从"方法数字更高"\textbf{抬升为"揭示结构分区"}
（见发现二）。在多个模型上 heuristic 与 BA-$k$ 接近，
本身就构成 regime map 的证据——在某些 (model, task) 区间里，
"保护哪些 $k$ 个位置"的选择空间很窄，无论 behavior profile 还是位置启发式
都会落到相似集合。

\paragraph{发现二：(Family, Scale, Task)-dependent Regime 地图。}
\label{sec:discussion-regime-map}
第~\ref{sec:exp-cross-model}~节的 Table~\ref{tab:t3-cross-model-main}
与 regime heatmap（图~\ref{fig:regime-map}）
在 4 模型 $\times$ 4 policy 空间上给出 per-model winner 分布，
4 行的最佳列落点各不相同：
\begin{itemize}[leftmargin=2em, itemsep=1pt, topsep=2pt]
  \item \textbf{Small-scale early-layer rescue}（Qwen2.5-3B）——
    BA-$k_1$ 保护 layer 0 相对 heuristic-$k_1$（保护中层）在 NarrativeQA 上回救 $\Delta \approx +4$ 分
    （§\ref{subsec:exp-per-model-3b}，Table~\ref{tab:t5-3b-early-layer}）；
  \item \textbf{Mid-scale top-$k$ 共识}（LLaMA-3.1-8B）——
    BA-$k_{11}$ 与 heuristic-$k_{11}$ 在 NarrativeQA / HotpotQA 上 top-tier 共存；
  \item \textbf{Large-scale wide-spread}（Qwen2.5-14B）——
    AutoK 标记 42/48 层 protected，top-3 policy 挤在 $3.5\%$ relative gap 内
    （§\ref{subsec:exp-per-model-14b}，Table~\ref{tab:t6-14b-toptier}）；
  \item \textbf{Mistral-specific AutoK win}——
    cov$80\%$ coverage 阈值下 AutoK 在 5-task 中赢 3 个，单模型最强正面案例
    （§\ref{subsec:exp-per-model-mistral}，Table~\ref{tab:t4-mistral-autok}）。
\end{itemize}
这一异质性\textbf{不否定} behavior framework 的价值，而是揭示其\emph{作用方式}：
framework 提供的是一个\emph{读数工具}让不同 regime 可被识别与命名，
而不是一个"永远最优"的 point 估计。

\paragraph{发现三：INT4 vs KIVI 的 Open Question（format vs calibration 耦合）。}
\label{sec:discussion-int4-openq}
第~\ref{sec:exp-int4-kivi}~节的 cross-model 对比显示
INT4-RoleAlign 与 KIVI 在同 format 下 PPL $\Delta \leq 0.15$、Needle 均 $100\%$。
由于两者共享 per-channel K + per-token V 格式，本文的对比
\textbf{只揭示}"同 format 下 calibration philosophy 差异有限"，
不能\emph{完全解耦} format 与 calibration 各自的独立贡献。
要完整回答"低比特 KV 量化的主要决定因素究竟是量化架构还是校准策略"，
未来需要 pure-format ablation（同 calibration、不同 format）
与 pure-calibration ablation（同 format、不同 calibration）的正交实验
——这构成 RoleAlign 的\textbf{第一个 open question}，
若 Hook（story §13）激活则可具体化为正式 claim。

\paragraph{发现四：INT8 canonical path 的 $\Delta{=}{+}0.02$ 闭环证据（C2 of RQ1）。}
\label{sec:discussion-int8}
在 Qwen2.5-1.5B 上，INT8-ours 相对 FP16 在 3 个 LongBench task 上的
mean $\Delta = +0.02$（Table~\ref{tab:t1-int8-canonical}），
在实证上不可区分。这不是"INT8 量化很容易"的废话——
它证明本文 framework 的 calibration 层在 canonical 规模下\textbf{可闭环}，
即 behavior-guided 原则存在一个\emph{干净的落地实例}，
剩下的工作是把同一原则推广到更激进的 bit-width 与更复杂的 allocator 层。

\paragraph{发现五：5 条 final-ready supported claims。}
\label{sec:discussion-final-ready}
本章总结的 5 条 claim 均由 clean-provenance pin=\code{ddada19} 的
md5-locked canonical 数据支撑（story 附 B）：
\begin{enumerate}[leftmargin=2em, itemsep=1pt, topsep=2pt]
  \item INT8 canonical path fidelity（int8$\leftrightarrow$fp16 $\Delta{=}+0.02$）；
  \item Mistral-specific AutoK win（cov$80\%$ 跨 core+extend task）；
  \item Qwen2.5-3B early-layer rescue regime；
  \item Qwen2.5-14B top-tier but no stable winner；
  \item Heuristic is a strong baseline（结构观察，非方法失败）。
\end{enumerate}

\paragraph{贡献总结（C1 / C2 / C3 镜像 §\ref{sec:ch1-contribution}）。}
本文从 attention 误差传播分析出发，
把\textbf{behavior 保持度}作为 KV Cache 量化的统一组织原则，
并通过两层实例化与跨模型实证验证其可用性：
\begin{itemize}[leftmargin=2em, itemsep=1pt, topsep=2pt]
  \item \textbf{C1}（framework）：behavior 原则同时贯通 calibration 与 allocation 两层，
    使两者不再是独立优化问题，而是共享同一 behavior sensitivity profile 的两层决策；
  \item \textbf{C2}（method instance）：INT8 canonical path 与 FP16 实证不可区分，
    INT4 RoleAlign 在低比特非对称 format 下与 KIVI 可比，
    AutoK 作为 profile-guided budget proposer 是 framework 自然长出的扩展；
  \item \textbf{C3}（empirical insight）：跨 5 模型、5 任务的 clean-provenance 实验
    揭示一张 (family, scale, task)-dependent 的 regime 地图，
    并在此基础上正面承认 heuristic 作为强基线——
    behavior-guided allocator 的真实价值从"击败弱对手"升级为"揭示结构分区"。
\end{itemize}

\section{局限性}
\label{sec:conclusion-limitations}
% ============================================================

诚实地列出本文研究的 5 条 limitations：

\begin{enumerate}[leftmargin=2em, itemsep=3pt, topsep=2pt]
  \item \textbf{实证范围有界}。
  本文在 5 模型 $\times$ 5 LongBench task $\times$ 单一 GPU（NVIDIA H20）
  的实验空间内展开；不覆盖所有 long-context 场景
  （如 complex reasoning、code completion long-form、multimodal KV cache）
  与所有硬件平台（A100 / TPU / 消费级 GPU）。
  跨硬件的浮点非结合律可能引入 $\leq 9\%$ 的准确率漂移
  \cite{song2025nondeterminism}，
  所有 PPL 与 quality 数据仅在本文硬件栈上保持位级确定。

  \item \textbf{INT4 vs KIVI 的 format--calibration 耦合未完全解耦}。
  如 §\ref{sec:discussion-int4-openq} 所述，
  RoleAlign 与 KIVI 共享 per-channel K + per-token V 格式，
  不能 isolate 出"同 format 下离线 calibration 相对运行时 absmax/min 的独立增益"。
  需要 pure-format ablation 与 pure-calibration ablation 的正交实验
  才能完整回答——这是\textbf{本文 Open Question 1}。

  \item \textbf{AutoK 仅在静态层面验证}。
  第~\ref{sec:exp-per-model}~节的 AutoK 是 \emph{model-level} 的 profile-guided
  proposer（给定模型 calibration，cov$80\%$ 输出固定 budget）；
  prompt-adaptive 路径（附录~\ref{sec:app-prompt-adaptive-8b}）
  在 Llama-3.1-8B 上 5-task mean 输 fixed-k（仅 lcc 独立赢 $+0.40$），
  Gate C verdict \textbf{Weak / Mixed}；
  selector 仍是 task-bucket fallback 实现，未来见第~\ref{sec:conclusion-future}~节。

  \item \textbf{Clean-provenance pin 固定的时效性}。
  所有正文数字基于 pin=\code{ddada19} 的 canonical 数据
  与 md5-locked manifest；
  该 pin 保证\emph{审计可追溯}但不保证\emph{泛化到未来 transformer 架构}。
  当新的长上下文注意力变体（如 sparse attention、hybrid-state）大量出现时，
  本文的 regime 地图需要在新架构上重新产出。

  \item \textbf{【条件性 Limitation】Allocator-vs-KIVI matched-budget formal compare 未完成}。
  本文的 cross-model regime 地图在\emph{每 policy 独立最优配置}上对比，
  尚未在\emph{matched memory budget}下对 Allocator（含 RoleAlign）与 KIVI 做正式
  formal comparison（story §13 Hook）；
  若 Hook 激活到 L1/L2，本条 limitation 可以从列表中删除，
  同时对应的 open question（§\ref{sec:discussion-int4-openq}）升级为正式 claim。
\end{enumerate}

\section{未来工作展望}
\label{sec:conclusion-future}
% ============================================================

基于本文的 regime 地图与 open questions，我们识别出 3 条最有价值的方向：

\paragraph{未来工作一：Per-Prompt Routing。}
附录~\ref{sec:app-prompt-adaptive-8b} 的 8B Gate C 数据揭示
\textbf{prompt\_adaptive 在 lcc 上独立赢 $+0.40$ 分}
（vs auto\_k），尽管 5-task mean 不及 fixed\_k；
附录~\ref{sec:app-prompt-adaptive-offprotocol} 的 1.5B/7B
off-protocol seed 亦指向 task-domain 相关的 routing signal。
当前 selector 实现是 \emph{task-bucket fallback}，未充分利用 prompt 级信号——
一个真正的 per-prompt selector（input prompt embedding $\to$ 最优 policy）
有可能把当前在 lcc 上的 positive gradient 扩展到更多 task。
这是 AutoK 的自然延伸方向。

\paragraph{未来工作二：更成熟的 Role-Aware Allocator。}
第~\ref{sec:ch3-allocator}~节的 allocator 按 K / V 分层分配提供了结构接口，
但当前的 K/V 非对称分配\emph{可行但未熟}——
L2 Phase A 的 role-aware kv-asymmetric 实验
（\code{results/l2\_kv\_asymmetric/}，108 CSV）显示一些局部改善，
但没有达到\emph{跨模型一致优于 AutoK} 的信号水平。
K 与 V 的 layer-wise 联合优化（例如 $(b_K^{(l)}, b_V^{(l)})$ 作为 pair 而非独立变量）
值得作为专项工作。

\paragraph{未来工作三：Allocator-vs-KIVI Matched-Budget Formal Compare。}
story §13 Hook 的正式对比包是本文的第一个\textbf{条件 Future Work}。
若在未来实验中 G0 Fairness Gate（format 相同 + memory $\pm 3\%$）、
G1 Main Matrix Validity、G2 Claim Strength 全部满足并达到 L1/L2，
则 framework 的\textbf{系统级优势}可以从当前的 open question
升级为正式 claim，
并把 §\ref{sec:discussion-int4-openq} 的 Open Question 1 具体化。
其 ExecPlan 骨架已在 \code{.agents/execplans/2026-04-20\_allocator-vs-kivi-claim-package.md} 冻结。

\section{结语}
\label{sec:conclusion-closing}
% ============================================================

KV Cache 量化不是一个"找最佳 bit-width"的优化问题，
而是一个需要\textbf{组织原则}的系统设计问题。
本文把\emph{behavior 保持度}作为 KV Cache 量化的 first principle，
完成了 calibration 与 allocation 两层的统一组织，
并通过 clean-provenance 跨 5 模型实证
交付了一张\textbf{可读数的 operating regime 地图}。

Regime 地图的价值不在于给出"通用最优"，
而在于使原本隐藏在"多方法跨模型平均数"背后的结构差异可\textbf{被命名、被复用}：
3B 的 early-layer rescue、14B 的 top-tier wide-spread、Mistral 的 AutoK cov80 胜利、
以及 heuristic 在多个 regime 区间里与 behavior-guided 重合的结构观察——
这 4 条命名规律构成 behavior framework 实证结果的\textbf{操作化入口}。

对 KV Cache 社区，本文的贡献是一组\textbf{工具与方法论}：
一个统一的 calibration-allocation framework、
一组可审计的实例化路径（INT8 canonical / INT4 RoleAlign / AutoK）、
以及一张跨模型 regime 地图与三条诚实的 open question。
我们希望这些工作能为未来的 KV Cache 研究提供\emph{读数尺度}，
而不仅是一个 point 估计。
