\chapter{总结与展望}
\label{chap:conclusion}

本章对全文的研究工作进行总结，回顾主要贡献，分析当前方法存在的局限性，
并对未来可能的研究方向进行展望。

% ------------------------------------------------------------
\section{研究工作总结}
\label{sec:conclusion-summary}
% ------------------------------------------------------------

如第四章实验所示，INT8 行为对齐量化在 PPL、Needle、LongBench 三个质量基准上
与 FP16 保持统计不可区分的水平，同时 Triton 融合核函数在解码延迟上实现了
8\%--38\% 的显著降低（解决了传统方案"省显存但掉速度"的矛盾），
KV Cache 显存节省约 44\%，使 INT8-ours 在效率--质量空间中占据 Pareto 最优位置。
同一校准算法在 Qwen2.5-7B 和 LLaMA-3.1-8B 上复现了一致的性能模式，验证了跨模型适用性。
在 11 项实验预期中，8 项达标，3 项未达标：
INT8 下 RULER 评测退化 $-$2.64\%（退化集中于 S-NIAH 和 MK-NIAH 检索子任务，且受配置聚合效应放大），
INT4 下 Needle 和 PPL 均出现显著退化，
后者确认了 4-bit 对称量化在长上下文场景下的适用性边界
（详见第~\ref{subsec:exp-int4-limitations}~节）。
实验还揭示了不同评测基准对量化噪声呈现差异化敏感度——
单点精确检索（Needle）在 INT8 下完全不受影响，
而多目标并发检索（MK-NIAH）和弱信号检测（S-NIAH）则更为敏感。

针对现有 KV Cache 量化方法校准目标与注意力行为脱节的问题，
本文提出了行为对齐量化框架（Behavior-Aligned Calibration Framework），
以注意力权重分布的 KL 散度作为校准目标，
结合逐层逐头温度校正机制（per-head $\tau^{-1}$），
在不增加推理开销的前提下恢复量化后注意力分布的形态特征。
在系统层面，基于 Triton 实现了 INT8/INT4 融合量化解码注意力核函数，
将反量化与注意力计算融合为单一 GPU 核函数调用，消除了中间张量的显存与访存开销。
在实证层面，本文在 3 个模型、7 种量化配置、5 个评测基准上
采用 Bootstrap 置信区间与 BH-FDR 校正的统计框架进行了全面评估。
KL 校准与温度校正的协同设计在 INT8 下稳定成立；
INT4 设置下，补充实验表明运行时 scale 精度显著影响部分模型的质量表现，
但当前证据尚不足以支持跨模型一致的非劣性结论。

\section{局限性}
\label{sec:conclusion-limitations}
% ------------------------------------------------------------

尽管本文提出的行为对齐量化框架在 INT8 的 PPL、Needle、LongBench 三项核心质量基准上维持了与 FP16 统计不可区分的水平（RULER 的退化集中于检索类子任务，详见第~\ref{para:c6-fail}~段分析），
但当前工作仍存在以下局限：

\textbf{INT4 量化的质量瓶颈。}
实验结果表明，INT4 量化在 32K 长上下文场景下存在系统性的质量退化：
Needle 通过率在 1.5B 模型上从 100\%（FP16）降至 0\%，
PPL 退化达 $-$15.9\%，
LongBench 综合评分在 1.5B 和 7B 模型上大幅下降。
第~\ref{subsec:exp-int4-limitations}~节的定量分析表明，
INT4 的 SQNR 仅 25.8~dB（INT8 为 49.9~dB），
逐元素量化噪声功率约为 INT8 的 289 倍，
远超 softmax 归一化可消解的范围。
随序列长度增大，极值统计效应进一步放大了噪声对检索信号的淹没概率，
导致 Needle 通过率从短上下文到 32K 的急剧下降。
在跨模型对比中，GQA 结构的量化误差稀释效应
为不同模型间的退化差异提供了定量解释：
LLaMA-3.1-8B（$H_{kv}=8$）的有效噪声功率仅为
Qwen2.5-1.5B（$H_{kv}=2$）的 $1/4$，
这解释了其在 INT4 下的异常稳健表现（Needle 100\%、PPL 退化仅 $-$9.8\%）。
补充实验进一步表明，运行时 scale 参数的精度链路（float32 vs float16）
也是影响 INT4 质量的重要因素，确保 float32 精度后部分模型的 PPL 获得显著改善，
但整体仍未达到跨模型一致的非劣性水平。

\textbf{KIVI-style 基线的实现差异。}
本文实现的 KIVI-style 基线采用了 KIVI~\citep{liu2024kivi}
的核心量化轴策略（Key 按通道量化、Value 按 token 量化），
但未完整复现原文的所有工程优化（如残差长度调度、异步量化流水线等）。
因此本文的 KIVI-style 实验结果应理解为对其核心算法思想的验证，
而非原文系统完整性能的严格复现。

\textbf{单 GPU 评测限制。}
本文的所有实验均在单张 NVIDIA H20 GPU（98GB 显存）上完成，
未涉及多卡分布式推理场景。在张量并行（Tensor Parallelism）或
流水线并行（Pipeline Parallelism）配置下，KV Cache 的分片、
同步与量化的交互可能引入新的性能瓶颈和数值问题，
当前框架尚未对此进行验证。

\textbf{校准产物的模型特异性。}
行为对齐校准过程产生的量化参数（per-layer scales 和 per-head $\tau^{-1}$）
与特定模型的权重分布和注意力模式紧密耦合。
当部署新的模型架构或不同规模的模型变体时，
需要重新执行完整的校准流程以获取适配的量化参数，
这在模型迭代频繁的场景下会带来额外的工程成本。
研究校准参数在同架构不同规模模型间的可迁移性，
是降低部署成本的一个重要方向。

\section{对工业实践的启示}
\label{sec:conclusion-implications}
% ------------------------------------------------------------

本文的实验结果对 LLM 推理系统的工程实践具有以下参考价值：

\textbf{INT8 量化在单 GPU 推理场景下已展现明确的部署可行性。}
INT8 行为对齐量化在 PPL、Needle、LongBench 三项核心质量基准上
均未引入统计显著退化（RULER 的退化集中于检索类子任务且受配置聚合效应放大，详见第~\ref{para:c6-fail}~段），
同时实现了 44\% 的 KV Cache 显存节省和 8\%--38\% 的解码延迟降低。
对于 PPL 和检索准确率是核心指标的在线推理服务，
这意味着在相同硬件条件下可以支持更长的上下文窗口或更大的并发批次。
表~\ref{tab:app-batch-capacity}（附录）显示，
int8\_ours 的可服务 batch 从 fp16 的 16 提升到 32（2$\times$），
直接倍增了单 GPU 的服务吞吐能力。

\textbf{INT4 量化需审慎评估应用场景。}
INT4 量化虽可将 KV Cache 进一步压缩至 FP16 的 $1/4$，
但 32K 长上下文下的质量退化使其不适合对精度要求高的任务
（如精确信息检索、长文档问答）。
对于对质量容忍度较高的场景（如大规模文本摘要、粗粒度分类），
INT4 量化的显存优势仍有实用价值。

\textbf{融合核函数是量化收益的放大器。}
本文的对比实验表明，仅做 KV Cache 量化而不优化计算路径
（如 int8\_baseline 的两阶段方案），
反而可能因中间张量分配导致 TPOT 增加。
Triton 融合核函数消除了这一开销，使量化的显存节省真正转化为端到端的性能提升。
这提示工业部署中应将量化与计算融合作为整体方案设计，而非独立优化。

\textbf{校准成本与部署收益的权衡。}
行为对齐校准是一次性离线过程：在单张 GPU 上，
1.5B 模型的全量校准（128 样本、逐层逐头搜索）约需 15 分钟，
7B/8B 模型约需 30--50 分钟。校准产出的 per-layer scale 和 per-head $\tau^{-1}$
参数以 JSON 文件形式保存（$<$10~KB），在推理时仅需加载一次即可永久复用。
对于在线推理服务，单次校准投入可在数小时的持续服务中
通过吞吐量提升和显存节省完全摊销，投入产出比极为可观。

\section{未来工作展望}
\label{sec:conclusion-future}
% ------------------------------------------------------------

基于本文的研究成果和上述局限性分析，以下方向值得进一步探索：

\textbf{基于 GQA 头数的自适应量化比特分配。}
第~\ref{subsec:exp-int4-limitations}~节的 GQA 误差稀释分析表明，
$H_{kv}$ 越大的模型对低比特量化的容忍度越高。
这一发现启发了一种架构感知的自适应策略：
根据每个模型的 $H_{kv}$ 和每个层的注意力头重要性评分
（如校准阶段的 KL 散度值），自动为不同层和不同 KV 组分配量化比特宽度——
对 $H_{kv}$ 较大或 KL 不敏感的组使用 INT4，
对 $H_{kv}$ 较小或关键层保持 INT8。
这种混合精度策略有望在总体显存预算不变的情况下
突破单一比特宽度的质量瓶颈。

\textbf{多 GPU 分布式推理。}
将行为对齐量化框架扩展到张量并行和流水线并行场景，
需要解决 KV Cache 分片粒度与量化粒度的对齐问题、
跨设备量化参数的一致性维护以及通信-计算重叠的优化。
与主流分布式推理框架的集成将是框架走向实际部署的关键一步。

\textbf{推理服务框架集成。}
当前实现基于 Transformers 库的自定义生成循环，尚未与
vLLM\citep{kwon2023vllm}、Text Generation Inference（TGI）等
生产级推理服务框架集成。这些框架已实现了 PagedAttention、
连续批处理（Continuous Batching）等系统级优化，
将行为对齐量化集成到其调度与内存管理机制中，
有望在真实服务场景下充分发挥量化的显存节省优势。

\textbf{自适应混合精度策略。}
本文的实验表明，不同注意力头对量化噪声的敏感度存在显著差异。
基于此观察，一个自然的改进方向是根据每个注意力头的重要性评分
（如校准阶段的 KL 散度值或梯度敏感度）动态分配量化比特宽度——
对敏感头保持较高精度，对鲁棒头采用更激进的压缩。
这种自适应混合精度策略有望在总体显存预算不变的情况下
进一步提升量化后的推理质量。

\textbf{校准目标函数的泛化探索。}
本文以 KL 散度作为校准目标，但在 INT4 等低比特场景下，
KL 散度优化景观可能存在多个局部最优解，
且 KL 散度对尾部分布的高敏感性在低比特下可能放大噪声干扰。
探索 Wasserstein 距离（地球搬运距离）或 R\'{e}nyi 散度等替代目标函数，
可能在低比特量化的校准优化中展现不同的收敛行为和解空间特性。
特别是 Wasserstein 距离在度量概率分布距离时不要求支撑集重叠，
在量化导致分布支撑发生离散化偏移时可能提供更稳定的梯度信号。
