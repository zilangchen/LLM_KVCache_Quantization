\chapter{总结与展望}
\label{chap:conclusion}

本章对全文的研究工作进行总结，回顾主要贡献，分析当前方法存在的局限性，
并对未来可能的研究方向进行展望。

% ------------------------------------------------------------
\section{研究工作总结}
\label{sec:conclusion-summary}
% ------------------------------------------------------------

如第四章实验所示，INT8 行为对齐量化在全部质量基准上保持与 FP16
统计不可区分的水平，同时实现了显著的解码延迟降低（8\%--38\%）
和 KV Cache 显存节省（44\%）。
在 11 项核心声明中，8 项通过验证，3 项 INT4 相关声明（C6--C8）未通过，
确认了 4-bit 对称量化的适用性边界
（详见第~\ref{subsec:exp-int4-limitations}~节）。

随着大语言模型参数规模和上下文窗口的持续增长，推理阶段的键值缓存（KV Cache）
已成为制约部署效率的核心瓶颈。KV Cache 的显存占用与序列长度线性增长，
在长上下文场景下极易触发显存溢出，严重限制了批处理吞吐量和可服务的最大上下文长度。
量化作为压缩 KV Cache 的主要手段，能够有效降低显存占用并提升访存效率，
但其成功与否高度依赖于校准策略的选择——不恰当的校准准则会导致量化后
注意力分布发生难以察觉的语义偏移，进而在长上下文推理中引发显著的质量退化。

针对现有 KV Cache 量化方法中校准策略不够精准、缺乏行为层面对齐的问题，
本文提出了行为对齐量化框架（Behavior-Aligned Calibration Framework）。
该框架的核心思想是：以 attention 权重分布的 KL 散度（而非传统的数值均方误差）
作为校准优化目标，在量化参数搜索过程中直接最小化量化前后注意力分布的差异，
从而确保量化操作不改变模型的注意力行为模式。
在此基础上，本文进一步引入逐层逐头温度校正机制（per-head $\tau^{-1}$），
通过为每个注意力头独立搜索最优的逆温度参数，恢复量化后注意力分布的尖锐度，
补偿量化噪声对 softmax 输出分布的平滑化效应。

在系统实现方面，本文设计并实现了基于 Triton 的融合量化解码注意力核函数
（Fused Quantized Decode Attention Kernel），将反量化、缩放、注意力计算
融合为单一 GPU 核函数调用，避免了中间张量的显存分配与多次全局内存访存，
在保证数值精度的同时实现了推理加速。该核函数同时支持 INT8 和 INT4
两种量化精度，并兼容分组查询注意力（GQA）机制。

本文在 3 个代表性模型（Qwen2.5-1.5B-Instruct、Qwen2.5-7B-Instruct、
LLaMA-3.1-8B-Instruct）、7 种量化配置（fp16 基线、INT8 基线、INT8-ours、
INT4 基线、INT4-fused、INT4-ours、KIVI-style）和 5 个评测基准
（困惑度、LongBench、RULER、Needle-in-a-Haystack、延迟性能）上
进行了全面的实证研究。实验采用 Bootstrap 置信区间与 BH-FDR 校正的
统计检验框架，确保结论的统计可靠性。

\section{主要贡献回顾}
\label{sec:conclusion-contributions}
% ------------------------------------------------------------

本文的主要贡献可归纳为以下三个方面：

\textbf{贡献一：行为对齐校准框架。}
本文提出了以 attention 权重分布的 KL 散度为优化目标的校准方法，
替代传统的均方误差（MSE）或百分位截断（percentile clipping）准则。
该方法在量化参数搜索过程中直接度量量化前后注意力行为的差异，
并结合逐层逐头温度校正（per-head $\tau^{-1}$）搜索机制，
在不增加推理时额外计算开销的前提下恢复量化后注意力分布的形态特征。
KL 校准与温度校正的协同设计在 INT8 主线上稳定成立，并在 INT4 设置下
提供了具有启发性的补充证据：后验工程分析表明，运行时 scale 精度会显著
影响部分模型的 INT4 质量表现。然而，当前证据仍显示 INT4 结果具有明显的
模型相关性与配置敏感性，尚不足以支持跨模型一致的非劣性结论。

\textbf{贡献二：Triton 融合量化解码注意力核函数。}
本文设计并实现了面向 INT8 和 INT4 两种精度的 Triton 融合核函数，
将 KV Cache 的反量化操作与注意力得分计算融合在同一核函数中执行。
对于 INT4 精度，核函数内部实现了 bit-packing 数据的在线解包与反量化，
避免了显式的中间缓冲区分配。融合实现消除了朴素量化方案中
"先反量化到 fp16、再做注意力"所带来的额外显存占用和访存开销，
在保持与 PyTorch 参考实现数值一致的同时实现了推理吞吐量的提升。

\textbf{贡献三：全面的多维度实证研究。}
本文构建了覆盖 3 个模型、7 种量化模式、5 个评测基准的全矩阵实验体系，
在语言建模质量（困惑度）、长文档理解（LongBench）、
合成检索能力（RULER、Needle-in-a-Haystack）以及推理延迟等多个维度
对各量化方案进行了系统比较。所有质量指标均报告了基于 Bootstrap 的
95\% 置信区间，并通过 BH-FDR 校正控制多重比较的假发现率，
确保实验结论的统计严谨性和可复现性。

\section{局限性}
\label{sec:conclusion-limitations}
% ------------------------------------------------------------

尽管本文提出的行为对齐量化框架在 INT8 全部质量基准上稳定维持了与 FP16 统计不可区分的质量水平，
但当前工作仍存在以下局限：

\textbf{INT4 量化的质量瓶颈。}
实验结果表明，INT4 量化在 32K 长上下文场景下存在系统性的质量退化：
Needle 通过率在 1.5B 模型上从 100\%（FP16）降至 0\%（Claim C7 FAIL），
PPL 退化达 $-$15.9\%（Claim C8 FAIL），
LongBench 综合评分在 1.5B 和 7B 模型上大幅下降。
这表明 4-bit 对称均匀量化的 $2^4 = 16$ 个表示级别
不足以捕捉长序列中关键注意力模式的精细差异，
尚不满足生产部署的质量要求。
在跨模型对比中，LLaMA-3.1-8B 的 INT4 退化幅度小于 Qwen 系列，
可能受益于其 8 个 KV 头对量化误差的更强稀释效应。
如何在 INT4 乃至更低比特下保持长上下文检索能力，仍是一个开放性问题。
后续工程分析进一步揭示，运行时 scale 参数的精度链路（float32 vs float16）
也是影响 INT4 质量的因素之一，修复后部分模型的 PPL 获得显著改善，
但整体仍未达到跨模型一致的非劣性水平（详见第~\ref{subsec:exp-int4-limitations} 节）。

\textbf{KIVI-style 基线的实现差异。}
本文实现的 KIVI-style 基线采用了 KIVI~\citep{liu2024kivi}
的核心量化轴策略（Key 按通道量化、Value 按 token 量化），
但未完整复现原文的所有工程优化（如残差长度调度、异步量化流水线等）。
因此本文的 KIVI-style 实验结果应理解为对其核心算法思想的验证，
而非原文系统完整性能的严格复现。

\textbf{单 GPU 评测限制。}
本文的所有实验均在单张 NVIDIA H20 GPU（98GB 显存）上完成，
未涉及多卡分布式推理场景。在张量并行（Tensor Parallelism）或
流水线并行（Pipeline Parallelism）配置下，KV Cache 的分片、
同步与量化的交互可能引入新的性能瓶颈和数值问题，
当前框架尚未对此进行验证。

\textbf{校准产物的模型特异性。}
行为对齐校准过程产生的量化参数（per-layer scales 和 per-head $\tau^{-1}$）
与特定模型的权重分布和注意力模式紧密耦合。
当部署新的模型架构或不同规模的模型变体时，
需要重新执行完整的校准流程以获取适配的量化参数，
这在模型迭代频繁的场景下会带来额外的工程成本。
研究校准参数在同架构不同规模模型间的可迁移性，
是降低部署成本的一个重要方向。

\section{对工业实践的启示}
\label{sec:conclusion-implications}
% ------------------------------------------------------------

本文的实验结果对 LLM 推理系统的工程实践具有以下参考价值：

\textbf{INT8 量化在单 GPU 推理场景下已展现明确的部署可行性。}
INT8 行为对齐量化在全部质量基准上均未引入统计显著退化，
同时实现了 44\% 的 KV Cache 显存节省和 8\%--38\% 的解码延迟降低。
对于在线推理服务，这意味着在相同硬件条件下可以支持
更长的上下文窗口或更大的并发批次，而无需承担质量风险。
表~\ref{tab:app-batch-capacity}（附录）显示，
int8\_ours 的可服务 batch 从 fp16 的 16 提升到 32（2$\times$），
直接倍增了单 GPU 的服务吞吐能力。

\textbf{INT4 量化需审慎评估应用场景。}
INT4 量化虽可将 KV Cache 进一步压缩至 FP16 的 $1/4$，
但 32K 长上下文下的质量退化使其不适合对精度要求高的任务
（如精确信息检索、长文档问答）。
对于对质量容忍度较高的场景（如大规模文本摘要、粗粒度分类），
INT4 量化的显存优势仍有实用价值。

\textbf{融合核函数是量化收益的放大器。}
本文的对比实验表明，仅做 KV Cache 量化而不优化计算路径
（如 int8\_baseline 的两阶段方案），
反而可能因中间张量分配导致 TPOT 增加。
Triton 融合核函数消除了这一开销，使量化的显存节省真正转化为端到端的性能提升。
这提示工业部署中应将量化与计算融合作为整体方案设计，而非独立优化。

\textbf{校准成本与部署收益的权衡。}
行为对齐校准是一次性离线过程：在单张 GPU 上，
1.5B 模型的全量校准（128 样本、逐层逐头搜索）约需 15 分钟，
7B/8B 模型约需 30--50 分钟。校准产出的 per-layer scale 和 per-head $\tau^{-1}$
参数以 JSON 文件形式保存（$<$10~KB），在推理时仅需加载一次即可永久复用。
对于在线推理服务，单次校准投入可在数小时的持续服务中
通过吞吐量提升和显存节省完全摊销，投入产出比极为可观。

\section{未来工作展望}
\label{sec:conclusion-future}
% ------------------------------------------------------------

基于本文的研究成果和上述局限性分析，以下方向值得进一步探索：

\textbf{更低比特量化的探索。}
INT2 乃至混合 2/4-bit 量化是进一步压缩 KV Cache 的自然延伸。
然而，极低比特量化面临表示能力急剧下降的挑战，
需要研究非均匀量化网格、基于重要性的自适应比特分配
以及量化感知微调（Quantization-Aware Training）等技术的协同应用，
以在极限压缩率下维持可接受的推理质量。

\textbf{多 GPU 分布式推理。}
将行为对齐量化框架扩展到张量并行和流水线并行场景，
需要解决 KV Cache 分片粒度与量化粒度的对齐问题、
跨设备量化参数的一致性维护以及通信-计算重叠的优化。
与主流分布式推理框架的集成将是框架走向实际部署的关键一步。

\textbf{推理服务框架集成。}
当前实现基于 Transformers 库的自定义生成循环，尚未与
vLLM\citep{kwon2023vllm}、Text Generation Inference（TGI）等
生产级推理服务框架集成。这些框架已实现了 PagedAttention、
连续批处理（Continuous Batching）等系统级优化，
将行为对齐量化集成到其调度与内存管理机制中，
有望在真实服务场景下充分发挥量化的显存节省优势。

\textbf{自适应混合精度策略。}
本文的实验表明，不同注意力头对量化噪声的敏感度存在显著差异。
基于此观察，一个自然的改进方向是根据每个注意力头的重要性评分
（如校准阶段的 KL 散度值或梯度敏感度）动态分配量化比特宽度——
对敏感头保持较高精度，对鲁棒头采用更激进的压缩。
这种自适应混合精度策略有望在总体显存预算不变的情况下
进一步提升量化后的推理质量。

\textbf{校准效率优化。}
当前的校准流程需要在代表性数据上逐层逐头地搜索最优量化参数，
其时间开销随模型层数和注意力头数线性增长。
研究更高效的搜索算法（如基于贝叶斯优化的参数搜索、
分层剪枝策略或一次性校准方法）以缩短校准时间，
将有助于提升框架在模型快速迭代场景下的实用性。
