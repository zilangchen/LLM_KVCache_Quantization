本章回扣引言提出的核心问题，
以 INT4-RoleAlign 的成功为论证终点，
总结行为对齐量化框架的方法论启示，
分析局限性并展望未来方向。

% ------------------------------------------------------------
\section{核心回答}
\label{sec:conclusion-answers}
% ------------------------------------------------------------

引言提出了三个核心问题，本文的实验证据给出了明确回答：

\textbf{应优化什么？}
不是数值重建误差，而是注意力行为。
以 attention-KL 作为校准目标，INT8 行为对齐量化
作为该原则的规范验证实例，
在三个模型族上保持质量的同时实现 44\% KV 显存降低——
表明注意力行为保真是正确的优化目标。

\textbf{低比特为何失败？}
受控 K/V 消融揭示对称 INT4 的系统性失效源于结构性根因：
Key 量化主导质量损失，且呈显著的 GQA 架构依赖特征。
Key 决定注意力的``看哪里''，
其通道间数值异质性在低比特下无法被对称格式有效捕捉。

\textbf{诊断导出了什么？}
INT4-RoleAlign——诊断驱动的角色感知低比特设计。
在三个模型上 Needle 从 0\%$\to$100\%，
在 73\% KV Cache 压缩率下恢复检索能力
（RULER 评测提供支持性证据，s\_niah $\geq$99\%）。
PPL 退化因模型规模而异（8B: 2.4\%，7B: 6.1\%，1.5B: 13.7\%），
揭示了局部检索对量化噪声的鲁棒性远高于全局语言建模质量。
从原则到诊断到设计到边界的闭环构成本文的核心贡献。

% ------------------------------------------------------------
\section{核心观察}
\label{sec:conclusion-summary}
% ------------------------------------------------------------

本文的实验证据支撑三个层层递进的核心观察：

\textbf{观察一：注意力行为保真是 KV Cache 量化的正确优化目标。}
以 attention-KL 作为校准目标，INT8 行为对齐量化在四个模型（三个模型族）上
保持 PPL、Needle、LongBench-style 合成评测与 FP16 统计不可区分的水平
（RULER 退化 $-$2.64\% 集中于检索类子任务），
同时实现 KV 显存降低约 44\%、解码延迟降低 8--38\%。
INT8 结果构成行为对齐原则的 canonical validated instance：
它表明以注意力行为而非数值重建误差作为校准目标，
能够在质量、显存与延迟间产出可部署的折中。

\textbf{观察二：低比特退化是 Key 主导、架构依赖的结构性问题。}
同一 attention-KL 诊断透镜揭示对称 INT4 的系统性失效
（Needle 0\%，PPL 退化 $>$15\%），
受控 K/V 消融在三个模型族上一致确认 Key 量化是退化的主导因素，
且该敏感性呈显著的 GQA 架构依赖特征
（$H_{kv}$ 越大，量化噪声稀释效应越强）。
这一核心发现揭示：低比特退化并非随机的精度损失，
而是具有明确结构——Key 决定注意力的``看哪里''，
其通道间数值异质性在低比特下无法被对称格式有效捕捉。

\textbf{观察三：检索恢复与能力边界——诊断驱动的低比特设计。}
基于上述诊断，INT4-RoleAlign 吸收 per-channel K + per-token V 非对称量化轴，
通过 BA-guided percentile 校准搜索最优参数。
在三个模型上 Needle 从 0\% 恢复至 100\%，
KV Cache 显存节省 73\%。
PPL 退化随模型规模递减（8B: 2.4\%，7B: 6.1\%，1.5B: 13.7\%），
揭示了 INT4 量化的结构性特征：
局部检索对量化噪声的鲁棒性远高于全局语言建模质量。
解码延迟因缺少 Triton 融合核函数增加约 2--2.5$\times$，
构成当前设计的明确效率边界。
这一从诊断到设计再到边界界定的完整链条构成本文的核心方法贡献，
验证了行为对齐原则的诊断驱动设计能力，
同时诚实地界定了低比特量化的能力前沿。

\paragraph{方法论启示}
上述三个观察共同指向一个方法论层面的启示：
行为对齐框架为 KV Cache 量化提供了从``试错''到``诊断驱动设计''的升级路径。
传统量化方法依赖数值误差指标和经验性的参数调整，
当遇到低比特失效时缺乏系统性的诊断手段。
Attention-KL 作为统一的校准与诊断原则，
既解释高比特设置下的成功（观察一），
也指导低比特设置下的设计演化（观察二 $\to$ 观察三）。

\section{局限性}
\label{sec:conclusion-limitations}
% ------------------------------------------------------------

当前工作存在以下局限：

\textbf{INT4-RoleAlign 的验证范围。}
INT4-RoleAlign 已在三个模型上通过 PPL、Needle、LongBench、RULER 四类质量评测
以及 TPOT 和 KV Cache 显存两类系统效率评测验证。
检索能力在 Needle 上完全恢复（100\%），RULER 子任务提供进一步支持（s\_niah $\geq$99\%），
但 PPL 退化显著（8B: 2.4\%，7B: 6.1\%，1.5B: 13.7\%），
且解码延迟因缺少 Triton 融合核函数增加约 2--2.5$\times$。
chunk\_size=1 全量化压力测试仅在 Qwen2.5-1.5B 单模型、单种子条件下验证，
跨模型推广仍需后续实验。

\textbf{评测方法边界。}
所有实验在单张 NVIDIA H20 GPU 上完成，未涉及多卡分布式推理。
LongBench 评测基于自行实现的合成数据源而非官方数据集，
绝对分数不可与排行榜直接对比，但量化方法间的相对差异仍有效。
Mistral-7B 的 LongBench 数据因合成源问题在正文核心对比中已排除
（Mistral 在 PPL、Needle、RULER 上的评测均正常有效）。
K/V 消融实验仅覆盖 RULER 和 LongBench，未纳入 PPL 指标。

\textbf{校准框架局限。}
1.5B 主模型校准产物的 Q 向量预处理缺少 input\_layernorm 和 RoPE 变换
（附录~\ref{sec:app-calib-provenance}），修正版对比验证尚未完成。
校准仅使用 WikiText-103 数据集，对代码、多语言等分布外数据的适用性未验证。
$\tau^{-1}$ 温度校正在非对称格式下无稳定收益甚至有害，
提示有效组件依赖量化格式，不可跨格式迁移。
校准参数与模型权重耦合，部署新模型需重新校准。

\textbf{KIVI-style 基线的实现差异。}
本文 KIVI-style 基线采用 KIVI~\citep{liu2024kivi} 的核心量化轴策略，
未完整复现原文全部工程优化（如残差长度调度等），
结果应理解为对核心算法思想的验证。

\section{工业实践启示}
\label{sec:conclusion-implications}
% ------------------------------------------------------------

上述结果对 LLM 推理系统的工程实践具有直接参考价值：
INT8 行为对齐量化在质量无损的前提下将可服务 batch 倍增至 2$\times$
（附录表~\ref{tab:app-batch-capacity}），
但 Triton 融合核函数是效率收益的前提——
仅做量化而不融合计算路径反而可能增加延迟。
INT4-RoleAlign 以 73\% 压缩率在 Needle、RULER、LongBench 评测上保持检索质量，
同时 KV Cache 显存降低约 3.7$\times$。
解码延迟因 \texttt{torch\_ref} 路径（无 Triton 融合核函数）比 FP16 增加约 2--2.5$\times$，
当前定位为``检索保持 + 显存优化方案''。
校准为一次性离线过程（1.5B 约 15 分钟，7B/8B 约 30--50 分钟），
产出 $<$10~KB 的 JSON 参数文件，部署成本极低。

\section{未来工作展望}
\label{sec:conclusion-future}
% ------------------------------------------------------------

基于本文的研究成果和上述局限性分析，以下方向值得进一步探索：

\textbf{基于 GQA 头数的自适应量化比特分配。}
第~\ref{subsec:exp-int4-limitations}~节和第~\ref{sec:exp-kv-sensitivity}~节的实验
共同表明，GQA 头数（$H_{kv}$）是决定低比特量化容忍度的关键架构因素，
且 K/V 消融进一步证实了 Key 与 Value 在量化敏感性上的根本不对称性。
这一发现启发了一种架构感知的自适应策略：
根据每个模型的 $H_{kv}$ 和每个层/头的 KL 散度值，
自动为不同层和不同 KV 维度（K vs V）分配量化比特宽度——
$H_{kv}$ 较大的模型可更激进地压缩 Value 至 INT4 甚至 INT2，
$H_{kv}$ 较小的模型则应优先保障 Key 精度。
第~\ref{subsec:exp-mixedkv}~节的 MixedKV 实验已初步验证了
该策略方向的可行性，更精细的逐层自适应分配是进一步研究的重点。

\textbf{多 GPU 分布式推理。}
将行为对齐量化框架扩展到张量并行和流水线并行场景，
需要解决 KV Cache 分片粒度与量化粒度的对齐问题、
跨设备量化参数的一致性维护以及通信-计算重叠的优化。
与主流分布式推理框架的集成将是框架走向实际部署的关键一步。

\textbf{推理服务框架集成。}
当前实现基于 Transformers 库的自定义生成循环，尚未与
vLLM~\cite{kwon2023vllm}、Text Generation Inference（TGI）\footnote{HuggingFace TGI: \url{https://github.com/huggingface/text-generation-inference}} 等
生产级推理服务框架集成。这些框架已实现了 PagedAttention、
连续批处理（Continuous Batching）等系统级优化，
将行为对齐量化集成到其调度与内存管理机制中，
有望在真实服务场景下充分发挥量化的显存节省优势。

\textbf{K/V 差异化精度的精细化（部分已实现）。}
第~\ref{sec:exp-kv-sensitivity}~节的 K/V 消融实验已验证了
MixedKV（K@INT8+V@INT4）在多模型上的可行性。
在此基础上，一个自然的改进方向是进一步精细化 K/V 差异化策略：
根据每一层的 K/V KL 散度值动态选择各层的 K/V 比特宽度——
对 Key 敏感层保持 INT8 甚至 FP16，
对 Key 鲁棒层可尝试 INT4。
第~\ref{subsec:exp-kv-arch-dependency}~节的热力图数据
为层级比特分配提供了初步的定量依据，
但将此策略自动化并集成到推理框架中仍需进一步工程工作。
INT4-RoleAlign 已初步实现了 K/V 差异化精度的核心思想
（per-channel K + per-token V 非对称量化），
但逐层自适应比特宽度分配尚未探索。

\textbf{校准目标函数的泛化探索。}
本文以 KL 散度作为校准目标，但在 INT4 等低比特场景下，
KL 散度优化景观可能存在多个局部最优解，
且 KL 散度对尾部分布的高敏感性在低比特下可能放大噪声干扰。
探索 Wasserstein 距离（地球搬运距离）或 R\'{e}nyi 散度等替代目标函数，
可能在低比特量化的校准优化中展现不同的收敛行为和解空间特性。
特别是 Wasserstein 距离在度量概率分布距离时不要求支撑集重叠，
在量化导致分布支撑发生离散化偏移时可能提供更稳定的梯度信号。

\textbf{chunk\_size=1 跨模型系统验证。}
本文在 Qwen2.5-1.5B 上观察到 KIVI-style 在 chunk\_size=1 下的灾难性退化
与 INT4-RoleAlign 的鲁棒性。
在更多模型（尤其是不同 GQA 头数的架构）上系统验证
chunk\_size 对不同量化方案的影响，
将为实际部署中的量化粒度选择提供更充分的指导。

\textbf{BA percentile 在更低比特下的可行性探索。}
INT4-RoleAlign 的 BA-guided percentile 校准在 4-bit 下展现了有效性。
探索该校准范式在 INT3 乃至 INT2 极低比特下的适用性，
以及与非均匀量化网格的结合潜力，
是进一步拓展行为对齐框架压缩能力的方向。

\section{结语}
\label{sec:conclusion-closing}
% ------------------------------------------------------------

KV Cache 量化的关键不是最小化数值重建误差，而是保持注意力行为。
Attention-KL 作为统一的校准目标与诊断透镜，
首先在 INT8 上得到验证，
进而揭示低比特退化的 Key 主导根因与 GQA 架构依赖，
最终导出 INT4-RoleAlign——
以 73\% 的 KV Cache 压缩率实现三模型 Needle 100\% 的检索恢复，
同时揭示 PPL 退化（2.4--13.7\%）与延迟代价的明确边界。
上述结果表明，行为对齐原则不仅是更优的量化目标，
更提供了一条从原则到诊断到设计到边界界定的完整方法论路径，
将 KV Cache 量化推进为可诊断、可分层讨论的问题。
