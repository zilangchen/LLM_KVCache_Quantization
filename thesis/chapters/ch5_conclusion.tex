\chapter{总结与展望}
\label{chap:conclusion}

本章对全文的研究工作进行总结，回顾主要贡献，分析当前方法存在的局限性，
并对未来可能的研究方向进行展望。

% ------------------------------------------------------------
\section{研究工作总结}
\label{sec:conclusion-summary}
% ------------------------------------------------------------

\textbf{INT8 行为对齐量化的基础验证。}
如第四章实验所示，INT8 行为对齐量化在 PPL、Needle、LongBench 三个核心质量基准上
与 FP16 保持统计不可区分的水平，同时 Triton 融合核函数实现了 8\%--38\% 的解码延迟降低，
KV Cache 显存节省约 44\%。
同一校准算法在 Qwen2.5-7B、LLaMA-3.1-8B 和 Mistral-7B 上复现了一致的性能模式，
验证了在四个模型（三个模型族）上的跨模型适用性。
自适应保护机制（Adaptive Static Scale Protection）是长上下文 Needle 通过率的关键影响因素，
校准样本数从 16 到 256 对量化质量影响极小，表明方法具有良好的实用鲁棒性（基于 PPL 和 Needle 两项指标在 1.5B 和 7B 上的观测）。

贯穿全文的技术路线遵循递进式扩展：INT8 行为对齐量化作为经过完整验证的规范实例，MixedKV 作为实用性扩展，INT4-RoleAlign 作为低比特修复扩展。三者共享同一诊断框架，但证据成熟度依次递减。

\textbf{从 KL 诊断到 INT4-RoleAlign。}
本文的主要技术贡献体现在 KL 散度的双重角色：
不仅作为校准目标，更作为诊断透镜。
KL 诊断揭示了对称 per-group INT4 的系统性失效模式
（Qwen2.5-1.5B Needle 0\%、PPL 退化超 15\%），
受控 K/V 消融实验在四个模型（三个模型族）上一致确认 Key 量化为退化主导因素，
且该敏感性呈明显的 GQA 架构依赖特征（$H_{kv}$ 越大，量化噪声稀释效应越强）。
基于此诊断链条，本文提出 INT4-RoleAlign：
吸收 per-channel K + per-token V 非对称量化轴设计，
通过 BA-guided percentile 校准确定最优参数。
INT4-RoleAlign 在 Qwen2.5-1.5B 上将 Needle 通过率从 0\% 恢复至 100\%，
PPL 退化从 KIVI-style 的 12\% 降至 1.2\%；
在三个模型（两个模型族）上对 KIVI-style INT4 呈现竞争性。

\textbf{chunk\_size 鲁棒性证据。}
在 Qwen2.5-1.5B 的 fully quantized PPL 评测协议下（chunk\_size=1，
所有历史 KV 均经量化存取），KIVI-style PPL 急剧退化至 9442（+101{,}000\%），
而 INT4-RoleAlign 仅为 9.49（+1.9\%），
表明 BA-guided percentile 校准在逐 token 粒度下仍然有效。
这一结果进一步佐证了行为对齐校准框架从 INT8 到 INT4、
从对称到非对称格式的适用性（在当前评测的模型范围内）。

在全部实验预期中，INT8 相关 10 项预期中 9 项达标
（RULER 退化 $-$2.64\% 集中于检索类子任务，受配置聚合效应放大），
MixedKV 和校准鲁棒性实验达标，
INT4 对称量化 3 项预期全部未达标——
确认了 4-bit 对称量化在长上下文场景下的适用性边界，
也正是 INT4-RoleAlign 的诊断驱动出发点。

% ------------------------------------------------------------
\section{局限性}
\label{sec:conclusion-limitations}
% ------------------------------------------------------------

尽管本文提出的行为对齐量化框架在 INT8 的 PPL、Needle、LongBench 三项核心质量基准上维持了与 FP16 统计不可区分的水平（RULER 的退化集中于检索类子任务，详见第~\ref{para:c6-fail}~段分析），
但当前工作仍存在以下局限：

\textbf{INT4 量化的质量瓶颈。}
实验结果表明，INT4 量化在 32K 长上下文场景下存在系统性的质量退化：
Needle 通过率在 1.5B 模型上从 100\%（FP16）降至 0\%，
PPL 退化达 $-$15.9\%，
LongBench 综合评分在 1.5B 和 7B 模型上大幅下降。
第~\ref{subsec:exp-int4-limitations}~节的定量分析表明，
INT4 的 SQNR 仅 25.8~dB（INT8 为 49.9~dB），
逐元素量化噪声功率约为 INT8 的 289 倍，
远超 softmax 归一化可消解的范围。
随序列长度增大，极值统计效应进一步放大了噪声对检索信号的淹没概率，
导致 Needle 通过率从短上下文到 32K 的急剧下降。
在跨模型对比中，GQA 结构的量化误差稀释效应
为不同模型间的退化差异提供了定量解释：
LLaMA-3.1-8B（$H_{kv}=8$）的有效噪声功率仅为
Qwen2.5-1.5B（$H_{kv}=2$）的 $1/4$，
这解释了其在 INT4 下的异常稳健表现（Needle 100\%、PPL 退化仅 $-$9.8\%）。
补充实验进一步表明，运行时 scale 参数的精度链路（float32 vs float16）
也是影响 INT4 质量的重要因素，确保 float32 精度后部分模型的 PPL 获得显著改善，
但整体仍未达到跨模型一致的非劣性水平。
INT4-RoleAlign 已部分解决上述瓶颈：
在三个模型上 PPL 退化从 KIVI-style 的 2.4\%--12\% 降至 0.3\%--1.2\%，
Needle 通过率在全部模型上恢复至 100\%。
但 chunk\_size=1 的跨模型系统验证仅在 Qwen2.5-1.5B 上完成，
推广至更多模型和更长上下文仍需进一步实验。

\textbf{KIVI-style 基线的实现差异。}
本文实现的 KIVI-style 基线采用了 KIVI~\citep{liu2024kivi}
的核心量化轴策略（Key 按通道量化、Value 按 token 量化），
但未完整复现原文的所有工程优化（如残差长度调度、异步量化流水线等）。
因此本文的 KIVI-style 实验结果应理解为对其核心算法思想的验证，
而非原文系统完整性能的严格复现。

\textbf{单 GPU 评测限制。}
本文的所有实验均在单张 NVIDIA H20 GPU（98GB 显存）上完成，
未涉及多卡分布式推理场景。在张量并行（Tensor Parallelism）或
流水线并行（Pipeline Parallelism）配置下，KV Cache 的分片、
同步与量化的交互可能引入新的性能瓶颈和数值问题，
当前框架尚未对此进行验证。

\textbf{校准产物的预处理局限与模型特异性。}
本文 1.5B 主模型的校准产物（v3\_quick）在生成时
Q 向量预处理流程尚未包含 input\_layernorm 和 RoPE 变换
（详见第三章和附录~\ref{sec:app-calib-provenance}），
后续版本已修正但重校准对比验证尚未完成。
此外，
行为对齐校准过程产生的量化参数（per-layer scales 和 per-head $\tau^{-1}$）
与特定模型的权重分布和注意力模式紧密耦合。
当部署新的模型架构或不同规模的模型变体时，
需要重新执行完整的校准流程以获取适配的量化参数，
这在模型迭代频繁的场景下会带来额外的工程成本。
研究校准参数在同架构不同规模模型间的可迁移性，
是降低部署成本的一个重要方向。

\textbf{Mistral-7B LongBench 评测的数据局限。}
Mistral-7B-Instruct-v0.3 作为第四验证模型，
其 LongBench 评测结果在所有量化配置下均为 1.0，
源于评测管线使用了合成数据源而非标准 LongBench 测试集。
该 artifact 导致 Mistral-7B 的 LongBench 数据
不具备与其他三个模型的可比性，
因此正文的跨模型 LongBench 核心对比中排除了 Mistral 的数据
（详见附录~\ref{sec:app-kv-ablation-full}）。
Mistral-7B 在 PPL、Needle 和 RULER 上的评测均正常有效。

\textbf{K/V 消融实验的评测覆盖范围。}
第~\ref{sec:exp-kv-sensitivity}~节的 K/V 精度消融实验
仅评测了 RULER 和 LongBench 两个长上下文基准，
未覆盖 PPL 指标。这一设计选择基于 K/V 消融主要关注
长上下文场景下的质量退化模式，
但 K/V 差异化量化对语言建模质量（PPL）的影响
尚需补充实验验证。

\textbf{校准样本来源单一。}
本文的行为对齐校准仅使用 WikiText-103 数据集，
未验证其对代码、多语言、数学推理等领域分布外数据的适用性。
尽管第~\ref{subsec:exp-ablation-b10}~节表明校准样本数量的影响极小，
但校准\emph{数据集}选择的影响尚未系统评估。

\textbf{chunk\_size=1 验证范围有限。}
第四章的 chunk\_size 鲁棒性分析仅在 Qwen2.5-1.5B 单模型、单种子（seed=1234）
条件下完成。虽然 INT4-RoleAlign 在该模型上表现出鲁棒性
（PPL 9.42→9.49，+0.75\%），但该结论能否推广至其他模型架构和参数规模
尚需系统性的跨模型 chunk\_size=1 验证。

\textbf{$\tau^{-1}$ 温度校正在非对称格式下的不可迁移性。}
实验表明，基于 INT8 对称量化校准搜索得到的 $\tau^{-1}$ 参数
在 INT4 非对称量化上无稳定收益甚至有害
（Needle 100\%→95\%，LongBench 下降 15\%），
提示行为对齐框架的有效组件依赖量化格式。
探索针对非对称量化噪声特性的温度校正方案是一个值得研究的方向。

\section{对工业实践的启示}
\label{sec:conclusion-implications}
% ------------------------------------------------------------

本文的实验结果对 LLM 推理系统的工程实践具有以下参考价值：

\textbf{INT8 量化在单 GPU 推理场景下已展现明确的部署可行性。}
INT8 行为对齐量化在 PPL、Needle、LongBench 三项核心质量基准上
均未引入统计显著退化（RULER 的退化集中于检索类子任务且受配置聚合效应放大，详见第~\ref{para:c6-fail}~段），
同时实现了 44\% 的 KV Cache 显存节省和 8\%--38\% 的解码延迟降低。
对于 PPL 和检索准确率是核心指标的在线推理服务，
这意味着在相同硬件条件下可以支持更长的上下文窗口或更大的并发批次。
表~\ref{tab:app-batch-capacity}（附录）显示，
int8\_ours 的可服务 batch 从 fp16 的 16 提升到 32（2$\times$），
直接倍增了单 GPU 的服务吞吐能力。

\textbf{INT4 量化需审慎评估应用场景。}
INT4 量化虽可将 KV Cache 进一步压缩至 FP16 的 $1/4$，
但 32K 长上下文下的质量退化使其不适合对精度要求高的任务
（如精确信息检索、长文档问答）。
对于对质量容忍度较高的场景（如大规模文本摘要、粗粒度分类），
INT4 量化的显存优势仍有实用价值。

\textbf{融合核函数是量化收益的放大器。}
本文的对比实验表明，仅做 KV Cache 量化而不优化计算路径
（如 int8\_baseline 的两阶段方案），
反而可能因中间张量分配导致 TPOT 增加。
Triton 融合核函数消除了这一开销，使量化的显存节省真正转化为端到端的性能提升。
这提示工业部署中应将量化与计算融合作为整体方案设计，而非独立优化。

\textbf{校准成本与部署收益的权衡。}
行为对齐校准是一次性离线过程：在单张 GPU 上，
1.5B 模型的全量校准（128 样本、逐层逐头搜索）约需 15 分钟，
7B/8B 模型约需 30--50 分钟。校准产出的 per-layer scale 和 per-head $\tau^{-1}$
参数以 JSON 文件形式保存（$<$10~KB），在推理时仅需加载一次，在模型权重不变的前提下可持续复用。
对于在线推理服务，单次校准投入可在数小时的持续服务中
通过吞吐量提升和显存节省完全摊销，投入产出比较为有利。

\section{未来工作展望}
\label{sec:conclusion-future}
% ------------------------------------------------------------

基于本文的研究成果和上述局限性分析，以下方向值得进一步探索：

\textbf{基于 GQA 头数的自适应量化比特分配。}
第~\ref{subsec:exp-int4-limitations}~节和第~\ref{sec:exp-kv-sensitivity}~节的实验
共同表明，GQA 头数（$H_{kv}$）是决定低比特量化容忍度的关键架构因素，
且 K/V 消融进一步证实了 Key 与 Value 在量化敏感性上的根本不对称性。
这一发现启发了一种架构感知的自适应策略：
根据每个模型的 $H_{kv}$ 和每个层/头的 KL 散度值，
自动为不同层和不同 KV 维度（K vs V）分配量化比特宽度——
$H_{kv}$ 较大的模型可更激进地压缩 Value 至 INT4 甚至 INT2，
$H_{kv}$ 较小的模型则应优先保障 Key 精度。
第~\ref{subsec:exp-mixedkv}~节的 MixedKV 实验已初步验证了
该策略方向的可行性，更精细的逐层自适应分配是进一步研究的重点。

\textbf{多 GPU 分布式推理。}
将行为对齐量化框架扩展到张量并行和流水线并行场景，
需要解决 KV Cache 分片粒度与量化粒度的对齐问题、
跨设备量化参数的一致性维护以及通信-计算重叠的优化。
与主流分布式推理框架的集成将是框架走向实际部署的关键一步。

\textbf{推理服务框架集成。}
当前实现基于 Transformers 库的自定义生成循环，尚未与
vLLM~\cite{kwon2023vllm}、Text Generation Inference（TGI）\footnote{HuggingFace TGI: \url{https://github.com/huggingface/text-generation-inference}} 等
生产级推理服务框架集成。这些框架已实现了 PagedAttention、
连续批处理（Continuous Batching）等系统级优化，
将行为对齐量化集成到其调度与内存管理机制中，
有望在真实服务场景下充分发挥量化的显存节省优势。

\textbf{K/V 差异化精度的精细化（部分已实现）。}
第~\ref{sec:exp-kv-sensitivity}~节的 K/V 消融实验已验证了
MixedKV（K@INT8+V@INT4）在多模型上的可行性。
在此基础上，一个自然的改进方向是进一步精细化 K/V 差异化策略：
根据每一层的 K/V KL 散度值动态选择各层的 K/V 比特宽度——
对 Key 敏感层保持 INT8 甚至 FP16，
对 Key 鲁棒层可尝试 INT4。
第~\ref{subsec:exp-kv-arch-dependency}~节的热力图数据
为层级比特分配提供了初步的定量依据，
但将此策略自动化并集成到推理框架中仍需进一步工程工作。
INT4-RoleAlign 已初步实现了 K/V 差异化精度的核心思想
（per-channel K + per-token V 非对称量化），
但逐层自适应比特宽度分配尚未探索。

\textbf{校准目标函数的泛化探索。}
本文以 KL 散度作为校准目标，但在 INT4 等低比特场景下，
KL 散度优化景观可能存在多个局部最优解，
且 KL 散度对尾部分布的高敏感性在低比特下可能放大噪声干扰。
探索 Wasserstein 距离（地球搬运距离）或 R\'{e}nyi 散度等替代目标函数，
可能在低比特量化的校准优化中展现不同的收敛行为和解空间特性。
特别是 Wasserstein 距离在度量概率分布距离时不要求支撑集重叠，
在量化导致分布支撑发生离散化偏移时可能提供更稳定的梯度信号。

\textbf{chunk\_size=1 跨模型系统验证。}
本文在 Qwen2.5-1.5B 上观察到 KIVI-style 在 chunk\_size=1 下的灾难性退化
与 INT4-RoleAlign 的鲁棒性。
在更多模型（尤其是不同 GQA 头数的架构）上系统验证
chunk\_size 对不同量化方案的影响，
将为实际部署中的量化粒度选择提供更充分的指导。

\textbf{BA percentile 在更低比特下的可行性探索。}
INT4-RoleAlign 的 BA-guided percentile 校准在 4-bit 下展现了有效性。
探索该校准范式在 INT3 乃至 INT2 极低比特下的适用性，
以及与非均匀量化网格的结合潜力，
是进一步拓展行为对齐框架压缩能力的方向。
