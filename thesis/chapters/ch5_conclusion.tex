\section{核心发现与收束}
\label{sec:conclusion-answers}
% ------------------------------------------------------------

\paragraph{Finding 1：校准目标灵敏度的 bit-width 与规模依赖（回应 RQ1）。}
在贪心解码与 robust selection 约束下，
attention-KL 与 MSE 两种校准目标的作用呈 bit-width 与规模的双重依赖。
INT8 下两者完全趋同，产生逐位一致的校准产物；
INT4 下在 1.5B（$H_{kv}=2$）上分化
（percentile 99.5 vs 99.0，scale 差异 12.35\%），
而在 7B（$H_{kv}=4$）上再次趋同。
在本文所测试的模型规模与协议下，
KL 至少不劣于 MSE，并在小模型 INT4 设置下表现更优。

\paragraph{Finding 2：Key 主导退化、$H_{kv}$ 结构性关联与低比特实例化验证（回应 RQ2--3）。}
通过 K-only / V-only 因子分解，四个模型一致观察到
INT4 退化由 Key 侧主导。
14B 上保持 Key 为 FP16 可恢复 93\% 的 PPL 退化，
保持 Value 为 FP16 仅恢复 64\%。
该诊断指向 per-channel Key + per-token Value 差异化格式——
与 KIVI~\cite{liu2024kivi} 从工程经验出发独立提出的格式一致。
在该格式上以离线 attention-KL 搜索实例化的 INT4-RoleAlign
在四个模型上保持 Needle 100\%，KV Cache 压缩约 73\%，
PPL 退化随规模减弱（1.5B: 13.7\%，7B: 6.1\%，8B: 2.4\%）。
14B 提供外部效度：Needle 在 4K--32K 全通过，
RULER 在 4K--16K 达到 96.6\%--98.5\%。

\paragraph{Finding 3：融合核效率的 $(H_{kv}, \text{seq\_len})$ 特征（回应 RQ3）。}
Triton 融合量化解码核函数的效率增益
在 $(H_{kv}, \text{seq\_len})$ 空间中呈规模依赖：
$H_{kv}=8$ 在 8K+ 序列下提供显著加速（32K 下延迟减少约 40\%），
$H_{kv}=2$ 下融合核在所有长度下均慢于 SDPA。
8B（$H_{kv}=8$, LLaMA）与 14B（$H_{kv}=8$, Qwen）
在短序列效率特征上高度一致，
为 $H_{kv}$ 而非模型规模主导提供支持证据。
部署建议（限 NVIDIA H20 平台）：
$H_{kv}\geq 4$ 且序列长度超过 8K 时使用融合核，$H_{kv}=2$ 使用 SDPA。

\section{局限性}
\label{sec:conclusion-limitations}
% ------------------------------------------------------------

当前工作存在以下局限：

\paragraph{INT4-RoleAlign 的验证范围}
INT4-RoleAlign 的核心 PPL/Needle 结果已扩展到四个模型（Qwen2.5-1.5B/7B/14B、LLaMA-3.1-8B）；
其中完整的四类质量矩阵（PPL、Needle、LongBench、RULER）
与系统效率矩阵（TPOT、KV 显存）目前覆盖三个模型（1.5B/7B/8B）。
检索能力在 Needle 上完全恢复（100\%），RULER 整体通过率作为补充证据，
但 PPL 退化显著（8B: 2.4\%，7B: 6.1\%，1.5B: 13.7\%），
参考实现（torch\_ref）路径下解码延迟增加约 2.4--2.6$\times$，
本文已验证 INT4 非对称 Triton 融合核函数的数值正确性
（第~\ref{subsec:ch3-triton-int4-asym}~节），
但尚未实现端到端延迟加速。
chunk\_size=1 全量化压力测试仅在 Qwen2.5-1.5B 单模型、单种子条件下验证，
跨模型推广仍需后续实验。此外，INT4-RoleAlign 的 PPL 对 chunk\_size 敏感：
标准评测条件下（cs=128）PPL=10.58（1.5B），
而在极限 cs=1 的压力测试中退化显著（PPL $> 10^4$）。
该现象源于 asymmetric 量化在极小 batch 解码时对
prefill-computed scale range 的依赖——
decode 阶段新 K 值若超出该范围则量化误差无缓冲空间。
当前实验提示 cs=1 风险极高，cs$\geq$8 的系统验证仍需后续补充；
后续工作可探索 decode-phase 动态 scale 更新机制。

\paragraph{评测完整性}
本研究过程中发现评测脚本存在静默回退缺陷
（eval\_ppl.py 缺少 int4\_ours\_asym 分支导致静默使用 INT8 路径），
导致早期版本的 INT4-RoleAlign PPL 数据（9.42）实为 INT8 结果。
该问题在后续验证中发现并修正，更正后的 PPL 为 10.58。
修复 bug 后，\textbf{PPL（1.5B/7B/8B/14B 四模型）、RULER 与 LongBench
均以 int4\_ours\_asym 作为独立 kv\_mode 完整重跑}，
对应结果分别汇总于第~\ref{subsec:exp-rolealign-results}~节
（PPL / Needle / LongBench）、表~\ref{tab:kivi-int4-threeway}
（RULER 4K--32K）及附录~\ref{sec:app-longbench-official}
（LongBench v2 bug 验证），论文所有公开数据均为修复后的结果。
推理引擎中此类"静默错误"虽报告率低但危害严重：
脚本正常完成、数据格式正确，唯一异常是量化精度不匹配。
本文的教训是：KV Cache 量化评测必须在结果 CSV 中记录实际使用的
cache 类型和量化参数，并在脚本层面对未知 kv\_mode 立即报错
而非静默回退。

\paragraph{评测方法边界}
所有实验在单张 NVIDIA H20 GPU 上完成，未涉及多卡分布式推理。
此外，本文未与 FP8 KV Cache 进行对比。
FP8（E4M3/E5M2）在 H100/H200 等新一代硬件上已获原生支持，
其浮点格式对异常值的天然容忍度可能在无需校准的情况下达到接近 INT8 的精度，
但仅提供 8~bit 压缩率而无 INT4 的 4~bit 选项。
LongBench 评测基于自行实现的合成数据源而非官方数据集，
绝对分数不可与排行榜直接对比，量化方法间的相对差异可提供方向性参考，
但仍需官方基准进一步校准。
Mistral-7B 的 LongBench 数据因合成源问题在正文核心对比中已排除
（Mistral 在 PPL、Needle、RULER 上的评测均正常有效）。
单指标 K/V 消融（保持一端 FP16、另一端量化）的完整 PPL 评测
（表~\ref{tab:kv-ablation-ppl}）仅在 Qwen2.5-1.5B 单模型上以 1 seed 验证；
但 K/V 角色差异化的扩展形式 \emph{非对称 MixedKV（K@INT8 + V@INT4）}
已在第~\ref{subsec:exp-mixedkv}~节横跨 Qwen2.5-1.5B / 7B、
LLaMA-3.1-8B 与 Mistral-7B 四模型以 3 seeds 验证，
因此"K/V 精度不对称"这一机制结论具有跨模型外部效度，
单 seed 只影响 FP16-基线受控消融的 PPL 数值精度，
不影响 Key 主导退化的方向性判断。

\paragraph{校准框架局限}
1.5B 主模型校准产物的 Q 向量预处理缺少 input\_layernorm 和 RoPE 变换
，
但修正版对比验证表明对 INT8 的 PPL 影响仅 0.05\%（$<$0.01 绝对值），
对 INT4-RoleAlign 影响为零（BA percentile 校准独立于 RoPE）。
LLaMA-3.1-8B 的 INT8 校准在 $n{=}10$ 种子验证中展现一定次优性
（相对 FP16 基线 PPL 偏高约 16\%，高于 1.5B/7B 的 $<$0.5\% 退化），
推测与校准产物对 8B 架构的稀疏分布适配有关，
该异常不影响主表（聚焦 1.5B）与 INT4-RoleAlign 的核心结论，
但提示后续工作应为 8B 规模专门调优校准流程。
校准仅使用 WikiText-2 数据集，对代码、多语言等分布外数据的适用性未验证。
提示有效组件依赖量化格式，不可跨格式迁移。
校准参数与模型权重耦合，部署新模型需重新校准。

\paragraph{KIVI-style 基线的实现差异}
本文 KIVI-style 基线采用 KIVI~\citep{liu2024kivi} 的核心量化轴策略，
未完整复现原文全部工程优化（如残差长度调度等），
结果应理解为对核心算法思想的验证。

\paragraph{BitDecoding 融合核的 GQA 兼容性}
在探索 Tensor Core 级别的 INT4 解码加速时，
本文评测了 BitDecoding~\cite{du2026bitdecoding} 的 CUTLASS 内核。
测试发现其在 GQA 架构下的数值输出与 FP16 参考存在显著偏差，
无法通过数值验证。
该问题发生在库层面，超出 wrapper 可修复的范围，
因此本文将 BitDecoding 作为工程案例记录，
而非可用的部署 backend（详见第四章讨论）。

\paragraph{Threat-to-Validity 总结}
第四章综合讨论部分给出了五条 Threat-to-Validity 声明，
覆盖 $H_{kv}$/规模共变、解码策略依赖（greedy only）、
硬件依赖（NVIDIA H20 only）、
校准数据分布依赖（WikiText-2 only）、
以及单实现威胁（KIVI 自实现）。
相关结论应在上述边界内理解。

\section{未来工作展望}
\label{sec:conclusion-future}
% ------------------------------------------------------------

基于本文的研究成果和上述局限性分析，以下方向值得进一步探索：

\paragraph{基于 GQA 头数的自适应量化比特分配}
第~\ref{subsec:exp-int4-limitations}~节和第~\ref{sec:exp-kv-sensitivity}~节的实验
共同表明，GQA 头数（$H_{kv}$）是决定低比特量化容忍度的关键架构因素，
且 K/V 消融进一步证实了 Key 与 Value 在量化敏感性上的根本不对称性。
这一发现启发了一种架构感知的自适应策略：
根据每个模型的 $H_{kv}$ 和每个层/头的 KL 散度值，
自动为不同层和不同 KV 维度（K vs V）分配量化比特宽度。
具体而言，$H_{kv}$ 较大的模型可更激进地压缩 Value 至 INT4 甚至 INT2，
$H_{kv}$ 较小的模型则应优先保障 Key 精度。
第~\ref{subsec:exp-mixedkv}~节的 MixedKV 实验已初步验证了
该策略方向的可行性，更精细的逐层自适应分配是进一步研究的重点。

\paragraph{多 GPU 分布式推理}
将行为对齐量化框架扩展到张量并行和流水线并行场景，
需要解决 KV Cache 分片粒度与量化粒度的对齐问题、
跨设备量化参数的一致性维护以及通信-计算重叠的优化。
与主流分布式推理框架的集成将是框架走向实际部署的关键一步。

\paragraph{推理服务框架集成}
当前实现基于 Transformers 库的自定义生成循环，尚未与
vLLM~\cite{kwon2023vllm}、Text Generation Inference（TGI）\footnote{HuggingFace TGI: \url{https://github.com/huggingface/text-generation-inference}} 等
生产级推理服务框架集成。这些框架已实现了 PagedAttention、
连续批处理（Continuous Batching）等系统级优化，
将行为对齐量化集成到其调度与内存管理机制中，
有望在真实服务场景下充分发挥量化的显存节省优势。

\paragraph{K/V 差异化精度的精细化}
第~\ref{sec:exp-kv-sensitivity}~节的 K/V 消融实验已验证了
MixedKV（K@INT8 + V@INT4）在多模型上的可行性。
在此基础上，一个自然的改进方向是进一步精细化 K/V 差异化策略：
根据每一层的 K/V KL 散度值动态选择各层的 K/V 比特宽度：
对 Key 敏感层保持 INT8 甚至 FP16，
对 Key 鲁棒层可尝试 INT4。
第~\ref{subsec:exp-kv-arch-dependency}~节的热力图数据
为层级比特分配提供了初步的定量依据，
但将此策略自动化并集成到推理框架中仍需进一步工程工作。
INT4-RoleAlign 已初步实现了 K/V 差异化精度的核心思想
（per-channel K + per-token V 非对称量化），
但逐层自适应比特宽度分配尚未探索。

\paragraph{校准目标函数的泛化探索}
本文以 KL 散度作为校准目标，但在 INT4 等低比特场景下，
KL 散度优化景观可能存在多个局部最优解，
且 KL 散度对尾部分布的高敏感性在低比特下可能放大噪声干扰。
探索 Wasserstein 距离（地球搬运距离）或 R\'{e}nyi 散度等替代目标函数，
可能在低比特量化的校准优化中展现不同的收敛行为和解空间特性。
特别是 Wasserstein 距离在度量概率分布距离时不要求支撑集重叠，
在量化导致分布支撑发生离散化偏移时可能提供更稳定的梯度信号。
此外，本文采用 forward KL（$D_{\text{KL}}(p_{\text{ref}} \| p_{\text{quant}})$）
的 zero-avoiding 特性优先保护关键 token 的注意力权重不被遗漏
（第~\ref{sec:ch3-calibration}~节），
但 reverse KL 的 zero-forcing 特性在抑制量化引入的虚假注意力峰值方面
可能具有互补优势。
对两种方向的系统消融对比（forward KL vs reverse KL vs Jensen-Shannon）
将为校准目标的最优选择提供更完整的理论指导。

\paragraph{chunk\_size=1 跨模型系统验证}
本文在 Qwen2.5-1.5B 上观察到 KIVI-style 与 INT4-RoleAlign 在 chunk\_size=1
全量化压力测试下均发生灾难性退化
（KIVI PPL 从 10.43 升至 9442，INT4-RoleAlign PPL $> 10^4$），
根因是 per-channel K Scale 仅基于首个 token 建立时 scale 范围极窄，
后续 K 值几乎全部溢出（详见第~\ref{subsec:exp-int4-boundary}~节与附录~\ref{sec:app-chunksize}）。
在更多模型（尤其是不同 GQA 头数的架构）上系统验证
chunk\_size 对不同量化方案的影响，并探索 decode 阶段动态 scale 更新机制，
将为实际部署中的量化粒度选择提供更充分的指导。

\paragraph{BA percentile 在更低比特下的可行性探索}
INT4-RoleAlign 的 BA-guided percentile 校准在 4-bit 下展现了有效性。
探索该校准范式在 INT3 乃至 INT2 极低比特下的适用性，
以及与非均匀量化网格的结合潜力，
是进一步拓展行为对齐框架压缩能力的方向。

\section{结语}
\label{sec:conclusion-closing}
% ------------------------------------------------------------

本文的核心主张是：
attention-KL 提供了 GQA 架构下 KV Cache 量化的统一校准与诊断原则，
而 $H_{kv}$ 是与低比特失效模式、温度校正分化和融合解码效率
一致性变化的结构性关联变量。
这一视角把 KV Cache 量化的问题从``先量化再看效果''的试错模式
转向以 $H_{kv}$ 为组织轴的系统分析：
校准目标的有效性何时依赖行为度量、
退化在 Key/Value 间如何分配、
以及融合核在何种 $(H_{kv}, \text{seq\_len})$ 条件下提供正向加速。
分层流程在其他中间表示压缩场景中是否能迁移，
取决于目标任务是否存在类似的
``数值保真 $\neq$ 行为保真''错位以及可直接观测的行为级度量。

在当前模型集合中，$H_{kv}$ 与模型规模部分共变，
因此相关结论应理解为结构性证据而非严格受控因果识别。
本文结论的适用范围限于四个模型（$\leq 14$B）、贪心解码、
robust selection 协议与自研 KIVI-style 实现这一具体协议。
这些边界同时也定义了后续工作的清晰起点。
