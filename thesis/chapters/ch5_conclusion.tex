\section{核心发现与问题回答}
\label{sec:conclusion-answers}
% ------------------------------------------------------------

引言提出了三个递进的研究问题（RQ1--RQ3），
本节将实验证据组织为三个核心发现。

\paragraph{发现一（回答 RQ1）：注意力行为保真是正确的优化目标}
INT8 行为对齐量化的质量损失极为有限：PPL 退化不到 0.3\%，
Needle 通过率在四个模型上始终保持 100\%，
LongBench 综合评分与 FP16 的差异在 $n=5$ 的检验中未达统计显著性，
效应量微小（$<$2\%，附录~\ref{sec:app-longbench-official} 的官方数据验证了该结论）。
RULER 是唯一出现退化的指标（$-$2.64\%），
退化集中于检索子任务中的非主线配置变体。
效率层面，KV 显存压缩 44\%，
Triton 融合核函数带来 8--38\% 的解码加速。
上述数据表明，以 attention-KL 为校准目标的行为保真策略
在 INT8 精度下是可行且有效的。

\paragraph{发现二（回答 RQ2）：低比特退化是 Key 主导的结构性问题}
对称 INT4 的失效并非偶然的精度损失，
而是呈现出清晰的结构性特征。
Needle 直接降至 0\%，PPL 退化超过 15\%；
受控 K/V 消融进一步表明，退化几乎全部来自 Key：
将 Key 单独降至 INT4 即可复现主要退化，
Value 单独降至 INT4 则影响甚微。
这一 Key 敏感性与 GQA 头数（$H_{kv}$）密切相关：
$H_{kv}=8$ 的 LLaMA-3.1-8B 对 INT4 量化噪声展现出明显的稀释效应，
$H_{kv}=2$ 的 Qwen2.5-1.5B 则最为脆弱。
Key 决定了注意力的``看哪里''，
其通道间的数值异质性在 4-bit 的 15 个离散级别下无法被对称格式保持。

\paragraph{发现三（回答 RQ3）：诊断导出角色感知设计，检索可恢复但存在代价}
诊断结论直接给出了设计方向：对 Key 和 Value 采用差异化的量化轴。
INT4-RoleAlign 在三个模型上将 Needle 从 0\% 恢复至 100\%，
KV Cache 压缩 73\%（RULER 评测提供支持性证据，s\_niah $\geq$99\%）。
代价同样明确：PPL 退化 2.4--13.7\%（随 $H_{kv}$ 减小而增大），
在参考实现（torch\_ref）路径下解码延迟增加约 2.4--2.6$\times$，
但本文已实现专用的 INT4 非对称 Triton 融合核函数
（第~\ref{subsec:ch3-triton-int4-asym}~节），
通过 split-channel 设计在 kernel 内部完成 INT4 解包与非对称反量化。
尽管 PPL 退化幅度不小，
Needle 检索在三模型上均完整恢复。
这一解耦现象表明，
局部检索仅依赖少数关键位置的注意力聚焦，
对量化噪声的容忍度远高于需要全局保真的语言建模。

\paragraph{方法论启示}
三个发现串联起来，指向一个更一般的判断：
KV Cache 量化可以从``先量化再看效果''的试错模式，
升级为``先诊断失效根因再针对性设计''的路径。
Attention-KL 在这条路径中承担双重功能：
在 INT8 上它是校准目标（发现一），在 INT4 诊断中它是分析工具（发现二），
两者共享同一套数学框架，后者的发现直接导出了发现三的设计。
从原则到诊断、从设计到边界的闭环构成本文的核心贡献。

\section{局限性}
\label{sec:conclusion-limitations}
% ------------------------------------------------------------

当前工作存在以下局限：

\paragraph{INT4-RoleAlign 的验证范围}
INT4-RoleAlign 已在三个模型上通过 PPL、Needle、LongBench、RULER 四类质量评测
以及 TPOT 和 KV Cache 显存两类系统效率评测验证。
检索能力在 Needle 上完全恢复（100\%），RULER 子任务提供进一步支持（s\_niah $\geq$99\%），
但 PPL 退化显著（8B: 2.4\%，7B: 6.1\%，1.5B: 13.7\%），
参考实现（torch\_ref）路径下解码延迟增加约 2.4--2.6$\times$，
但本文已实现 INT4 非对称 Triton 融合核函数（第~\ref{subsec:ch3-triton-int4-asym}~节）以缓解此问题。
chunk\_size=1 全量化压力测试仅在 Qwen2.5-1.5B 单模型、单种子条件下验证，
跨模型推广仍需后续实验。

\paragraph{评测完整性}
本研究过程中发现评测脚本存在静默回退缺陷
（eval\_ppl.py 缺少 int4\_ours\_asym 分支导致静默使用 INT8 路径），
导致早期版本的 INT4-RoleAlign PPL 数据（9.42）实为 INT8 结果。
该问题在后续验证中发现并修正，更正后的 PPL 为 10.58。
推理引擎中此类"静默错误"虽报告率低但危害严重：
脚本正常完成、数据格式正确，唯一异常是量化精度不匹配。
本文的教训是：KV Cache 量化评测必须在结果 CSV 中记录实际使用的
cache 类型和量化参数，并在脚本层面对未知 kv\_mode 执行 fail-fast
而非静默回退。

\paragraph{评测方法边界}
所有实验在单张 NVIDIA H20 GPU 上完成，未涉及多卡分布式推理。
此外，本文未与 FP8 KV Cache 进行对比。
FP8（E4M3/E5M2）在 H100/H200 等新一代硬件上已获原生支持，
其浮点格式对异常值的天然容忍度可能在无需校准的情况下达到接近 INT8 的精度，
但仅提供 8~bit 压缩率而无 INT4 的 4~bit 选项。
LongBench 评测基于自行实现的合成数据源而非官方数据集，
绝对分数不可与排行榜直接对比，但量化方法间的相对差异仍有效。
Mistral-7B 的 LongBench 数据因合成源问题在正文核心对比中已排除
（Mistral 在 PPL、Needle、RULER 上的评测均正常有效）。
K/V 消融实验覆盖了 RULER、LongBench 和 PPL
（表~\ref{tab:kv-ablation-ppl}），但仅在 Qwen2.5-1.5B 单模型上验证。

\paragraph{校准框架局限}
1.5B 主模型校准产物的 Q 向量预处理缺少 input\_layernorm 和 RoPE 变换
（附录~\ref{sec:app-calib-provenance}），修正版对比验证尚未完成。
校准仅使用 WikiText-103 数据集，对代码、多语言等分布外数据的适用性未验证。
$\tau^{-1}$ 温度校正在非对称格式下无稳定收益甚至有害，
提示有效组件依赖量化格式，不可跨格式迁移。
校准参数与模型权重耦合，部署新模型需重新校准。

\paragraph{KIVI-style 基线的实现差异}
本文 KIVI-style 基线采用 KIVI~\citep{liu2024kivi} 的核心量化轴策略，
未完整复现原文全部工程优化（如残差长度调度等），
结果应理解为对核心算法思想的验证。

\section{工业实践启示}
\label{sec:conclusion-implications}
% ------------------------------------------------------------

上述结果对 LLM 推理系统的工程实践具有直接参考价值：
INT8 行为对齐量化在质量无损的前提下将可服务 batch 倍增至 2$\times$
（附录表~\ref{tab:app-batch-capacity}），
但 Triton 融合核函数是效率收益的前提；
仅做量化而不融合计算路径反而可能增加延迟。
INT4-RoleAlign 以 73\% 压缩率在 Needle、RULER、LongBench 评测上保持检索质量，
同时 KV Cache 显存降低约 3.7$\times$。
解码延迟因 \code{torch\_ref} 路径（无 Triton 融合核函数）比 FP16 增加约 2--2.5$\times$，
当前定位为``检索保持 + 显存优化方案''。
校准为一次性离线过程（1.5B 约 15 分钟，7B/8B 约 30--50 分钟），
产出 $<$10~KB 的 JSON 参数文件，部署成本极低。

\section{未来工作展望}
\label{sec:conclusion-future}
% ------------------------------------------------------------

基于本文的研究成果和上述局限性分析，以下方向值得进一步探索：

\paragraph{基于 GQA 头数的自适应量化比特分配}
第~\ref{subsec:exp-int4-limitations}~节和第~\ref{sec:exp-kv-sensitivity}~节的实验
共同表明，GQA 头数（$H_{kv}$）是决定低比特量化容忍度的关键架构因素，
且 K/V 消融进一步证实了 Key 与 Value 在量化敏感性上的根本不对称性。
这一发现启发了一种架构感知的自适应策略：
根据每个模型的 $H_{kv}$ 和每个层/头的 KL 散度值，
自动为不同层和不同 KV 维度（K vs V）分配量化比特宽度。
具体而言，$H_{kv}$ 较大的模型可更激进地压缩 Value 至 INT4 甚至 INT2，
$H_{kv}$ 较小的模型则应优先保障 Key 精度。
第~\ref{subsec:exp-mixedkv}~节的 MixedKV 实验已初步验证了
该策略方向的可行性，更精细的逐层自适应分配是进一步研究的重点。

\paragraph{多 GPU 分布式推理}
将行为对齐量化框架扩展到张量并行和流水线并行场景，
需要解决 KV Cache 分片粒度与量化粒度的对齐问题、
跨设备量化参数的一致性维护以及通信-计算重叠的优化。
与主流分布式推理框架的集成将是框架走向实际部署的关键一步。

\paragraph{推理服务框架集成}
当前实现基于 Transformers 库的自定义生成循环，尚未与
vLLM~\cite{kwon2023vllm}、Text Generation Inference（TGI）\footnote{HuggingFace TGI: \url{https://github.com/huggingface/text-generation-inference}} 等
生产级推理服务框架集成。这些框架已实现了 PagedAttention、
连续批处理（Continuous Batching）等系统级优化，
将行为对齐量化集成到其调度与内存管理机制中，
有望在真实服务场景下充分发挥量化的显存节省优势。

\paragraph{K/V 差异化精度的精细化}
第~\ref{sec:exp-kv-sensitivity}~节的 K/V 消融实验已验证了
MixedKV（K@INT8+V@INT4）在多模型上的可行性。
在此基础上，一个自然的改进方向是进一步精细化 K/V 差异化策略：
根据每一层的 K/V KL 散度值动态选择各层的 K/V 比特宽度：
对 Key 敏感层保持 INT8 甚至 FP16，
对 Key 鲁棒层可尝试 INT4。
第~\ref{subsec:exp-kv-arch-dependency}~节的热力图数据
为层级比特分配提供了初步的定量依据，
但将此策略自动化并集成到推理框架中仍需进一步工程工作。
INT4-RoleAlign 已初步实现了 K/V 差异化精度的核心思想
（per-channel K + per-token V 非对称量化），
但逐层自适应比特宽度分配尚未探索。

\paragraph{校准目标函数的泛化探索}
本文以 KL 散度作为校准目标，但在 INT4 等低比特场景下，
KL 散度优化景观可能存在多个局部最优解，
且 KL 散度对尾部分布的高敏感性在低比特下可能放大噪声干扰。
探索 Wasserstein 距离（地球搬运距离）或 R\'{e}nyi 散度等替代目标函数，
可能在低比特量化的校准优化中展现不同的收敛行为和解空间特性。
特别是 Wasserstein 距离在度量概率分布距离时不要求支撑集重叠，
在量化导致分布支撑发生离散化偏移时可能提供更稳定的梯度信号。
此外，本文采用 forward KL（$D_{\text{KL}}(p_{\text{ref}} \| p_{\text{quant}})$）
的 zero-avoiding 特性优先保护关键 token 的注意力权重不被遗漏
（第~\ref{sec:ch3-calibration}~节），
但 reverse KL 的 zero-forcing 特性在抑制量化引入的虚假注意力峰值方面
可能具有互补优势。
对两种方向的系统消融对比（forward KL vs reverse KL vs Jensen-Shannon）
将为校准目标的最优选择提供更完整的理论指导。

\paragraph{chunk\_size=1 跨模型系统验证}
本文在 Qwen2.5-1.5B 上观察到 KIVI-style 在 chunk\_size=1 下的灾难性退化
与 INT4-RoleAlign 的鲁棒性。
在更多模型（尤其是不同 GQA 头数的架构）上系统验证
chunk\_size 对不同量化方案的影响，
将为实际部署中的量化粒度选择提供更充分的指导。

\paragraph{BA percentile 在更低比特下的可行性探索}
INT4-RoleAlign 的 BA-guided percentile 校准在 4-bit 下展现了有效性。
探索该校准范式在 INT3 乃至 INT2 极低比特下的适用性，
以及与非均匀量化网格的结合潜力，
是进一步拓展行为对齐框架压缩能力的方向。

\section{结语}
\label{sec:conclusion-closing}
% ------------------------------------------------------------

本文的实践表明，将校准目标从数值层面提升到行为层面，
不仅改善了量化质量，更使失效分析成为可能。
同一度量既是优化的目标函数，也是诊断的分析工具。
以行为级度量替代数值级度量这一方法论路径，
在原理上可能适用于其他中间表示压缩场景（如激活值量化、特征蒸馏、稀疏化），
但需在具体问题中识别恰当的行为级度量，
其有效性取决于目标任务是否存在类似``数值保真 $\neq$ 行为保真''的错位。

本文的局限同样明确：模型规模受限于 $\leq$8B，
INT4 的 PPL 代价在小模型上依然显著，
效率收益依赖于融合核函数的完成度。
这些局限界定了当前工作的适用边界，
也为后续在更大模型、更低比特、更成熟系统框架上的延伸提供了清晰的起点。
